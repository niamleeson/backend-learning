Unary and Streaming RPCs with grpc-js

Intuition

gRPC's four communication patterns are like different phone call types. Unary is a regular call (ask a question, get an answer). Server streaming is like calling a radio station that plays songs continuously after you request a genre. Client streaming is like dictating a long message to voicemail. Bidirectional streaming is a normal conversation where both sides talk freely.

Unary and Streaming RPCs with grpc-js

gRPC supports four communication patterns: unary, server streaming, client streaming, and bidirectional streaming. This lesson covers implementing all four with the @grpc/grpc-js library in Node.js.

gRPC Communication Patterns

1. Unary RPC (most common)
   Client ---[request]---> Server
   Client <--[response]--- Server

2. Server Streaming RPC
   Client ---[request]---> Server
   Client <--[response 1]- Server
   Client <--[response 2]- Server
   Client <--[response N]- Server

3. Client Streaming RPC
   Client ---[request 1]-> Server
   Client ---[request 2]-> Server
   Client ---[request N]-> Server
   Client <--[response]--- Server

4. Bidirectional Streaming RPC
   Client ---[request 1]-> Server
   Client <--[response 1]- Server
   Client ---[request 2]-> Server
   Client <--[response 2]- Server
   (interleaved, no fixed order)

Setting Up the Server

import * as grpc from '@grpc/grpc-js';
import * as protoLoader from '@grpc/proto-loader';
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

const PROTO_PATH = './protos/blog.proto';
const packageDefinition = protoLoader.loadSync(PROTO_PATH, {
  keepCase: true,
  longs: String,
  enums: String,
  defaults: true,
  oneofs: true,
});

const blogProto = grpc.loadPackageDefinition(packageDefinition) as any;

1. Unary RPC Implementation

// Server implementation
const blogService: grpc.UntypedServiceImplementation = {
  // Unary: GetPost
  GetPost: async (
    call: grpc.ServerUnaryCall<any, any>,
    callback: grpc.sendUnaryData<any>
  ) => {
    try {
      const { id } = call.request;
      const post = await prisma.post.findUnique({
        where: { id },
        include: { author: true },
      });

      if (!post) {
        return callback({
          code: grpc.status.NOT_FOUND,
          message: `Post with id '${id}' not found`,
        });
      }

      callback(null, {
        id: post.id,
        title: post.title,
        body: post.body,
        author_id: post.authorId,
        status: `POST_STATUS_${post.status}`,
        tags: post.tags,
        created_at: { seconds: Math.floor(post.createdAt.getTime() / 1000) },
      });
    } catch (error) {
      callback({
        code: grpc.status.INTERNAL,
        message: 'Internal server error',
      });
    }
  },

  // Unary: CreatePost
  CreatePost: async (call, callback) => {
    try {
      const { title, body, tags } = call.request;

      // Get user from metadata (like HTTP headers)
      const userId = call.metadata.get('user-id')[0] as string;
      if (!userId) {
        return callback({
          code: grpc.status.UNAUTHENTICATED,
          message: 'user-id metadata required',
        });
      }

      const post = await prisma.post.create({
        data: { title, body, tags, authorId: userId, status: 'DRAFT' },
      });

      callback(null, {
        id: post.id,
        title: post.title,
        body: post.body,
        author_id: post.authorId,
        status: 'POST_STATUS_DRAFT',
        tags: post.tags,
      });
    } catch (error) {
      callback({ code: grpc.status.INTERNAL, message: 'Failed to create post' });
    }
  },
};

2. Server Streaming RPC

Server streaming is perfect for: live feeds, long-running queries, real-time notifications.

// Server: stream posts matching a filter
blogService.WatchPosts = (call: grpc.ServerWritableStream<any, any>) => {
  const { status_filter } = call.request;
  console.log(`Client watching posts with status: ${status_filter}`);

  // Simulate streaming new posts
  const interval = setInterval(async () => {
    try {
      const recentPosts = await prisma.post.findMany({
        where: {
          status: status_filter?.replace('POST_STATUS_', '') || undefined,
          createdAt: { gte: new Date(Date.now() - 5000) }, // last 5 seconds
        },
        orderBy: { createdAt: 'desc' },
        take: 10,
      });

      for (const post of recentPosts) {
        call.write({
          id: post.id,
          title: post.title,
          body: post.body,
          status: `POST_STATUS_${post.status}`,
        });
      }
    } catch (error) {
      console.error('Stream error:', error);
    }
  }, 5000);

  // Clean up when client disconnects
  call.on('cancelled', () => {
    console.log('Client cancelled stream');
    clearInterval(interval);
  });

  // Or end the stream after some condition
  // call.end();
};

3. Client Streaming RPC

Client streaming is ideal for: batch uploads, file uploads, aggregating data.

// Server: receive stream of posts and return summary
blogService.BatchCreatePosts = (
  call: grpc.ServerReadableStream<any, any>,
  callback: grpc.sendUnaryData<any>
) => {
  const posts: any[] = [];

  call.on('data', (request) => {
    console.log(`Received post: ${request.title}`);
    posts.push(request);
  });

  call.on('end', async () => {
    try {
      const created = await prisma.post.createMany({
        data: posts.map(p => ({
          title: p.title,
          body: p.body,
          tags: p.tags || [],
          authorId: call.metadata.get('user-id')[0] as string,
          status: 'DRAFT',
        })),
      });

      callback(null, {
        posts: [], // could return created posts
        total_count: created.count,
      });
    } catch (error) {
      callback({ code: grpc.status.INTERNAL, message: 'Batch create failed' });
    }
  });

  call.on('error', (error) => {
    console.error('Client stream error:', error);
  });
};

4. Bidirectional Streaming RPC

Bidirectional streaming enables real-time, two-way communication:

// Server: chat service
blogService.Chat = (call: grpc.ServerDuplexStream<any, any>) => {
  console.log('Bidirectional chat stream opened');

  call.on('data', (message) => {
    console.log(`[${message.sender_id}]: ${message.text}`);

    // Echo back or broadcast to other streams
    call.write({
      sender_id: 'server',
      text: `Received: ${message.text}`,
      timestamp: { seconds: Math.floor(Date.now() / 1000) },
    });
  });

  call.on('end', () => {
    console.log('Client ended stream');
    call.end();
  });

  call.on('error', (error) => {
    console.error('Stream error:', error);
  });
};

Starting the Server

function startServer() {
  const server = new grpc.Server();

  server.addService(
    blogProto.myapp.blog.BlogService.service,
    blogService
  );

  server.bindAsync(
    '0.0.0.0:50051',
    grpc.ServerCredentials.createInsecure(),
    (error, port) => {
      if (error) {
        console.error('Server failed to start:', error);
        return;
      }
      console.log(`gRPC server running on port ${port}`);
    }
  );
}

startServer();

Client Implementation

import * as grpc from '@grpc/grpc-js';
import * as protoLoader from '@grpc/proto-loader';

const packageDefinition = protoLoader.loadSync('./protos/blog.proto', {
  keepCase: true, longs: String, enums: String, defaults: true, oneofs: true,
});
const blogProto = grpc.loadPackageDefinition(packageDefinition) as any;

const client = new blogProto.myapp.blog.BlogService(
  'localhost:50051',
  grpc.credentials.createInsecure()
);

// Unary call
client.GetPost({ id: '123' }, (error, response) => {
  if (error) {
    console.error('Error:', error.message);
    return;
  }
  console.log('Post:', response);
});

// Promise wrapper for unary calls
function getPost(id: string): Promise<any> {
  return new Promise((resolve, reject) => {
    client.GetPost({ id }, (error, response) => {
      if (error) reject(error);
      else resolve(response);
    });
  });
}

// Server streaming client
const stream = client.WatchPosts({ status_filter: 'POST_STATUS_PUBLISHED' });
stream.on('data', (post) => console.log('New post:', post.title));
stream.on('end', () => console.log('Stream ended'));
stream.on('error', (err) => console.error('Stream error:', err));

// Client streaming
const batchStream = client.BatchCreatePosts((error, response) => {
  if (error) console.error(error);
  else console.log(`Created ${response.total_count} posts`);
});
batchStream.write({ title: 'Post 1', body: 'Content 1' });
batchStream.write({ title: 'Post 2', body: 'Content 2' });
batchStream.end();

// Bidirectional streaming
const chatStream = client.Chat();
chatStream.on('data', (msg) => console.log(`Server: ${msg.text}`));
chatStream.write({ sender_id: 'user1', text: 'Hello!' });
chatStream.write({ sender_id: 'user1', text: 'How are you?' });
setTimeout(() => chatStream.end(), 5000);

Exercises

  1. Build a Full Unary gRPC Service

    Implement a UserService with gRPC unary RPCs: GetUser, CreateUser, UpdateUser, DeleteUser, ListUsers. Use @grpc/grpc-js with proto-loader. Include metadata-based authentication. Write both server and client code.

    Hint

    Define request/response messages in a .proto file. Load with protoLoader.loadSync. Implement each RPC as a method in the service object.

  2. Implement Server Streaming for Live Updates

    Create a gRPC server streaming RPC that watches a database table for changes and pushes new/updated records to connected clients. Simulate changes with setInterval. Handle client cancellation gracefully.

    Hint

    Use call.write() to send each update and call.on('cancelled') to detect disconnection. Clean up intervals on cancel.

  3. Build a Chat Service with Bidirectional Streaming

    Create a Chat gRPC service where multiple clients can send and receive messages in real-time. Maintain a list of active streams and broadcast messages to all connected clients.

    Hint

    Store active ServerDuplexStream instances in an array. When one client sends a message, iterate over all streams and write the message to each.

Common Problems & Solutions

1 gRPC server silently does nothing when started
Cause: server.bindAsync is called but the callback error is not checked, or the port is already in use.
Solution: Always check the error parameter in the bindAsync callback. Log the port on success. Check for EADDRINUSE errors.
2 Client receives UNAVAILABLE error
Cause: The server is not running, the port is wrong, or there is a network issue.
Solution: Verify the server is running on the expected port. Use grpc.credentials.createInsecure() for development. Check firewalls and DNS resolution.
3 Streaming calls hang indefinitely
Cause: The server never calls call.end() for server streams, or the client never calls stream.end() for client streams.
Solution: Always call end() when streaming is complete. Set deadlines on the client side. Use call.on('cancelled') on the server to detect and clean up abandoned streams.
