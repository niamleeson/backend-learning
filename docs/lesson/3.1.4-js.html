Express Router: Modular Route Organization

Intuition

Express Router is like a phone extension system. The main switchboard (app) routes calls to departments (routers), and each department has its own internal directory. A caller dials the main number plus an extension, and the switchboard routes them to the right department where local rules apply.

Express Router: Modular Route Organization

Introduction

As your Express application grows, putting all routes in a single file becomes unmaintainable. Express Router is a mini-application that provides isolated route handling. Each Router instance is a complete middleware and routing system, which lets you break your application into modular, self-contained pieces.

Creating a Router

import { Router, Request, Response } from 'express';

const router = Router();

router.get('/', (req: Request, res: Response) => {
  res.json({ users: [] });
});

router.get('/:id', (req: Request, res: Response) => {
  res.json({ user: { id: req.params.id } });
});

router.post('/', (req: Request, res: Response) => {
  res.status(201).json({ user: req.body });
});

export default router;

Mounting Routers

In your main app.ts, mount each router at a base path:

import express from 'express';
import userRouter from './modules/users/users.routes';
import postRouter from './modules/posts/posts.routes';
import commentRouter from './modules/comments/comments.routes';

const app = express();
app.use(express.json());

// Mount routers
app.use('/api/users', userRouter);
app.use('/api/posts', postRouter);
app.use('/api/comments', commentRouter);

export default app;

Now routes defined in the user router as / and /:id respond to /api/users and /api/users/:id respectively.

Route Registration:

app.use('/api/users', userRouter)
                          |
                          +-- GET /          -> GET /api/users
                          +-- GET /:id       -> GET /api/users/:id
                          +-- POST /         -> POST /api/users
                          +-- PUT /:id       -> PUT /api/users/:id
                          +-- DELETE /:id    -> DELETE /api/users/:id

Router-Level Middleware

Routers can have their own middleware that only applies to routes within that router:

import { Router, Request, Response, NextFunction } from 'express';

const router = Router();

// This middleware runs for ALL routes in this router
router.use((req: Request, res: Response, next: NextFunction) => {
  console.log(`[Users] ${req.method} ${req.path} at ${new Date().toISOString()}`);
  next();
});

// Authentication middleware for this router only
const requireAuth = (req: Request, res: Response, next: NextFunction) => {
  if (!req.headers.authorization) {
    return res.status(401).json({ error: 'Authentication required' });
  }
  next();
};

// Public routes
router.get('/', (req, res) => {
  res.json({ users: [] });
});

// Protected routes - apply auth middleware
router.use(requireAuth); // Everything below requires auth

router.post('/', (req, res) => {
  res.status(201).json({ user: req.body });
});

router.delete('/:id', (req, res) => {
  res.status(204).send();
});

export default router;

Nested Routers

Routers can be nested within routers for hierarchical resource structures:

// src/modules/users/users.routes.ts
import { Router } from 'express';
import userPostRouter from './user-posts.routes';

const router = Router();

router.get('/', (req, res) => {
  res.json({ users: [] });
});

router.get('/:userId', (req, res) => {
  res.json({ user: { id: req.params.userId } });
});

// Nest the posts router under /users/:userId/posts
router.use('/:userId/posts', userPostRouter);

export default router;

// src/modules/users/user-posts.routes.ts
import { Router } from 'express';

// mergeParams: true allows access to parent router's params
const router = Router({ mergeParams: true });

router.get('/', (req, res) => {
  const { userId } = req.params; // Available because of mergeParams
  res.json({ posts: [], userId });
});

router.get('/:postId', (req, res) => {
  const { userId, postId } = req.params;
  res.json({ post: { userId, postId } });
});

router.post('/', (req, res) => {
  const { userId } = req.params;
  res.status(201).json({ post: { ...req.body, userId } });
});

export default router;

The key is Router({ mergeParams: true }). Without this, the child router cannot access :userId from the parent.

URL Resolution with Nested Routers:

app.use('/api/users', userRouter)
                          |
                          +-- GET /                    -> GET /api/users
                          +-- GET /:userId             -> GET /api/users/42
                          |
                          +-- use('/:userId/posts', userPostRouter)
                                    |
                                    +-- GET /          -> GET /api/users/42/posts
                                    +-- GET /:postId   -> GET /api/users/42/posts/7
                                    +-- POST /         -> POST /api/users/42/posts

Controller Pattern with Router

A clean pattern is to separate route definitions from handler logic:

// src/modules/users/users.controller.ts
import { Request, Response, NextFunction } from 'express';

export class UsersController {
  async getAll(req: Request, res: Response, next: NextFunction) {
    try {
      const users = [{ id: 1, name: 'Alice' }, { id: 2, name: 'Bob' }];
      res.json({ users });
    } catch (error) {
      next(error);
    }
  }

  async getById(req: Request, res: Response, next: NextFunction) {
    try {
      const { id } = req.params;
      const user = { id: parseInt(id, 10), name: 'Alice' };
      if (!user) {
        return res.status(404).json({ error: 'User not found' });
      }
      res.json({ user });
    } catch (error) {
      next(error);
    }
  }

  async create(req: Request, res: Response, next: NextFunction) {
    try {
      const { name, email } = req.body;
      const user = { id: Date.now(), name, email };
      res.status(201).json({ user });
    } catch (error) {
      next(error);
    }
  }

  async update(req: Request, res: Response, next: NextFunction) {
    try {
      const { id } = req.params;
      const updates = req.body;
      res.json({ user: { id: parseInt(id, 10), ...updates } });
    } catch (error) {
      next(error);
    }
  }

  async delete(req: Request, res: Response, next: NextFunction) {
    try {
      res.status(204).send();
    } catch (error) {
      next(error);
    }
  }
}

// src/modules/users/users.routes.ts
import { Router } from 'express';
import { UsersController } from './users.controller';

const router = Router();
const controller = new UsersController();

router.get('/', controller.getAll.bind(controller));
router.get('/:id', controller.getById.bind(controller));
router.post('/', controller.create.bind(controller));
router.put('/:id', controller.update.bind(controller));
router.delete('/:id', controller.delete.bind(controller));

export default router;

Important: When using class methods as route handlers, you must use .bind(controller) or arrow functions to preserve the correct this context.

Route Prefix Patterns

API Versioning with Routers

import { Router } from 'express';
import v1Routes from './v1/routes';
import v2Routes from './v2/routes';

const app = express();

const apiRouter = Router();
apiRouter.use('/v1', v1Routes);
apiRouter.use('/v2', v2Routes);

app.use('/api', apiRouter);

// Results in: /api/v1/users, /api/v2/users, etc.

Dynamic Route Registration

// src/routes/index.ts
import { Router } from 'express';
import userRoutes from '../modules/users/users.routes';
import postRoutes from '../modules/posts/posts.routes';
import authRoutes from '../modules/auth/auth.routes';

const router = Router();

const moduleRoutes = [
  { path: '/users', route: userRoutes },
  { path: '/posts', route: postRoutes },
  { path: '/auth', route: authRoutes },
];

moduleRoutes.forEach(({ path, route }) => {
  router.use(path, route);
});

export default router;

// app.ts
import apiRoutes from './routes';
app.use('/api', apiRoutes);

router.param() Middleware

Execute middleware when a specific parameter is present:

const router = Router();

// Runs whenever :userId appears in a route
router.param('userId', async (req, res, next, userId) => {
  try {
    const id = parseInt(userId, 10);
    if (isNaN(id)) {
      return res.status(400).json({ error: 'Invalid user ID' });
    }
    // Attach the loaded user to the request
    req.user = { id, name: 'Alice', email: 'alice@example.com' };
    next();
  } catch (error) {
    next(error);
  }
});

// req.user is available in all routes with :userId
router.get('/:userId', (req, res) => {
  res.json({ user: req.user });
});

router.put('/:userId', (req, res) => {
  res.json({ user: { ...req.user, ...req.body } });
});

Common Pitfalls

Forgetting mergeParams

// WRONG: child router cannot access parent params
const childRouter = Router();
childRouter.get('/', (req, res) => {
  req.params.userId; // undefined!
});

// CORRECT: enable mergeParams
const childRouter = Router({ mergeParams: true });
childRouter.get('/', (req, res) => {
  req.params.userId; // "42"
});

Losing 'this' Context with Class Methods

// WRONG: 'this' is undefined inside the method
router.get('/', controller.getAll);

// CORRECT: bind the context
router.get('/', controller.getAll.bind(controller));

// ALSO CORRECT: use arrow function wrapper
router.get('/', (req, res, next) => controller.getAll(req, res, next));

Ordering Middleware in Routers

// WRONG: auth middleware runs for ALL routes
router.use(requireAuth);
router.get('/public', publicHandler);   // Also requires auth!
router.get('/private', privateHandler);

// CORRECT: apply selectively
router.get('/public', publicHandler);
router.use(requireAuth); // Only routes below require auth
router.get('/private', privateHandler);

Summary

Express Router is the building block for modular applications. It lets you group related routes, apply route-specific middleware, nest resources hierarchically, and keep each module self-contained. Use mergeParams for nested routers, the controller pattern for separation of concerns, and careful middleware ordering to build clean, maintainable route hierarchies.

Exercises

  1. Modularize a Monolithic App

    Take an Express app with all routes in a single file (users CRUD, posts CRUD, comments CRUD) and refactor it into three separate router modules. Each module should have its own routes file and controller. Mount them all under /api in app.ts.

    Hint

    Create a Router() in each module's routes file, define routes relative to the mount point (use '/' not '/api/users'), and export the router.

  2. Nested Resources with mergeParams

    Build nested routes: /api/courses/:courseId/lessons/:lessonId/comments. Create three routers with proper nesting. The comments router should have access to both courseId and lessonId from parent routers.

    Hint

    Use Router({ mergeParams: true }) for child routers. Mount with parent.use('/:courseId/lessons', lessonRouter) and lesson.use('/:lessonId/comments', commentRouter).

  3. API Version Router

    Create a versioned API structure where /api/v1/users returns { version: 1, users: [...] } and /api/v2/users returns { version: 2, users: [...], total: N }. Use nested routers to organize the versions.

    Hint

    Create separate routers for v1 and v2. Mount them under a parent API router: apiRouter.use('/v1', v1Router); apiRouter.use('/v2', v2Router); app.use('/api', apiRouter).

Common Problems & Solutions

1 Child router cannot access parent route parameters (req.params.userId is undefined)
Cause: By default, Express routers have their own params scope. Child routers do not inherit params from parent routers.
Solution: Create the child router with Router({ mergeParams: true }). This merges parent params into the child's req.params.
2 Class controller method throws 'Cannot read property of undefined' for this.service
Cause: When passing a class method as a route handler (router.get('/', controller.method)), the 'this' context is lost.
Solution: Use .bind(): router.get('/', controller.method.bind(controller)). Or wrap in an arrow function: (req, res, next) => controller.method(req, res, next).
3 Router middleware runs on routes it should not affect
Cause: router.use(middleware) applies to all routes registered AFTER it in the same router. If registered too early, it affects public routes too.
Solution: Register public routes before the middleware, or apply middleware only to specific routes: router.get('/private', authMiddleware, handler).
