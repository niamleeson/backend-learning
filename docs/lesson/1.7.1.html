Reading and Writing Text Files

Intuition

File I/O in Python is a stream: open returns a file object you read from or write to, and the with statement guarantees the stream is closed when you're done.

Reading and writing text files in Python is straightforward with the built-in open() function. Understanding encoding and mode flags ensures correct behavior across platforms.

# Writing text
with open("output.txt", "w", encoding="utf-8") as f:
    f.write("Hello, World!\n")
    f.writelines(["Line 2\n", "Line 3\n"])

# Reading text
with open("output.txt", "r", encoding="utf-8") as f:
    content = f.read()          # entire file as string

with open("output.txt", encoding="utf-8") as f:
    for line in f:              # memory-efficient line iteration
        print(line.strip())

# Appending
with open("output.txt", "a", encoding="utf-8") as f:
    f.write("Line 4\n")
  • Always specify encoding="utf-8" explicitly — the default varies by platform.
  • Use with to ensure files are closed even if an exception occurs.
  • Iterate over the file object directly for memory-efficient line-by-line reading.
  • Mode flags: r (read), w (write/truncate), a (append), x (exclusive create).

Exercises

  1. Line Counter

    Write a function count_lines(filepath) that returns the number of non-empty lines in a text file.

    Hint

    Iterate line by line and count lines where line.strip() is truthy.

  2. Search and Replace in File

    Write a function replace_in_file(filepath, old, new) that reads a file, replaces all occurrences of old with new, and writes the result back.

    Hint

    Read the whole file, use str.replace(), then write it back in 'w' mode.

Common Problems & Solutions

1 `UnicodeDecodeError` when reading a file.
Cause: The file's encoding doesn't match the encoding argument (or the platform default).
Solution: Specify the correct encoding explicitly, e.g., `encoding='utf-8'` or `encoding='latin-1'`.
2 Opening a file in `'w'` mode erases its contents immediately.
Cause: `'w'` mode truncates the file to zero length on open.
Solution: Use `'a'` mode to append, or `'r+'` to read and write without truncation.
