*args and **kwargs

Intuition

Think of *args as a catch-all basket for unnamed extras and **kwargs as a catch-all basket for named extras. They let a function accept anything without knowing the specifics in advance.

*args collects extra positional arguments into a tuple, and **kwargs collects extra keyword arguments into a dictionary. Together they allow fully flexible function signatures.

def log(message, *args, **kwargs):
    """Log with optional positional context and keyword metadata."""
    print(f"[LOG] {message}")
    if args:
        print(f"  Context: {args}")
    if kwargs:
        print(f"  Metadata: {kwargs}")

log("User login", "web", "v2", user_id=42, ip="10.0.0.1")
# [LOG] User login
#   Context: ('web', 'v2')
#   Metadata: {'user_id': 42, 'ip': '10.0.0.1'}

# Unpacking in function calls
def add(a, b, c):
    return a + b + c

nums = [1, 2, 3]
print(add(*nums))           # 6

config = {"a": 10, "b": 20, "c": 30}
print(add(**config))        # 60
  • *args is a tuple, **kwargs is a dict; the names are convention, not required
  • Order in signature: regular, *args, keyword-only, **kwargs
  • Use * and ** to unpack sequences and dicts into function calls
  • Common in decorators and wrapper functions for forwarding arguments

Exercises

  1. Flexible Sum

    Write a function total(*args) that sums any number of numeric arguments. Handle the case of zero arguments by returning 0.

    Hint

    Use sum(args) inside the function.

  2. Config Merger

    Write merge_configs(**kwargs) that accepts any number of keyword arguments and returns them as a dictionary. Then call it by unpacking two existing dicts.

    Hint

    Use {**dict1, **dict2} or pass both with merge_configs(**d1, **d2) if keys don't overlap.

Common Problems & Solutions

1 TypeError: func() got an unexpected keyword argument
Cause: Passing keyword arguments to a function that doesn't accept `**kwargs` or that specific parameter.
Solution: Add `**kwargs` to accept arbitrary keywords, or check the function signature.
2 Modifying kwargs dict affects caller's dictionary
Cause: When unpacking with `**`, the dict values are shared references, not deep copies.
Solution: Make a copy inside the function if you need to mutate: `opts = {**kwargs}`.
