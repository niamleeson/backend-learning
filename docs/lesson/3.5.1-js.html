Integrating Prisma with Express

Intuition

Prisma with Express is like having a perfectly organized filing system. Prisma is the filing clerk who knows exactly where every document is (type-safe queries), ensures documents follow the rules (schema validation), and handles complex filing tasks atomically (transactions). Express is the front desk that takes requests from visitors.

Integrating Prisma with Express

Introduction

Prisma is a modern ORM for Node.js and TypeScript that provides type-safe database access, automatic migrations, and an intuitive data modeling language. Integrating Prisma with Express gives you a robust, type-safe data layer for your API. This lesson covers setting up Prisma in an Express project, structuring the integration, and implementing common data access patterns.

Setup

npm install @prisma/client
npm install -D prisma
npx prisma init

This creates a prisma/ directory with a schema.prisma file and a .env file.

Define Your Schema

// prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id           Int       @id @default(autoincrement())
  email        String    @unique
  name         String
  passwordHash String    @map("password_hash")
  role         Role      @default(USER)
  posts        Post[]
  createdAt    DateTime  @default(now()) @map("created_at")
  updatedAt    DateTime  @updatedAt @map("updated_at")

  @@map("users")
}

model Post {
  id          Int       @id @default(autoincrement())
  title       String
  content     String
  published   Boolean   @default(false)
  author      User      @relation(fields: [authorId], references: [id])
  authorId    Int       @map("author_id")
  tags        Tag[]
  createdAt   DateTime  @default(now()) @map("created_at")
  updatedAt   DateTime  @updatedAt @map("updated_at")

  @@map("posts")
}

model Tag {
  id    Int    @id @default(autoincrement())
  name  String @unique
  posts Post[]

  @@map("tags")
}

enum Role {
  USER
  ADMIN
  EDITOR
}

Prisma Client Singleton

Create a single Prisma Client instance shared across your application:

// src/config/database.ts
import { PrismaClient } from '@prisma/client';

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined;
};

export const prisma = globalForPrisma.prisma ?? new PrismaClient({
  log: process.env.NODE_ENV === 'development'
    ? ['query', 'info', 'warn', 'error']
    : ['error'],
});

if (process.env.NODE_ENV !== 'production') {
  globalForPrisma.prisma = prisma;
}

The global assignment prevents creating multiple Prisma Client instances during hot reloading in development.

Service Layer with Prisma

// src/modules/users/users.service.ts
import { prisma } from '../../config/database';
import { Prisma, User } from '@prisma/client';
import bcrypt from 'bcrypt';

// Type for user without sensitive fields
type SafeUser = Omit<User, 'passwordHash'>;

const safeUserSelect = {
  id: true,
  email: true,
  name: true,
  role: true,
  createdAt: true,
  updatedAt: true,
} satisfies Prisma.UserSelect;

export const userService = {
  async findAll(params: {
    page: number;
    limit: number;
    search?: string;
    role?: string;
  }): Promise<{ users: SafeUser[]; total: number }> {
    const { page, limit, search, role } = params;
    const where: Prisma.UserWhereInput = {
      ...(search && {
        OR: [
          { name: { contains: search, mode: 'insensitive' } },
          { email: { contains: search, mode: 'insensitive' } },
        ],
      }),
      ...(role && { role: role as any }),
    };

    const [users, total] = await Promise.all([
      prisma.user.findMany({
        where,
        select: safeUserSelect,
        skip: (page - 1) * limit,
        take: limit,
        orderBy: { createdAt: 'desc' },
      }),
      prisma.user.count({ where }),
    ]);

    return { users, total };
  },

  async findById(id: number): Promise<SafeUser | null> {
    return prisma.user.findUnique({
      where: { id },
      select: safeUserSelect,
    });
  },

  async findByEmail(email: string): Promise<User | null> {
    return prisma.user.findUnique({ where: { email } });
  },

  async create(data: { email: string; name: string; password: string }): Promise<SafeUser> {
    const passwordHash = await bcrypt.hash(data.password, 12);
    return prisma.user.create({
      data: {
        email: data.email,
        name: data.name,
        passwordHash,
      },
      select: safeUserSelect,
    });
  },

  async update(id: number, data: Prisma.UserUpdateInput): Promise<SafeUser> {
    return prisma.user.update({
      where: { id },
      data,
      select: safeUserSelect,
    });
  },

  async delete(id: number): Promise<void> {
    await prisma.user.delete({ where: { id } });
  },
};

Controller Layer

// src/modules/users/users.controller.ts
import { Request, Response, NextFunction } from 'express';
import { userService } from './users.service';
import { NotFoundError } from '../../utils/ApiError';

export const usersController = {
  async getAll(req: Request, res: Response, next: NextFunction) {
    try {
      const page = parseInt(req.query.page as string) || 1;
      const limit = Math.min(parseInt(req.query.limit as string) || 20, 100);
      const { users, total } = await userService.findAll({
        page, limit,
        search: req.query.search as string,
        role: req.query.role as string,
      });

      res.json({
        success: true,
        data: users,
        meta: { total, page, limit, totalPages: Math.ceil(total / limit) },
      });
    } catch (error) {
      next(error);
    }
  },

  async getById(req: Request, res: Response, next: NextFunction) {
    try {
      const id = parseInt(req.params.id, 10);
      const user = await userService.findById(id);
      if (!user) throw new NotFoundError('User', id);
      res.json({ success: true, data: user });
    } catch (error) {
      next(error);
    }
  },

  async create(req: Request, res: Response, next: NextFunction) {
    try {
      const user = await userService.create(req.body);
      res.status(201).json({ success: true, data: user });
    } catch (error) {
      if ((error as any).code === 'P2002') {
        return res.status(409).json({ success: false, error: { message: 'Email already exists' } });
      }
      next(error);
    }
  },

  async update(req: Request, res: Response, next: NextFunction) {
    try {
      const id = parseInt(req.params.id, 10);
      const user = await userService.update(id, req.body);
      res.json({ success: true, data: user });
    } catch (error) {
      if ((error as any).code === 'P2025') {
        return next(new NotFoundError('User', req.params.id));
      }
      next(error);
    }
  },

  async remove(req: Request, res: Response, next: NextFunction) {
    try {
      const id = parseInt(req.params.id, 10);
      await userService.delete(id);
      res.status(204).send();
    } catch (error) {
      next(error);
    }
  },
};

Transactions

async function createPostWithTags(data: {
  title: string;
  content: string;
  authorId: number;
  tagNames: string[];
}) {
  return prisma.$transaction(async (tx) => {
    // Create or find tags
    const tags = await Promise.all(
      data.tagNames.map(name =>
        tx.tag.upsert({
          where: { name },
          create: { name },
          update: {},
        })
      )
    );

    // Create post with tags
    return tx.post.create({
      data: {
        title: data.title,
        content: data.content,
        authorId: data.authorId,
        tags: { connect: tags.map(t => ({ id: t.id })) },
      },
      include: { tags: true, author: { select: safeUserSelect } },
    });
  });
}

Graceful Shutdown

// src/server.ts
import { prisma } from './config/database';

const server = app.listen(config.port, () => {
  console.log(`Server running on port ${config.port}`);
});

async function gracefulShutdown() {
  console.log('Shutting down...');
  server.close();
  await prisma.$disconnect();
  process.exit(0);
}

process.on('SIGTERM', gracefulShutdown);
process.on('SIGINT', gracefulShutdown);

Error Handling for Prisma Errors

// src/middleware/prismaErrorHandler.ts
import { Prisma } from '@prisma/client';

export function handlePrismaError(error: unknown): never {
  if (error instanceof Prisma.PrismaClientKnownRequestError) {
    switch (error.code) {
      case 'P2002': // Unique constraint violation
        throw new ApiError(409, `Duplicate value for ${(error.meta?.target as string[])?.join(', ')}`);
      case 'P2025': // Record not found
        throw new ApiError(404, 'Record not found');
      case 'P2003': // Foreign key constraint
        throw new ApiError(400, 'Related record not found');
      default:
        throw new ApiError(500, 'Database error');
    }
  }
  throw error;
}

Common Pitfalls

Creating Multiple Prisma Clients

// WRONG: new client on every import
export function getDb() { return new PrismaClient(); }

// CORRECT: singleton pattern
export const prisma = new PrismaClient();

Not Handling Prisma-Specific Errors

// WRONG: generic 500 for all errors
catch (error) { res.status(500).json({ error: 'Server error' }); }

// CORRECT: translate Prisma errors to appropriate HTTP errors
catch (error) {
  if (error.code === 'P2002') return res.status(409).json({ error: 'Duplicate' });
  if (error.code === 'P2025') return res.status(404).json({ error: 'Not found' });
  next(error);
}

Summary

Integrating Prisma with Express gives you type-safe database access with an intuitive API. Use a singleton Prisma Client, organize code into service and controller layers, handle Prisma-specific errors, use transactions for multi-step operations, and disconnect gracefully on shutdown.

Exercises

  1. CRUD API with Prisma

    Build a complete REST API for a blog with Prisma: users, posts, and tags. Implement all CRUD operations with proper error handling, pagination, and filtering. Use Prisma's relation queries for eager loading.

    Hint

    Use Prisma's include for relation loading. Use skip and take for pagination. Handle P2002 (unique constraint) and P2025 (not found) errors specifically.

  2. Prisma Transactions

    Implement a 'publish post' feature that atomically: validates the post exists, sets published=true, creates a notification record, and updates the author's post count. Use Prisma's interactive transaction.

    Hint

    Use prisma.$transaction(async (tx) => { ... }). All operations inside use the tx client, not the global prisma. If any operation fails, all are rolled back.

  3. Advanced Filtering and Search

    Build a search endpoint that supports: text search across title and content, filtering by multiple tags (AND/OR), date range filters, author filter, and published status. Construct dynamic Prisma where clauses.

    Hint

    Build the where object conditionally using spread: { ...(search && { OR: [...] }), ...(tags && { tags: { some: { name: { in: tags } } } }) }.

Common Problems & Solutions

1 PrismaClientInitializationError: Can't reach database server
Cause: The DATABASE_URL in .env is incorrect, the database server is not running, or network/firewall issues.
Solution: Verify DATABASE_URL format: postgresql://user:password@host:port/dbname. Ensure the database server is running. Check that the database exists: npx prisma db push.
2 Multiple Prisma Client instances warning in development
Cause: Hot reloading creates a new Prisma Client on each reload, exhausting database connections.
Solution: Use the global singleton pattern: store the client on globalThis and reuse it. This prevents connection pool exhaustion during development.
3 Prisma query returns null for relations
Cause: Relations are not included by default in Prisma queries. You must explicitly request them.
Solution: Use include: { posts: true } or select: { posts: { select: { id: true, title: true } } } to load relations.
