Memory Profiling: tracemalloc and memory_profiler

Intuition

Memory profiling is like auditing a warehouse. tracemalloc tells you which department ordered the most storage and how inventory changed over time, revealing leaks before you run out of space.

Memory Profiling: tracemalloc and memory_profiler

Memory profiling helps find memory leaks, excessive allocations, and opportunities to reduce memory usage in Python programs.

import tracemalloc

# Start tracing memory allocations
tracemalloc.start()

def create_large_data():
    data = [dict(id=i, name=f"item_{i}", values=list(range(100)))
            for i in range(10_000)]
    return data

data = create_large_data()

# Take a snapshot and analyze
snapshot = tracemalloc.take_snapshot()
top_stats = snapshot.statistics('lineno')

print("Top 5 memory allocations:")
for stat in top_stats[:5]:
    print(stat)

# Compare two snapshots to find leaks
tracemalloc.start()
snapshot1 = tracemalloc.take_snapshot()

leaky_list = []
for i in range(1000):
    leaky_list.append(" " * 10000)  # Growing allocation

snapshot2 = tracemalloc.take_snapshot()
diffs = snapshot2.compare_to(snapshot1, 'lineno')
for diff in diffs[:5]:
    print(diff)  # Shows size increase per line

# memory_profiler (pip install memory_profiler)
# @profile
# def my_func():
#     ...
# Run: python -m memory_profiler script.py

Key Takeaways: - tracemalloc is built-in and tracks where allocations originate - Compare two snapshots to identify memory growth (leak detection) - memory_profiler shows per-line memory usage (like line_profiler for memory) - Use sys.getsizeof() for quick size checks on individual objects - Watch for hidden references (closures, caches, global lists) that prevent garbage collection

Exercises

  1. Find a Memory Leak

    Write a program with an intentional memory leak (e.g., appending to a global list in a loop). Use tracemalloc to take snapshots before and after, and identify the leaking line.

    Hint

    Use snapshot2.compare_to(snapshot1, 'lineno') to see which lines increased memory. The diff shows size change per allocation site.

  2. Optimize Memory Usage

    Create a class storing 100K records. Profile memory with tracemalloc. Optimize using __slots__, generators, or more efficient data structures. Compare before/after.

    Hint

    Adding __slots__ eliminates __dict__ per instance, saving ~50% memory. Use tracemalloc.get_traced_memory() for peak measurement.

Common Problems & Solutions

1 `tracemalloc` shows large allocations from internal CPython code, not your code.
Cause: tracemalloc tracks all allocations including interpreter internals and imported modules.
Solution: Filter by your source files: `snapshot.filter_traces([tracemalloc.Filter(True, '*/my_module.py')])` before analyzing.
2 Memory usage keeps growing even though objects should be garbage collected.
Cause: Circular references or lingering references in caches, closures, or global variables prevent GC.
Solution: Use `gc.get_referrers(obj)` to find what holds references. Clear caches with `lru_cache.cache_clear()`. Use `weakref` for caches.
