Concurrency Limits and Backpressure Management

Intuition

Concurrency limits are like a restaurant that only seats 50 people at a time. Without limits, 500 people rush in and nobody gets served (system overload). With limits, customers wait in a queue and are seated as tables free up. Backpressure is when the kitchen (consumer) tells the waiter (producer) to slow down taking orders because they're falling behind.

Concurrency Limits and Backpressure Management

Introduction

In Node.js, it is dangerously easy to overwhelm your system by launching too many concurrent operations. Without concurrency limits and backpressure, your server can exhaust memory, database connections, or external API rate limits. This lesson covers techniques for controlling concurrency in async operations, managing backpressure in streams, and protecting your system from cascading overload.

The Problem: Unbounded Concurrency

// DANGEROUS: 10,000 concurrent database queries
const userIds = getThousandsOfIds(); // 10,000 IDs
const users = await Promise.all(
  userIds.map(id => prisma.user.findUnique({ where: { id } }))
);
// Exhausts connection pool, crashes database, OOM

// DANGEROUS: Unlimited concurrent HTTP requests
const urls = getThousandsOfUrls();
const results = await Promise.all(
  urls.map(url => fetch(url))
);
// Exhausts file descriptors, rate limited, timeouts

Concurrency Limiter with p-limit

import pLimit from 'p-limit';

const limit = pLimit(10); // Max 10 concurrent operations

const userIds = getThousandsOfIds();
const users = await Promise.all(
  userIds.map(id => limit(() => prisma.user.findUnique({ where: { id } })))
);
// Only 10 queries run at a time

Custom Concurrency Limiter

class ConcurrencyLimiter {
  constructor(maxConcurrent) {
    this.maxConcurrent = maxConcurrent;
    this.running = 0;
    this.queue = [];
  }

  async execute(fn) {
    if (this.running >= this.maxConcurrent) {
      // Wait in queue
      await new Promise(resolve => this.queue.push(resolve));
    }

    this.running++;
    try {
      return await fn();
    } finally {
      this.running--;
      // Release next in queue
      if (this.queue.length > 0) {
        const next = this.queue.shift();
        next();
      }
    }
  }

  get stats() {
    return {
      running: this.running,
      queued: this.queue.length,
      maxConcurrent: this.maxConcurrent,
    };
  }
}

// Usage
const dbLimiter = new ConcurrencyLimiter(20);   // Max 20 DB queries
const apiLimiter = new ConcurrencyLimiter(5);    // Max 5 external API calls

app.get('/api/users/:id/full-profile', async (req, res) => {
  const [user, orders, analytics] = await Promise.all([
    dbLimiter.execute(() => prisma.user.findUnique({ where: { id: req.params.id } })),
    dbLimiter.execute(() => prisma.order.findMany({ where: { userId: req.params.id } })),
    apiLimiter.execute(() => fetch(`https://analytics.api/user/${req.params.id}`)),
  ]);
  res.json({ user, orders, analytics: await analytics.json() });
});

Batch Processing with Concurrency Control

async function processBatch(items, batchSize, concurrency, processFn) {
  const limit = pLimit(concurrency);
  const results = [];

  for (let i = 0; i < items.length; i += batchSize) {
    const batch = items.slice(i, i + batchSize);
    const batchResults = await Promise.all(
      batch.map(item => limit(() => processFn(item)))
    );
    results.push(...batchResults);

    // Optional: log progress
    console.log(`Processed ${Math.min(i + batchSize, items.length)}/${items.length}`);
  }

  return results;
}

// Process 10,000 users, 100 at a time, 10 concurrent
await processBatch(userIds, 100, 10, async (userId) => {
  const user = await prisma.user.findUnique({ where: { id: userId } });
  await sendEmail(user.email, 'Newsletter');
  return { userId, sent: true };
});

Backpressure in Node.js Streams

Backpressure prevents fast producers from overwhelming slow consumers.

Without backpressure:
Producer (fast) ──> ████████████████ ──> Consumer (slow)
                    Buffer grows unbounded = OOM

With backpressure:
Producer (paused) ──> ████ ──> Consumer (slow)
                    Buffer stays bounded
                    Producer waits for consumer

import { Transform, pipeline } from 'node:stream';
import { promisify } from 'node:util';

const pipelineAsync = promisify(pipeline);

// Transform stream with built-in backpressure
class BatchProcessor extends Transform {
  constructor(processFn, options = {}) {
    super({
      objectMode: true,
      highWaterMark: options.highWaterMark || 16, // Buffer up to 16 objects
    });
    this.processFn = processFn;
    this.concurrency = options.concurrency || 5;
    this.limiter = new ConcurrencyLimiter(this.concurrency);
  }

  async _transform(chunk, encoding, callback) {
    try {
      const result = await this.limiter.execute(() => this.processFn(chunk));
      this.push(result);
      callback();
    } catch (err) {
      callback(err);
    }
  }
}

// Usage: Process database cursor with backpressure
import { Readable } from 'node:stream';

async function processLargeDataset() {
  // Readable stream from database cursor
  const source = new Readable({
    objectMode: true,
    async read(size) {
      const batch = await prisma.order.findMany({
        take: size,
        skip: this._offset || 0,
        cursor: this._cursor ? { id: this._cursor } : undefined,
        orderBy: { id: 'asc' },
      });

      if (batch.length === 0) {
        this.push(null); // End stream
        return;
      }

      this._cursor = batch[batch.length - 1].id;
      this._offset = (this._offset || 0) + batch.length;

      for (const item of batch) {
        const canContinue = this.push(item);
        if (!canContinue) break; // Backpressure! Stop reading
      }
    },
  });

  const processor = new BatchProcessor(
    async (order) => {
      await processOrder(order);
      return { orderId: order.id, processed: true };
    },
    { concurrency: 10, highWaterMark: 32 }
  );

  let processed = 0;
  const sink = new Transform({
    objectMode: true,
    transform(chunk, enc, cb) {
      processed++;
      if (processed % 1000 === 0) {
        console.log(`Processed ${processed} orders`);
      }
      cb();
    },
  });

  await pipelineAsync(source, processor, sink);
  console.log(`Total processed: ${processed}`);
}

Rate Limiting Outbound Requests

class RateLimiter {
  constructor(maxRequests, intervalMs) {
    this.maxRequests = maxRequests;
    this.intervalMs = intervalMs;
    this.tokens = maxRequests;
    this.queue = [];

    // Refill tokens periodically
    setInterval(() => {
      this.tokens = this.maxRequests;
      this._processQueue();
    }, this.intervalMs);
  }

  async acquire() {
    if (this.tokens > 0) {
      this.tokens--;
      return;
    }

    // Wait for token
    return new Promise(resolve => {
      this.queue.push(resolve);
    });
  }

  _processQueue() {
    while (this.queue.length > 0 && this.tokens > 0) {
      this.tokens--;
      const resolve = this.queue.shift();
      resolve();
    }
  }
}

// Usage: max 100 requests per second to external API
const rateLimiter = new RateLimiter(100, 1000);

async function callExternalAPI(data) {
  await rateLimiter.acquire();
  return fetch('https://external-api.com/endpoint', {
    method: 'POST',
    body: JSON.stringify(data),
  });
}

Express Request Queue with Backpressure

import express from 'express';

function createBackpressureMiddleware(maxConcurrent = 100, maxQueue = 500) {
  let active = 0;
  let queued = 0;

  return (req, res, next) => {
    if (active >= maxConcurrent) {
      if (queued >= maxQueue) {
        // Shed load
        return res.status(503).json({
          error: 'Service temporarily overloaded',
          retryAfter: 5,
        });
      }
      queued++;
    }

    active++;

    const cleanup = () => {
      active--;
      if (queued > 0) queued--;
    };

    res.on('finish', cleanup);
    res.on('close', cleanup);

    next();
  };
}

const app = express();
app.use(createBackpressureMiddleware(100, 500));

Monitoring Concurrency

// Middleware to track concurrent requests
let activeRequests = 0;
let maxActiveRequests = 0;

app.use((req, res, next) => {
  activeRequests++;
  maxActiveRequests = Math.max(maxActiveRequests, activeRequests);

  res.on('finish', () => {
    activeRequests--;
  });
  next();
});

function getEventLoopLag() {
  const start = Date.now();
  return new Promise(resolve => {
    setImmediate(() => resolve(Date.now() - start));
  });
}

Common Pitfalls

  1. Promise.all with unbounded arrays: Always limit concurrency with p-limit or batching.
  2. Ignoring stream backpressure: Not checking the return value of push() or write() leads to unbounded buffering.
  3. No request shedding: When overloaded, accept-everything queuing just delays the failure. Return 503 early.
  4. Rate limiting only on inbound: Don't forget to rate-limit outbound calls to external APIs and databases.

Summary

Concurrency control and backpressure are essential for building robust Node.js applications. Use p-limit or custom limiters to bound concurrent operations. Use Node.js streams with highWaterMark for backpressure-aware data processing. Implement request shedding (503) when the system is overloaded. Monitor active requests, queue depth, and event loop lag to detect overload before it becomes an outage.

Common Problems & Solutions

1 Application runs out of memory processing large datasets
Cause: Using Promise.all or Array.map to process millions of records loads everything into memory simultaneously.
Solution: Use streaming with backpressure or batch processing with concurrency limits. Process chunks of 100-1000 records at a time.
2 External API returning 429 (rate limit) errors
Cause: Application sends too many requests too quickly to an external API.
Solution: Implement a rate limiter (token bucket) for outbound requests. Respect Retry-After headers. Use exponential backoff on 429 responses.
3 Event loop lag increasing under load
Cause: Too many concurrent operations competing for the event loop, or large data being serialized/deserialized synchronously.
Solution: Add concurrency limits. Use worker threads for CPU-bound work. Monitor event loop lag and shed load when lag exceeds threshold.
