The Iterator Protocol: __iter__ and __next__

Intuition

The iterator protocol is a simple contract: give me the next item or tell me you're done. This abstraction lets for-loops work with files, generators, ranges, and any custom sequence.

The iterator protocol is the foundation of Python's for loops. Any object that implements __iter__ and __next__ can be iterated over.

class Countdown:
    def __init__(self, start):
        self.current = start

    def __iter__(self):
        return self  # the object is its own iterator

    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        val = self.current
        self.current -= 1
        return val

# Usage
for n in Countdown(5):
    print(n)  # 5, 4, 3, 2, 1

# Manual iteration
it = iter([10, 20, 30])  # calls __iter__
print(next(it))          # 10 — calls __next__
print(next(it))          # 20
print(next(it, "done"))  # 30
print(next(it, "done"))  # "done" (default)
  • __iter__ returns the iterator object; __next__ returns the next value or raises StopIteration.
  • for x in obj is syntactic sugar for calling iter(obj) then repeatedly calling next().
  • An iterable has __iter__; an iterator has both __iter__ and __next__.
  • Use next(it, default) to avoid StopIteration exceptions when you're unsure if items remain.

Common Problems & Solutions

1 Iterating over an iterator a second time yields nothing.
Cause: Iterators are consumed in a single pass; once exhausted, they keep raising `StopIteration`.
Solution: Create a new iterator from the iterable each time, or convert to a list if you need multiple passes.
2 Confusing iterables with iterators: not all iterables are iterators.
Cause: A list is iterable (has `__iter__`) but is not an iterator (no `__next__`).
Solution: Call `iter(obj)` to get an iterator from any iterable.
