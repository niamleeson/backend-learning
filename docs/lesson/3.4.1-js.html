Passport.js: Strategies and Session Management

Intuition

Passport.js is like a security checkpoint at a building. The strategy is the type of ID check (badge, fingerprint, key card). When you pass, you get a visitor badge (session cookie) that lets you move freely through the building without showing ID again — until the badge expires.

Passport.js: Strategies and Session Management

Introduction

Authentication is the process of verifying who a user is. Passport.js is the most popular authentication middleware for Node.js, with over 500 strategies for different authentication methods. It provides a clean, modular architecture that separates authentication logic from your application code. In this lesson you will learn Passport's core concepts, implement local (username/password) authentication, and manage user sessions.

How Passport Works

Passport uses a strategy pattern. Each authentication method (local, JWT, OAuth, etc.) is implemented as a separate strategy:

Client Request
      |
      v
passport.authenticate('strategy-name')
      |
      v
Strategy verifies credentials
      |
      +--> Success: attaches user to req.user, calls next()
      |
      +--> Failure: returns 401 or redirects

Local Strategy (Username/Password)

Configuring the Strategy

// src/config/passport.ts
import passport from 'passport';
import { Strategy as LocalStrategy } from 'passport-local';
import bcrypt from 'bcrypt';
import { userService } from '../modules/users/users.service';

passport.use(new LocalStrategy(
  {
    usernameField: 'email',    // Use email instead of username
    passwordField: 'password',
  },
  async (email, password, done) => {
    try {
      // Find user by email
      const user = await userService.findByEmail(email);
      if (!user) {
        return done(null, false, { message: 'Invalid email or password' });
      }

      // Verify password
      const isMatch = await bcrypt.compare(password, user.passwordHash);
      if (!isMatch) {
        return done(null, false, { message: 'Invalid email or password' });
      }

      // Success
      return done(null, user);
    } catch (error) {
      return done(error);
    }
  }
));

The done() Callback

done(error, user, info)

done(null, user)             // Success: user is authenticated
done(null, false, { message }) // Failure: invalid credentials
done(error)                  // Error: something went wrong

Session Management

Sessions let the server remember authenticated users across requests:

Session Serialization

// Serialize: store user ID in the session
passport.serializeUser((user: any, done) => {
  done(null, user.id);
});

// Deserialize: fetch full user from the ID stored in session
passport.deserializeUser(async (id: number, done) => {
  try {
    const user = await userService.findById(id);
    done(null, user);
  } catch (error) {
    done(error);
  }
});

Session Flow:

Login Request
    |
    v
passport.authenticate('local')
    |
    v
Strategy verifies credentials -> user object
    |
    v
serializeUser(user) -> stores user.id in session
    |
    v
Cookie sent to client (connect.sid)

Subsequent Requests
    |
    v
Cookie sent by client
    |
    v
Session middleware reads session from store
    |
    v
deserializeUser(id) -> fetches full user
    |
    v
req.user is populated

Express Session Configuration

// src/app.ts
import session from 'express-session';
import passport from 'passport';
import './config/passport'; // Import to register strategies

const app = express();

app.use(express.json());

// Session middleware (BEFORE passport)
app.use(session({
  secret: process.env.SESSION_SECRET || 'change-me',
  resave: false,
  saveUninitialized: false,
  cookie: {
    secure: process.env.NODE_ENV === 'production', // HTTPS only in production
    httpOnly: true,         // Not accessible via JavaScript
    maxAge: 24 * 60 * 60 * 1000, // 1 day
    sameSite: 'lax',
  },
  // In production, use a session store (Redis, PostgreSQL)
  // store: new RedisStore({ client: redisClient }),
}));

// Passport middleware (AFTER session)
app.use(passport.initialize());
app.use(passport.session());

Authentication Routes

// src/modules/auth/auth.routes.ts
import { Router, Request, Response, NextFunction } from 'express';
import passport from 'passport';
import bcrypt from 'bcrypt';

const router = Router();

// POST /auth/register
router.post('/register', async (req: Request, res: Response, next: NextFunction) => {
  try {
    const { email, password, name } = req.body;

    // Check if user exists
    const existing = await userService.findByEmail(email);
    if (existing) {
      return res.status(409).json({ error: 'Email already registered' });
    }

    // Hash password
    const passwordHash = await bcrypt.hash(password, 12);

    // Create user
    const user = await userService.create({ email, passwordHash, name });

    // Auto-login after registration
    req.login(user, (err) => {
      if (err) return next(err);
      res.status(201).json({
        data: { id: user.id, email: user.email, name: user.name }
      });
    });
  } catch (error) {
    next(error);
  }
});

// POST /auth/login
router.post('/login', (req: Request, res: Response, next: NextFunction) => {
  passport.authenticate('local', (err: Error, user: any, info: any) => {
    if (err) return next(err);
    if (!user) {
      return res.status(401).json({ error: info?.message || 'Authentication failed' });
    }

    req.login(user, (err) => {
      if (err) return next(err);
      res.json({
        data: { id: user.id, email: user.email, name: user.name }
      });
    });
  })(req, res, next);
});

// POST /auth/logout
router.post('/logout', (req: Request, res: Response, next: NextFunction) => {
  req.logout((err) => {
    if (err) return next(err);
    req.session.destroy((err) => {
      if (err) return next(err);
      res.clearCookie('connect.sid');
      res.json({ message: 'Logged out' });
    });
  });
});

// GET /auth/me - Get current user
router.get('/me', (req: Request, res: Response) => {
  if (!req.isAuthenticated()) {
    return res.status(401).json({ error: 'Not authenticated' });
  }
  res.json({ data: req.user });
});

export default router;

Authentication Middleware

// src/middleware/auth.ts
export function isAuthenticated(req: Request, res: Response, next: NextFunction) {
  if (req.isAuthenticated()) {
    return next();
  }
  res.status(401).json({ error: 'Authentication required' });
}

export function isAdmin(req: Request, res: Response, next: NextFunction) {
  if (req.isAuthenticated() && (req.user as any).role === 'admin') {
    return next();
  }
  res.status(403).json({ error: 'Admin access required' });
}

Session Stores for Production

In-memory sessions are lost on server restart. Use a persistent store:

// Redis session store
import RedisStore from 'connect-redis';
import { createClient } from 'redis';

const redisClient = createClient({ url: process.env.REDIS_URL });
redisClient.connect();

app.use(session({
  store: new RedisStore({ client: redisClient }),
  secret: process.env.SESSION_SECRET!,
  resave: false,
  saveUninitialized: false,
  cookie: { secure: true, httpOnly: true, maxAge: 86400000 }
}));

Security Best Practices

  1. Hash passwords with bcrypt (cost factor 12+)
  2. Use secure cookies in production (secure, httpOnly, sameSite)
  3. Store sessions in Redis or a database, not memory
  4. Rotate session secrets periodically
  5. Implement account lockout after failed attempts
  6. Never reveal whether an email exists (use generic "invalid credentials" message)

Common Pitfalls

Middleware Order

// WRONG: passport before session
app.use(passport.initialize());
app.use(passport.session());
app.use(session({ ... }));  // Too late!

// CORRECT: session before passport
app.use(session({ ... }));
app.use(passport.initialize());
app.use(passport.session());

Forgetting to Call req.login()

// After registration, you must explicitly log in the user
const user = await userService.create(userData);
// user is created but NOT logged in yet!

req.login(user, (err) => {
  // NOW the user is logged in and session is created
  res.json({ data: user });
});

Summary

Passport.js provides a modular, strategy-based approach to authentication. The Local Strategy handles username/password authentication, sessions maintain login state across requests, and serialization/deserialization bridge the gap between sessions and user objects. Always use bcrypt for passwords, secure cookies in production, and a persistent session store.

Exercises

  1. Implement Local Auth with Sessions

    Build a complete auth system with Passport Local Strategy: register, login, logout, and get-current-user endpoints. Hash passwords with bcrypt, manage sessions with express-session, and create isAuthenticated middleware.

    Hint

    Remember the middleware order: express.json() -> session() -> passport.initialize() -> passport.session(). Use bcrypt.hash(password, 12) for registration and bcrypt.compare() for login.

  2. Account Lockout System

    Extend the login system to lock accounts after 5 failed login attempts within 15 minutes. Store attempt counts and lockout timestamps. Return appropriate error messages and remaining lockout time.

    Hint

    Track failed attempts per email in a Map or database. Check lockout status before authenticating. Reset the counter on successful login.

  3. Session Store with Redis

    Replace the default in-memory session store with Redis using connect-redis. Configure proper TTL, handle Redis connection errors gracefully, and verify sessions survive server restarts.

    Hint

    Install connect-redis and redis packages. Create a Redis client, pass it to RedisStore, and use that as the session store. Handle the 'error' event on the Redis client.

Common Problems & Solutions

1 req.user is undefined after successful login
Cause: Session middleware is not configured, or passport.session() is not registered, or serializeUser/deserializeUser are not defined.
Solution: Ensure session middleware is registered before passport middleware. Implement both serializeUser and deserializeUser. Check that the session cookie is being set (look for connect.sid in browser cookies).
2 Sessions are lost on server restart
Cause: Using the default in-memory session store, which loses all data when the process exits.
Solution: Use a persistent session store like connect-redis, connect-pg-simple, or connect-mongo for production environments.
3 passport.authenticate callback not firing
Cause: Missing (req, res, next) call at the end of the custom callback pattern.
Solution: When using the custom callback pattern, passport.authenticate returns a middleware function that must be invoked: passport.authenticate('local', callback)(req, res, next).
