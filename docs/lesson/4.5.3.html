Flask-Login and OAuth2 Basics

Intuition

Flask-Login manages the 'who is logged in' question for server-rendered apps using sessions, while OAuth2 lets you outsource the 'prove who you are' question to providers like Google. Together they cover both traditional and social login flows.

Flask-Login and OAuth2 Basics

Flask-Login manages user sessions for traditional web applications with login/logout flows. It handles remembering users, protecting views, and managing the current user. OAuth2 extends authentication by allowing users to log in with external providers like Google or GitHub.

from flask import Flask, redirect, url_for, request, jsonify
from flask_login import (
    LoginManager, UserMixin, login_user, logout_user,
    login_required, current_user
)

app = Flask(__name__)
app.config['SECRET_KEY'] = 'dev-secret'

login_manager = LoginManager(app)
login_manager.login_view = 'login'

class User(UserMixin):
    def __init__(self, id, username, email):
        self.id = id
        self.username = username
        self.email = email

# Simulated user store
USERS = {1: User(1, 'alice', 'alice@example.com')}

@login_manager.user_loader
def load_user(user_id):
    return USERS.get(int(user_id))

@app.route('/login', methods=['POST'])
def login():
    data = request.get_json()
    # In production: query DB, check hashed password
    user = USERS.get(1)
    if user and data.get('username') == user.username:
        login_user(user, remember=data.get('remember', False))
        return jsonify({'message': f'Welcome, {user.username}'})
    return jsonify({'error': 'Invalid credentials'}), 401

@app.route('/dashboard')
@login_required
def dashboard():
    return jsonify({
        'user': current_user.username,
        'email': current_user.email
    })

@app.route('/logout', methods=['POST'])
@login_required
def logout():
    logout_user()
    return jsonify({'message': 'Logged out'})

Key Takeaways: - UserMixin provides default implementations for is_authenticated, is_active, is_anonymous, and get_id(). - The user_loader callback is called on every request to load the user from the session. - OAuth2 delegates authentication to trusted providers; Authlib or Flask-Dance simplify the integration.

Exercises

  1. Remember Me Functionality

    Implement login with a 'remember me' option using Flask-Login. Configure the remember cookie duration and verify that closing the browser does not log out remembered users.

    Hint

    Pass remember=True to login_user() and set REMEMBER_COOKIE_DURATION in app.config to control the cookie lifetime.

  2. GitHub OAuth Login

    Set up GitHub OAuth2 login using Authlib or Flask-Dance. Register an OAuth app on GitHub, implement the authorize and callback routes, and create a local user on first login.

    Hint

    Use Flask-Dance's make_github_blueprint() for the simplest setup. The callback gives you the GitHub user profile from which you create a local account.

Common Problems & Solutions

1 current_user is always AnonymousUserMixin even after calling login_user().
Cause: The user_loader callback is not returning the correct user, or SECRET_KEY is not set so the session is not persisting.
Solution: Verify that user_loader returns a User object for valid IDs and that SECRET_KEY is configured. Check that the user ID type matches between login and the loader.
2 OAuth callback returns an error: redirect_uri_mismatch.
Cause: The redirect URI in your code does not exactly match the one registered with the OAuth provider.
Solution: Ensure the redirect URI in your OAuth provider settings matches url_for('callback', _external=True) exactly, including http vs https and trailing slashes.
