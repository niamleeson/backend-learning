Routing, URL Building, and HTTP Methods

Intuition

Routing is the front door of your application -- it decides which function handles each incoming request based on the URL pattern and HTTP method. url_for() is the reverse lookup that keeps your internal links consistent even when URLs change.

Routing, URL Building, and HTTP Methods

Flask's routing system maps URLs to Python functions using decorators. Each route can accept specific HTTP methods and dynamic URL segments. The url_for() function generates URLs by endpoint name, keeping your code resilient to URL changes.

from flask import Flask, url_for, redirect, request, jsonify

app = Flask(__name__)

@app.route('/users', methods=['GET', 'POST'])
def users():
    if request.method == 'POST':
        data = request.get_json()
        return jsonify({'created': data['name']}), 201
    return jsonify({'users': ['alice', 'bob']})

@app.route('/users/<int:user_id>', methods=['GET', 'PUT', 'DELETE'])
def user_detail(user_id):
    if request.method == 'DELETE':
        return '', 204
    return jsonify({'id': user_id, 'name': 'alice'})

@app.route('/go-home')
def go_home():
    return redirect(url_for('users'))

# Converters: string (default), int, float, path, uuid
@app.route('/files/<path:filepath>')
def serve_file(filepath):
    return jsonify({'file': filepath})

Key Takeaways: - Use methods to restrict which HTTP verbs a route responds to. - URL converters like <int:user_id> enforce type checking on path segments. - url_for('endpoint_name') builds URLs dynamically, so renaming paths won't break links.

Exercises

  1. RESTful Route Design

    Create routes for a 'books' resource: list all books (GET /books), create a book (POST /books), get one book (GET /books/), update (PUT /books/), and delete (DELETE /books/).

    Hint

    Use a dictionary as an in-memory store and request.get_json() to parse incoming JSON payloads.

  2. URL Building in Templates

    Create a small Flask app with three routes and a Jinja2 template that uses url_for() to generate navigation links to each route.

    Hint

    In Jinja2 templates, use {{ url_for('endpoint') }} inside href attributes.

Common Problems & Solutions

1 405 Method Not Allowed when sending a POST request.
Cause: The route decorator does not include 'POST' in the methods list; Flask defaults to GET only.
Solution: Add methods=['GET', 'POST'] to the @app.route() decorator for that endpoint.
2 BuildError: Could not build url for endpoint 'xyz'.
Cause: The endpoint name passed to url_for() does not match any registered route function name.
Solution: Check the function name (not the URL path) and if using blueprints, prefix with 'blueprint_name.function_name'.
