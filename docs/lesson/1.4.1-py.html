Lists: Operations, Slicing, and Performance

Intuition

A Python list is a dynamic array of pointers. Appending is cheap, but inserting or deleting near the front is expensive because elements must be shifted.

Lists are Python's most versatile sequence type, supporting mutation, slicing, and a rich set of operations. Understanding their underlying array-based implementation is key to writing performant code.

nums = [1, 2, 3, 4, 5, 6, 7, 8]

# Slicing: start:stop:step
print(nums[2:6])      # [3, 4, 5, 6]
print(nums[::2])      # [1, 3, 5, 7]
print(nums[::-1])     # [8, 7, 6, 5, 4, 3, 2, 1]

# In-place mutation
nums[1:3] = [20, 30]  # [1, 20, 30, 4, 5, 6, 7, 8]
nums.extend([9, 10])  # O(k) amortized
nums.insert(0, 0)     # O(n) — shifts all elements
  • Append is O(1) amortized; insert at index 0 is O(n) because every element must shift.
  • Slicing creates a shallow copy — nested mutable objects share references.
  • Use del nums[i] or nums.pop(i) to remove by index; both are O(n) except pop() from the end.
  • List comprehensions are faster than equivalent for loops with append.

Exercises

  1. Rotate a List

    Write a function rotate(lst, k) that rotates a list k positions to the right using only slicing. For example, rotate([1,2,3,4,5], 2) returns [4,5,1,2,3].

    Hint

    Use k % len(lst) to handle k larger than the list length, then concatenate two slices.

  2. Flatten Nested Lists

    Write a function flatten(nested) that takes an arbitrarily nested list of integers and returns a flat list. For example, flatten([1, [2, [3, 4], 5]]) returns [1, 2, 3, 4, 5].

    Hint

    Use recursion: check isinstance(item, list) to decide whether to recurse or append.

Common Problems & Solutions

1 Modifying a list while iterating over it causes skipped elements or IndexError.
Cause: The iterator's internal index becomes stale when elements are inserted or removed.
Solution: Iterate over a copy (`for x in lst[:]`) or build a new list with a comprehension.
2 `nums.sort()` returns `None`, not the sorted list.
Cause: `sort()` mutates in place and returns `None` by design to signal no new object is created.
Solution: Use `sorted(nums)` when you need a new sorted list without mutating the original.
