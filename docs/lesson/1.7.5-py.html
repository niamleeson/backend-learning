Pickle, shelve, and Binary Formats

Intuition

Pickle is Python's native serialization — it can save almost anything, but the trade-off is that the output is Python-only and carries a security risk when loading untrusted data.

pickle serializes arbitrary Python objects to a binary format. shelve provides a dict-like interface backed by a pickle database. Both are Python-specific and not suitable for cross-language data exchange.

import pickle
import shelve

# Pickle: serialize any Python object
data = {"model": [1, 2, 3], "accuracy": 0.95}
with open("model.pkl", "wb") as f:
    pickle.dump(data, f)

with open("model.pkl", "rb") as f:
    loaded = pickle.load(f)

# Shelve: persistent dict-like storage
with shelve.open("mydb") as db:
    db["users"] = [{"name": "Alice"}, {"name": "Bob"}]
    db["config"] = {"theme": "dark"}

with shelve.open("mydb") as db:
    print(db["users"])  # [{"name": "Alice"}, ...]
  • Pickle can serialize most Python objects: classes, functions, nested structures.
  • Never unpickle data from untrusted sources — it can execute arbitrary code.
  • shelve is a quick-and-dirty persistent key-value store for prototyping.
  • For cross-language or long-term storage, prefer JSON, MessagePack, or Parquet.

Common Problems & Solutions

1 Unpickling a file from an untrusted source executes arbitrary code.
Cause: Pickle's `__reduce__` method can invoke any callable during deserialization.
Solution: Never unpickle untrusted data. Use JSON or another safe format for external data.
2 Pickled files break after renaming or moving the class definition.
Cause: Pickle stores the module path and class name; if they change, unpickling fails.
Solution: Keep class definitions stable, or use `copyreg` / `__reduce__` to customize deserialization.
