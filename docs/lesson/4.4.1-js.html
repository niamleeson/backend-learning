@nestjs/passport: Strategies and Guards

Intuition

Passport strategies are like different types of keys to a building. A local strategy is like showing your employee badge (username/password) to a security guard. A JWT strategy is like swiping an access card (token). An API key strategy is like entering a PIN code. Each method is different, but they all result in the same thing: proving who you are to get through the door.

@nestjs/passport: Strategies and Guards

Introduction

@nestjs/passport is the official NestJS wrapper around Passport.js, the most popular authentication library for Node.js. It integrates Passport's strategy-based authentication model with NestJS's guard and module systems, providing a clean and declarative way to handle authentication.

Installation

npm i @nestjs/passport passport
npm i -D @types/passport
# Plus strategy-specific packages:
npm i passport-local passport-jwt
npm i -D @types/passport-local @types/passport-jwt

How Passport Works in NestJS

Request
  |
  v
AuthGuard('strategy-name')
  |
  v
PassportStrategy
  |-- validate() returns user
  |-- or throws UnauthorizedException
  |
  v
request.user = validated user
  |
  v
Route Handler (has access to request.user)

Passport uses strategies -- pluggable authentication mechanisms. Each strategy validates credentials differently (username/password, JWT, OAuth, API key, etc.) but all follow the same pattern.

Setting Up the Auth Module

// auth/auth.module.ts
import { Module } from '@nestjs/common';
import { PassportModule } from '@nestjs/passport';
import { JwtModule } from '@nestjs/jwt';
import { AuthService } from './auth.service';
import { AuthController } from './auth.controller';
import { LocalStrategy } from './strategies/local.strategy';
import { JwtStrategy } from './strategies/jwt.strategy';
import { UsersModule } from '../users/users.module';

@Module({
  imports: [
    UsersModule,
    PassportModule.register({ defaultStrategy: 'jwt' }),
    JwtModule.register({
      secret: process.env.JWT_SECRET || 'dev-secret',
      signOptions: { expiresIn: '1h' },
    }),
  ],
  controllers: [AuthController],
  providers: [AuthService, LocalStrategy, JwtStrategy],
  exports: [AuthService],
})
export class AuthModule {}

Local Strategy (Username/Password)

// auth/strategies/local.strategy.ts
import { Strategy } from 'passport-local';
import { PassportStrategy } from '@nestjs/passport';
import { Injectable, UnauthorizedException } from '@nestjs/common';
import { AuthService } from '../auth.service';

@Injectable()
export class LocalStrategy extends PassportStrategy(Strategy) {
  constructor(private authService: AuthService) {
    super({
      usernameField: 'email', // Use 'email' instead of default 'username'
    });
  }

  async validate(email: string, password: string): Promise<any> {
    const user = await this.authService.validateUser(email, password);
    if (!user) {
      throw new UnauthorizedException('Invalid email or password');
    }
    return user; // This becomes request.user
  }
}

Auth Service

// auth/auth.service.ts
import { Injectable, UnauthorizedException } from '@nestjs/common';
import { JwtService } from '@nestjs/jwt';
import { UsersService } from '../users/users.service';
import * as bcrypt from 'bcrypt';

@Injectable()
export class AuthService {
  constructor(
    private usersService: UsersService,
    private jwtService: JwtService,
  ) {}

  async validateUser(email: string, password: string): Promise<any> {
    const user = await this.usersService.findByEmail(email);
    if (user && (await bcrypt.compare(password, user.password))) {
      const { password: _, ...result } = user;
      return result;
    }
    return null;
  }

  async login(user: any) {
    const payload = {
      sub: user.id,
      email: user.email,
      roles: user.roles,
    };

    return {
      accessToken: this.jwtService.sign(payload),
      user: {
        id: user.id,
        email: user.email,
        name: user.name,
      },
    };
  }

  async register(dto: RegisterDto) {
    const hashedPassword = await bcrypt.hash(dto.password, 10);
    const user = await this.usersService.create({
      ...dto,
      password: hashedPassword,
    });

    return this.login(user);
  }
}

Auth Controller

// auth/auth.controller.ts
import { Controller, Post, Body, UseGuards, Request } from '@nestjs/common';
import { AuthGuard } from '@nestjs/passport';
import { AuthService } from './auth.service';
import { RegisterDto } from './dto/register.dto';

@Controller('auth')
export class AuthController {
  constructor(private authService: AuthService) {}

  @UseGuards(AuthGuard('local'))
  @Post('login')
  async login(@Request() req: any) {
    // req.user is set by LocalStrategy.validate()
    return this.authService.login(req.user);
  }

  @Post('register')
  async register(@Body() dto: RegisterDto) {
    return this.authService.register(dto);
  }
}

Named Guards

Create reusable, named guard classes instead of inline AuthGuard('strategy'):

// auth/guards/local-auth.guard.ts
import { Injectable } from '@nestjs/common';
import { AuthGuard } from '@nestjs/passport';

@Injectable()
export class LocalAuthGuard extends AuthGuard('local') {}

// auth/guards/jwt-auth.guard.ts
@Injectable()
export class JwtAuthGuard extends AuthGuard('jwt') {}

// Usage:
@UseGuards(LocalAuthGuard)
@Post('login')
async login(@Request() req: any) { /* ... */ }

@UseGuards(JwtAuthGuard)
@Get('profile')
async getProfile(@Request() req: any) { /* ... */ }

Customizing Guard Behavior

@Injectable()
export class JwtAuthGuard extends AuthGuard('jwt') {
  // Custom error handling
  handleRequest(err: any, user: any, info: any, context: ExecutionContext) {
    if (err || !user) {
      throw err || new UnauthorizedException({
        message: 'Authentication required',
        reason: info?.message || 'Invalid token',
      });
    }
    return user;
  }

  // Optionally skip auth for public routes
  canActivate(context: ExecutionContext) {
    const isPublic = this.reflector.getAllAndOverride<boolean>('isPublic', [
      context.getHandler(),
      context.getClass(),
    ]);
    if (isPublic) return true;
    return super.canActivate(context);
  }

  constructor(private reflector: Reflector) {
    super();
  }
}

API Key Strategy

// strategies/api-key.strategy.ts
import { Strategy } from 'passport-custom';
import { PassportStrategy } from '@nestjs/passport';
import { Injectable, UnauthorizedException } from '@nestjs/common';
import { ApiKeysService } from '../api-keys/api-keys.service';

@Injectable()
export class ApiKeyStrategy extends PassportStrategy(Strategy, 'api-key') {
  constructor(private apiKeysService: ApiKeysService) {
    super();
  }

  async validate(req: Request): Promise<any> {
    const apiKey =
      req.headers['x-api-key'] || req.query['api_key'];

    if (!apiKey) {
      throw new UnauthorizedException('API key is required');
    }

    const keyRecord = await this.apiKeysService.validateKey(apiKey as string);
    if (!keyRecord) {
      throw new UnauthorizedException('Invalid API key');
    }

    return {
      id: keyRecord.userId,
      apiKeyId: keyRecord.id,
      permissions: keyRecord.permissions,
    };
  }
}

Usage:

// Protect with API key
@UseGuards(AuthGuard('api-key'))
@Get('data')
getData(@Request() req: any) {
  return { userId: req.user.id };
}

Multiple Strategies (Either/Or Auth)

@Injectable()
export class CombinedAuthGuard extends AuthGuard(['jwt', 'api-key']) {
  handleRequest(err: any, user: any, info: any) {
    // Accept if either strategy succeeds
    if (user) return user;
    throw new UnauthorizedException('Authentication required (JWT or API key)');
  }
}

Complete Login Flow

1. Client sends POST /auth/login with { email, password }
2. LocalAuthGuard triggers LocalStrategy
3. LocalStrategy calls AuthService.validateUser()
4. AuthService checks credentials against database
5. If valid, LocalStrategy returns user -> sets request.user
6. Controller calls AuthService.login(request.user)
7. AuthService creates JWT token
8. Client receives { accessToken, user }

Subsequent requests:
1. Client sends request with Authorization: Bearer <token>
2. JwtAuthGuard triggers JwtStrategy
3. JwtStrategy verifies token, returns payload
4. payload becomes request.user
5. Controller handles the request with user context

Common Pitfalls

  1. Missing strategy registration -- The strategy class must be a provider in the module. If LocalStrategy isn't in providers, AuthGuard('local') fails.
  2. Wrong field names -- Passport-local expects username and password by default. Override with usernameField: 'email' in super().
  3. Returning null from validate() -- Always throw an UnauthorizedException instead of returning null/undefined. Null causes confusing errors.
  4. Not hashing passwords -- Store bcrypt hashes, never plain text. Compare with bcrypt.compare(), not ===.

Exercises

  1. Implement Local + JWT Authentication

    Build a complete auth system with: LocalStrategy for login (email/password), JwtStrategy for protected routes, AuthService with validateUser and login methods, and AuthController with login and register endpoints. Use bcrypt for password hashing.

    Hint

    Follow the setup in this lesson. Make sure to add bcrypt password hashing in register and comparison in validateUser. The JWT payload should include sub (userId) and email.

  2. Add an API Key Strategy

    Create an ApiKeyStrategy that reads the key from the X-API-Key header. Store API keys in a database table with associated permissions. Build an endpoint to generate new API keys for authenticated users.

    Hint

    Use passport-custom for the strategy. Generate API keys with crypto.randomBytes(32).toString('hex'). Store the hash of the key, not the key itself.

  3. Build a Combined Auth Guard

    Create a guard that accepts either JWT or API key authentication. If both fail, return a descriptive error. Test with both authentication methods on the same endpoint.

    Hint

    Extend AuthGuard with an array of strategy names: AuthGuard(['jwt', 'api-key']). Override handleRequest to accept any successful strategy.

Common Problems & Solutions

1 AuthGuard('local') returns 401 even with correct credentials
Cause: The request body field names don't match what passport-local expects. By default, it expects 'username' and 'password', but the client sends 'email' and 'password'.
Solution: Pass `{ usernameField: 'email' }` to `super()` in the LocalStrategy constructor.

constructor(private authService: AuthService) {\n super({ usernameField: 'email' }); // Map 'email' field to username\n}

2 LocalStrategy validate() is never called
Cause: The LocalStrategy class is not registered as a provider in the AuthModule.
Solution: Add `LocalStrategy` to the `providers` array in AuthModule. Passport strategies must be registered as NestJS providers to be discovered.
3 request.user is undefined in the controller
Cause: The guard is not applied to the route, or the strategy's validate() method returns undefined/null instead of a user object.
Solution: Ensure @UseGuards() is on the route. Verify that validate() returns a truthy value. In validate(), throw UnauthorizedException for failures instead of returning null.
