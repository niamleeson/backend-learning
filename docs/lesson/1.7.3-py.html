CSV and Data Files

Intuition

The csv module handles the tricky edge cases of CSV parsing (quoted fields, embedded commas, newlines in values) so you don't have to split on commas yourself.

The csv module provides readers and writers for comma-separated value files, handling quoting, escaping, and different dialects automatically.

import csv

# Writing CSV
rows = [
    ["name", "age", "city"],
    ["Alice", 30, "NYC"],
    ["Bob", 25, "LA"],
]
with open("people.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.writer(f)
    writer.writerows(rows)

# Reading CSV as dicts (uses first row as keys)
with open("people.csv", newline="", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(f"{row['name']} is {row['age']}")

# Writing CSV from dicts
with open("out.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.DictWriter(f, fieldnames=["name", "age"])
    writer.writeheader()
    writer.writerow({"name": "Carol", "age": 28})
  • Always pass newline="" to open() when using the csv module — it handles line endings internally.
  • DictReader and DictWriter are more readable than index-based access.
  • Use delimiter='\t' for TSV files or dialect='excel' for Excel-compatible output.
  • All values are read as strings; convert numeric fields explicitly.

Exercises

  1. CSV Aggregator

    Read a CSV file with columns product, quantity, price. Write a function that returns the total revenue (sum of quantity * price for each row).

    Hint

    Use DictReader and convert quantity and price to float before multiplying.

  2. CSV to JSON Converter

    Write a function csv_to_json(csv_path, json_path) that reads a CSV file and writes the equivalent JSON array of objects.

    Hint

    Use list(csv.DictReader(f)) to get a list of dicts, then json.dump it.

Common Problems & Solutions

1 Extra blank rows appear in CSV output on Windows.
Cause: Without `newline=""`, Python adds extra `\r` characters on Windows.
Solution: Always open CSV files with `newline=""` as specified in the csv module docs.
2 Numeric values come back as strings after reading.
Cause: The csv module reads everything as strings; it does not infer types.
Solution: Explicitly convert: `int(row['age'])` or `float(row['price'])`.
