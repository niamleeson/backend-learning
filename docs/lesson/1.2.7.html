Lambda Functions and First-Class Functions

Intuition

A lambda is a throwaway function for one-off jobs, like a sticky note instead of a full letter. First-class functions mean you can treat functions like any other data: store them, pass them, combine them.

Lambda functions are anonymous, single-expression functions. Combined with Python's first-class function support, they enable powerful functional programming patterns.

# Lambda: anonymous single-expression function
square = lambda x: x ** 2
print(square(5))  # 25

# Common use: sorting key
students = [("Alice", 90), ("Bob", 85), ("Charlie", 92)]
by_score = sorted(students, key=lambda s: s[1], reverse=True)

# First-class functions: pass and return functions
def apply_operation(func, values):
    return [func(v) for v in values]

result = apply_operation(lambda x: x * 2, [1, 2, 3])  # [2, 4, 6]

# map, filter, reduce
from functools import reduce
nums = [1, 2, 3, 4, 5]
squared = list(map(lambda x: x**2, nums))
evens = list(filter(lambda x: x % 2 == 0, nums))
total = reduce(lambda a, b: a + b, nums)
  • Lambdas are limited to a single expression; use def for anything complex
  • Functions are first-class: they can be assigned, passed, and returned
  • Prefer comprehensions over map/filter for readability in most cases
  • functools.reduce applies a function cumulatively to sequence elements

Exercises

  1. Sort by Multiple Criteria

    Given a list of dicts with 'name' and 'age' keys, sort first by age descending, then by name ascending, using a lambda key.

    Hint

    Use sorted(people, key=lambda p: (-p['age'], p['name'])).

  2. Function Pipeline

    Write a pipeline(*funcs) function that returns a new function composing all input functions left to right. Test with add1, double, str.

    Hint

    Use functools.reduce to compose: reduce(lambda f, g: lambda x: g(f(x)), funcs).

Common Problems & Solutions

1 SyntaxError in lambda: can't use statements
Cause: Lambdas only support expressions, not statements like `print`, `if/else` blocks, or assignments.
Solution: Use a conditional expression `x if cond else y` in lambda, or switch to `def`.
2 Lambda in loop captures variable by reference
Cause: All lambdas created in a loop reference the same variable, ending up with the final value.
Solution: Bind the value with a default arg: `lambda x, i=i: x + i`.
