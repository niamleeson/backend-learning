Operators: Arithmetic, Comparison, Logical, Bitwise

Intuition

Operators are just shortcuts for calling special methods on objects. When you write a + b, Python calls a.__add__(b). This is why you can define + for your own classes.

Python provides arithmetic, comparison, logical, and bitwise operators. Understanding operator precedence and behavior prevents subtle bugs in expressions.

# Arithmetic
print(7 / 2)    # 3.5  (true division)
print(7 // 2)   # 3    (floor division)
print(7 % 2)    # 1    (modulo)
print(2 ** 10)  # 1024 (exponentiation)

# Comparison chaining (unique to Python)
x = 5
print(1 < x < 10)        # True
print(1 < x > 3 != 7)    # True

# Bitwise operators
a, b = 0b1100, 0b1010
print(bin(a & b))  # 0b1000 (AND)
print(bin(a | b))  # 0b1110 (OR)
print(bin(a ^ b))  # 0b0110 (XOR)
print(bin(~a))     # -0b1101 (NOT)
print(bin(a << 2)) # 0b110000 (left shift)
  • / always returns float; // returns floor-divided int (or float if either operand is float)
  • Comparison chaining like a < b < c is equivalent to a < b and b < c
  • Bitwise operators work on the integer's binary representation
  • ** has higher precedence than unary -: -2**2 is -4, not 4

Exercises

  1. Floor Division vs True Division

    Compute 17 / 3, 17 // 3, -17 // 3, and 17 % 3. Explain why -17 // 3 is -6 and not -5.

    Hint

    Floor division rounds toward negative infinity, not toward zero.

  2. Bitwise Flag System

    Define permission flags READ=1, WRITE=2, EXECUTE=4. Combine them with OR, check for a flag with AND, and remove a flag with AND+NOT.

    Hint

    Check: perms & READ. Remove: perms & ~WRITE.

Common Problems & Solutions

1 Unexpected result: `-2 ** 2` gives `-4` instead of `4`
Cause: `**` binds tighter than unary `-`, so it's parsed as `-(2**2)`.
Solution: Use parentheses: `(-2) ** 2` gives `4`.
2 ZeroDivisionError with `//` or `%`
Cause: Floor division and modulo by zero are not allowed, just like regular division.
Solution: Guard with `if divisor != 0:` or handle the exception.
