Flask-Smorest: OpenAPI-First Development

Intuition

Flask-Smorest bridges the gap between code and documentation by making your schemas the single source of truth. Write your Marshmallow schemas once, and you get validation, serialization, and API documentation all generated automatically.

Flask-Smorest and OpenAPI-First Development

Flask-Smorest is a modern Flask extension that generates OpenAPI (Swagger) documentation from your code. It uses Marshmallow schemas for request/response validation and provides automatic interactive API docs. This is the OpenAPI-first approach to Flask API development.

from flask import Flask
from flask_smorest import Api, Blueprint, abort
from marshmallow import Schema, fields

app = Flask(__name__)
app.config['API_TITLE'] = 'My Store API'
app.config['API_VERSION'] = 'v1'
app.config['OPENAPI_VERSION'] = '3.0.3'
app.config['OPENAPI_URL_PREFIX'] = '/'
app.config['OPENAPI_SWAGGER_UI_PATH'] = '/swagger-ui'
app.config['OPENAPI_SWAGGER_UI_URL'] = 'https://cdn.jsdelivr.net/npm/swagger-ui-dist/'

api = Api(app)

class ItemSchema(Schema):
    id = fields.Int(dump_only=True)
    name = fields.Str(required=True)
    price = fields.Float(required=True)

class ItemQuerySchema(Schema):
    name = fields.Str()
    min_price = fields.Float()

blp = Blueprint('items', __name__, url_prefix='/api/items',
                description='Operations on items')

@blp.route('/')
class ItemList(blp.MethodView):
    @blp.arguments(ItemQuerySchema, location='query')
    @blp.response(200, ItemSchema(many=True))
    def get(self, args):
        """List all items with optional filtering."""
        return [{'id': 1, 'name': 'Widget', 'price': 9.99}]

    @blp.arguments(ItemSchema)
    @blp.response(201, ItemSchema)
    def post(self, new_item):
        """Create a new item."""
        new_item['id'] = 1
        return new_item

api.register_blueprint(blp)

Key Takeaways: - Flask-Smorest auto-generates OpenAPI 3.x docs from your schemas and docstrings. - @blp.arguments() validates input and @blp.response() documents and serializes output. - Swagger UI is served automatically, providing interactive API testing at /swagger-ui.

Exercises

  1. Complete CRUD with Swagger Docs

    Build a full CRUD API for a 'tasks' resource using Flask-Smorest. Include query parameters for filtering, proper error responses documented with @blp.alt_response, and verify the Swagger UI shows everything.

    Hint

    Use @blp.alt_response(404, description='Task not found') to document error cases in the OpenAPI spec.

  2. Pagination with Flask-Smorest

    Implement cursor-based or page-based pagination using Flask-Smorest's built-in pagination support. Return paginated results with metadata.

    Hint

    Use @blp.paginate() decorator or create a PaginationSchema for query args with page and per_page fields.

Common Problems & Solutions

1 Swagger UI shows no endpoints or displays a blank page.
Cause: The OPENAPI configuration keys are missing or incorrect, or the blueprint was not registered with the Api object.
Solution: Verify that API_TITLE, API_VERSION, and OPENAPI_VERSION are set in app.config. Register blueprints with api.register_blueprint(blp), not app.register_blueprint().
2 Request validation errors return 422 with unexpected error format.
Cause: Flask-Smorest uses its own error format for validation errors from Marshmallow schemas.
Solution: Customize the error handler with @api.errorhandler(422) or accept Flask-Smorest's default format and document it for clients.
