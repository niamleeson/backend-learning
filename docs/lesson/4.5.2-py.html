JWT Authentication: Tokens, Refresh, and Revocation

Intuition

JWTs are like concert wristbands -- they prove you have been authenticated without going back to the ticket booth each time. The refresh token is your original ticket that lets you get a new wristband when the old one expires.

JWT Authentication: Tokens, Refresh, and Revocation

JSON Web Tokens (JWTs) enable stateless authentication for APIs. A JWT contains encoded claims (user ID, roles, expiration) signed by the server. Flask-JWT-Extended provides comprehensive JWT support including access tokens, refresh tokens, and token revocation for building secure APIs.

from flask import Flask, jsonify, request
from flask_jwt_extended import (
    JWTManager, create_access_token, create_refresh_token,
    jwt_required, get_jwt_identity, get_jwt
)
from datetime import timedelta

app = Flask(__name__)
app.config['JWT_SECRET_KEY'] = 'jwt-secret-change-in-production'
app.config['JWT_ACCESS_TOKEN_EXPIRES'] = timedelta(minutes=15)
app.config['JWT_REFRESH_TOKEN_EXPIRES'] = timedelta(days=30)

jwt = JWTManager(app)

# Simple blocklist for revoked tokens
BLOCKLIST = set()

@jwt.token_in_blocklist_loader
def check_if_revoked(jwt_header, jwt_payload):
    return jwt_payload['jti'] in BLOCKLIST

@app.route('/auth/login', methods=['POST'])
def login():
    data = request.get_json()
    if data['username'] == 'admin' and data['password'] == 'secret':
        access = create_access_token(identity=data['username'],
                                      additional_claims={'role': 'admin'})
        refresh = create_refresh_token(identity=data['username'])
        return jsonify(access_token=access, refresh_token=refresh)
    return jsonify(error='Bad credentials'), 401

@app.route('/auth/refresh', methods=['POST'])
@jwt_required(refresh=True)
def refresh():
    identity = get_jwt_identity()
    new_access = create_access_token(identity=identity)
    return jsonify(access_token=new_access)

@app.route('/auth/logout', methods=['POST'])
@jwt_required()
def logout():
    jti = get_jwt()['jti']
    BLOCKLIST.add(jti)
    return jsonify(message='Token revoked'), 200

@app.route('/protected')
@jwt_required()
def protected():
    return jsonify(user=get_jwt_identity(), claims=get_jwt())

Key Takeaways: - Access tokens are short-lived (minutes); refresh tokens are long-lived (days/weeks) and used to get new access tokens. - Token revocation uses a blocklist checked on every request via token_in_blocklist_loader. - Store additional user data (roles, permissions) in JWT claims to avoid database lookups.

Exercises

  1. Role-Based Access Control

    Extend the JWT system with role-based access. Create a custom decorator that checks the 'role' claim in the JWT and restricts certain endpoints to 'admin' users only.

    Hint

    Use get_jwt() to access custom claims inside a decorator, and abort with 403 Forbidden if the role does not match.

  2. Token Refresh Rotation

    Implement refresh token rotation: when a refresh token is used, revoke it and issue both a new access token and a new refresh token. This limits the window of abuse for stolen refresh tokens.

    Hint

    In the refresh endpoint, get the current refresh token's JTI, add it to the blocklist, and return both new access and refresh tokens.

Common Problems & Solutions

1 401 Unauthorized even though a valid token is sent.
Cause: The token is sent in the wrong location. Flask-JWT-Extended expects the token in the Authorization header as 'Bearer ' by default.
Solution: Send the header as: Authorization: Bearer . Alternatively configure JWT_TOKEN_LOCATION to accept tokens from cookies or query strings.
2 Token revocation does not persist after server restart.
Cause: The blocklist is stored in a Python set in memory, which is cleared on restart.
Solution: Use Redis or a database table to store revoked token JTIs. Redis with a TTL matching the token expiration is the most common production approach.
