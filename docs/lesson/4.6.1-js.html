Gateways: @WebSocketGateway and Events

Intuition

A WebSocket gateway is like a live phone line to a call center. Unlike HTTP (where you call, get an answer, and hang up), WebSockets keep the line open. The gateway operator (@SubscribeMessage) listens for different phrases ('message', 'typing') and responds accordingly. The server can also call you back anytime without you asking (@WebSocketServer emit).

Gateways: @WebSocketGateway and Events

Introduction

WebSockets provide full-duplex, persistent communication between client and server. Unlike HTTP's request-response model, WebSockets allow the server to push data to clients at any time. NestJS provides first-class support through gateways -- classes decorated with @WebSocketGateway() that handle WebSocket connections and events.

Installation

npm i @nestjs/websockets @nestjs/platform-socket.io
npm i socket.io
npm i -D @types/socket.io

NestJS supports both Socket.IO and plain WebSockets (ws). Socket.IO is the default and provides features like rooms, namespaces, and automatic reconnection.

Creating a Gateway

// events/events.gateway.ts
import {
  WebSocketGateway, WebSocketServer,
  SubscribeMessage, MessageBody,
  ConnectedSocket, OnGatewayInit,
  OnGatewayConnection, OnGatewayDisconnect,
} from '@nestjs/websockets';
import { Server, Socket } from 'socket.io';
import { Logger } from '@nestjs/common';

@WebSocketGateway({
  cors: {
    origin: '*',
  },
})
export class EventsGateway
  implements OnGatewayInit, OnGatewayConnection, OnGatewayDisconnect
{
  @WebSocketServer()
  server: Server;

  private logger = new Logger('EventsGateway');

  afterInit(server: Server) {
    this.logger.log('WebSocket Gateway initialized');
  }

  handleConnection(client: Socket) {
    this.logger.log(`Client connected: ${client.id}`);
  }

  handleDisconnect(client: Socket) {
    this.logger.log(`Client disconnected: ${client.id}`);
  }

  @SubscribeMessage('message')
  handleMessage(
    @MessageBody() data: { text: string },
    @ConnectedSocket() client: Socket,
  ): string {
    this.logger.log(`Message from ${client.id}: ${data.text}`);
    return data.text; // Sends acknowledgement back to sender
  }

  @SubscribeMessage('broadcast')
  handleBroadcast(@MessageBody() data: { text: string }): void {
    // Send to ALL connected clients (including sender)
    this.server.emit('message', {
      text: data.text,
      timestamp: new Date().toISOString(),
    });
  }
}

Event Handling Patterns

Return Value (Acknowledgement)

@SubscribeMessage('request-data')
handleRequest(@MessageBody() query: any): { event: string; data: any } {
  // Returns to the sender as an acknowledgement
  return {
    event: 'response-data',
    data: { results: [1, 2, 3] },
  };
}

Emit to All Clients

@SubscribeMessage('announce')
handleAnnounce(@MessageBody() data: any): void {
  this.server.emit('announcement', data);
}

Emit to Specific Client

@SubscribeMessage('private-message')
handlePrivateMessage(
  @MessageBody() data: { to: string; message: string },
  @ConnectedSocket() client: Socket,
) {
  this.server.to(data.to).emit('private-message', {
    from: client.id,
    message: data.message,
  });
}

Broadcast (to all except sender)

@SubscribeMessage('typing')
handleTyping(
  @MessageBody() data: { username: string },
  @ConnectedSocket() client: Socket,
) {
  client.broadcast.emit('user-typing', data);
}

WebSocket Auth Guard

@Injectable()
export class WsAuthGuard implements CanActivate {
  constructor(private jwtService: JwtService) {}

  canActivate(context: ExecutionContext): boolean {
    const client = context.switchToWs().getClient<Socket>();
    const token =
      client.handshake.auth.token ||
      client.handshake.headers.authorization?.split(' ')[1];

    if (!token) throw new WsException('Authentication required');

    try {
      const payload = this.jwtService.verify(token);
      client.data.user = payload; // Store user data on socket
      return true;
    } catch {
      throw new WsException('Invalid token');
    }
  }
}

Exception Filters for WebSockets

@Catch(WsException)
export class WsExceptionFilter implements ExceptionFilter {
  catch(exception: WsException, host: ArgumentsHost) {
    const client = host.switchToWs().getClient<Socket>();
    const error = exception.getError();

    client.emit('error', {
      message: typeof error === 'string' ? error : (error as any).message,
      timestamp: new Date().toISOString(),
    });
  }
}

Emitting from Services

Inject the gateway into a service to emit events from anywhere:

@Injectable()
export class NotificationsService {
  constructor(private eventsGateway: EventsGateway) {}

  notifyUser(userId: string, notification: any) {
    // Emit to a specific room (user's personal room)
    this.eventsGateway.server
      .to(`user:${userId}`)
      .emit('notification', notification);
  }

  broadcastUpdate(data: any) {
    this.eventsGateway.server.emit('update', data);
  }
}

Client-Side Connection

// Client (browser or Node.js)
import { io } from 'socket.io-client';

const socket = io('http://localhost:3000', {
  auth: { token: 'jwt-token-here' },
});

socket.on('connect', () => {
  console.log('Connected:', socket.id);
});

socket.on('message', (data) => {
  console.log('Received:', data);
});

socket.emit('message', { text: 'Hello!' }, (ack) => {
  console.log('Server acknowledged:', ack);
});

socket.on('disconnect', (reason) => {
  console.log('Disconnected:', reason);
});

Common Pitfalls

  1. CORS not configured -- WebSocket connections fail silently without proper CORS settings. Always configure cors in the gateway decorator.
  2. Not handling disconnections -- Clean up resources (rooms, state) when clients disconnect.
  3. Using HTTP exceptions in WebSocket context -- Use WsException instead of HttpException in gateways.
  4. Missing @WebSocketServer() -- Without this decorator, this.server is undefined. You need it to emit events to clients.

Exercises

  1. Build a Basic WebSocket Echo Server

    Create a gateway that: accepts connections with logging, handles a 'ping' event by responding with 'pong' and a timestamp, handles a 'broadcast' event that sends the message to all connected clients, tracks connected client count and emits it on connect/disconnect.

    Hint

    Use a Set to track client IDs. In handleConnection, add the ID and emit the count. In handleDisconnect, remove it. Use this.server.emit for broadcasts.

  2. Add Authentication to WebSockets

    Create a WsAuthGuard that validates JWT tokens from the handshake auth object. Store the authenticated user on client.data.user. Create a @WsCurrentUser() decorator to extract the user in handlers. Reject unauthenticated connections.

    Hint

    Access the token via client.handshake.auth.token. Use JwtService to verify. For the decorator, use createParamDecorator with context.switchToWs().getClient().data.user.

  3. Emit Events from HTTP Endpoints

    Create a notification system where: an HTTP POST endpoint creates a notification, the gateway emits the notification to the specific user in real-time, the user's socket joins a personal room on connection (e.g., 'user:userId').

    Hint

    In handleConnection, verify the user and call client.join(user:${userId}). In the HTTP service, inject the gateway and call this.gateway.server.to(user:${userId}).emit(...).

Common Problems & Solutions

1 WebSocket connection fails with CORS error
Cause: The gateway doesn't have CORS configured, or the allowed origins don't match the client's origin.
Solution: Add cors configuration to @WebSocketGateway(): `@WebSocketGateway({ cors: { origin: 'http://localhost:3000' } })`.
2 this.server is undefined in the gateway
Cause: Missing `@WebSocketServer()` decorator on the server property.
Solution: Add `@WebSocketServer() server: Server;` as a class property. NestJS populates it after the gateway initializes.
3 Guard throws HttpException instead of WsException
Cause: Reusing an HTTP auth guard for WebSocket context. HTTP exceptions aren't properly serialized for WebSocket clients.
Solution: Create a dedicated WsAuthGuard that throws WsException. Use context.switchToWs() instead of context.switchToHttp().
