Queries: Type-Safe SQL with Drizzle

Intuition

Drizzle's query API is like writing SQL but with autocomplete and spell-check. Every table name, column name, and operator is checked by TypeScript as you type. It is SQL with training wheels that you never need to remove, because the training wheels make you faster, not slower.

Queries: Type-Safe SQL with Drizzle

Introduction

Drizzle provides two query APIs: the SQL-like core API (which mirrors SQL syntax closely) and the relational queries API (which handles JOINs and nested data automatically). Both are fully type-safe. The core API gives you maximum control; the relational API gives you convenience.

Core API: SQL-Like Queries

import { eq, ne, gt, gte, lt, lte, and, or, not,
         like, ilike, inArray, notInArray, isNull, isNotNull,
         between, sql, desc, asc, count, sum, avg } from 'drizzle-orm';
import { db } from './db';
import { users, orders, orderItems, products } from './db/schema';

// SELECT
const allUsers = await db.select().from(users);

// Select specific columns
const names = await db.select({
  id: users.id,
  name: users.name,
  email: users.email,
}).from(users);

// WHERE
const activeAdmins = await db.select()
  .from(users)
  .where(
    and(
      eq(users.role, 'admin'),
      isNull(users.deletedAt)
    )
  );

// Complex filtering
const filtered = await db.select()
  .from(users)
  .where(
    and(
      or(
        ilike(users.name, '%alice%'),
        ilike(users.email, '%alice%')
      ),
      gte(users.createdAt, new Date('2024-01-01')),
      isNull(users.deletedAt)
    )
  )
  .orderBy(desc(users.createdAt))
  .limit(20)
  .offset(0);

JOINs

// INNER JOIN
const ordersWithUsers = await db.select({
  orderId: orders.id,
  total: orders.total,
  customerName: users.name,
  customerEmail: users.email,
})
  .from(orders)
  .innerJoin(users, eq(orders.userId, users.id))
  .where(eq(orders.status, 'completed'))
  .orderBy(desc(orders.createdAt));

// LEFT JOIN
const usersWithOrderCount = await db.select({
  userId: users.id,
  name: users.name,
  orderCount: count(orders.id),
  totalSpent: sum(orders.total),
})
  .from(users)
  .leftJoin(orders, eq(users.id, orders.userId))
  .groupBy(users.id, users.name)
  .orderBy(desc(count(orders.id)));

// Multi-table JOIN
const orderDetails = await db.select({
  orderId: orders.id,
  product: products.name,
  quantity: orderItems.quantity,
  unitPrice: orderItems.unitPrice,
  customer: users.name,
})
  .from(orderItems)
  .innerJoin(orders, eq(orderItems.orderId, orders.id))
  .innerJoin(users, eq(orders.userId, users.id))
  .innerJoin(products, eq(orderItems.productId, products.id))
  .where(eq(orders.id, orderId));

Relational Queries API

// Find many with relations (like Prisma's include)
const usersWithOrders = await db.query.users.findMany({
  with: {
    orders: {
      with: {
        items: {
          with: { product: true },
        },
      },
      orderBy: (orders, { desc }) => [desc(orders.createdAt)],
      limit: 5,
    },
  },
  where: (users, { isNull }) => isNull(users.deletedAt),
  limit: 20,
});

// Find first
const user = await db.query.users.findFirst({
  where: (users, { eq }) => eq(users.email, 'alice@example.com'),
  with: {
    orders: { limit: 10 },
  },
});

// Select specific columns with relations
const lightweight = await db.query.users.findMany({
  columns: {
    id: true,
    name: true,
    email: true,
    passwordHash: false, // exclude
  },
  with: {
    orders: {
      columns: { id: true, total: true, status: true },
    },
  },
});

Dynamic Query Building

function buildProductQuery(filters: ProductFilters) {
  const conditions = [];

  if (filters.search) {
    conditions.push(
      or(
        ilike(products.name, `%${filters.search}%`),
        ilike(products.description, `%${filters.search}%`)
      )
    );
  }
  if (filters.minPrice) conditions.push(gte(products.price, filters.minPrice.toString()));
  if (filters.maxPrice) conditions.push(lte(products.price, filters.maxPrice.toString()));
  if (filters.categoryId) conditions.push(eq(products.categoryId, filters.categoryId));
  if (filters.inStock) conditions.push(gt(products.stock, 0));

  conditions.push(eq(products.isActive, true));

  let query = db.select().from(products)
    .where(and(...conditions))
    .limit(filters.limit || 20)
    .offset(filters.offset || 0);

  if (filters.sortBy === 'price') {
    query = query.orderBy(filters.sortDir === 'desc' ? desc(products.price) : asc(products.price));
  } else {
    query = query.orderBy(desc(products.createdAt));
  }

  return query;
}

Transactions

const order = await db.transaction(async (tx) => {
  const product = await tx.select()
    .from(products)
    .where(eq(products.id, productId))
    .for('update')  // SELECT FOR UPDATE
    .then(rows => rows[0]);

  if (!product || product.stock < quantity) {
    throw new Error('Insufficient stock');
  }

  await tx.update(products)
    .set({ stock: sql`${products.stock} - ${quantity}` })
    .where(eq(products.id, productId));

  const [newOrder] = await tx.insert(orders)
    .values({ userId, total: (parseFloat(product.price) * quantity).toString(), status: 'pending' })
    .returning();

  await tx.insert(orderItems)
    .values({ orderId: newOrder.id, productId, quantity, unitPrice: product.price });

  return newOrder;
});

Common Problems & Solutions

1 Type error when using operators like eq() with wrong column types
Cause: Drizzle enforces strict type checking. Comparing a UUID column with a number, or a decimal with a number literal, causes type errors.
Solution: Ensure types match: use strings for uuid and decimal columns. For decimal, pass string values: `eq(products.price, '9.99')` not `eq(products.price, 9.99)`.
2 Relational queries return empty relations even though data exists
Cause: Relations are not defined or not passed to the drizzle() constructor.
Solution: Define relations using the `relations()` function, import them, and spread into the schema parameter: `drizzle(pool, { schema: { ...schema, ...relations } })`.
3 Cannot use .orderBy() with relational queries
Cause: The relational API uses a different syntax for ordering than the core API.
Solution: In relational queries, use the callback syntax: `orderBy: (table, { desc }) => [desc(table.createdAt)]` instead of the standalone `desc()` function.
