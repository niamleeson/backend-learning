Metaclasses: type, __new__, and Custom Metaclasses

Intuition

If classes are cookie cutters and instances are cookies, then metaclasses are the factories that build cookie cutters. They control how classes themselves are constructed.

Metaclasses: type, new, and Custom Metaclasses

Metaclasses are classes that create classes. Just as a class defines how instances behave, a metaclass defines how classes behave. type is the default metaclass.

# type is a metaclass: it creates classes
MyClass = type('MyClass', (object,), {'x': 10, 'greet': lambda self: 'hi'})

# Custom metaclass
class SingletonMeta(type):
    _instances = {}

    def __call__(cls, *args, **kwargs):
        if cls not in cls._instances:
            cls._instances[cls] = super().__call__(*args, **kwargs)
        return cls._instances[cls]

class Database(metaclass=SingletonMeta):
    def __init__(self):
        self.connection = "connected"

db1 = Database()
db2 = Database()
assert db1 is db2  # Same instance

# Metaclass with __new__ for class validation
class ValidatedMeta(type):
    def __new__(mcs, name, bases, namespace):
        if 'validate' not in namespace and name != 'Base':
            raise TypeError(f"{name} must implement validate()")
        return super().__new__(mcs, name, bases, namespace)

class Base(metaclass=ValidatedMeta):
    pass

class User(Base):
    def validate(self):  # Required by metaclass
        return True

Key Takeaways: - type(name, bases, dict) dynamically creates classes at runtime - Metaclass __new__ controls class creation; __call__ controls instantiation - Use metaclasses for singletons, validation, registration, or ORM patterns - Metaclasses are powerful but complex; prefer __init_subclass__ or decorators when possible - Most Python developers rarely need custom metaclasses directly

Exercises

  1. Registry Metaclass

    Create a metaclass that automatically registers every class that uses it into a global registry dict mapping class names to classes. Test with 3 different classes.

    Hint

    In __new__, after creating the class with super().__new__(), add it to a class-level dict: cls._registry[name] = cls.

  2. Interface Enforcement Metaclass

    Write a metaclass that checks subclasses implement specific abstract methods (save, load). Raise TypeError at class creation time if methods are missing.

    Hint

    In __new__, check if 'save' not in namespace and any base has the metaclass. Skip the check for the base class itself.

Common Problems & Solutions

1 `TypeError: metaclass conflict` when using multiple inheritance with different metaclasses.
Cause: Two base classes use different metaclasses, and Python cannot determine which to use.
Solution: Create a new metaclass that inherits from both conflicting metaclasses: `class CombinedMeta(Meta1, Meta2): pass`.
2 Metaclass `__new__` is called for every subclass, including ones that should be exempt.
Cause: Metaclass logic applies to all classes in the hierarchy, including intermediate base classes.
Solution: Add a check for the base class name or use a flag attribute to skip validation for base classes.
