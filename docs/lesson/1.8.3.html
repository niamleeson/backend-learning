Generator Expressions

Intuition

Generator expressions are list comprehensions with a lazy evaluation strategy — they compute each element on demand rather than materializing the entire list upfront.

Generator expressions are the lazy counterpart of list comprehensions. They use parentheses instead of brackets and produce values on demand without allocating a full list.

# List comprehension — builds entire list in memory
squares_list = [x**2 for x in range(1_000_000)]

# Generator expression — lazy, O(1) memory
squares_gen = (x**2 for x in range(1_000_000))

# Use directly in functions that accept iterables
total = sum(x**2 for x in range(1_000_000))  # no extra parens needed

# Chaining with conditions
even_squares = (x**2 for x in range(100) if x % 2 == 0)
print(list(even_squares))  # [0, 4, 16, 36, ...]

# Generator expressions are single-use
gen = (x for x in [1, 2, 3])
print(list(gen))  # [1, 2, 3]
print(list(gen))  # [] — exhausted
  • Use generator expressions when you only need to iterate once — they save memory.
  • When passed as the sole argument to a function, the outer parentheses double as the function call: sum(x for x in items).
  • Generator expressions are single-use; convert to list if you need multiple passes.
  • For complex logic (multiple statements, try/except), use a full generator function instead.

Exercises

  1. Memory Comparison

    Write a script that compares memory usage of [x**2 for x in range(10_000_000)] vs (x**2 for x in range(10_000_000)) using sys.getsizeof.

    Hint

    sys.getsizeof shows the generator object is tiny; the list is huge. Note: getsizeof on a generator doesn't show yielded values.

  2. Pipeline Processing

    Chain generator expressions to: (1) read lines from a file, (2) strip whitespace, (3) filter non-empty lines, (4) convert to uppercase. Process lazily.

    Hint

    Assign each step to a variable: lines = (line.strip() for line in f), then filter, then transform.

Common Problems & Solutions

1 Trying to index a generator expression raises `TypeError`.
Cause: Generators don't support random access; they only support sequential iteration.
Solution: Convert to a list first if you need indexing, or use `itertools.islice`.
2 A generator expression used twice produces results only the first time.
Cause: Generator expressions are single-use iterators; once exhausted, they yield nothing.
Solution: Recreate the generator expression or convert to a list if you need multiple passes.
