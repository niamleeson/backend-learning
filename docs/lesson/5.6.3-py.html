Broadcasting and Room-Based Messaging

Intuition

Rooms are like separate conference calls on the same phone system: each room has its own participant list, and when someone speaks, only people in that room hear it. The room manager is the operator connecting and disconnecting callers.

Broadcasting and Room-Based Messaging

Room-based messaging allows users to join specific channels, receiving only messages from rooms they belong to. This scales better than global broadcasting and enables features like topic-based discussions.

from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from collections import defaultdict

app = FastAPI()

class RoomManager:
    def __init__(self):
        self.rooms: dict[str, dict[str, WebSocket]] = defaultdict(dict)

    async def join_room(self, room: str, username: str, ws: WebSocket):
        await ws.accept()
        self.rooms[room][username] = ws
        await self.broadcast(room, {
            "type": "system",
            "content": f"{username} joined {room}"
        })

    async def leave_room(self, room: str, username: str):
        self.rooms[room].pop(username, None)
        if not self.rooms[room]:
            del self.rooms[room]  # Clean up empty rooms
        else:
            await self.broadcast(room, {
                "type": "system",
                "content": f"{username} left {room}"
            })

    async def broadcast(self, room: str, message: dict):
        disconnected = []
        for user, ws in self.rooms.get(room, {}).items():
            try:
                await ws.send_json(message)
            except Exception:
                disconnected.append(user)
        for user in disconnected:
            self.rooms[room].pop(user, None)

rooms = RoomManager()

@app.websocket("/ws/{room}/{username}")
async def room_ws(ws: WebSocket, room: str, username: str):
    await rooms.join_room(room, username, ws)
    try:
        while True:
            text = await ws.receive_text()
            await rooms.broadcast(room, {
                "type": "message", "sender": username, "content": text
            })
    except WebSocketDisconnect:
        await rooms.leave_room(room, username)

Key Takeaways: - Rooms are logical groupings; use a dict mapping room names to their members - Clean up empty rooms to prevent memory leaks - Handle send failures gracefully by removing disconnected clients during broadcast - Users can potentially join multiple rooms using separate WebSocket connections

Exercises

  1. Multi-Room Support

    Modify the system so a single WebSocket connection can subscribe to multiple rooms. Use JSON messages with a join_room action and a send action with a room field. Track which rooms each connection belongs to.

    Hint

    Maintain a reverse mapping: user_rooms: dict[str, set[str]] tracking which rooms each user is in. Parse incoming JSON messages to handle {"action": "join", "room": "general"} and {"action": "send", "room": "general", "content": "hello"}.

  2. Redis Pub/Sub Rooms

    Replace the in-memory room manager with Redis pub/sub so rooms work across multiple server instances. Each room maps to a Redis channel. Workers subscribe to channels and forward messages to local connections.

    Hint

    Use redis.asyncio with pubsub.subscribe(room_name). Run a background task per subscription that reads messages from Redis and forwards them to local WebSocket connections in that room.

Common Problems & Solutions

1 Memory usage grows as rooms accumulate without cleanup
Cause: Empty rooms remain in the rooms dictionary after all users leave, and stale connections are not removed.
Solution: Delete room entries when the last user leaves. Implement periodic cleanup that iterates through rooms, attempts a ping on each connection, and removes dead connections. Log room creation and deletion for monitoring.
2 Broadcast to large rooms causes timeouts for other rooms
Cause: Broadcasting sequentially to a room with many connections blocks the event loop, delaying message delivery to other rooms.
Solution: Use `asyncio.gather()` to send messages to all room members concurrently. For very large rooms, batch the sends or offload message delivery to a background task with `asyncio.create_task()`.
