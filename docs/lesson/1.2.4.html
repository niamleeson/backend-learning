Scope and the LEGB Rule

Intuition

LEGB is like searching for a name in ever-widening circles: first your room (local), then your house (enclosing), then your neighborhood (global), then the city directory (built-in).

Python resolves variable names using the LEGB rule: Local, Enclosing, Global, Built-in. Understanding scope prevents hard-to-trace bugs with variable access.

x = "global"

def outer():
    x = "enclosing"

    def inner():
        x = "local"
        print(x)       # "local" (L)

    inner()
    print(x)           # "enclosing" (E)

outer()
print(x)               # "global" (G)

# Using global and nonlocal keywords
counter = 0
def increment():
    global counter
    counter += 1

def make_counter():
    count = 0
    def inc():
        nonlocal count
        count += 1
        return count
    return inc
  • LEGB: Local -> Enclosing -> Global -> Built-in
  • global keyword lets you modify a module-level variable from inside a function
  • nonlocal keyword lets you modify a variable in an enclosing (but not global) scope
  • Avoid global; prefer passing values as arguments and returning results

Exercises

  1. Scope Detective

    Without running, predict the output of a function that defines x at global, enclosing, and local levels, then prints x at each level. Verify by running.

    Hint

    Each print(x) resolves to the nearest scope where x is defined.

  2. Counter with Nonlocal

    Create a make_counter() function that returns increment and get_count functions sharing a nonlocal count variable.

    Hint

    Define count = 0 in make_counter, use nonlocal count in both inner functions.

Common Problems & Solutions

1 UnboundLocalError: local variable referenced before assignment
Cause: Assigning to a variable inside a function makes Python treat it as local, even if the assignment is after a read.
Solution: Use `global` or `nonlocal` to declare intent, or restructure to avoid reassignment.
2 Accidentally shadowing a built-in like `print` or `len`
Cause: Assigning `print = 'hello'` hides the built-in function in the current scope.
Solution: Avoid using built-in names as variables. Use `del print` to restore access.
