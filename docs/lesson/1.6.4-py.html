Dependency Management: pip, Poetry, uv

Intuition

pip is the foundation everyone builds on; Poetry adds structure and lockfiles; uv gives you pip's interface at Rust speed. Pick the right tool for your project's complexity.

Python's dependency management landscape includes pip (the standard installer), Poetry (all-in-one tool with lockfiles), and uv (a fast Rust-based replacement for pip and venv).

# pip: the standard
# $ pip install requests
# $ pip install -r requirements.txt

# Poetry: manages deps, venvs, and packaging
# $ poetry init
# $ poetry add requests
# $ poetry add --group dev pytest
# $ poetry lock           # generates poetry.lock
# $ poetry install         # installs from lockfile

# uv: fast, drop-in pip replacement
# $ uv venv
# $ uv pip install requests
# $ uv pip compile requirements.in -o requirements.txt
# $ uv pip sync requirements.txt
  • pip is universal but lacks a built-in lockfile — pair it with pip-tools for reproducibility.
  • Poetry provides dependency resolution, lockfiles, venv management, and publishing in one tool.
  • uv is 10-100x faster than pip, written in Rust, and compatible with pip's interface.
  • All three can coexist; choose based on project needs and team preferences.

Exercises

  1. Poetry Workflow

    Initialize a new Poetry project, add httpx as a dependency and pytest as a dev dependency, then examine pyproject.toml and poetry.lock.

    Hint

    Run poetry init --no-interaction, then poetry add httpx and poetry add --group dev pytest.

  2. uv Speed Test

    Install uv, create a virtual environment with uv venv, install 5 packages, and compare the time with the same operation using regular pip.

    Hint

    Use time uv pip install ... vs time pip install ... to compare.

Common Problems & Solutions

1 Dependency conflicts: package A requires `foo>=2.0` but package B requires `foo<2.0`.
Cause: Incompatible version constraints between two dependencies.
Solution: Use `poetry` or `uv` for better dependency resolution. If unresolvable, pin compatible versions or find alternative packages.
2 `poetry install` is slow on large projects.
Cause: Poetry's resolver can be slow with many transitive dependencies.
Solution: Use `poetry lock --no-update` to avoid re-resolving, or consider migrating to `uv` for faster installs.
