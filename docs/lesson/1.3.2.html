Instance, Class, and Static Methods

Intuition

Instance methods know about a specific object, class methods know about the class itself (useful for creating objects), and static methods know about neither, they just live in the class for organizational purposes.

Python classes support three method types: instance methods (operate on an instance), class methods (operate on the class), and static methods (independent utility functions namespaced to the class).

class Pizza:
    def __init__(self, radius: float, toppings: list):
        self.radius = radius
        self.toppings = toppings

    # Instance method: accesses self
    def area(self) -> float:
        import math
        return math.pi * self.radius ** 2

    # Class method: accesses cls, often used as factory
    @classmethod
    def margherita(cls, radius: float) -> "Pizza":
        return cls(radius, ["mozzarella", "tomato", "basil"])

    # Static method: no access to self or cls
    @staticmethod
    def validate_topping(topping: str) -> bool:
        forbidden = ["pineapple"]
        return topping.lower() not in forbidden

p = Pizza.margherita(12)
print(p.area())
print(Pizza.validate_topping("mushroom"))  # True
  • Instance methods take self; they can access and modify instance state
  • Class methods take cls; ideal for alternative constructors (factory methods)
  • Static methods take neither; they're utility functions scoped to the class
  • Class methods respect inheritance: cls refers to the subclass when called on one

Exercises

  1. Date Factory Methods

    Create a Date class with year, month, day. Add a @classmethod from_string(cls, s) that parses 'YYYY-MM-DD' and a @staticmethod is_leap_year(year).

    Hint

    Use s.split('-') to parse the string. Leap year: divisible by 4, not by 100 unless by 400.

  2. Method Type Identification

    Create a class with one of each method type. Call each method from both the class and an instance. Note which calls work and which fail.

    Hint

    Calling an instance method from the class requires passing an instance manually.

Common Problems & Solutions

1 TypeError: method() takes 0 positional arguments but 1 was given
Cause: Defining a method without `self` parameter but calling it on an instance, which passes `self` automatically.
Solution: Add `self` as the first parameter, or use `@staticmethod` if `self` isn't needed.
2 classmethod factory returns parent class instead of subclass
Cause: Using the class name directly (e.g., `return Pizza(...)`) instead of `cls(...)`.
Solution: Always use `cls(...)` in classmethods so subclasses get instances of their own type.
