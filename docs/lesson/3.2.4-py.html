JSONB: Storing and Querying Semi-Structured Data

Intuition

JSONB gives you the flexibility of a document database inside PostgreSQL. You get schemaless storage for parts of your data while keeping relational integrity everywhere else.

JSONB: Semi-Structured Data in PostgreSQL

JSONB stores binary-parsed JSON, enabling flexible schemas while retaining the power of SQL queries, indexing, and constraints.

CREATE TABLE events (
    id SERIAL PRIMARY KEY,
    data JSONB NOT NULL
);

INSERT INTO events (data) VALUES
  ('{"type": "click", "page": "/home", "user_id": 42}');

-- Access nested fields
SELECT data->>'type' AS event_type,
       data->'metadata'->>'source' AS source
FROM events;

-- Containment query (uses GIN index)
SELECT * FROM events
WHERE data @> '{"type": "click"}';

-- Update nested value
UPDATE events
SET data = jsonb_set(data, '{metadata,processed}', 'true')
WHERE id = 1;

Key Takeaways: - Use -> to get JSON object, ->> to get text. Use @> for containment checks. - GIN indexes on JSONB columns make containment and existence queries fast. - jsonb_set, jsonb_insert, and || operator allow partial updates without rewriting the whole value.

Exercises

  1. Query Nested JSONB

    Given a JSONB column 'profile' with nested address fields, write a query to find all users in a specific city.

    Hint

    Use profile->'address'->>'city' = 'Seattle' or containment: profile @> '{"address":{"city":"Seattle"}}'.

  2. JSONB Aggregation

    Build a JSONB object from query results using jsonb_agg and jsonb_build_object.

    Hint

    SELECT jsonb_agg(jsonb_build_object('name', name, 'total', total)) FROM orders GROUP BY category.

Common Problems & Solutions

1 JSONB query is slow despite having data indexed.
Cause: Using ->> operator with = instead of @> containment, which bypasses the GIN index.
Solution: Rewrite as a containment query with @>, or create an expression index on (data->>'field').
2 Lost data when updating JSONB with || operator.
Cause: The || merge is shallow and replaces entire nested objects rather than deep-merging.
Solution: Use jsonb_set() for targeted nested updates, or use || only for top-level keys.
