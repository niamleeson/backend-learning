Protocol and Structural Subtyping

Intuition

Protocol is Python's way of formalizing duck typing. If it has the right methods with the right signatures, it fits the Protocol, no inheritance needed.

Protocol and Structural Subtyping

Protocols enable structural subtyping (duck typing with static checks). Instead of requiring inheritance, a class satisfies a Protocol if it has the required methods and attributes.

from typing import Protocol, runtime_checkable

class Drawable(Protocol):
    def draw(self, x: int, y: int) -> None: ...

class Circle:
    def draw(self, x: int, y: int) -> None:
        print(f"Circle at ({x}, {y})")

class Square:
    def draw(self, x: int, y: int) -> None:
        print(f"Square at ({x}, {y})")

def render(shape: Drawable) -> None:
    shape.draw(0, 0)  # Both Circle and Square work!

render(Circle())  # OK - Circle has draw()
render(Square())  # OK - Square has draw()

@runtime_checkable
class Sized(Protocol):
    def __len__(self) -> int: ...

assert isinstance([1, 2], Sized)  # True at runtime

Key Takeaways: - Protocols define interfaces without requiring inheritance - Classes satisfy a Protocol by having matching methods (structural subtyping) - Use @runtime_checkable to enable isinstance() checks - Protocols are ideal for dependency injection and decoupled designs - Prefer Protocol over ABC when you cannot control the implementing classes

Exercises

  1. Serializable Protocol

    Define a Serializable Protocol with to_dict(self) -> dict[str, Any] and from_dict(cls, data: dict) -> Self. Write two classes that satisfy it and a function save_all(items: list[Serializable]).

    Hint

    Classes do not need to explicitly inherit from Serializable. Just implement to_dict and from_dict with matching signatures.

  2. Comparable Protocol

    Create a Comparable Protocol with __lt__ and __eq__. Write a generic min_item(items: list[T]) -> T where T is bound to Comparable.

    Hint

    Use TypeVar('T', bound=Comparable) to restrict T to types implementing comparison methods.

Common Problems & Solutions

1 `isinstance()` check fails with `TypeError: Protocols with non-method members don't support isinstance()`.
Cause: A Protocol with attributes (not just methods) cannot be runtime-checked even with `@runtime_checkable`.
Solution: Only use `@runtime_checkable` on Protocols that define methods. For attribute-only protocols, rely on static checking.
2 mypy says a class does not implement a Protocol despite having the right methods.
Cause: Method signatures do not match exactly (e.g., different parameter names or types).
Solution: Ensure method signatures match precisely, including parameter types and return type annotations.
