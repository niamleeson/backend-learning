Configuration Management and Environment Variables

Intuition

Configuration management is about separating what changes between environments from the code itself. Environment variables act as the bridge between your deployment environment and your application's behavior.

Configuration Management and Environment Variables

Flask applications need different settings for development, testing, and production. Flask's config system supports loading from Python objects, files, and environment variables. The python-dotenv package helps manage environment-specific values.

import os
from flask import Flask

class BaseConfig:
    SECRET_KEY = os.environ.get('SECRET_KEY', 'change-me')
    SQLALCHEMY_TRACK_MODIFICATIONS = False

class DevelopmentConfig(BaseConfig):
    DEBUG = True
    SQLALCHEMY_DATABASE_URI = 'sqlite:///dev.db'

class ProductionConfig(BaseConfig):
    DEBUG = False
    SQLALCHEMY_DATABASE_URI = os.environ['DATABASE_URL']

class TestingConfig(BaseConfig):
    TESTING = True
    SQLALCHEMY_DATABASE_URI = 'sqlite:///:memory:'

def create_app(config_class=None):
    app = Flask(__name__)

    if config_class is None:
        env = os.environ.get('FLASK_ENV', 'development')
        config_map = {
            'development': DevelopmentConfig,
            'production': ProductionConfig,
            'testing': TestingConfig
        }
        config_class = config_map.get(env, DevelopmentConfig)

    app.config.from_object(config_class)
    app.config.from_prefixed_env()  # loads FLASK_* env vars
    return app

Key Takeaways: - Use config classes with inheritance to share common settings and override per environment. - Never hardcode secrets; load them from environment variables or a .env file. - app.config.from_prefixed_env() (Flask 2.2+) auto-loads FLASK_-prefixed environment variables into config.

Exercises

  1. Dotenv Configuration

    Install python-dotenv, create a .env file with SECRET_KEY and DATABASE_URL, and modify your app factory to load these values automatically when the app starts.

    Hint

    Flask automatically loads .env files if python-dotenv is installed. You can also call load_dotenv() explicitly in your factory.

  2. Config Validation

    Write a function that runs after app creation and raises an error if required config keys (SECRET_KEY, SQLALCHEMY_DATABASE_URI) are missing or set to default placeholder values.

    Hint

    Iterate over a list of required keys and check app.config.get(key). Raise RuntimeError with a helpful message for missing values.

Common Problems & Solutions

1 KeyError when accessing os.environ['DATABASE_URL'] in production.
Cause: The environment variable is not set in the deployment environment, and os.environ[] raises KeyError for missing keys.
Solution: Use os.environ.get('DATABASE_URL') with a default or validate required variables at startup with a clear error message.
2 Changes to .env file are not picked up by the running app.
Cause: Environment variables from .env are loaded once at startup. Changes require a restart.
Solution: Restart the Flask development server after modifying .env. For auto-reload, ensure FLASK_DEBUG=1 is set so the reloader picks up file changes.
