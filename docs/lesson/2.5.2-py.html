functools: partial, lru_cache, wraps, reduce

Intuition

functools is your toolbox for function manipulation. partial pre-fills arguments, lru_cache remembers results, and wraps keeps decorators transparent. They make functions more composable and efficient.

functools: partial, lru_cache, wraps, and reduce

The functools module provides higher-order functions and tools for working with callables. These are essential building blocks for functional Python.

from functools import partial, lru_cache, wraps, reduce
import time

# partial: fix some arguments of a function
def power(base: int, exponent: int) -> int:
    return base ** exponent

square = partial(power, exponent=2)
cube = partial(power, exponent=3)
print(square(5))  # 25
print(cube(3))    # 27

# lru_cache: memoize expensive function calls
@lru_cache(maxsize=128)
def fibonacci(n: int) -> int:
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(100))  # Instant with caching
print(fibonacci.cache_info())  # Hits, misses, size

# wraps: preserve metadata in decorators
def timer(func):
    @wraps(func)  # Preserves __name__, __doc__
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.perf_counter()-start:.4f}s")
        return result
    return wrapper

@timer
def slow_func():
    """A slow function."""
    time.sleep(0.1)

Key Takeaways: - partial creates specialized versions of functions by fixing arguments - lru_cache memoizes results; use maxsize=None for unlimited cache - Always use @wraps(func) in decorators to preserve the wrapped function's metadata - reduce accumulates values; provide an initial value for empty iterables - Use cache_info() and cache_clear() to monitor and manage lru_cache

Exercises

  1. Build a Config Loader with partial

    Create a generic load_config(path, format, strict) function. Use partial to create load_yaml, load_json, and load_toml variants with format pre-filled.

    Hint

    Each variant is partial(load_config, format='yaml') etc. The caller only needs to pass path and optionally strict.

  2. Memoized Web Fetcher

    Write a function that fetches a URL and returns the response body. Use lru_cache to cache results for up to 32 URLs. Add a way to invalidate the cache for a specific URL.

    Hint

    You cannot invalidate a single key in lru_cache. Use a dict-based cache instead, or call cache_clear() for full invalidation.

Common Problems & Solutions

1 `lru_cache` raises `TypeError: unhashable type: 'list'` for list arguments.
Cause: lru_cache uses arguments as dictionary keys, which requires all arguments to be hashable.
Solution: Convert mutable arguments to tuples before passing: `func(tuple(my_list))`. Or use a custom cache with string keys.
2 Decorator hides the original function's name and docstring.
Cause: Without `@wraps`, the wrapper function replaces the original's `__name__` and `__doc__`.
Solution: Add `@functools.wraps(func)` to the inner wrapper function in every decorator you write.
