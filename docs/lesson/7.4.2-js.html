Joi: Validation Schemas and Integration Patterns

Intuition

If Zod is a bouncer with a typed guest list, Joi is an experienced customs officer who has been checking passports for years. The officer knows every edge case, can apply different rules based on your nationality (conditional validation with .when()), and speaks every language when explaining why you cannot enter (custom error messages). Joi may not predict your type like Zod, but its experience handling real-world complexity is unmatched.

Joi: Validation Schemas and Integration Patterns

What Is Joi?

Joi is one of the most established validation libraries in the Node.js ecosystem. Originally part of the hapi framework, Joi has been the go-to validation solution for Express and other frameworks for over a decade. Unlike Zod, Joi predates TypeScript's dominance and focuses purely on runtime validation with an expressive, chainable API.

Installing Joi

npm install joi

Joi has zero peer dependencies and works in any Node.js environment. For TypeScript projects, types are included in the package.

Joi vs Zod: When to Use Which

┌──────────────────┬───────────────────┬──────────────────┐
│     Feature      │       Joi         │       Zod        │
├──────────────────┼───────────────────┼──────────────────┤
│ Type inference   │ Limited           │ Excellent        │
│ Bundle size      │ ~150KB            │ ~50KB            │
│ Ecosystem        │ Mature, huge      │ Growing fast     │
│ Error messages   │ Very customizable │ Good defaults    │
│ Async validation │ Built-in          │ parseAsync       │
│ Conditional      │ .when() powerful  │ Requires refine  │
│ Best for         │ JS projects,      │ TS projects,     │
│                  │ complex rules     │ type safety      │
└──────────────────┴───────────────────┴──────────────────┘

Use Joi when you are working with a plain JavaScript project, need highly customized error messages, or need powerful conditional validation logic. Use Zod when TypeScript type inference is a priority.

Basic Schema Definition

import Joi from 'joi';

// Primitive schemas
const nameSchema = Joi.string().min(1).max(100).required();
const ageSchema = Joi.number().integer().min(0).max(150);
const emailSchema = Joi.string().email();
const isActiveSchema = Joi.boolean();

// Validate
const { error, value } = nameSchema.validate('Alice');
console.log(value);  // 'Alice'
console.log(error);  // undefined (no error)

const result = nameSchema.validate(42);
console.log(result.error.details[0].message);
// '"value" must be a string'

Object Schemas

const userSchema = Joi.object({
  name: Joi.string().min(1).max(100).required(),
  email: Joi.string().email().required(),
  age: Joi.number().integer().min(13).max(120).required(),
  role: Joi.string().valid('user', 'admin', 'moderator').default('user'),
  bio: Joi.string().max(500).allow('').optional(),
  tags: Joi.array().items(Joi.string()).max(10).default([]),
});

const input = {
  name: 'Alice',
  email: 'alice@example.com',
  age: 28,
};

const { error, value } = userSchema.validate(input);
// value now has defaults applied: { ...input, role: 'user', tags: [] }

Validation Options

Joi's validation behavior is highly configurable:

const schema = Joi.object({
  name: Joi.string().required(),
  email: Joi.string().email().required(),
});

const input = {
  name: 'Alice',
  email: 'alice@example.com',
  extraField: 'should this be here?',
};

// Default: allows unknown keys
schema.validate(input);  // passes, extraField is in value

// stripUnknown: remove unknown keys silently
schema.validate(input, { stripUnknown: true });
// value = { name: 'Alice', email: 'alice@example.com' }

// abortEarly: false -- collect ALL errors, not just the first
schema.validate({ name: '', email: 'bad' }, { abortEarly: false });
// error.details contains both errors

// allowUnknown: false (reject unknown keys)
schema.validate(input, { allowUnknown: false });
// error: '"extraField" is not allowed'

Conditional Validation with .when()

Joi's conditional validation is one of its strongest features:

const paymentSchema = Joi.object({
  method: Joi.string().valid('credit_card', 'bank_transfer', 'paypal').required(),

  // Conditionally required based on method
  cardNumber: Joi.string().creditCard().when('method', {
    is: 'credit_card',
    then: Joi.required(),
    otherwise: Joi.forbidden(),
  }),

  cardExpiry: Joi.string().pattern(/^(0[1-9]|1[0-2])\/\d{2}$/).when('method', {
    is: 'credit_card',
    then: Joi.required(),
    otherwise: Joi.forbidden(),
  }),

  bankAccount: Joi.string().when('method', {
    is: 'bank_transfer',
    then: Joi.required(),
    otherwise: Joi.forbidden(),
  }),

  paypalEmail: Joi.string().email().when('method', {
    is: 'paypal',
    then: Joi.required(),
    otherwise: Joi.forbidden(),
  }),

  amount: Joi.number().positive().precision(2).required(),
  currency: Joi.string().length(3).uppercase().required(),
});

// Valid credit card payment
paymentSchema.validate({
  method: 'credit_card',
  cardNumber: '4111111111111111',
  cardExpiry: '12/25',
  amount: 99.99,
  currency: 'USD',
});

// Invalid: credit_card method but providing bankAccount
paymentSchema.validate({
  method: 'credit_card',
  bankAccount: '12345',  // Error: "bankAccount" is not allowed
  amount: 99.99,
  currency: 'USD',
});

Cross-Field Validation

const dateRangeSchema = Joi.object({
  startDate: Joi.date().iso().required(),
  endDate: Joi.date().iso().min(Joi.ref('startDate')).required(),
});

// Field references
const accountSchema = Joi.object({
  password: Joi.string().min(8).required(),
  confirmPassword: Joi.string()
    .valid(Joi.ref('password'))
    .required()
    .messages({
      'any.only': 'Passwords do not match',
    }),
}).with('password', 'confirmPassword');

// .with() means if password is present, confirmPassword must be too
// .without() means if A is present, B must not be
// .xor() means exactly one of the fields must be present
// .or() means at least one must be present
// .nand() means not both can be present

Custom Error Messages

Joi provides extensive message customization:

const schema = Joi.object({
  username: Joi.string()
    .alphanum()
    .min(3)
    .max(30)
    .required()
    .messages({
      'string.base': 'Username must be text',
      'string.alphanum': 'Username can only contain letters and numbers',
      'string.min': 'Username must be at least {#limit} characters',
      'string.max': 'Username cannot exceed {#limit} characters',
      'any.required': 'Username is required',
    }),

  email: Joi.string()
    .email()
    .required()
    .messages({
      'string.email': 'Please provide a valid email address',
      'any.required': 'Email is required',
    }),

  age: Joi.number()
    .integer()
    .min(13)
    .max(120)
    .messages({
      'number.base': 'Age must be a number',
      'number.min': 'You must be at least {#limit} years old',
    }),
});

Custom Validators

// Using .custom() for complex validations
const passwordSchema = Joi.string()
  .min(8)
  .custom((value, helpers) => {
    if (!/[A-Z]/.test(value)) {
      return helpers.error('password.uppercase');
    }
    if (!/[0-9]/.test(value)) {
      return helpers.error('password.number');
    }
    if (!/[!@#$%^&*]/.test(value)) {
      return helpers.error('password.special');
    }
    return value;  // return the value if valid
  })
  .messages({
    'password.uppercase': 'Password must contain at least one uppercase letter',
    'password.number': 'Password must contain at least one number',
    'password.special': 'Password must contain at least one special character (!@#$%^&*)',
  });

// Async custom validator (e.g., database lookup)
const uniqueEmailSchema = Joi.string()
  .email()
  .external(async (value) => {
    const exists = await db.user.findUnique({ where: { email: value } });
    if (exists) {
      throw new Error('Email already registered');
    }
    return value;
  });

// Use validateAsync for external validators
const result = await schema.validateAsync(input, { externals: true });

Express.js Integration

import Joi from 'joi';
import express from 'express';

// Validation middleware factory
function validateRequest(schemas) {
  return (req, res, next) => {
    const errors = {};
    const options = {
      abortEarly: false,
      stripUnknown: true,
      errors: { wrap: { label: false } },  // cleaner error messages
    };

    for (const [source, schema] of Object.entries(schemas)) {
      if (!schema) continue;

      const { error, value } = schema.validate(req[source], options);

      if (error) {
        errors[source] = error.details.reduce((acc, detail) => {
          const key = detail.path.join('.');
          if (!acc[key]) acc[key] = [];
          acc[key].push(detail.message);
          return acc;
        }, {});
      } else {
        req[source] = value;  // replace with validated/stripped data
      }
    }

    if (Object.keys(errors).length > 0) {
      return res.status(400).json({
        status: 'error',
        message: 'Validation failed',
        errors,
      });
    }

    next();
  };
}

// Define route schemas
const createUserSchemas = {
  body: Joi.object({
    name: Joi.string().trim().min(1).max(100).required(),
    email: Joi.string().email().lowercase().required(),
    password: Joi.string().min(8).max(128).required(),
    role: Joi.string().valid('user', 'admin').default('user'),
  }),
};

const listUsersSchemas = {
  query: Joi.object({
    page: Joi.number().integer().min(1).default(1),
    limit: Joi.number().integer().min(1).max(100).default(20),
    search: Joi.string().trim().max(200).allow(''),
    role: Joi.string().valid('user', 'admin'),
    sortBy: Joi.string().valid('name', 'email', 'createdAt').default('createdAt'),
    order: Joi.string().valid('asc', 'desc').default('desc'),
  }),
};

const getUserSchemas = {
  params: Joi.object({
    id: Joi.string().uuid().required(),
  }),
};

// Apply to routes
const app = express();
app.use(express.json());

const router = express.Router();

router.post('/users', validateRequest(createUserSchemas), (req, res) => {
  const { name, email, password, role } = req.body;
  res.status(201).json({ message: 'User created', data: { name, email, role } });
});

router.get('/users', validateRequest(listUsersSchemas), (req, res) => {
  const { page, limit, search, role, sortBy, order } = req.query;
  res.json({ page, limit, search, role, sortBy, order });
});

router.get('/users/:id', validateRequest(getUserSchemas), (req, res) => {
  const { id } = req.params;
  res.json({ id });
});

app.use('/api', router);

NestJS Integration with Joi

NestJS has built-in Joi support for configuration validation:

import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';
import Joi from 'joi';

@Module({
  imports: [
    ConfigModule.forRoot({
      validationSchema: Joi.object({
        NODE_ENV: Joi.string().valid('development', 'production', 'test').required(),
        PORT: Joi.number().port().default(3000),
        DATABASE_URL: Joi.string().uri().required(),
        JWT_SECRET: Joi.string().min(32).required(),
        REDIS_URL: Joi.string().uri().required(),
        LOG_LEVEL: Joi.string().valid('error', 'warn', 'info', 'debug').default('info'),
      }),
      validationOptions: {
        abortEarly: false,
        allowUnknown: true,
      },
    }),
  ],
})
export class AppModule {}

Alternatives and Extending Joi

// Custom Joi extension for phone numbers
const customJoi = Joi.extend((joi) => ({
  type: 'phone',
  base: joi.string(),
  messages: {
    'phone.format': '{{#label}} must be a valid E.164 phone number',
  },
  validate(value, helpers) {
    if (!/^\+[1-9]\d{1,14}$/.test(value)) {
      return { value, errors: helpers.error('phone.format') };
    }
    return { value };
  },
}));

// Use the extension
const contactSchema = Joi.object({
  name: Joi.string().required(),
  phone: customJoi.phone().required(),
});

Error Handling Patterns

// Centralized error handler that understands Joi errors
function errorHandler(err, req, res, next) {
  if (err.isJoi) {
    const errors = err.details.reduce((acc, detail) => {
      const key = detail.path.join('.');
      if (!acc[key]) acc[key] = [];
      acc[key].push(detail.message);
      return acc;
    }, {});

    return res.status(400).json({
      status: 'error',
      message: 'Validation failed',
      errors,
    });
  }

  // Handle other errors
  res.status(500).json({
    status: 'error',
    message: 'Internal server error',
  });
}

app.use(errorHandler);

Summary

Joi is a battle-tested validation library with exceptional conditional validation through .when(), deeply customizable error messages, and a mature ecosystem. While it lacks Zod's TypeScript type inference, its rich API and long track record make it an excellent choice for JavaScript projects and complex validation scenarios where conditional logic and custom messages are paramount.

Exercises

  1. Payment Form Validator

    Build a Joi schema for a payment form that validates differently based on the payment method (credit_card, bank_transfer, crypto). For credit cards, require card number (Luhn check), expiry (MM/YY format, not expired), and CVV. For bank transfers, require account number and routing number. For crypto, require a wallet address. Use .when() for conditional fields and .forbidden() to prevent irrelevant fields.

    Hint

    Use Joi.string().creditCard() for card validation. Use .when('method', { is: 'credit_card', then: Joi.required(), otherwise: Joi.forbidden() }) for conditional fields.

  2. Express Validation Middleware with Joi

    Create a validation middleware that uses Joi schemas for body, query, and params. It should collect all errors (abortEarly: false), strip unknown fields, and return a structured error response. Test with at least 5 different invalid inputs and verify the error format is consistent.

    Hint

    Set options { abortEarly: false, stripUnknown: true, errors: { wrap: { label: false } } } and group error.details by path.

  3. Configuration Validator

    Create a Joi schema that validates your application's environment variables on startup. Include NODE_ENV, PORT, DATABASE_URL, JWT_SECRET (min 32 chars), JWT_EXPIRY (duration pattern), REDIS_URL, SMTP_HOST (required in production only), and LOG_LEVEL. If any required variable is missing, the app should print all errors and exit.

    Hint

    Use Joi.when() referencing NODE_ENV for production-only requirements. Use Joi.string().uri({ scheme: ['postgres', 'postgresql'] }) for DATABASE_URL.

  4. Joi vs Zod Comparison

    Implement the same validation schema in both Joi and Zod: a complex order schema with items array (each with product ID, quantity, price), shipping address (with conditional fields based on country), and discount code (optional, specific format). Compare the developer experience, error output, and TypeScript integration.

    Hint

    Focus on conditional validation (.when() in Joi vs .refine()/.superRefine() in Zod) and how each handles type inference.

Common Problems & Solutions

1 Joi strips fields that were expected in the handler
Cause: Using { stripUnknown: true } removes any field not defined in the schema. If you forgot to include a field in the schema, it silently disappears.
Solution: Double-check that your schema includes all expected fields. During development, use { allowUnknown: true } first, then switch to { stripUnknown: true } once the schema is complete. Log the validated value to verify all fields are present.
2 Only the first validation error is returned to the client
Cause: Joi's default behavior is abortEarly: true, which stops validation at the first error.
Solution: Pass { abortEarly: false } as a validation option: `schema.validate(input, { abortEarly: false })`. This collects all validation errors so the client can fix everything in one attempt.
3 Joi error messages include ugly quotes and label prefixes
Cause: By default, Joi wraps field names in quotes in error messages, producing output like '"name" is required'.
Solution: Set errors.wrap.label to false in options: `schema.validate(input, { errors: { wrap: { label: false } } })`. For even cleaner messages, use the .messages() method on each field to provide fully custom error text.
