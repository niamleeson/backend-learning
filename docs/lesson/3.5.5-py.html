Advanced Queries: Hybrid Properties, Query API, and Filtering

Intuition

Hybrid properties give your model methods that speak two languages: Python for instances and SQL for queries. You define the logic once and use it everywhere.

Advanced Queries and Hybrid Properties

Hybrid properties work at both the Python instance level and SQL expression level, enabling clean model APIs that translate to efficient queries.

from sqlalchemy.ext.hybrid import hybrid_property
from sqlalchemy import select, Numeric

class Product(Base):
    __tablename__ = 'products'
    id: Mapped[int] = mapped_column(primary_key=True)
    price: Mapped[float] = mapped_column(Numeric(10, 2))
    discount: Mapped[float] = mapped_column(Numeric(5, 2), default=0)

    @hybrid_property
    def sale_price(self):
        return self.price * (1 - self.discount)

    @sale_price.expression
    @classmethod
    def sale_price(cls):
        return cls.price * (1 - cls.discount)

    @hybrid_property
    def is_expensive(self):
        return self.price > 100

    @is_expensive.expression
    @classmethod
    def is_expensive(cls):
        return cls.price > 100

# Use in queries just like a column
stmt = select(Product).where(Product.sale_price < 50)
cheap = session.execute(stmt).scalars().all()

Key Takeaways: - Hybrid properties define logic once that works in both Python and SQL contexts. - The @expression classmethod defines how the property translates to a SQL expression. - Useful for computed fields like full_name, age, sale_price without denormalizing.

Exercises

  1. Full Name Hybrid

    Create a User model with first_name and last_name, plus a full_name hybrid property that works in both Python and SQL.

    Hint

    Python: f'{self.first_name} {self.last_name}'. Expression: cls.first_name + ' ' + cls.last_name.

  2. Status Hybrid Method

    Create a hybrid_method on an Order model that accepts a status string and returns True/False in Python, or a filter expression in SQL.

    Hint

    @hybrid_method def has_status(self, status): return self.status == status.

Common Problems & Solutions

1 Hybrid property works in Python but fails in SQL queries.
Cause: Missing @expression classmethod, so SQLAlchemy cannot translate it to SQL.
Solution: Define the @.expression classmethod that returns a SQL-compatible expression.
2 Hybrid property with complex Python logic cannot be expressed in SQL.
Cause: Some Python operations have no direct SQL equivalents.
Solution: Use case() expressions for conditionals, func.concat() for strings, or move logic to a SQL function.
