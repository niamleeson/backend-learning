GCP and Heroku: Alternative Platforms

Intuition

Heroku is like a fully furnished apartment -- move in immediately but pay premium rent. AWS is like buying land and building from scratch -- maximum control but requires expertise. GCP Cloud Run is like a modern co-working space -- bring your laptop (container) and everything else is handled for you.

GCP and Heroku: Alternative Platforms

AWS isn't the only option. GCP offers strong data/ML services, while Heroku provides the simplest path from code to production.

Heroku - Simplicity First

heroku create my-python-app
git push heroku main
heroku addons:create heroku-postgresql:essential-0
heroku config:set SECRET_KEY=mysecret
heroku logs --tail

# Procfile
# web: gunicorn app:create_app()
# worker: celery -A tasks worker
# release: python manage.py migrate

GCP Cloud Run - Serverless Containers

gcloud builds submit --tag gcr.io/my-project/myapp
gcloud run deploy myapp \
  --image gcr.io/my-project/myapp \
  --platform managed \
  --allow-unauthenticated

GCP Cloud Functions

import functions_framework

@functions_framework.http
def hello(request):
    name = request.args.get('name', 'World')
    return f'Hello, {name}!'

gcloud functions deploy hello --runtime python312 --trigger-http

Platform Comparison

Feature Heroku AWS GCP
Ease of use Easiest Complex Moderate
Cost Higher per unit Pay-as-you-go Pay-as-you-go
Scale Limited Unlimited Unlimited
Best for MVPs, small apps Enterprise Data/ML workloads

Common Mistakes

  • Using Heroku free tier for production (sleeps after inactivity)
  • Not setting Cloud Run minimum instances, causing cold starts

Key Takeaways

  • Heroku is fastest to production but most expensive at scale
  • Cloud Run: bring your container, get autoscaling
  • Choose your cloud based on team expertise, not hype

Exercises

  1. Deploy to Heroku

    Deploy a FastAPI application to Heroku with a PostgreSQL database. Create a Procfile, runtime.txt, and configure environment variables. Verify the app works with the managed database.

    Hint

    Use gunicorn with uvicorn workers in the Procfile: web: uvicorn app.main:app --host 0.0.0.0 --port $PORT

  2. Deploy to Cloud Run

    Containerize a Python application and deploy it to GCP Cloud Run. Configure environment variables, set up a Cloud SQL PostgreSQL instance, and connect them.

    Hint

    Use the Cloud SQL Auth Proxy sidecar in Cloud Run to connect to Cloud SQL securely.

  3. Cost Comparison

    For a small application serving 100k requests/month with a PostgreSQL database, calculate the monthly cost on Heroku, AWS (EC2+RDS), and GCP (Cloud Run+Cloud SQL). Document trade-offs.

    Hint

    Check the pricing calculators for each platform. Consider compute, database, bandwidth, and storage costs.

Common Problems & Solutions

1 Heroku app crashes with H10 error
Symptoms: ['H10 - App crashed', 'Web process failed to bind to $PORT']
Cause: The application failed to bind to the PORT environment variable that Heroku assigns dynamically.
Solution: Use `--port $PORT` in your Procfile. Never hardcode a port number on Heroku.

Procfile

web: uvicorn app.main:app --host 0.0.0.0 --port $PORT

2 Cloud Run cold starts are too slow
Symptoms: ['First request takes 5-15 seconds', 'Intermittent timeouts']
Cause: Cloud Run scales to zero by default. The first request after idle time must start a new container.
Solution: Set minimum instances to 1: `gcloud run services update myapp --min-instances 1`. Optimize your Docker image size and application startup time.
