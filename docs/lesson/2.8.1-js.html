Document Model and CRUD Operations

Intuition

MongoDB is like a filing cabinet where each folder can contain different types of documents. In a relational database, every row must have the exact same columns, like a spreadsheet. In MongoDB, each document can have whatever fields it needs, like a folder that might contain a letter, a photo, or a contract — each with different properties.

Document Model and CRUD Operations

Introduction

MongoDB is a document database that stores data as flexible, JSON-like documents (BSON). Unlike relational databases with rigid schemas and rows, MongoDB lets each document have a different structure. This makes it excellent for rapid prototyping, content management, IoT data, and domains where the data shape varies.

When to Use MongoDB vs PostgreSQL

MongoDB shines when:
  - Schema varies between documents (product catalogs with different attributes)
  - You need flexible, nested data (CMS content, user-generated data)
  - Read-heavy workloads with denormalized data
  - Rapid prototyping (no migrations needed)
  - Horizontal scaling (built-in sharding)

PostgreSQL shines when:
  - Data has clear relationships (users, orders, payments)
  - You need ACID transactions across multiple tables
  - Complex queries with JOINs and aggregations
  - Data integrity is critical (financial, medical)
  - You need full-text search + relational queries

Setting Up the MongoDB Driver

npm install mongodb

import { MongoClient, Db, Collection, ObjectId } from 'mongodb';

const client = new MongoClient(process.env.MONGODB_URI || 'mongodb://localhost:27017', {
  maxPoolSize: 10,
  minPoolSize: 2,
  serverSelectionTimeoutMS: 5000,
});

await client.connect();
const db: Db = client.db('myapp');

// Collections (like tables)
const users: Collection = db.collection('users');
const products: Collection = db.collection('products');
const orders: Collection = db.collection('orders');

Document Structure

// MongoDB document (BSON)
const userDoc = {
  _id: new ObjectId(),            // auto-generated unique ID
  name: 'Alice',
  email: 'alice@example.com',
  age: 30,
  address: {                       // nested object
    street: '123 Main St',
    city: 'New York',
    state: 'NY',
    zip: '10001',
  },
  tags: ['developer', 'gamer'],   // array
  preferences: {                   // flexible structure
    theme: 'dark',
    notifications: { email: true, push: false },
  },
  createdAt: new Date(),
  updatedAt: new Date(),
};

CREATE Operations

// Insert one
const result = await users.insertOne({
  name: 'Alice',
  email: 'alice@example.com',
  role: 'customer',
  createdAt: new Date(),
});
console.log('Inserted ID:', result.insertedId);

// Insert many
const result = await products.insertMany([
  { name: 'Widget', price: 9.99, category: 'tools', inStock: true },
  { name: 'Gadget', price: 19.99, category: 'electronics', inStock: true },
  { name: 'Doohickey', price: 4.99, category: 'tools', inStock: false },
]);
console.log('Inserted:', result.insertedCount);

READ Operations

// Find one
const user = await users.findOne({ email: 'alice@example.com' });

// Find by ID
const user = await users.findOne({ _id: new ObjectId('507f1f77bcf86cd799439011') });

// Find many with filter
const activeUsers = await users.find({ role: 'customer' }).toArray();

// Projection (select fields)
const names = await users.find(
  { role: 'customer' },
  { projection: { name: 1, email: 1, _id: 0 } }  // include/exclude
).toArray();

// Comparison operators
const expensive = await products.find({
  price: { $gt: 10 },           // greater than
  inStock: { $eq: true },       // equal
}).toArray();

// Logical operators
const result = await users.find({
  $or: [
    { role: 'admin' },
    { age: { $gte: 21 } },
  ],
  $and: [
    { status: 'active' },
    { email: { $exists: true } },
  ],
}).toArray();

// Array queries
const jsDevs = await users.find({
  tags: 'javascript',                    // array contains
  tags: { $all: ['javascript', 'react'] }, // contains all
  tags: { $size: 3 },                     // exact array length
  'tags.0': 'javascript',                // first element
}).toArray();

// Nested object queries (dot notation)
const nyUsers = await users.find({
  'address.city': 'New York',
  'preferences.notifications.email': true,
}).toArray();

// Regex
const alices = await users.find({
  name: { $regex: /^alice/i },
}).toArray();

// Sorting, limiting, skipping
const page = await products.find({ inStock: true })
  .sort({ price: 1 })          // 1 = asc, -1 = desc
  .skip(20)                     // offset
  .limit(10)                    // limit
  .toArray();

// Count
const total = await products.countDocuments({ inStock: true });
const estimated = await products.estimatedDocumentCount(); // faster, approximate

UPDATE Operations

// Update one
await users.updateOne(
  { _id: userId },
  {
    $set: { name: 'Alice Updated', updatedAt: new Date() },
    $inc: { loginCount: 1 },
  }
);

// Update operators
await users.updateOne(
  { _id: userId },
  {
    $set: { 'address.city': 'Boston' },    // set field
    $unset: { tempField: '' },              // remove field
    $inc: { age: 1 },                       // increment
    $push: { tags: 'newTag' },              // add to array
    $pull: { tags: 'oldTag' },              // remove from array
    $addToSet: { tags: 'uniqueTag' },       // add if not exists
    $rename: { oldName: 'newName' },        // rename field
    $currentDate: { updatedAt: true },      // set to now
  }
);

// Update many
const result = await products.updateMany(
  { category: 'tools' },
  { $mul: { price: 1.1 } }  // 10% price increase
);
console.log('Modified:', result.modifiedCount);

// FindOneAndUpdate (atomic read + update)
const updated = await users.findOneAndUpdate(
  { _id: userId },
  { $set: { name: 'New Name' }, $inc: { version: 1 } },
  { returnDocument: 'after' }  // return updated document
);

// Upsert
await users.updateOne(
  { email: 'alice@example.com' },
  {
    $set: { name: 'Alice', lastSeen: new Date() },
    $setOnInsert: { createdAt: new Date(), role: 'customer' },
  },
  { upsert: true }
);

DELETE Operations

// Delete one
await users.deleteOne({ _id: userId });

// Delete many
const result = await sessions.deleteMany({
  expiresAt: { $lt: new Date() },
});
console.log('Deleted:', result.deletedCount);

// FindOneAndDelete
const deleted = await users.findOneAndDelete({ _id: userId });
console.log('Deleted user:', deleted);

Indexes

// Create indexes
await users.createIndex({ email: 1 }, { unique: true });
await products.createIndex({ name: 'text', description: 'text' }); // text search
await products.createIndex({ category: 1, price: -1 }); // compound
await orders.createIndex({ createdAt: 1 }, { expireAfterSeconds: 86400 * 90 }); // TTL index

// List indexes
const indexes = await users.indexes();

TypeScript with MongoDB

interface User {
  _id?: ObjectId;
  name: string;
  email: string;
  role: 'customer' | 'admin';
  tags: string[];
  address?: { city: string; state: string };
  createdAt: Date;
}

const users = db.collection<User>('users');

const user = await users.findOne({ email: 'alice@example.com' });
// user is WithId<User> | null — fully typed!

await users.insertOne({
  name: 'Bob',
  email: 'bob@example.com',
  role: 'customer',
  tags: ['new'],
  createdAt: new Date(),
});

Exercises

  1. Product Catalog CRUD

    Build CRUD operations for a product catalog where products have different attributes based on category (electronics have specs.ram, clothing has specs.size). Demonstrate the flexibility of the document model.

    Hint

    Use nested objects for category-specific attributes. Query with dot notation: {'specs.ram': {$gte: 8}}. Use $exists to find products with specific attributes.

  2. Bulk Operations

    Implement a bulk import that: inserts new products, updates existing ones (matched by SKU), and deletes discontinued ones. Use bulkWrite for efficiency. Handle partial failures.

    Hint

    Use bulkWrite with an array of insertOne, updateOne, and deleteOne operations. Set ordered: false for parallel execution. Check result.insertedCount, modifiedCount, deletedCount.

  3. Query Builder

    Build a dynamic query builder for MongoDB that accepts a filter object with: search (text), priceRange, categories (array), inStock (boolean), sortBy, page, limit. Convert to MongoDB query syntax.

    Hint

    Build the filter object conditionally. Use $text for search (requires text index). Use $in for categories. Combine with $and.

Common Problems & Solutions

1 ObjectId comparison fails — documents not found
Cause: Comparing a string with an ObjectId. `{ _id: '507f...' }` won't match `{ _id: ObjectId('507f...') }`.
Solution: Always convert string IDs to ObjectId: `{ _id: new ObjectId(stringId) }`. Handle invalid ObjectId strings with try-catch.
2 Query returns all documents instead of filtered results
Cause: Empty filter object `{}` matches all documents. A typo in the field name silently matches nothing, causing the $and to be effectively empty.
Solution: Log your query filter before executing. Use TypeScript interfaces to catch field name typos. Test with explain() to verify the query plan.
3 Updates silently do nothing
Cause: The filter doesn't match any documents, or the update operators are wrong. updateOne returns modifiedCount: 0 but doesn't throw.
Solution: Always check result.matchedCount and modifiedCount. Use findOneAndUpdate with returnDocument: 'after' to see the result. Log the filter and update for debugging.
