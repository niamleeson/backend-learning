Import System: Modules, Packages, __init__.py

Intuition

The import system is just a file lookup mechanism: Python searches directories in sys.path for modules and uses init.py files to recognize packages.

Python's import system resolves modules and packages using sys.path. Understanding __init__.py, relative imports, and the module search order is essential for structuring projects.

# Project structure:
# mypackage/
#   __init__.py        <- makes it a package
#   utils.py
#   subpkg/
#     __init__.py
#     helpers.py

# Absolute imports (preferred)
from mypackage.utils import clean_data
from mypackage.subpkg.helpers import format_output

# Relative imports (within the package only)
# In mypackage/subpkg/helpers.py:
from ..utils import clean_data      # go up one level
from . import other_module           # same package

# __init__.py can re-export for cleaner API
# mypackage/__init__.py:
from .utils import clean_data
  • __init__.py marks a directory as a Python package. It can be empty or re-export names.
  • Python searches sys.path in order: script directory, PYTHONPATH, site-packages.
  • Relative imports use dots (. for current package, .. for parent) and only work inside packages.
  • Avoid circular imports by restructuring or using local imports inside functions.

Exercises

  1. Create a Package

    Create a package mathtools with modules arithmetic.py (add, subtract) and geometry.py (area_circle). Add an __init__.py that re-exports all functions.

    Hint

    In __init__.py, use from .arithmetic import add, subtract and similar for geometry.

  2. Inspect sys.path

    Write a script that prints each entry in sys.path, one per line, and identifies which entry contains the json standard library module.

    Hint

    Use import json; print(json.__file__) to find its location and match against sys.path entries.

Common Problems & Solutions

1 `ImportError: attempted relative import with no known parent package`.
Cause: Running a module directly (`python file.py`) sets `__name__` to `'__main__'`, breaking relative imports.
Solution: Run as a module from the parent: `python -m mypackage.subpkg.helpers`.
2 Circular import causes `ImportError` or `AttributeError` on partially loaded module.
Cause: Module A imports B which imports A — at that point A is only partially initialized.
Solution: Move the import inside the function that needs it, or refactor shared code into a third module.
