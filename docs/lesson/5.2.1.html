Depends() and Function Dependencies

Intuition

Dependency injection in FastAPI works like a smart assistant that prepares everything your endpoint needs before it runs. Instead of repeating setup code in every endpoint, you declare what you need and FastAPI assembles it for you.

Depends() and Function Dependencies

FastAPI's dependency injection system uses Depends() to declare reusable components that are resolved automatically when a request is handled. Dependencies can be functions or callable classes.

from fastapi import FastAPI, Depends, Query, HTTPException

app = FastAPI()

def common_parameters(
    skip: int = Query(0, ge=0),
    limit: int = Query(10, le=100)
) -> dict:
    return {"skip": skip, "limit": limit}

def get_current_user(token: str = Query(...)) -> dict:
    if token != "secret-token":
        raise HTTPException(status_code=401, detail="Invalid token")
    return {"user_id": 1, "username": "johndoe"}

@app.get("/items/")
async def read_items(
    commons: dict = Depends(common_parameters),
    user: dict = Depends(get_current_user)
):
    return {"user": user, "pagination": commons}

@app.get("/users/")
async def read_users(commons: dict = Depends(common_parameters)):
    return {"pagination": commons}

Key Takeaways: - Depends() injects the return value of a callable into your endpoint - Dependencies are reusable across multiple endpoints - Dependencies can raise HTTPException to short-circuit request handling - Dependencies are resolved per-request, ensuring isolation between requests

Exercises

  1. Class-Based Dependency

    Create a callable class Paginator that accepts page and size query parameters, validates them, and provides offset and limit properties. Use it as a dependency in two different endpoints.

    Hint

    Define __init__ with the query parameters and use Depends(Paginator). FastAPI will call the class constructor, and the instance becomes the injected value.

  2. Database Session Dependency

    Create a dependency function using yield that creates a fake database connection dict, yields it to the endpoint, and prints 'Connection closed' after the request completes.

    Hint

    Use def get_db(): db = {"connected": True}; yield db; print("Connection closed"). The code after yield runs during cleanup.

Common Problems & Solutions

1 Dependency function is called but returns None
Cause: The dependency function uses `yield` but is missing the actual yield statement, or it returns early before yielding.
Solution: Ensure the function contains a `yield` statement that yields the actual value. If using `return`, switch from a generator dependency to a regular function dependency.
2 Dependency is instantiated only once instead of per-request
Cause: The dependency is being created at module level and passed as a value rather than as a callable to `Depends()`.
Solution: Pass the callable itself, not an instance: use `Depends(get_db)` not `Depends(get_db())`. The parentheses inside Depends cause immediate evaluation.
