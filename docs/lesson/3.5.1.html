Declarative Models and Mapped Columns

Intuition

Declarative models let you define database tables as Python classes. Each instance is a row, each class attribute is a column, and Python types map to SQL types.

Declarative Models and Mapped Columns

SQLAlchemy ORM maps Python classes to database tables using a declarative style. Each class attribute becomes a column.

from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column, relationship
from sqlalchemy import String, DateTime, func
from datetime import datetime

class Base(DeclarativeBase):
    pass

class User(Base):
    __tablename__ = 'users'

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(100))
    email: Mapped[str] = mapped_column(String(255), unique=True)
    active: Mapped[bool] = mapped_column(default=True)
    created_at: Mapped[datetime] = mapped_column(
        DateTime, server_default=func.now()
    )
    bio: Mapped[str | None] = mapped_column(String)

    posts: Mapped[list['Post']] = relationship(back_populates='author')

    def __repr__(self):
        return f'<User(id={self.id}, name={self.name})>'

Base.metadata.create_all(engine)

Key Takeaways: - Mapped[type] with mapped_column() is the modern SQLAlchemy 2.0 pattern replacing Column(). - Type annotations control nullability: Mapped[str] is NOT NULL, Mapped[str | None] is nullable. - tablename sets the table name; Base.metadata.create_all() generates DDL.

Exercises

  1. Create a Product Model

    Define a Product model with id, name, price (Numeric), description (optional), and created_at using Mapped and mapped_column.

    Hint

    price: Mapped[float] = mapped_column(Numeric(10, 2)), description: Mapped[str | None].

  2. Mixin for Timestamps

    Create a TimestampMixin class with created_at and updated_at columns that can be inherited by any model.

    Hint

    Define a mixin class with Mapped columns, then class MyModel(TimestampMixin, Base).

Common Problems & Solutions

1 Model class not creating the expected table.
Cause: Forgot to inherit from Base, or __tablename__ is missing.
Solution: Ensure the model inherits from your DeclarativeBase subclass and defines __tablename__.
2 Column is nullable when it should not be.
Cause: Using Mapped[str | None] or Optional[str] which makes the column nullable.
Solution: Use Mapped[str] (without None) to enforce NOT NULL at the database level.
