File Uploads, Filtering, and Advanced Features

Production APIs often need file uploads, complex filtering, sorting, and search. FastAPI handles multipart file uploads natively and supports flexible query patterns for data retrieval.

from fastapi import FastAPI, UploadFile, File, Query, Depends
from enum import Enum
import shutil
from pathlib import Path

app = FastAPI()

class SortOrder(str, Enum):
    asc = "asc"
    desc = "desc"

@app.post("/upload")
async def upload_file(
    file: UploadFile = File(..., description="File to upload"),
    user=Depends(get_current_user),
):
    upload_dir = Path("uploads") / str(user.id)
    upload_dir.mkdir(parents=True, exist_ok=True)
    file_path = upload_dir / file.filename
    with file_path.open("wb") as buffer:
        shutil.copyfileobj(file.file, buffer)
    return {"filename": file.filename, "size": file_path.stat().st_size}

@app.get("/tasks/search")
async def search_tasks(
    q: str | None = Query(None, min_length=1),
    priority: int | None = Query(None, ge=1, le=5),
    sort_by: str = Query("created_at"),
    order: SortOrder = Query(SortOrder.desc),
    db=Depends(get_db), user=Depends(get_current_user),
):
    query = select(Task).where(Task.owner_id == user.id)
    if q:
        query = query.where(Task.title.ilike(f"%{q}%"))
    if priority:
        query = query.where(Task.priority == priority)
    col = getattr(Task, sort_by, Task.created_at)
    query = query.order_by(col.desc() if order == SortOrder.desc else col)
    result = await db.execute(query)
    return result.scalars().all()

Key Takeaways: - UploadFile handles multipart file uploads with streaming support for large files - Use Enum classes for validated sort/filter options - Build dynamic queries by conditionally adding .where() clauses - Always validate and sanitize filenames to prevent path traversal attacks

Exercises

  1. Multi-File Upload with Validation

    Create an endpoint that accepts multiple file uploads, validates file types (only images), limits total upload size to 10MB, and returns metadata for each uploaded file including a generated thumbnail URL.

    Hint

    Use files: list[UploadFile] = File(...) for multiple files. Check file.content_type.startswith('image/') and track cumulative size. Use file.seek(0, 2) and file.tell() to get size before reading.

  2. Advanced Filter Builder

    Create a reusable filter dependency class that accepts arbitrary filter parameters like priority__gte=3, status__in=active,pending, title__contains=report and builds the corresponding SQLAlchemy query conditions dynamically.

    Hint

    Parse query parameters by splitting on __ to get field name and operator. Map operators to SQLAlchemy methods: gte -> >=, in -> .in_(), contains -> .ilike(). Return a list of filter conditions.

Common Problems & Solutions

1 Large file upload causes server to run out of memory
Cause: The entire file is being read into memory at once using `await file.read()` instead of streaming to disk.
Solution: Use `shutil.copyfileobj(file.file, destination)` to stream the file to disk in chunks. For async streaming, read in chunks with `while chunk := await file.read(1024 * 1024): dest.write(chunk)`.
2 SQL injection through dynamic sort_by parameter
Cause: User input is used directly to construct column names in the query without validation.
Solution: Validate `sort_by` against an explicit allowlist of column names. Use `getattr(Model, sort_by, None)` and return 400 if the attribute doesn't exist. Never interpolate user input into raw SQL.
