OAuth2 Basics with Passport.js

Intuition

OAuth2 is like checking into a hotel with your passport. Instead of the hotel (your app) running its own ID verification system, it trusts the government (Google/GitHub) to vouch for you. The hotel just needs to verify the passport is real and record your details.

OAuth2 Basics with Passport.js

Introduction

OAuth2 allows users to log in to your application using their existing accounts from providers like Google, GitHub, Facebook, and others. Instead of managing passwords, you delegate authentication to a trusted provider. Passport.js makes OAuth2 integration straightforward with dedicated strategies for each provider.

How OAuth2 Works

OAuth2 Authorization Code Flow:

1. User clicks "Login with Google"
       |
       v
2. Your app redirects to Google's auth page
   https://accounts.google.com/o/oauth2/auth?
     client_id=YOUR_ID&
     redirect_uri=http://localhost:3000/auth/google/callback&
     scope=email+profile&
     response_type=code
       |
       v
3. User logs in and grants permission on Google
       |
       v
4. Google redirects back to YOUR app with a code
   http://localhost:3000/auth/google/callback?code=abc123
       |
       v
5. Your server exchanges the code for an access token
   (server-to-server request to Google)
       |
       v
6. Your server uses the access token to fetch user profile
       |
       v
7. User is authenticated, session/JWT created

Setup: Google OAuth2

npm install passport-google-oauth20
npm install -D @types/passport-google-oauth20

Register Your App with Google

  1. Go to Google Cloud Console
  2. Create OAuth2 credentials
  3. Set authorized redirect URI: http://localhost:3000/auth/google/callback
  4. Save the Client ID and Client Secret in .env
# .env
GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-client-secret
GOOGLE_CALLBACK_URL=http://localhost:3000/auth/google/callback

Configure the Strategy

// src/config/passport.ts
import passport from 'passport';
import { Strategy as GoogleStrategy, Profile } from 'passport-google-oauth20';
import { config } from './index';
import { userService } from '../modules/users/users.service';

passport.use(new GoogleStrategy(
  {
    clientID: config.google.clientId,
    clientSecret: config.google.clientSecret,
    callbackURL: config.google.callbackUrl,
    scope: ['email', 'profile'],
  },
  async (accessToken, refreshToken, profile: Profile, done) => {
    try {
      // Find or create user
      let user = await userService.findByGoogleId(profile.id);

      if (!user) {
        user = await userService.create({
          googleId: profile.id,
          email: profile.emails?.[0]?.value || '',
          name: profile.displayName,
          avatar: profile.photos?.[0]?.value,
          provider: 'google',
        });
      }

      done(null, user);
    } catch (error) {
      done(error as Error);
    }
  }
));

// Serialization (same as before)
passport.serializeUser((user: any, done) => done(null, user.id));
passport.deserializeUser(async (id: number, done) => {
  try {
    const user = await userService.findById(id);
    done(null, user);
  } catch (err) {
    done(err);
  }
});

OAuth Routes

// src/modules/auth/oauth.routes.ts
import { Router } from 'express';
import passport from 'passport';

const router = Router();

// Initiate Google OAuth flow
router.get('/google',
  passport.authenticate('google', { scope: ['email', 'profile'] })
);

// Google callback handler
router.get('/google/callback',
  passport.authenticate('google', {
    failureRedirect: '/login?error=auth_failed',
  }),
  (req, res) => {
    // Successful authentication
    // For SPAs: redirect with token or set cookie
    res.redirect(`${config.frontendUrl}/auth/callback?success=true`);
  }
);

OAuth + JWT (For API-First Apps)

// Instead of sessions, generate JWT after OAuth
router.get('/google/callback',
  passport.authenticate('google', { session: false, failureRedirect: '/login' }),
  (req, res) => {
    const user = req.user as any;
    const tokens = tokenService.generateTokenPair({
      userId: user.id,
      email: user.email,
      role: user.role,
    });

    // Redirect to frontend with tokens
    const params = new URLSearchParams({
      accessToken: tokens.accessToken,
      refreshToken: tokens.refreshToken,
    });
    res.redirect(`${config.frontendUrl}/auth/callback?${params}`);
  }
);

import { Strategy as GitHubStrategy } from 'passport-github2';

passport.use(new GitHubStrategy(
  {
    clientID: config.github.clientId,
    clientSecret: config.github.clientSecret,
    callbackURL: config.github.callbackUrl,
    scope: ['user:email'],
  },
  async (accessToken: string, refreshToken: string, profile: any, done: Function) => {
    try {
      let user = await userService.findByGithubId(profile.id);
      if (!user) {
        user = await userService.create({
          githubId: profile.id,
          email: profile.emails?.[0]?.value || '',
          name: profile.displayName || profile.username,
          avatar: profile.photos?.[0]?.value,
          provider: 'github',
        });
      }
      done(null, user);
    } catch (error) {
      done(error);
    }
  }
));

// Routes
router.get('/github', passport.authenticate('github', { scope: ['user:email'] }));
router.get('/github/callback',
  passport.authenticate('github', { failureRedirect: '/login' }),
  (req, res) => res.redirect(config.frontendUrl)
);

Account Linking

Users may sign up with email/password then later want to link their Google account:

// Link Google account to existing user
router.get('/link/google',
  isAuthenticated,
  passport.authorize('google', { scope: ['email', 'profile'] })
);

router.get('/link/google/callback',
  isAuthenticated,
  passport.authorize('google', { failureRedirect: '/settings' }),
  async (req, res) => {
    const googleProfile = (req as any).account;
    await userService.linkGoogleAccount(req.user!.id, googleProfile.id);
    res.redirect('/settings?linked=google');
  }
);

Security Considerations

  1. Validate the state parameter to prevent CSRF attacks (Passport handles this)
  2. Use HTTPS in production for callback URLs
  3. Validate email from OAuth provider (not all providers verify emails)
  4. Handle account conflicts (same email from different providers)
  5. Store provider tokens securely if you need ongoing API access
  6. Limit scopes to only what you need

Common Pitfalls

Callback URL Mismatch

Error: redirect_uri_mismatch

The callback URL in your code MUST exactly match the one registered
with the OAuth provider, including protocol, hostname, port, and path.

Registered: http://localhost:3000/auth/google/callback
Code:       http://localhost:3000/auth/google/callback  // Must match exactly!

Not Handling Email Conflicts

// User signs up with email, then tries Google OAuth with same email
// You need a strategy:
// 1. Link accounts automatically (if email is verified)
// 2. Reject and suggest linking manually
// 3. Create a separate account (confusing for users)

Summary

OAuth2 with Passport.js lets users authenticate via external providers like Google and GitHub. The Authorization Code flow redirects users to the provider, then exchanges a code for an access token on your server. Passport strategies handle the complexity, letting you focus on user creation and session/token management. Always handle account linking and email conflicts gracefully.

Exercises

  1. Google OAuth Integration

    Set up Google OAuth2 with Passport.js. Register your app in Google Cloud Console, configure the strategy, create login/callback routes, and handle user creation for new Google users. Use sessions for auth state.

    Hint

    Set up credentials in Google Cloud Console under APIs & Services > Credentials. The callback URL must match exactly. Use passport.authenticate('google') for both the login route and the callback route.

  2. Multi-Provider Auth

    Extend your auth system to support both Google and GitHub OAuth alongside local (email/password) authentication. Handle the case where a user registers with one method and later wants to use another with the same email.

    Hint

    Add a 'provider' column to your users table. Check for existing users by email during OAuth callback. If found with a different provider, offer to link accounts.

  3. OAuth + JWT for SPAs

    Implement OAuth authentication that returns JWTs instead of using sessions. After successful OAuth callback, generate access and refresh tokens and redirect to the frontend with tokens in URL parameters or set them as httpOnly cookies.

    Hint

    Use passport.authenticate('google', { session: false }) to skip session creation. Generate tokens in the callback handler and redirect to frontend.

Common Problems & Solutions

1 OAuth callback returns 'redirect_uri_mismatch' error
Cause: The callback URL in your code does not exactly match the one registered in the OAuth provider's console.
Solution: Check that protocol (http vs https), hostname, port, and path match exactly. In development, use http://localhost:3000. In production, use your actual domain with https.
2 User email is undefined from OAuth profile
Cause: The email scope was not requested, or the provider does not guarantee email in the profile.
Solution: Request the email scope explicitly: scope: ['email', 'profile']. For GitHub, emails may need a separate API call. Always handle the case where email is undefined.
3 CORS errors when redirecting to OAuth provider from SPA
Cause: Trying to initiate OAuth from a frontend fetch/axios call instead of a full page redirect.
Solution: OAuth requires a full browser redirect, not an AJAX call. Use window.location.href = '/auth/google' or an anchor tag. The server then redirects to Google's auth page.
