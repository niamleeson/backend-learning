Arguments: Positional, Keyword, Default

Intuition

Arguments are like filling out a form. Positional args fill fields left to right. Keyword args let you fill any field by name. Defaults pre-fill fields you commonly leave unchanged.

Python supports positional, keyword, and default arguments, giving you flexible function signatures. Understanding the rules prevents confusing errors.

def create_user(name, email, role="viewer", active=True):
    return {"name": name, "email": email, "role": role, "active": active}

# Positional
create_user("Alice", "alice@example.com")

# Keyword (order doesn't matter)
create_user(email="bob@example.com", name="Bob", role="admin")

# Positional-only (/) and keyword-only (*)
def strict(pos_only, /, normal, *, kw_only):
    return pos_only + normal + kw_only

strict(1, 2, kw_only=3)        # OK
# strict(pos_only=1, 2, kw_only=3)  # Error!

# Default values are evaluated once
def add_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items
  • Positional args come first, then keyword args with defaults
  • Use / to enforce positional-only, * to enforce keyword-only
  • Never use mutable default arguments (lists, dicts); use None instead
  • Keyword arguments improve readability at the call site

Exercises

  1. Flexible Formatter

    Write a function format_name(first, last, /, *, title=None, suffix=None) that builds a full name string. first and last must be positional.

    Hint

    Concatenate parts conditionally: f"{title + ' ' if title else ''}{first} {last}{', ' + suffix if suffix else ''}".

  2. Mutable Default Trap

    Demonstrate the mutable default bug: write a function with def append_to(item, target=[]) and call it 3 times. Then fix it.

    Hint

    Call append_to(1), append_to(2), append_to(3) and notice the list grows across calls.

Common Problems & Solutions

1 SyntaxError: non-default argument follows default argument
Cause: A parameter without a default value is placed after one with a default.
Solution: Put all required parameters before optional ones in the signature.
2 TypeError: got multiple values for argument
Cause: Passing the same argument both positionally and as a keyword.
Solution: Pass each argument only once. Use positional-only (`/`) to avoid ambiguity.
