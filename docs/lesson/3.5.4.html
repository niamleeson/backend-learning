Eager vs Lazy Loading: joinedload, selectinload, subqueryload

Intuition

Lazy loading is like ordering one item at a time from a menu. Eager loading is like ordering everything up front. Choose based on whether you know you will need the related data.

Eager vs Lazy Loading

Loading strategies control when related objects are fetched from the database. The wrong strategy leads to N+1 queries or loading too much data.

from sqlalchemy.orm import joinedload, selectinload, lazyload
from sqlalchemy import select

# N+1 problem with lazy loading (default)
with Session(engine) as session:
    authors = session.execute(select(Author)).scalars().all()
    for a in authors:
        print(a.books)  # each access fires a separate SELECT

# Fix: eager load with joinedload (single JOIN)
stmt = select(Author).options(joinedload(Author.books))
authors = session.execute(stmt).unique().scalars().all()

# selectinload: separate SELECT ... WHERE id IN (...)
stmt = select(Author).options(selectinload(Author.books))

# Nested eager loading
stmt = select(Author).options(
    selectinload(Author.books).joinedload(Book.publisher)
)

# Override at relationship level
class Author(Base):
    __tablename__ = 'authors'
    id: Mapped[int] = mapped_column(primary_key=True)
    books: Mapped[list['Book']] = relationship(
        back_populates='author', lazy='selectin'
    )

Key Takeaways: - Lazy loading (default) fires one query per access. Fine for single objects, terrible in loops. - joinedload uses a SQL JOIN; best for single-child or mandatory relationships. - selectinload fires a second SELECT with IN; best for collections to avoid cartesian products.

Exercises

  1. Eliminate N+1 Queries

    Given code that loops over authors and prints their books (causing N+1), refactor it to use selectinload.

    Hint

    Add .options(selectinload(Author.books)) to the select statement before executing.

  2. Nested Eager Loading

    Load authors with their books and each book's reviews in two levels of eager loading.

    Hint

    selectinload(Author.books).selectinload(Book.reviews) chains loading strategies.

Common Problems & Solutions

1 joinedload produces duplicate parent rows.
Cause: A JOIN with a one-to-many relationship multiplies parent rows.
Solution: Call .unique() on the result before .scalars().all() to deduplicate, or use selectinload instead.
2 Loading too much data with overly aggressive eager loading.
Cause: Eager loading large collections when only the parent data is needed.
Solution: Use raiseload() to make accidental lazy loads raise an error, forcing explicit loading decisions.
