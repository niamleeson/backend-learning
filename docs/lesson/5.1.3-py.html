Response Models and Status Codes

Intuition

Response models are your API's security filter, ensuring only intended data leaves your server. Status codes are the universal language between client and server, telling the client exactly what happened without parsing the body.

Response Models and Status Codes

FastAPI lets you control exactly what data your API returns and which HTTP status codes it uses. Response models filter sensitive fields, and explicit status codes communicate intent clearly.

from fastapi import FastAPI, HTTPException, status
from pydantic import BaseModel

app = FastAPI()

class UserIn(BaseModel):
    username: str
    password: str
    email: str

class UserOut(BaseModel):
    username: str
    email: str

users_db: dict[str, UserIn] = {}

@app.post("/users/", response_model=UserOut, status_code=status.HTTP_201_CREATED)
async def create_user(user: UserIn):
    if user.username in users_db:
        raise HTTPException(
            status_code=status.HTTP_409_CONFLICT,
            detail="Username already exists"
        )
    users_db[user.username] = user
    return user  # password is filtered out by response_model

@app.delete("/users/{username}", status_code=status.HTTP_204_NO_CONTENT)
async def delete_user(username: str):
    if username not in users_db:
        raise HTTPException(status_code=404, detail="User not found")
    del users_db[username]

Key Takeaways: - response_model filters the output, preventing sensitive fields like passwords from leaking - Use status constants for readability instead of raw integers - HTTPException lets you return error status codes with detail messages - 201 for creation, 204 for deletion with no content, 404 for not found

Common Problems & Solutions

1 Response model strips fields that should be included
Cause: The response model does not include all the fields you want to return, or `response_model_exclude_unset=True` is filtering out fields with default values.
Solution: Ensure the response model includes all desired fields. Remove `response_model_exclude_unset` or explicitly set the fields in your returned object to avoid them being treated as unset.
2 HTTPException returns HTML instead of JSON
Cause: A middleware or ASGI server is intercepting the response before FastAPI's exception handler processes it.
Solution: Ensure FastAPI's exception handlers are not overridden incorrectly. Check that middleware is not catching and re-raising exceptions in a way that bypasses FastAPI's JSON error formatting.
