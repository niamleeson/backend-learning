SSL/TLS Termination and HTTPS Setup

Intuition

SSL termination at Nginx is like having a security guard at the building entrance who handles all the ID checks and pat-downs. Once visitors pass through security, they can walk freely inside the building (internal network). The employees (your app) don't need to worry about security checks -- that's all handled at the door.

SSL/TLS Termination and HTTPS Setup

HTTPS is mandatory for production. SSL/TLS termination at Nginx means Nginx handles encryption, and your Python app receives plain HTTP.

Let's Encrypt with Certbot

sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d myapp.com -d www.myapp.com
sudo certbot renew --dry-run  # Test auto-renewal

Nginx HTTPS Configuration

server {
    listen 80;
    server_name myapp.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name myapp.com;

    ssl_certificate /etc/letsencrypt/live/myapp.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/myapp.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;

    add_header Strict-Transport-Security "max-age=63072000" always;
    add_header X-Content-Type-Options nosniff;
    add_header X-Frame-Options DENY;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Testing SSL

curl -vI https://myapp.com
openssl s_client -connect myapp.com:443 | openssl x509 -noout -dates
# Grade your config at https://www.ssllabs.com/ssltest/

Common Mistakes

  • Forgetting to redirect HTTP to HTTPS
  • Not setting up automatic renewal -- certs expire every 90 days
  • Missing X-Forwarded-Proto, causing redirect loops

Key Takeaways

  • Use Let's Encrypt + Certbot for free, automated SSL
  • Always redirect HTTP to HTTPS and set HSTS headers
  • Terminate SSL at Nginx so your app handles only plain HTTP

Common Problems & Solutions

1 Infinite redirect loop after enabling HTTPS
Symptoms: ['ERR_TOO_MANY_REDIRECTS', 'Browser shows redirect loop']
Cause: The application is checking the protocol and redirecting to HTTPS, but Nginx is proxying as HTTP internally. The app doesn't know it's behind HTTPS.
Solution: Pass the `X-Forwarded-Proto` header from Nginx: `proxy_set_header X-Forwarded-Proto $scheme;`. Configure your framework to trust this header (e.g., Django's SECURE_PROXY_SSL_HEADER).
2 Certificate expired and site shows security warning
Symptoms: ['NET::ERR_CERT_DATE_INVALID', 'Your connection is not private']
Cause: Let's Encrypt certificates expire every 90 days and auto-renewal failed.
Solution: Run `sudo certbot renew` manually. Check renewal timer: `sudo systemctl status certbot.timer`. Test with `sudo certbot renew --dry-run`.
3 Mixed content warnings after HTTPS migration
Cause: Some resources (images, scripts, CSS) are still loaded via HTTP.
Solution: Update all internal URLs to use HTTPS or protocol-relative URLs (//). Use Content-Security-Policy headers to enforce HTTPS. Check browser console for specific mixed content errors.
