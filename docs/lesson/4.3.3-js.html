Interceptors: Logging, Caching, and Response Transformation

Intuition

Interceptors are like a two-way mirror in an interview room. Before the interview (handler), you can prepare the room and set up recording. After the interview, you can review the recording, edit it, add timestamps, or flag issues. The mirror (interceptor) sees everything going in and coming out without the interviewee (handler) knowing.

Interceptors: Logging, Caching, and Response Transformation

Introduction

Interceptors are one of NestJS's most powerful features. Inspired by Aspect-Oriented Programming (AOP), they can: - Execute logic before and after the route handler - Transform the result returned by the handler - Transform exceptions thrown by the handler - Extend basic handler behavior (e.g., caching, logging, timing) - Completely override the handler under certain conditions

Interceptors in the Request Pipeline

Middleware -> Guards -> INTERCEPTORS (before) -> Pipes -> Handler -> INTERCEPTORS (after) -> Filters
                            ^                                              ^
                            |______________________________________________|
                                    Interceptors wrap the handler

Interceptors wrap the handler execution. They see both the incoming request and the outgoing response.

Creating an Interceptor

An interceptor implements NestInterceptor with an intercept() method:

import {
  Injectable, NestInterceptor, ExecutionContext, CallHandler,
} from '@nestjs/common';
import { Observable } from 'rxjs';
import { tap, map } from 'rxjs/operators';

@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const request = context.switchToHttp().getRequest();
    const method = request.method;
    const url = request.url;
    const now = Date.now();

    console.log(`[${method}] ${url} - Request started`);

    return next.handle().pipe(
      tap(() => {
        console.log(`[${method}] ${url} - ${Date.now() - now}ms`);
      }),
    );
  }
}

The key concept: next.handle() returns an Observable of the handler's response. Using RxJS operators, you can transform, delay, cache, or replace the response.

Common Interceptor Patterns

Response Transformation

Wrap all responses in a standard envelope:

export interface ApiResponse<T> {
  data: T;
  timestamp: string;
  path: string;
}

@Injectable()
export class TransformInterceptor<T> implements NestInterceptor<T, ApiResponse<T>> {
  intercept(
    context: ExecutionContext,
    next: CallHandler,
  ): Observable<ApiResponse<T>> {
    const request = context.switchToHttp().getRequest();

    return next.handle().pipe(
      map((data) => ({
        data,
        timestamp: new Date().toISOString(),
        path: request.url,
      })),
    );
  }
}

Before: { "id": 1, "name": "Alice" } After: { "data": { "id": 1, "name": "Alice" }, "timestamp": "2024-...", "path": "/users/1" }

Timeout Interceptor

import { Observable, throwError, TimeoutError } from 'rxjs';
import { catchError, timeout } from 'rxjs/operators';
import { RequestTimeoutException } from '@nestjs/common';

@Injectable()
export class TimeoutInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    return next.handle().pipe(
      timeout(5000),
      catchError((err) => {
        if (err instanceof TimeoutError) {
          return throwError(() => new RequestTimeoutException());
        }
        return throwError(() => err);
      }),
    );
  }
}

Caching Interceptor

@Injectable()
export class CacheInterceptor implements NestInterceptor {
  private cache = new Map<string, { data: any; expiry: number }>();

  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const request = context.switchToHttp().getRequest();

    // Only cache GET requests
    if (request.method !== 'GET') {
      return next.handle();
    }

    const key = request.url;
    const cached = this.cache.get(key);

    if (cached && cached.expiry > Date.now()) {
      console.log(`Cache HIT: ${key}`);
      return of(cached.data);
    }

    console.log(`Cache MISS: ${key}`);
    return next.handle().pipe(
      tap((data) => {
        this.cache.set(key, {
          data,
          expiry: Date.now() + 60000, // 1 minute TTL
        });
      }),
    );
  }
}

Error Mapping Interceptor

@Injectable()
export class ErrorMappingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    return next.handle().pipe(
      catchError((error) => {
        if (error.code === '23505') {
          // PostgreSQL unique violation
          return throwError(
            () => new ConflictException('Resource already exists'),
          );
        }
        if (error.code === '23503') {
          // PostgreSQL foreign key violation
          return throwError(
            () => new BadRequestException('Referenced resource not found'),
          );
        }
        return throwError(() => error);
      }),
    );
  }
}

Serialization Interceptor

Exclude sensitive fields from responses:

import { ClassSerializerInterceptor } from '@nestjs/common';

// Use the built-in ClassSerializerInterceptor with class-transformer
// In your entity:
import { Exclude, Expose } from 'class-transformer';

export class UserEntity {
  id: number;
  name: string;
  email: string;

  @Exclude()
  password: string;

  @Exclude()
  internalNotes: string;

  @Expose()
  get displayName(): string {
    return `${this.name} (${this.email})`;
  }

  constructor(partial: Partial<UserEntity>) {
    Object.assign(this, partial);
  }
}

// In controller:
@Controller('users')
@UseInterceptors(ClassSerializerInterceptor)
export class UsersController {
  @Get(':id')
  findOne(@Param('id') id: string) {
    const user = this.usersService.findOne(id);
    return new UserEntity(user); // password and internalNotes are excluded
  }
}

Performance Monitoring Interceptor

@Injectable()
export class PerformanceInterceptor implements NestInterceptor {
  private readonly logger = new Logger('Performance');

  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const request = context.switchToHttp().getRequest();
    const { method, url } = request;
    const start = process.hrtime.bigint();

    return next.handle().pipe(
      tap(() => {
        const end = process.hrtime.bigint();
        const durationMs = Number(end - start) / 1_000_000;

        if (durationMs > 1000) {
          this.logger.warn(`SLOW: [${method}] ${url} - ${durationMs.toFixed(2)}ms`);
        } else {
          this.logger.log(`[${method}] ${url} - ${durationMs.toFixed(2)}ms`);
        }
      }),
    );
  }
}

Binding Interceptors

// Method level
@UseInterceptors(LoggingInterceptor)
@Get()
findAll() {}

// Controller level
@Controller('users')
@UseInterceptors(LoggingInterceptor)
export class UsersController {}

// Global (main.ts)
app.useGlobalInterceptors(new LoggingInterceptor());

// Global with DI (preferred)
@Module({
  providers: [
    { provide: APP_INTERCEPTOR, useClass: LoggingInterceptor },
    { provide: APP_INTERCEPTOR, useClass: TransformInterceptor },
  ],
})
export class AppModule {}

Multiple global interceptors execute in registration order for the "before" phase and reverse order for the "after" phase (like an onion).

Interceptor Execution Order

Request
  -> Interceptor A (before)
    -> Interceptor B (before)
      -> Handler
    -> Interceptor B (after)
  -> Interceptor A (after)
Response

This is the onion model -- interceptors wrap around each other.

Skipping Interceptors with Metadata

export const SKIP_TRANSFORM = 'skipTransform';
export const SkipTransform = () => SetMetadata(SKIP_TRANSFORM, true);

@Injectable()
export class TransformInterceptor implements NestInterceptor {
  constructor(private reflector: Reflector) {}

  intercept(context: ExecutionContext, next: CallHandler) {
    const skip = this.reflector.getAllAndOverride<boolean>(SKIP_TRANSFORM, [
      context.getHandler(),
      context.getClass(),
    ]);

    if (skip) return next.handle();

    return next.handle().pipe(
      map((data) => ({ data, timestamp: new Date().toISOString() })),
    );
  }
}

// Usage:
@SkipTransform()
@Get('raw')
getRawData() { return { raw: true }; }

Common Pitfalls

  1. Forgetting RxJS -- Interceptors use Observables. You need tap (side effects), map (transform), catchError (handle errors), etc.
  2. Not returning next.handle() -- If you don't call or return next.handle(), the handler never executes.
  3. Mutating response in tap -- tap is for side effects. Use map to transform the response.
  4. Order confusion -- Global interceptors run in registration order. Controller/method interceptors run after global ones.

Exercises

  1. Build a Response Envelope Interceptor

    Create an interceptor that wraps all responses in a standard format: { success: true, data: , meta: { timestamp, requestId, duration } }. For errors, the format should be: { success: false, error: }. Support a @SkipEnvelope() decorator for raw responses.

    Hint

    Use map to transform successful responses. Use catchError to handle errors. Generate requestId with crypto.randomUUID(). Track duration with Date.now().

  2. Implement an ETag Caching Interceptor

    Build an interceptor that generates ETags for GET responses (using a hash of the response body). If the client sends an If-None-Match header matching the ETag, return 304 Not Modified instead of the full response.

    Hint

    Use crypto.createHash('md5').update(JSON.stringify(data)).digest('hex') for ETag generation. Set the ETag header on response. Check If-None-Match in the request.

  3. Create a Request/Response Logger

    Build a comprehensive logging interceptor that logs: HTTP method, URL, request body (redacted sensitive fields), response status code, response time, response size, and user ID (if authenticated). Output as structured JSON for log aggregation.

    Hint

    Create a list of sensitive field names (password, token, secret) to redact. Use tap for logging. Calculate response size from JSON.stringify().length.

Common Problems & Solutions

1 Interceptor doesn't modify the response
Cause: Using `tap()` instead of `map()`. The `tap` operator performs side effects but doesn't modify the stream value.
Solution: Use `map()` to transform the response: `next.handle().pipe(map(data => ({ wrapped: data })))`.

// WRONG: tap doesn't transform\nnext.handle().pipe(tap(data => ({ wrapped: data })))\n\n// CORRECT: map transforms\nnext.handle().pipe(map(data => ({ wrapped: data })))

2 Interceptor runs but handler response is empty
Cause: The interceptor creates a new Observable instead of chaining from `next.handle()`, so the handler never actually executes.
Solution: Always chain from `next.handle()`. The `CallHandler.handle()` triggers the route handler. Without it, the handler never runs.
3 Global interceptor can't inject services
Cause: Registered with `app.useGlobalInterceptors(new MyInterceptor())` which bypasses DI.
Solution: Register via a module: `{ provide: APP_INTERCEPTOR, useClass: MyInterceptor }`. This allows constructor injection.
