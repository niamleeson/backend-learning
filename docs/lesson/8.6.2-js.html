API Gateway and Service Discovery

Intuition

An API Gateway is like the front desk of a hotel. Guests (clients) don't wander the halls looking for housekeeping, room service, or concierge (individual services). They call the front desk, which routes their request to the right department. Service discovery is the hotel's internal phone directory that automatically updates when staff move to different extensions.

API Gateway and Service Discovery

Introduction

In a microservices architecture, clients should not need to know about individual services. An API Gateway provides a single entry point that routes requests to the appropriate service, handles cross-cutting concerns, and simplifies the client interface. Service discovery allows services to find each other dynamically without hardcoded addresses.

API Gateway Pattern

Without Gateway:                     With Gateway:
Client must know all services        Client knows one endpoint

Client ──> User Service :3001        Client ──> API Gateway :443
Client ──> Order Service :3002                    │
Client ──> Payment Service :3003                  ├── /users → User Service
Client ──> Search Service :3004                   ├── /orders → Order Service
                                                  ├── /payments → Payment Service
                                                  └── /search → Search Service

API Gateway with Express

import express from 'express';
import { createProxyMiddleware } from 'http-proxy-middleware';

const app = express();

// Rate limiting
import rateLimit from 'express-rate-limit';
app.use(rateLimit({ windowMs: 60000, max: 100 }));

// Authentication (cross-cutting concern)
app.use('/api', async (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'Unauthorized' });

  try {
    const user = await verifyJWT(token);
    req.headers['X-User-Id'] = user.id;
    req.headers['X-User-Role'] = user.role;
    next();
  } catch {
    res.status(401).json({ error: 'Invalid token' });
  }
});

// Route to services
app.use('/api/users', createProxyMiddleware({
  target: process.env.USER_SERVICE_URL || 'http://user-service:3001',
  pathRewrite: { '^/api/users': '/users' },
  changeOrigin: true,
}));

app.use('/api/orders', createProxyMiddleware({
  target: process.env.ORDER_SERVICE_URL || 'http://order-service:3002',
  pathRewrite: { '^/api/orders': '/orders' },
  changeOrigin: true,
}));

app.use('/api/products', createProxyMiddleware({
  target: process.env.PRODUCT_SERVICE_URL || 'http://product-service:3003',
  pathRewrite: { '^/api/products': '/products' },
  changeOrigin: true,
}));

// Response aggregation
app.get('/api/dashboard', async (req, res) => {
  const userId = req.headers['X-User-Id'];

  const [profile, orders, recommendations] = await Promise.all([
    fetch(`http://user-service:3001/users/${userId}`).then(r => r.json()),
    fetch(`http://order-service:3002/orders?userId=${userId}&limit=5`).then(r => r.json()),
    fetch(`http://product-service:3003/recommendations/${userId}`).then(r => r.json()),
  ]);

  res.json({ profile, recentOrders: orders, recommendations });
});

app.listen(443);

NestJS API Gateway

// gateway/src/main.ts
import { NestFactory } from '@nestjs/core';
import { GatewayModule } from './gateway.module';

async function bootstrap() {
  const app = await NestFactory.create(GatewayModule);
  await app.listen(3000);
}
bootstrap();

// gateway/src/gateway.module.ts
@Module({
  imports: [
    ClientsModule.register([
      {
        name: 'USER_SERVICE',
        transport: Transport.TCP,
        options: { host: 'user-service', port: 3001 },
      },
      {
        name: 'ORDER_SERVICE',
        transport: Transport.TCP,
        options: { host: 'order-service', port: 3002 },
      },
    ]),
  ],
  controllers: [GatewayController],
})
export class GatewayModule {}

// gateway/src/gateway.controller.ts
@Controller('api')
export class GatewayController {
  constructor(
    @Inject('USER_SERVICE') private userClient: ClientProxy,
    @Inject('ORDER_SERVICE') private orderClient: ClientProxy,
  ) {}

  @Get('users/:id')
  getUser(@Param('id') id: string) {
    return this.userClient.send('get_user', { id });
  }

  @Post('orders')
  createOrder(@Body() data: CreateOrderDto, @Req() req) {
    return this.orderClient.send('create_order', {
      ...data,
      userId: req.headers['x-user-id'],
    });
  }

  @Get('dashboard')
  async getDashboard(@Req() req) {
    const userId = req.headers['x-user-id'];

    const [profile, orders] = await Promise.all([
      firstValueFrom(this.userClient.send('get_user', { id: userId })),
      firstValueFrom(this.orderClient.send('get_orders', { userId, limit: 5 })),
    ]);

    return { profile, recentOrders: orders };
  }
}

Client-Side Discovery with Registry

import Redis from 'ioredis';

class ServiceRegistry {
  constructor(redis) {
    this.redis = redis;
    this.ttl = 30; // 30 second heartbeat
  }

  // Service registers itself on startup
  async register(serviceName, host, port) {
    const instanceId = `${host}:${port}`;
    const key = `service:${serviceName}:instances`;

    await this.redis.hset(key, instanceId, JSON.stringify({
      host, port,
      registeredAt: Date.now(),
      lastHeartbeat: Date.now(),
    }));

    // Heartbeat
    const heartbeat = setInterval(async () => {
      await this.redis.hset(key, instanceId, JSON.stringify({
        host, port,
        registeredAt: Date.now(),
        lastHeartbeat: Date.now(),
      }));
    }, (this.ttl / 2) * 1000);

    // Deregister on shutdown
    const cleanup = async () => {
      clearInterval(heartbeat);
      await this.redis.hdel(key, instanceId);
    };

    process.on('SIGTERM', cleanup);
    process.on('SIGINT', cleanup);
  }

  // Client discovers available instances
  async discover(serviceName) {
    const key = `service:${serviceName}:instances`;
    const instances = await this.redis.hgetall(key);

    const healthy = [];
    const now = Date.now();

    for (const [id, data] of Object.entries(instances)) {
      const instance = JSON.parse(data);
      if (now - instance.lastHeartbeat < this.ttl * 1000) {
        healthy.push(instance);
      } else {
        // Remove stale instance
        await this.redis.hdel(key, id);
      }
    }

    return healthy;
  }

  // Get one instance (round-robin)
  async getInstance(serviceName) {
    const instances = await this.discover(serviceName);
    if (instances.length === 0) throw new Error(`No instances of ${serviceName}`);

    const counterKey = `service:${serviceName}:counter`;
    const index = await this.redis.incr(counterKey);
    return instances[index % instances.length];
  }
}

// Usage in service startup
const registry = new ServiceRegistry(redis);
await registry.register('user-service', 'host1', 3001);

// Usage in client
const instance = await registry.getInstance('user-service');
const response = await fetch(`http://${instance.host}:${instance.port}/users/123`);

Common Pitfalls

  1. Gateway becomes a bottleneck: The gateway is a single point of failure. Deploy multiple instances with a load balancer in front.
  2. Too much logic in the gateway: Keep the gateway thin. Business logic belongs in services, not the gateway.
  3. Hardcoded service addresses: Use service discovery or environment variables. Never hardcode IPs.
  4. Missing circuit breaker: If a downstream service is down, the gateway should fail fast instead of hanging.

Summary

API Gateways provide a single entry point for microservices, handling cross-cutting concerns like authentication, rate limiting, and logging. Service discovery enables dynamic service-to-service communication without hardcoded addresses. Use DNS-based discovery in Docker/Kubernetes for simplicity, or build a registry with Redis for more control. Keep the gateway thin and deploy it with redundancy.

Exercises

  1. Express API Gateway

    Build an API Gateway using Express and http-proxy-middleware that routes to 3 backend services. Add JWT authentication, rate limiting, and request logging as middleware.

    Hint

    Use createProxyMiddleware for routing. Add auth middleware before proxy routes. Use express-rate-limit for rate limiting.

  2. Service Registry with Redis

    Implement a service registry where services register on startup, send heartbeats, and deregister on shutdown. Build a discovery client that finds healthy instances with round-robin selection.

    Hint

    Use Redis hashes for instance storage. Track lastHeartbeat timestamps. Remove stale instances.

  3. Response Aggregation

    Build a /dashboard endpoint in the API Gateway that aggregates data from User, Order, and Product services in parallel, with timeout handling for individual service failures.

    Hint

    Use Promise.allSettled to handle partial failures. Return available data with error flags for failed services.

Common Problems & Solutions

1 API Gateway becomes a single point of failure
Cause: Running only one gateway instance means all traffic stops if it goes down.
Solution: Deploy multiple gateway instances behind a load balancer (Nginx, ALB). Make the gateway stateless so any instance can handle any request.
2 Service discovery returns stale instances
Cause: A service instance crashed without deregistering, and its heartbeat has not expired yet.
Solution: Use short heartbeat intervals (10-15s). Implement health check endpoints. Remove instances that miss 2-3 heartbeats.
