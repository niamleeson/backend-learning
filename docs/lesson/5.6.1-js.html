Repository Pattern

Intuition

The Repository Pattern is like a librarian. You never go into the back room to search the shelves yourself (database). Instead, you ask the librarian (repository) to 'find me a book by this author' or 'add this new book to the collection.' If the library reorganizes its shelves (switches databases), your experience at the front desk stays exactly the same.

Repository Pattern

The Repository Pattern abstracts data access behind a clean interface, decoupling your business logic from database specifics. It is one of the most important patterns for building testable, maintainable backend applications.

Why Repository Pattern?

Without it, business logic is tightly coupled to the database:

// BAD: Business logic directly uses Prisma
class PostService {
  async publishPost(id: string) {
    const post = await prisma.post.findUnique({ where: { id } });
    if (!post) throw new Error('Not found');
    if (post.status !== 'DRAFT') throw new Error('Can only publish drafts');

    // What if we switch from Prisma to Drizzle? Or to MongoDB?
    // Every line that calls prisma.* must change.
    return prisma.post.update({
      where: { id },
      data: { status: 'PUBLISHED', publishedAt: new Date() },
    });
  }
}

With the Repository Pattern:

+------------------+     +------------------+     +------------------+
| Controller       |---->| Service          |---->| Repository       |
| (HTTP concern)   |     | (business logic) |     | (data access)    |
+------------------+     +------------------+     +------------------+
                                                         |
                                                         v
                                                  +------------------+
                                                  | Database         |
                                                  | (Prisma/Drizzle) |
                                                  +------------------+

Defining the Interface

// repositories/interfaces/post.repository.interface.ts
export interface IPostRepository {
  findById(id: string): Promise<Post | null>;
  findAll(options: FindAllOptions): Promise<{ posts: Post[]; total: number }>;
  findByAuthor(authorId: string): Promise<Post[]>;
  findByStatus(status: PostStatus): Promise<Post[]>;
  create(data: CreatePostData): Promise<Post>;
  update(id: string, data: UpdatePostData): Promise<Post>;
  delete(id: string): Promise<void>;
  exists(id: string): Promise<boolean>;
}

export interface FindAllOptions {
  page?: number;
  limit?: number;
  status?: PostStatus;
  authorId?: string;
  search?: string;
  orderBy?: string;
  order?: 'asc' | 'desc';
}

// Domain types (independent of ORM)
export interface Post {
  id: string;
  title: string;
  body: string;
  status: PostStatus;
  authorId: string;
  tags: string[];
  createdAt: Date;
  updatedAt: Date;
  publishedAt: Date | null;
}

export type CreatePostData = Pick<Post, 'title' | 'body' | 'authorId' | 'tags'>;
export type UpdatePostData = Partial<Pick<Post, 'title' | 'body' | 'status' | 'tags'>>;

Prisma Implementation

// repositories/prisma/post.repository.ts
import { PrismaClient } from '@prisma/client';
import { IPostRepository, Post, CreatePostData, UpdatePostData, FindAllOptions } from '../interfaces/post.repository.interface';

export class PrismaPostRepository implements IPostRepository {
  constructor(private prisma: PrismaClient) {}

  async findById(id: string): Promise<Post | null> {
    return this.prisma.post.findUnique({ where: { id } });
  }

  async findAll(options: FindAllOptions): Promise<{ posts: Post[]; total: number }> {
    const { page = 1, limit = 20, status, authorId, search, orderBy = 'createdAt', order = 'desc' } = options;
    const where: any = {};

    if (status) where.status = status;
    if (authorId) where.authorId = authorId;
    if (search) {
      where.OR = [
        { title: { contains: search, mode: 'insensitive' } },
        { body: { contains: search, mode: 'insensitive' } },
      ];
    }

    const [posts, total] = await Promise.all([
      this.prisma.post.findMany({
        where,
        skip: (page - 1) * limit,
        take: limit,
        orderBy: { [orderBy]: order },
      }),
      this.prisma.post.count({ where }),
    ]);

    return { posts, total };
  }

  async findByAuthor(authorId: string): Promise<Post[]> {
    return this.prisma.post.findMany({ where: { authorId } });
  }

  async findByStatus(status: PostStatus): Promise<Post[]> {
    return this.prisma.post.findMany({ where: { status } });
  }

  async create(data: CreatePostData): Promise<Post> {
    return this.prisma.post.create({
      data: { ...data, status: 'DRAFT' },
    });
  }

  async update(id: string, data: UpdatePostData): Promise<Post> {
    return this.prisma.post.update({ where: { id }, data });
  }

  async delete(id: string): Promise<void> {
    await this.prisma.post.delete({ where: { id } });
  }

  async exists(id: string): Promise<boolean> {
    const count = await this.prisma.post.count({ where: { id } });
    return count > 0;
  }
}

In-Memory Implementation (for testing)

// repositories/memory/post.repository.ts
export class InMemoryPostRepository implements IPostRepository {
  private posts: Post[] = [];
  private idCounter = 1;

  async findById(id: string): Promise<Post | null> {
    return this.posts.find(p => p.id === id) || null;
  }

  async findAll(options: FindAllOptions): Promise<{ posts: Post[]; total: number }> {
    let filtered = [...this.posts];

    if (options.status) filtered = filtered.filter(p => p.status === options.status);
    if (options.authorId) filtered = filtered.filter(p => p.authorId === options.authorId);
    if (options.search) {
      const q = options.search.toLowerCase();
      filtered = filtered.filter(p =>
        p.title.toLowerCase().includes(q) || p.body.toLowerCase().includes(q)
      );
    }

    const total = filtered.length;
    const page = options.page || 1;
    const limit = options.limit || 20;
    const posts = filtered.slice((page - 1) * limit, page * limit);

    return { posts, total };
  }

  async create(data: CreatePostData): Promise<Post> {
    const post: Post = {
      id: String(this.idCounter++),
      ...data,
      status: 'DRAFT',
      createdAt: new Date(),
      updatedAt: new Date(),
      publishedAt: null,
    };
    this.posts.push(post);
    return post;
  }

  async update(id: string, data: UpdatePostData): Promise<Post> {
    const index = this.posts.findIndex(p => p.id === id);
    if (index === -1) throw new Error('Not found');
    this.posts[index] = { ...this.posts[index], ...data, updatedAt: new Date() };
    return this.posts[index];
  }

  async delete(id: string): Promise<void> {
    this.posts = this.posts.filter(p => p.id !== id);
  }

  async exists(id: string): Promise<boolean> {
    return this.posts.some(p => p.id === id);
  }

  async findByAuthor(authorId: string): Promise<Post[]> {
    return this.posts.filter(p => p.authorId === authorId);
  }

  async findByStatus(status: PostStatus): Promise<Post[]> {
    return this.posts.filter(p => p.status === status);
  }

  // Test helper
  clear() { this.posts = []; }
}

Using in Services

// services/post.service.ts
export class PostService {
  constructor(private postRepo: IPostRepository) {} // depends on interface

  async publishPost(id: string, userId: string): Promise<Post> {
    const post = await this.postRepo.findById(id);
    if (!post) throw new NotFoundError('Post', id);
    if (post.authorId !== userId) throw new ForbiddenError('Not your post');
    if (post.status !== 'DRAFT') throw new BadRequestError('Only drafts can be published');

    return this.postRepo.update(id, {
      status: 'PUBLISHED',
    });
  }
}

Testing with In-Memory Repository

describe('PostService', () => {
  let service: PostService;
  let repo: InMemoryPostRepository;

  beforeEach(() => {
    repo = new InMemoryPostRepository();
    service = new PostService(repo);
  });

  it('should publish a draft post', async () => {
    const post = await repo.create({
      title: 'Test', body: 'Content', authorId: 'user-1', tags: [],
    });

    const published = await service.publishPost(post.id, 'user-1');
    expect(published.status).toBe('PUBLISHED');
  });

  it('should not publish if not the author', async () => {
    const post = await repo.create({
      title: 'Test', body: 'Content', authorId: 'user-1', tags: [],
    });

    await expect(service.publishPost(post.id, 'user-2'))
      .rejects.toThrow('Not your post');
  });
});

NestJS with Dependency Injection

// In NestJS, use custom provider tokens
export const POST_REPOSITORY = Symbol('POST_REPOSITORY');

@Module({
  providers: [
    PostService,
    {
      provide: POST_REPOSITORY,
      useFactory: (prisma: PrismaService) => new PrismaPostRepository(prisma),
      inject: [PrismaService],
    },
  ],
})
export class PostModule {}

@Injectable()
export class PostService {
  constructor(
    @Inject(POST_REPOSITORY) private postRepo: IPostRepository,
  ) {}
}

Exercises

  1. Implement a Generic Base Repository

    Create a generic BaseRepository interface and PrismaBaseRepository implementation that provides findById, findAll, create, update, delete, and count for any Prisma model. Then extend it for a specific UserRepository with custom methods like findByEmail.

    Hint

    Use generics for the entity type. The base class can accept the Prisma model delegate in its constructor.

  2. Test Service with In-Memory Repository

    Build an OrderService with business logic for creating, paying, shipping, and cancelling orders. Write an InMemoryOrderRepository and comprehensive tests for the service using it. No database required.

    Hint

    The in-memory repository stores orders in an array. The service tests should verify state transitions and business rules.

  3. Swap Implementations at Runtime

    Create a NestJS module where the repository implementation can be swapped via configuration: PrismaPostRepository for production, InMemoryPostRepository for testing. Use NestJS custom providers with useFactory.

    Hint

    Use ConfigService to read the implementation type and return the appropriate repository from useFactory.

Common Problems & Solutions

1 Repository interface leaks ORM-specific types
Cause: The interface uses Prisma-specific types (like PrismaPromise or Prisma.PostWhereInput) instead of plain domain types.
Solution: Define domain types (Post, CreatePostData, etc.) that are ORM-independent. The repository interface should only use these types. Map between Prisma types and domain types inside the implementation.
2 Repository becomes a 'god object' with too many methods
Cause: Every data access need is added as a new method on the repository.
Solution: Use a specification/criteria pattern for complex queries. Keep repository methods for common operations and use a query builder for ad-hoc queries.
3 Transactions across repositories are difficult
Cause: Each repository has its own database client instance.
Solution: Pass a transaction client through context or use a Unit of Work pattern that coordinates transactions across repositories.
