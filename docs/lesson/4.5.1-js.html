TypeORM with NestJS: Entities and Repositories

Intuition

TypeORM entities are like blueprints for database tables. Each class property marked with @Column() becomes a column in the table. The Repository is like a specialized filing clerk for that table -- it knows how to find, create, update, and delete records. TypeOrmModule.forFeature() is how you tell NestJS which filing clerks a department (module) needs.

TypeORM with NestJS: Entities and Repositories

Introduction

TypeORM is a mature ORM that supports both Active Record and Data Mapper patterns. NestJS has first-class integration with TypeORM through @nestjs/typeorm, providing seamless entity management, repository injection, and transaction support. While Prisma is gaining popularity, TypeORM remains widely used especially in enterprise NestJS applications.

Installation

npm i @nestjs/typeorm typeorm pg

Configuration

// app.module.ts
import { Module } from '@nestjs/common';
import { TypeOrmModule } from '@nestjs/typeorm';
import { ConfigModule, ConfigService } from '@nestjs/config';

@Module({
  imports: [
    ConfigModule.forRoot({ isGlobal: true }),
    TypeOrmModule.forRootAsync({
      imports: [ConfigModule],
      useFactory: (config: ConfigService) => ({
        type: 'postgres',
        host: config.get('DB_HOST'),
        port: config.get<number>('DB_PORT'),
        username: config.get('DB_USERNAME'),
        password: config.get('DB_PASSWORD'),
        database: config.get('DB_NAME'),
        entities: [__dirname + '/**/*.entity{.ts,.js}'],
        synchronize: config.get('NODE_ENV') !== 'production',
        logging: config.get('NODE_ENV') === 'development',
      }),
      inject: [ConfigService],
    }),
  ],
})
export class AppModule {}

Warning: synchronize: true auto-creates/modifies tables based on entities. Never use this in production -- use migrations instead.

Defining Entities

Entities map to database tables:

// users/entities/user.entity.ts
import {
  Entity, PrimaryGeneratedColumn, Column, CreateDateColumn,
  UpdateDateColumn, OneToMany, Index,
} from 'typeorm';
import { Post } from '../../posts/entities/post.entity';

@Entity('users')
export class User {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column({ length: 100 })
  name: string;

  @Column({ unique: true })
  @Index()
  email: string;

  @Column({ select: false }) // Excluded from SELECT by default
  password: string;

  @Column({ type: 'enum', enum: ['user', 'admin', 'moderator'], default: 'user' })
  role: string;

  @Column({ default: true })
  isActive: boolean;

  @Column({ type: 'jsonb', nullable: true })
  metadata: Record<string, any>;

  @OneToMany(() => Post, (post) => post.author)
  posts: Post[];

  @CreateDateColumn()
  createdAt: Date;

  @UpdateDateColumn()
  updatedAt: Date;
}

// posts/entities/post.entity.ts
import {
  Entity, PrimaryGeneratedColumn, Column, ManyToOne,
  ManyToMany, JoinTable, CreateDateColumn,
} from 'typeorm';
import { User } from '../../users/entities/user.entity';
import { Tag } from '../../tags/entities/tag.entity';

@Entity('posts')
export class Post {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column()
  title: string;

  @Column('text')
  content: string;

  @Column({ default: false })
  published: boolean;

  @ManyToOne(() => User, (user) => user.posts, { onDelete: 'CASCADE' })
  author: User;

  @Column()
  authorId: string;

  @ManyToMany(() => Tag, (tag) => tag.posts)
  @JoinTable()
  tags: Tag[];

  @CreateDateColumn()
  createdAt: Date;
}

Registering Entities in Modules

// users/users.module.ts
import { Module } from '@nestjs/common';
import { TypeOrmModule } from '@nestjs/typeorm';
import { User } from './entities/user.entity';
import { UsersService } from './users.service';
import { UsersController } from './users.controller';

@Module({
  imports: [TypeOrmModule.forFeature([User])],
  controllers: [UsersController],
  providers: [UsersService],
  exports: [UsersService],
})
export class UsersModule {}

TypeOrmModule.forFeature([User]) registers the User repository for injection within this module.

Using Repositories

Injecting the Repository

// users/users.service.ts
import { Injectable, NotFoundException } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { User } from './entities/user.entity';

@Injectable()
export class UsersService {
  constructor(
    @InjectRepository(User)
    private usersRepo: Repository<User>,
  ) {}

  async findAll(page = 1, limit = 20): Promise<{ data: User[]; total: number }> {
    const [data, total] = await this.usersRepo.findAndCount({
      skip: (page - 1) * limit,
      take: limit,
      order: { createdAt: 'DESC' },
    });
    return { data, total };
  }

  async findOne(id: string): Promise<User> {
    const user = await this.usersRepo.findOne({
      where: { id },
      relations: ['posts'],
    });
    if (!user) throw new NotFoundException(`User #${id} not found`);
    return user;
  }

  async findByEmail(email: string): Promise<User | null> {
    return this.usersRepo.findOne({
      where: { email },
      select: ['id', 'email', 'password', 'name', 'role'],
    });
  }

  async create(dto: CreateUserDto): Promise<User> {
    const user = this.usersRepo.create(dto);
    return this.usersRepo.save(user);
  }

  async update(id: string, dto: UpdateUserDto): Promise<User> {
    const user = await this.findOne(id);
    Object.assign(user, dto);
    return this.usersRepo.save(user);
  }

  async remove(id: string): Promise<void> {
    const result = await this.usersRepo.delete(id);
    if (result.affected === 0) {
      throw new NotFoundException(`User #${id} not found`);
    }
  }
}

Query Builder

For complex queries:

async searchUsers(query: SearchUsersDto) {
  const qb = this.usersRepo.createQueryBuilder('user')
    .leftJoinAndSelect('user.posts', 'post');

  if (query.search) {
    qb.where('user.name ILIKE :search OR user.email ILIKE :search', {
      search: `%${query.search}%`,
    });
  }

  if (query.role) {
    qb.andWhere('user.role = :role', { role: query.role });
  }

  if (query.isActive !== undefined) {
    qb.andWhere('user.isActive = :isActive', { isActive: query.isActive });
  }

  qb.orderBy(`user.${query.sortBy || 'createdAt'}`, query.sortOrder || 'DESC')
    .skip((query.page - 1) * query.limit)
    .take(query.limit);

  const [data, total] = await qb.getManyAndCount();
  return { data, total, page: query.page, limit: query.limit };
}

Custom Repositories

// users/users.repository.ts
import { Injectable } from '@nestjs/common';
import { DataSource, Repository } from 'typeorm';
import { User } from './entities/user.entity';

@Injectable()
export class UsersRepository extends Repository<User> {
  constructor(private dataSource: DataSource) {
    super(User, dataSource.createEntityManager());
  }

  async findByEmailWithPassword(email: string): Promise<User | null> {
    return this.createQueryBuilder('user')
      .addSelect('user.password')
      .where('user.email = :email', { email })
      .getOne();
  }

  async findActiveUsersWithPosts(): Promise<User[]> {
    return this.createQueryBuilder('user')
      .leftJoinAndSelect('user.posts', 'post', 'post.published = :published', {
        published: true,
      })
      .where('user.isActive = :active', { active: true })
      .getMany();
  }
}

Register it:

@Module({
  imports: [TypeOrmModule.forFeature([User])],
  providers: [UsersService, UsersRepository],
  exports: [UsersService],
})
export class UsersModule {}

Relations

One-to-Many / Many-to-One

// User has many Posts
@OneToMany(() => Post, (post) => post.author, { cascade: true })
posts: Post[];

// Post belongs to User
@ManyToOne(() => User, (user) => user.posts, { onDelete: 'CASCADE' })
author: User;

Many-to-Many

@ManyToMany(() => Tag, (tag) => tag.posts, { cascade: true })
@JoinTable({ name: 'post_tags' })
tags: Tag[];

Eager vs Lazy Loading

// Eager: always loaded (use sparingly)
@OneToMany(() => Post, (post) => post.author, { eager: true })
posts: Post[];

// Explicit loading with find options
const user = await this.usersRepo.findOne({
  where: { id },
  relations: ['posts', 'posts.tags'],
});

Entity Subscribers

Listen to entity lifecycle events:

import { EntitySubscriberInterface, EventSubscriber, InsertEvent } from 'typeorm';
import { User } from './user.entity';

@EventSubscriber()
export class UserSubscriber implements EntitySubscriberInterface<User> {
  listenTo() {
    return User;
  }

  beforeInsert(event: InsertEvent<User>) {
    console.log('Before user insert:', event.entity.email);
  }

  afterInsert(event: InsertEvent<User>) {
    console.log('User created:', event.entity.id);
  }
}

Common Pitfalls

  1. synchronize: true in production -- This can drop columns and lose data. Always use migrations in production.
  2. N+1 query problem -- Loading relations individually instead of with relations or leftJoinAndSelect. Always load needed relations in the initial query.
  3. Forgetting select: false for passwords -- The password field is returned by default. Use @Column({ select: false }) and explicitly select when needed.
  4. Not using create() before save() -- save() works with plain objects, but create() properly instantiates the entity with transformations and defaults.

Exercises

  1. Build an Entity-Based CRUD Module

    Create a ProductsModule with a Product entity (id, name, description, price, stock, category enum, isActive, createdAt, updatedAt). Implement full CRUD operations using the injected Repository. Add pagination, search by name, and filter by category.

    Hint

    Use @PrimaryGeneratedColumn('uuid') for the id. Use the QueryBuilder for search with ILIKE. Use findAndCount for pagination.

  2. Implement Related Entities

    Add a Category entity with a one-to-many relationship to Product. Add a Tag entity with a many-to-many relationship to Product. Create endpoints to: create a product with category and tags, find products by category, find products by tag.

    Hint

    Use @ManyToOne for Product->Category, @ManyToMany with @JoinTable for Product->Tag. Use relations option in find queries to load related data.

  3. Create and Run Migrations

    Set up a data-source.ts file for TypeORM CLI. Generate a migration from your entities. Run the migration against a PostgreSQL database. Then modify an entity (add a new column) and generate a new migration.

    Hint

    Add a script to package.json: "typeorm": "ts-node -r tsconfig-paths/register node_modules/typeorm/cli.js". Use migration:generate to create migrations from entity changes.

Common Problems & Solutions

1 Repository not found for entity
Symptoms: Error: No repository for 'YourEntity' was found.
Cause: The entity is not registered with TypeOrmModule.forFeature() in the module, or the module is not imported.
Solution: Add `TypeOrmModule.forFeature([YourEntity])` to the module's imports array. Ensure the entity is in the forRoot() entities array or uses autoLoadEntities.
2 Relation data is not loaded
Cause: Relations are lazy by default. Accessing a relation property returns undefined unless explicitly loaded.
Solution: Use `relations: ['posts']` in find options, or use QueryBuilder with leftJoinAndSelect. Avoid `eager: true` as it loads relations on every query.
3 Column with password is exposed in API responses
Cause: The password column doesn't have `select: false`, so it's included in all SELECT queries.
Solution: Add `@Column({ select: false })` to the password field. When you need the password (for auth), explicitly select it: `createQueryBuilder('user').addSelect('user.password')`.
