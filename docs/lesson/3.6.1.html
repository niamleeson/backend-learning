Alembic Setup and Configuration

Intuition

Alembic is version control for your database schema. Just as git tracks code changes, Alembic tracks CREATE TABLE, ALTER COLUMN, and other DDL as sequential scripts.

Alembic Setup and Configuration

Alembic is the migration tool for SQLAlchemy. It tracks schema changes as versioned migration scripts, enabling reproducible database evolution.

# Initialize Alembic in your project
# $ pip install alembic
# $ alembic init alembic

# alembic.ini - set the database URL
# sqlalchemy.url = postgresql://user:pass@localhost/mydb

# alembic/env.py - configure target metadata
from myapp.models import Base
target_metadata = Base.metadata

def run_migrations_online():
    connectable = engine_from_config(
        config.get_section(config.config_ini_section),
        prefix="sqlalchemy."
    )
    with connectable.connect() as connection:
        context.configure(
            connection=connection,
            target_metadata=target_metadata
        )
        with context.begin_transaction():
            context.run_migrations()

# Using environment variable for DB URL
import os
config.set_main_option('sqlalchemy.url', os.environ['DATABASE_URL'])

Key Takeaways: - alembic init creates the alembic/ directory with env.py and a versions/ folder. - Set target_metadata in env.py to your Base.metadata so Alembic can compare models to the database. - Store the database URL in environment variables, not in alembic.ini, for security.

Exercises

  1. Initialize Alembic

    Set up Alembic in an existing SQLAlchemy project: initialize, configure env.py with your models' metadata, and verify the connection.

    Hint

    Run alembic init alembic, edit env.py to import your Base, set target_metadata = Base.metadata.

  2. Environment-Based Config

    Modify env.py to read the database URL from an environment variable with a fallback to a local SQLite database.

    Hint

    config.set_main_option('sqlalchemy.url', os.environ.get('DATABASE_URL', 'sqlite:///local.db')).

Common Problems & Solutions

1 Alembic cannot find the models or target_metadata is None.
Cause: The import path for your models is incorrect or the Base class is not the same instance.
Solution: Ensure env.py imports the correct Base. Add your project to sys.path if needed.
2 Alembic connects to the wrong database.
Cause: sqlalchemy.url in alembic.ini still has the default placeholder.
Solution: Set the URL in alembic.ini or override it in env.py using config.set_main_option().
