Dunder Methods: __str__, __repr__, __eq__, __hash__

Intuition

Dunder methods are hooks into Python's machinery. When you write a + b, Python secretly calls a.__add__(b). Defining these methods makes your objects feel like built-in types.

Dunder (double underscore) methods let you define how objects behave with built-in operations: printing, comparison, hashing, arithmetic, and more.

class Money:
    def __init__(self, amount: float, currency: str = "USD"):
        self.amount = amount
        self.currency = currency

    def __repr__(self) -> str:
        return f"Money({self.amount}, {self.currency!r})"

    def __str__(self) -> str:
        return f"${self.amount:.2f} {self.currency}"

    def __eq__(self, other) -> bool:
        if not isinstance(other, Money):
            return NotImplemented
        return self.amount == other.amount and self.currency == other.currency

    def __hash__(self) -> int:
        return hash((self.amount, self.currency))

    def __add__(self, other):
        if self.currency != other.currency:
            raise ValueError("Cannot add different currencies")
        return Money(self.amount + other.amount, self.currency)

print(repr(Money(9.99)))       # Money(9.99, 'USD')
print(Money(10) + Money(20))   # $30.00 USD
  • __repr__ is for developers (unambiguous); __str__ is for users (readable)
  • If you define __eq__, define __hash__ too (or set __hash__ = None for unhashable)
  • Return NotImplemented (not raise) from comparison methods for unsupported types
  • __repr__ should ideally be valid Python that recreates the object

Exercises

  1. Comparable Vector

    Create a Vector class with x, y that supports __add__, __eq__, __abs__ (magnitude), __repr__, and __mul__ (scalar multiplication).

    Hint

    For __abs__, return (self.x**2 + self.y**2)**0.5. For __mul__, check if other is a number.

  2. Custom Container

    Create a Deck class that implements __len__, __getitem__, and __contains__ so you can use len(), indexing, and in with it.

    Hint

    Store cards in a list. __getitem__ enables both indexing and iteration for free.

Common Problems & Solutions

1 Object becomes unhashable after defining __eq__
Cause: Python sets `__hash__` to None when you define `__eq__`, making instances unusable in sets/dicts.
Solution: Define `__hash__` based on the same fields used in `__eq__`.
2 Comparison returns NotImplemented instead of False
Cause: Returning `NotImplemented` from `__eq__` is correct; Python will try the other operand's `__eq__`.
Solution: This is expected behavior. `NotImplemented` is a signal, not an error. Python handles it.
