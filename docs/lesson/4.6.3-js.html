Room-Based Messaging and Presence

Intuition

Rooms are like conference rooms in an office. You walk in (join), hear everything said (events), speak to everyone there (emit), and walk out (leave). Presence is like the office directory board showing who's in the building. When you badge in (connect), your status lights up green. When you leave (disconnect), it goes dark.

Room-Based Messaging and Presence

Introduction

Socket.IO rooms are a powerful abstraction for grouping connected clients. Combined with presence tracking (knowing who's online), they form the backbone of any real-time collaborative application. This lesson dives deep into room management patterns, presence systems, and scaling considerations.

Socket.IO Rooms Explained

A room is a server-side concept -- a named channel that sockets can join and leave. Events emitted to a room are delivered to all sockets in that room.

Room: "project-alpha"
+----------------------------------+
|  Socket A (Alice)                |
|  Socket B (Bob)                  |
|  Socket C (Charlie)              |
+----------------------------------+

server.to("project-alpha").emit("update", data)
// Alice, Bob, and Charlie all receive "update"

Key points: - Rooms exist only on the server (clients don't know about them) - A socket can be in multiple rooms simultaneously - Every socket automatically joins a room with its own ID (socket.id) - Rooms are automatically cleaned up when empty

Room Management Patterns

Joining and Leaving Rooms

@SubscribeMessage('join-room')
async handleJoinRoom(
  @MessageBody() data: { roomId: string },
  @ConnectedSocket() client: AuthenticatedSocket,
) {
  const user = client.data.user;

  // Authorization check
  const isMember = await this.roomService.isMember(user.id, data.roomId);
  if (!isMember) {
    throw new WsException('Not a member of this room');
  }

  client.join(`room:${data.roomId}`);

  // Notify others in the room
  client.to(`room:${data.roomId}`).emit('member-joined', {
    userId: user.id,
    name: user.name,
  });

  // Send current room state to the joining client
  const members = await this.getOnlineRoomMembers(data.roomId);
  client.emit('room-state', {
    roomId: data.roomId,
    onlineMembers: members,
  });
}

@SubscribeMessage('leave-room')
handleLeaveRoom(
  @MessageBody() data: { roomId: string },
  @ConnectedSocket() client: AuthenticatedSocket,
) {
  client.leave(`room:${data.roomId}`);
  client.to(`room:${data.roomId}`).emit('member-left', {
    userId: client.data.user.id,
  });
}

Broadcasting Patterns

// To everyone in a room (including sender)
this.server.to('room:123').emit('event', data);

// To everyone in a room (except sender)
client.to('room:123').emit('event', data);

// To everyone in multiple rooms
this.server.to('room:123').to('room:456').emit('event', data);

// To everyone except those in a specific room
this.server.except('room:123').emit('event', data);

// To a specific socket by ID
this.server.to(socketId).emit('event', data);

Presence System

Presence tracks which users are online and their current status.

In-Memory Presence Store

// presence/presence.service.ts
interface UserPresence {
  userId: string;
  name: string;
  status: 'online' | 'away' | 'busy' | 'offline';
  lastSeen: Date;
  sockets: Set<string>;
}

@Injectable()
export class PresenceService {
  private presence = new Map<string, UserPresence>();

  setOnline(userId: string, name: string, socketId: string) {
    const existing = this.presence.get(userId);
    if (existing) {
      existing.sockets.add(socketId);
      existing.status = 'online';
      existing.lastSeen = new Date();
    } else {
      this.presence.set(userId, {
        userId,
        name,
        status: 'online',
        lastSeen: new Date(),
        sockets: new Set([socketId]),
      });
    }
  }

  setOffline(userId: string, socketId: string): boolean {
    const existing = this.presence.get(userId);
    if (!existing) return false;

    existing.sockets.delete(socketId);

    if (existing.sockets.size === 0) {
      existing.status = 'offline';
      existing.lastSeen = new Date();
      return true; // User is fully offline
    }
    return false; // User still has other connections
  }

  setStatus(userId: string, status: 'online' | 'away' | 'busy') {
    const existing = this.presence.get(userId);
    if (existing) {
      existing.status = status;
    }
  }

  isOnline(userId: string): boolean {
    const p = this.presence.get(userId);
    return !!p && p.status !== 'offline' && p.sockets.size > 0;
  }

  getOnlineUsers(): Array<{ userId: string; name: string; status: string }> {
    return Array.from(this.presence.values())
      .filter((p) => p.sockets.size > 0)
      .map(({ userId, name, status }) => ({ userId, name, status }));
  }

  getOnlineUsersInSet(userIds: string[]) {
    return userIds.filter((id) => this.isOnline(id));
  }
}

Integration with Gateway

@WebSocketGateway({ cors: { origin: '*' } })
export class ChatGateway implements OnGatewayConnection, OnGatewayDisconnect {
  @WebSocketServer() server: Server;

  constructor(
    private chatService: ChatService,
    private presenceService: PresenceService,
  ) {}

  async handleConnection(client: AuthenticatedSocket) {
    const user = client.data.user;

    this.presenceService.setOnline(user.id, user.name, client.id);
    client.join(`user:${user.id}`);

    // Join all user's rooms
    const rooms = await this.chatService.getUserRooms(user.id);
    for (const room of rooms) {
      client.join(`room:${room.id}`);
      // Notify room that user is online
      client.to(`room:${room.id}`).emit('presence-update', {
        userId: user.id,
        status: 'online',
      });
    }
  }

  handleDisconnect(client: AuthenticatedSocket) {
    const user = client.data?.user;
    if (!user) return;

    const fullyOffline = this.presenceService.setOffline(user.id, client.id);

    if (fullyOffline) {
      // Notify all rooms the user was in
      this.server.emit('presence-update', {
        userId: user.id,
        status: 'offline',
        lastSeen: new Date(),
      });
    }
  }

  @SubscribeMessage('set-status')
  handleSetStatus(
    @MessageBody() data: { status: 'online' | 'away' | 'busy' },
    @ConnectedSocket() client: AuthenticatedSocket,
  ) {
    const user = client.data.user;
    this.presenceService.setStatus(user.id, data.status);
    this.server.emit('presence-update', {
      userId: user.id,
      status: data.status,
    });
  }

  @SubscribeMessage('get-presence')
  handleGetPresence(
    @MessageBody() data: { userIds: string[] },
  ) {
    const online = this.presenceService.getOnlineUsersInSet(data.userIds);
    return {
      event: 'presence-list',
      data: data.userIds.map((id) => ({
        userId: id,
        online: online.includes(id),
      })),
    };
  }
}

Idle Detection

Auto-set users as "away" after inactivity:

@WebSocketGateway()
export class ChatGateway {
  private activityTimers = new Map<string, NodeJS.Timeout>();
  private readonly IDLE_TIMEOUT = 5 * 60 * 1000; // 5 minutes

  @SubscribeMessage('activity')
  handleActivity(@ConnectedSocket() client: AuthenticatedSocket) {
    const userId = client.data.user.id;

    // Reset idle timer
    const existing = this.activityTimers.get(userId);
    if (existing) clearTimeout(existing);

    // If user was away, set back to online
    if (this.presenceService.getStatus(userId) === 'away') {
      this.presenceService.setStatus(userId, 'online');
      this.server.emit('presence-update', {
        userId,
        status: 'online',
      });
    }

    // Start new idle timer
    this.activityTimers.set(
      userId,
      setTimeout(() => {
        this.presenceService.setStatus(userId, 'away');
        this.server.emit('presence-update', {
          userId,
          status: 'away',
        });
      }, this.IDLE_TIMEOUT),
    );
  }
}

Scaling with Redis Adapter

For multi-server deployments, use the Redis adapter so rooms work across instances:

npm i @socket.io/redis-adapter redis

// main.ts
import { IoAdapter } from '@nestjs/platform-socket.io';
import { createAdapter } from '@socket.io/redis-adapter';
import { createClient } from 'redis';

export class RedisIoAdapter extends IoAdapter {
  private adapterConstructor: ReturnType<typeof createAdapter>;

  async connectToRedis(): Promise<void> {
    const pubClient = createClient({ url: process.env.REDIS_URL });
    const subClient = pubClient.duplicate();
    await Promise.all([pubClient.connect(), subClient.connect()]);
    this.adapterConstructor = createAdapter(pubClient, subClient);
  }

  createIOServer(port: number, options?: any) {
    const server = super.createIOServer(port, options);
    server.adapter(this.adapterConstructor);
    return server;
  }
}

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  const redisIoAdapter = new RedisIoAdapter(app);
  await redisIoAdapter.connectToRedis();
  app.useWebSocketAdapter(redisIoAdapter);
  await app.listen(3000);
}

When using the Redis adapter, the presence service should also use Redis instead of in-memory Maps.

Common Pitfalls

  1. Room naming collisions -- Prefix room names with their type: room:123, user:456, project:789 to avoid collisions.
  2. Not cleaning up on disconnect -- Leave rooms, clear timers, and update presence on every disconnect.
  3. In-memory state doesn't scale -- If running multiple server instances, presence and room state must be shared (Redis).
  4. Broadcasting to large rooms -- Emitting to rooms with thousands of members can be slow. Consider batching or using Redis pub/sub.

Exercises

  1. Build a Complete Presence System

    Implement a PresenceService that tracks online/away/busy/offline status, handles multiple connections per user, and provides room-scoped presence queries. Integrate it with the chat gateway.

    Hint

    Use a Map where UserPresence contains a Set of socket IDs. A user is online if they have at least one socket. Provide methods for room-scoped queries by intersecting room members with online users.

  2. Add Idle Detection

    Implement automatic idle detection: the client sends an 'activity' event on mouse move/keypress (throttled to once per 30 seconds). If no activity for 5 minutes, set status to 'away'. On next activity, revert to 'online'.

    Hint

    Use setTimeout on the server to track inactivity. Store timers in a Map. Clear and reset the timer on each 'activity' event.

  3. Scale with Redis Adapter

    Configure the Socket.IO Redis adapter for multi-server support. Move the presence service to use Redis (HSET/HGET) instead of in-memory Maps. Verify that messages and presence updates work across two server instances.

    Hint

    Use @socket.io/redis-adapter for rooms. For presence, use Redis HSET to store user status and SADD to track socket IDs. Use Redis pub/sub for cross-instance presence updates.

Common Problems & Solutions

1 Rooms don't work across multiple server instances
Symptoms: Messages sent in a room only reach clients connected to the same server instance.
Cause: Socket.IO rooms are in-memory by default. Each server instance has its own set of rooms.
Solution: Use the Redis adapter (@socket.io/redis-adapter). It synchronizes rooms across instances through Redis pub/sub.
2 Presence shows user as online after server restart
Cause: In-memory presence data is lost on restart. If the client reconnects quickly but the server's presence map is empty, inconsistencies arise.
Solution: Persist presence in Redis with TTL. On server start, don't assume anyone is online. Let clients reconnect and re-register their presence.
3 Memory leak from activity timers
Cause: Timers are created for idle detection but not cleared when the user disconnects.
Solution: In handleDisconnect, clear the timer: `clearTimeout(this.activityTimers.get(userId))` and delete the entry from the map.
