Abstract Base Classes (ABC)

Intuition

An ABC is a contract: it says 'any class claiming to be a Shape must provide area() and perimeter().' It catches missing implementations at instantiation time, not at call time.

Abstract Base Classes (ABCs) define interfaces that subclasses must implement. They prevent instantiation of incomplete classes and make contracts explicit.

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self) -> float:
        """Calculate the area of the shape."""
        pass

    @abstractmethod
    def perimeter(self) -> float:
        """Calculate the perimeter of the shape."""
        pass

    def describe(self) -> str:
        return f"{type(self).__name__}: area={self.area():.2f}"

class Circle(Shape):
    def __init__(self, radius: float):
        self.radius = radius
    def area(self) -> float:
        import math
        return math.pi * self.radius ** 2
    def perimeter(self) -> float:
        import math
        return 2 * math.pi * self.radius

# Shape()         # TypeError: Can't instantiate abstract class
c = Circle(5)
print(c.describe())  # "Circle: area=78.54"
  • ABCs cannot be instantiated directly; subclasses must implement all abstract methods
  • ABCs can have concrete methods that use abstract methods (template pattern)
  • Use @abstractmethod with @property, @classmethod, @staticmethod too
  • Python's collections.abc provides ABCs like Iterable, Mapping, Sequence

Common Problems & Solutions

1 TypeError: Can't instantiate abstract class with abstract method
Cause: A subclass didn't implement all abstract methods from the ABC.
Solution: Implement every `@abstractmethod` from the parent. Check the error message for which methods are missing.
2 ABC subclass doesn't enforce the method signature
Cause: Python ABCs enforce method existence, not signatures. A subclass can use a different signature.
Solution: Use type hints and runtime checks, or use a Protocol (typing module) for structural subtyping.
