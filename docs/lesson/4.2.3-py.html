CSRF Protection and File Uploads

Intuition

CSRF protection ensures that form submissions originate from your own application, not from a malicious third-party page. File uploads add another attack surface, so secure filenames and size limits are your first lines of defense.

CSRF Protection and File Uploads

Cross-Site Request Forgery (CSRF) protection prevents attackers from submitting forms on behalf of authenticated users. Flask-WTF provides CSRF protection by default. File uploads require careful handling of size limits, allowed types, and secure filenames.

from flask import Flask, request, jsonify, render_template
from flask_wtf import FlaskForm, CSRFProtect
from flask_wtf.file import FileField, FileRequired, FileAllowed
from wtforms import StringField
from werkzeug.utils import secure_filename
import os

app = Flask(__name__)
app.config['SECRET_KEY'] = 'dev-secret-key'
app.config['MAX_CONTENT_LENGTH'] = 16 * 1024 * 1024  # 16 MB
app.config['UPLOAD_FOLDER'] = '/tmp/uploads'

csrf = CSRFProtect(app)

class UploadForm(FlaskForm):
    title = StringField('Title')
    document = FileField('Document', validators=[
        FileRequired(),
        FileAllowed(['pdf', 'png', 'jpg'], 'Only PDF and images allowed')
    ])

@app.route('/upload', methods=['GET', 'POST'])
def upload():
    form = UploadForm()
    if form.validate_on_submit():
        f = form.document.data
        filename = secure_filename(f.filename)
        filepath = os.path.join(app.config['UPLOAD_FOLDER'], filename)
        f.save(filepath)
        return jsonify({'saved': filename}), 201
    return render_template('upload.html', form=form)

@app.errorhandler(413)
def too_large(e):
    return jsonify({'error': 'File too large. Max 16MB.'}), 413

Key Takeaways: - CSRFProtect(app) enables CSRF globally; use {{ csrf_token() }} or {{ form.hidden_tag() }} in templates. - secure_filename() strips dangerous characters from user-supplied filenames to prevent path traversal. - Set MAX_CONTENT_LENGTH to reject uploads that exceed your size limit before they consume memory.

Exercises

  1. CSRF-Protected API Endpoint

    Create a JSON API endpoint protected by CSRF. Send a POST request from a simple HTML page that includes the CSRF token in a header. Verify that requests without the token are rejected.

    Hint

    Set the X-CSRFToken header in your JavaScript fetch() call. Get the token from a meta tag rendered by {{ csrf_token() }}.

  2. Multi-File Upload

    Build an upload form that accepts multiple files at once, validates each file's type and size, and returns a summary of successfully saved files.

    Hint

    Use request.files.getlist('files') and loop through each file, applying secure_filename() and validation individually.

Common Problems & Solutions

1 400 Bad Request: The CSRF token is missing.
Cause: The form submission does not include the CSRF token, either because hidden_tag() is missing or the form enctype is wrong.
Solution: Add {{ form.hidden_tag() }} inside your
tag. For multipart forms, ensure enctype='multipart/form-data' is set.
2 413 Request Entity Too Large with no custom error message.
Cause: Flask rejects the request before your route runs when MAX_CONTENT_LENGTH is exceeded, returning a generic 413.
Solution: Register an @app.errorhandler(413) to return a user-friendly error response.
