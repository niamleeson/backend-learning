Design a News Feed (Twitter/Instagram)

Intuition

A news feed system is a content distribution network that balances pre-computation (push) for fast reads against on-demand assembly (pull) for high-follower accounts. The hybrid fanout model is the key insight that makes platforms like Twitter and Instagram scale.

Design a News Feed (Twitter/Instagram)

Requirements

  • Users post content and see a personalized feed of posts from followed accounts
  • Feed generation must handle celebrity accounts with millions of followers
  • Support ranking by relevance, not just chronological order
  • Handle text, images, and video content with real-time updates

Architecture

A post service stores new content and triggers fanout. A feed service assembles personalized timelines. For regular users, fan-out on write pre-computes feeds in a cache (Redis sorted sets). For celebrity accounts, fan-out on read merges their posts at query time. A ranking service scores and orders posts.

Key Components

  • Post Service: Stores posts in a database and publishes events to Kafka
  • Fanout Service: Pushes post IDs to follower feed caches for normal users
  • Feed Cache: Redis sorted sets per user storing ranked post IDs
  • Feed Service: Assembles feed by merging cached posts with celebrity posts at read time
  • Ranking Service: ML-based scoring considering engagement, recency, and user preferences

Scaling

Use a hybrid fanout approach: push for users with fewer followers, pull for celebrities. Shard feed caches by user ID. Use a graph database or adjacency list for the follow relationship. Pre-compute and cache trending content.

Trade-offs

  • Fan-out on write is fast for reads but slow and expensive for celebrity posts with millions of followers
  • Chronological feeds are simple but less engaging; ranked feeds increase engagement but add ML complexity

Exercises

  1. Implement Hybrid Fanout Logic

    Build a fanout service that decides between push and pull based on follower count. Users with < 10,000 followers get fan-out on write. Celebrity posts are merged at read time. Implement the feed assembly that combines both sources.

    Hint

    Maintain a set of celebrity user IDs. When assembling a feed, merge the user's pre-computed cache with a real-time query of recent posts from followed celebrities, then sort by rank score.

  2. Design a Feed Ranking System

    Create a ranking function that scores posts based on: recency (time decay), engagement (likes, comments, shares), relationship strength (interaction frequency with the author), and content type preference. Return the top N posts.

    Hint

    Use a weighted scoring formula: score = w1recency + w2engagement + w3affinity + w4content_type_boost. Normalize each factor to [0,1] before combining.

Common Problems & Solutions

1 A celebrity with 50 million followers publishes a post, and the fanout service takes minutes to update all follower feeds.
Cause: Fan-out on write for high-follower accounts creates massive write amplification, overwhelming the cache layer and delaying all other fanouts.
Solution: Implement hybrid fanout: skip cache pre-computation for users above a follower threshold. At read time, merge the user's cached feed with recent posts from followed celebrities fetched on demand.
2 Feed cache memory usage grows unbounded as users accumulate thousands of posts in their timelines.
Cause: Every post fanout adds entries to follower caches without eviction, and inactive users' caches consume memory without being read.
Solution: Limit feed cache to the most recent N posts (e.g., 800). Evict caches for users inactive beyond a threshold. When an inactive user returns, rebuild their feed on demand and re-cache it.
