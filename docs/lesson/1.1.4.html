Strings and String Methods

Intuition

Strings are like frozen text: you can look at any part and build new strings from them, but you can never change a character in place. Every operation gives you a fresh copy.

Strings in Python are immutable sequences of Unicode characters. The rich set of built-in string methods makes text processing straightforward without importing anything.

name = "Hello, World!"

# Common methods (all return new strings)
print(name.lower())              # "hello, world!"
print(name.replace("World", "Python"))  # "Hello, Python!"
print(name.split(", "))          # ['Hello', 'World!']

# f-strings (Python 3.6+)
user, score = "Alice", 95.5
print(f"{user} scored {score:.1f}%")  # "Alice scored 95.5%"

# Multiline and raw strings
path = r"C:\Users\new"           # raw: no escape processing
query = """SELECT *
FROM users
WHERE active = 1"""
  • Strings are immutable: methods return new strings, never modify in place
  • Use f-strings for readable interpolation; avoid % formatting and .format() in new code
  • str.join() is far more efficient than concatenation in loops
  • Use in for substring checks: "py" in "python" returns True

Exercises

  1. Word Frequency Counter

    Given a sentence string, write code that lowercases it, splits into words, and counts occurrences of each word using a dictionary.

    Hint

    Use .lower().split() and a dict to count, or collections.Counter.

  2. String Formatting Challenge

    Using f-strings, format the number 42.5678 to 2 decimal places, pad the string 'hi' to 10 characters (centered), and print an integer in binary.

    Hint

    Use format specs like :.2f, :^10, and :#b inside f-string braces.

Common Problems & Solutions

1 TypeError: 'str' object does not support item assignment
Cause: Trying to modify a character in a string like `s[0] = 'H'` fails because strings are immutable.
Solution: Create a new string: `s = 'H' + s[1:]` or use `s.replace()`.
2 Slow string concatenation in a loop
Cause: Using `+=` in a loop creates a new string every iteration, leading to O(n^2) performance.
Solution: Collect parts in a list and use `''.join(parts)` at the end.
