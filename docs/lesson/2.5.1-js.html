Prisma Schema: Models, Relations, and Data Types

Intuition

The Prisma schema is like an architect's blueprint. You draw the blueprint once (the schema), and from it you can generate the actual building (database tables via migrations) and a perfect 3D model for virtual tours (the type-safe client). Change the blueprint, regenerate, and everything stays in sync.

Prisma Schema: Models, Relations, and Data Types

Introduction

Prisma is a next-generation ORM for Node.js and TypeScript that takes a schema-first approach. You define your data model in a .prisma file, and Prisma generates a fully type-safe client. Unlike traditional ORMs where you define models in code, Prisma's schema is the single source of truth for your database structure.

Setting Up Prisma

npm install prisma @prisma/client
npx prisma init

This creates: - prisma/schema.prisma — your schema file - .env — database connection URL

The Prisma Schema

// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id            String    @id @default(uuid())
  email         String    @unique
  name          String
  passwordHash  String    @map("password_hash")
  role          Role      @default(CUSTOMER)
  createdAt     DateTime  @default(now()) @map("created_at")
  updatedAt     DateTime  @updatedAt @map("updated_at")
  deletedAt     DateTime? @map("deleted_at")

  // Relations
  orders        Order[]
  reviews       Review[]
  addresses     Address[]

  @@map("users")  // table name in database
}

enum Role {
  CUSTOMER
  ADMIN
  VENDOR
}

Data Types

model Example {
  // Scalar types
  id          Int       @id @default(autoincrement())
  uuid        String    @id @default(uuid())
  name        String                    // text / varchar
  description String?                   // nullable
  price       Decimal   @db.Decimal(10, 2) // exact decimal
  weight      Float                     // double precision
  quantity    Int                       // integer
  bigNum      BigInt                    // bigint
  active      Boolean   @default(true)
  data        Json                      // jsonb
  bytes       Bytes                     // bytea
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  // PostgreSQL-specific types
  tags        String[]                  // text array
  scores      Int[]                     // integer array
}

Prisma Type     -> PostgreSQL Type
-----------------------------------------
String          -> text
Int             -> integer
BigInt          -> bigint
Float           -> double precision
Decimal         -> decimal / numeric
Boolean         -> boolean
DateTime        -> timestamp(3) with tz
Json            -> jsonb
Bytes           -> bytea
String[]        -> text[]

Field Modifiers and Attributes

model Product {
  id        String   @id @default(uuid())     // primary key + default
  sku       String   @unique                   // unique constraint
  name      String                              // required
  subtitle  String?                             // optional (nullable)
  price     Decimal  @db.Decimal(10, 2)        // specific DB type
  tags      String[] @default([])              // array with default
  metadata  Json     @default("{}")            // JSON default
  slug      String   @unique                   // unique

  // Computed / generated
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  // Mapping to different column/table names
  categoryId String  @map("category_id")
  category   Category @relation(fields: [categoryId], references: [id])

  @@map("products")
  @@index([categoryId])          // database index
  @@index([name, price])         // composite index
}

Defining Relations

One-to-Many

model User {
  id     String  @id @default(uuid())
  email  String  @unique
  name   String
  orders Order[]   // one user has many orders

  @@map("users")
}

model Order {
  id        String   @id @default(uuid())
  total     Decimal  @db.Decimal(10, 2)
  status    String   @default("pending")
  userId    String   @map("user_id")
  user      User     @relation(fields: [userId], references: [id])
  items     OrderItem[]
  createdAt DateTime @default(now()) @map("created_at")

  @@map("orders")
  @@index([userId])
}

One-to-One

model User {
  id      String   @id @default(uuid())
  email   String   @unique
  profile Profile?
}

model Profile {
  id     String @id @default(uuid())
  bio    String?
  avatar String?
  userId String @unique @map("user_id")
  user   User   @relation(fields: [userId], references: [id])

  @@map("profiles")
}

Many-to-Many (Implicit)

model Post {
  id   String @id @default(uuid())
  title String
  tags  Tag[]   // Prisma creates junction table automatically
}

model Tag {
  id    String @id @default(uuid())
  name  String @unique
  posts Post[]
}
// Creates _PostToTag junction table automatically

Many-to-Many (Explicit Junction Table)

model Student {
  id          String       @id @default(uuid())
  name        String
  enrollments Enrollment[]
}

model Course {
  id          String       @id @default(uuid())
  title       String
  enrollments Enrollment[]
}

model Enrollment {
  id         String   @id @default(uuid())
  studentId  String   @map("student_id")
  courseId   String   @map("course_id")
  grade      String?
  enrolledAt DateTime @default(now()) @map("enrolled_at")

  student    Student  @relation(fields: [studentId], references: [id])
  course     Course   @relation(fields: [courseId], references: [id])

  @@unique([studentId, courseId])
  @@map("enrollments")
}

Self-Relation

model Employee {
  id        String     @id @default(uuid())
  name      String
  managerId String?    @map("manager_id")
  manager   Employee?  @relation("Management", fields: [managerId], references: [id])
  reports   Employee[] @relation("Management")

  @@map("employees")
}

Generating and Using the Client

# Generate client after schema changes
npx prisma generate

# Push schema to DB (development only, no migration files)
npx prisma db push

// Usage
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

// Full type safety!
const user = await prisma.user.create({
  data: {
    email: 'alice@example.com',
    name: 'Alice',
    passwordHash: 'hashed...',
  },
});
// user is fully typed: { id: string, email: string, name: string, ... }

Full E-Commerce Schema Example

model User {
  id           String    @id @default(uuid())
  email        String    @unique
  name         String
  passwordHash String    @map("password_hash")
  role         Role      @default(CUSTOMER)
  orders       Order[]
  reviews      Review[]
  addresses    Address[]
  createdAt    DateTime  @default(now()) @map("created_at")
  updatedAt    DateTime  @updatedAt @map("updated_at")

  @@map("users")
}

model Product {
  id          String      @id @default(uuid())
  name        String
  slug        String      @unique
  description String?
  price       Decimal     @db.Decimal(10, 2)
  stock       Int         @default(0)
  categoryId  String?     @map("category_id")
  category    Category?   @relation(fields: [categoryId], references: [id])
  images      ProductImage[]
  orderItems  OrderItem[]
  reviews     Review[]
  tags        Tag[]
  isActive    Boolean     @default(true) @map("is_active")
  metadata    Json        @default("{}")
  createdAt   DateTime    @default(now()) @map("created_at")

  @@map("products")
  @@index([categoryId])
  @@index([isActive, price])
}

model Category {
  id       String     @id @default(uuid())
  name     String
  slug     String     @unique
  parentId String?    @map("parent_id")
  parent   Category?  @relation("CategoryTree", fields: [parentId], references: [id])
  children Category[] @relation("CategoryTree")
  products Product[]

  @@map("categories")
}

model Order {
  id        String      @id @default(uuid())
  userId    String      @map("user_id")
  user      User        @relation(fields: [userId], references: [id])
  status    OrderStatus @default(PENDING)
  items     OrderItem[]
  total     Decimal     @db.Decimal(10, 2)
  createdAt DateTime    @default(now()) @map("created_at")

  @@map("orders")
  @@index([userId])
}

enum Role {
  CUSTOMER
  ADMIN
  VENDOR
}

enum OrderStatus {
  PENDING
  PROCESSING
  SHIPPED
  DELIVERED
  CANCELLED
}

Exercises

  1. Design a Blog Schema

    Create a Prisma schema for a blog with: users (with roles), posts (draft/published status, slug, tags), comments (nested/threaded), likes (on posts and comments). Use appropriate data types, relations, and indexes.

    Hint

    Use enum for post status. Self-relation on Comment for threading. Explicit junction table for post-tag M:N with a created_at field.

  2. Prisma Schema from Existing DB

    Use npx prisma db pull to introspect an existing PostgreSQL database and generate a schema. Then modify the generated schema to add proper relations, enums, and @map annotations.

    Hint

    Set DATABASE_URL to point to your existing database. After pulling, Prisma generates models matching your tables. Add relation fields manually.

  3. Multi-Model Schema with Constraints

    Design a schema for a project management tool: organizations, teams, users (with team membership), projects, tasks, and comments. Include proper cascading deletes and unique constraints.

    Hint

    Use onDelete: Cascade for child records. Create explicit junction table for user-team membership with a role field.

Common Problems & Solutions

1 Error: Prisma schema validation error - relation field must have @relation attribute
Cause: When you have multiple relations between the same two models, Prisma can't disambiguate them automatically.
Solution: Add named relations: `@relation("AuthoredPosts")` and `@relation("ReviewedPosts")` to disambiguate. Both sides must use the same name.
2 @updatedAt field doesn't update on nested writes
Cause: Prisma's @updatedAt only updates when the model itself is directly updated, not when related models change.
Solution: Manually set updatedAt in your update queries for parent models, or use a database trigger to update the timestamp.
3 Decimal values come back as Prisma.Decimal objects, not numbers
Cause: Prisma uses a Decimal.js wrapper to preserve precision for decimal/numeric columns.
Solution: Convert to number with `.toNumber()` or to string with `.toString()`. For JSON serialization, add a custom toJSON method or use a transform.
