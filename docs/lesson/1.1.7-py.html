Control Flow: if, elif, else

Intuition

Control flow is like a series of gates. Python checks each gate (condition) in order and enters the first one that's open. Once inside, it skips all remaining gates.

Conditional statements let your program make decisions. Python uses if, elif, and else with indentation-based blocks, keeping code clean and readable.

status_code = 404

if status_code == 200:
    message = "OK"
elif status_code == 404:
    message = "Not Found"
elif status_code >= 500:
    message = "Server Error"
else:
    message = "Unknown"

# Ternary expression
result = "even" if 10 % 2 == 0 else "odd"

# Match statement (Python 3.10+)
match status_code:
    case 200:
        print("Success")
    case 404:
        print("Not found")
    case _:
        print("Other")
  • Python has no switch statement before 3.10; use match/case in 3.10+
  • Ternary syntax: value_if_true if condition else value_if_false
  • Avoid deeply nested ifs; use early returns or guard clauses
  • Combine conditions with and, or, not rather than nesting

Exercises

  1. Grade Calculator

    Write a function that takes a score (0-100) and returns a letter grade: A (90+), B (80+), C (70+), D (60+), F (below 60). Handle invalid inputs.

    Hint

    Use if/elif/else chain, checking from highest to lowest.

  2. FizzBuzz with Ternary

    Write FizzBuzz for 1-20 using a single ternary expression (nested ternaries are fine). Print each result.

    Hint

    You can nest: 'FizzBuzz' if ... else ('Fizz' if ... else ('Buzz' if ... else n)).

Common Problems & Solutions

1 IndentationError: expected an indented block
Cause: The body of an `if` block is empty or incorrectly indented.
Solution: Use `pass` as a placeholder or ensure the body is indented consistently (4 spaces).
2 Using `=` instead of `==` in a condition
Cause: Assignment (`=`) inside an `if` is a SyntaxError in Python, unlike C/Java.
Solution: Use `==` for comparison. For assignment expressions, use walrus operator `:=`.
