Subqueries and Correlated Subqueries

Intuition

A subquery is a question inside a question. A correlated subquery is like checking each student's grade against their own class average rather than the school average.

Subqueries and Correlated Subqueries

Subqueries are queries nested inside another query. They can appear in SELECT, FROM, or WHERE clauses. Correlated subqueries reference columns from the outer query.

-- Scalar subquery in WHERE
SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

-- Subquery in FROM (derived table)
SELECT dept, avg_sal
FROM (SELECT department AS dept, AVG(salary) AS avg_sal
      FROM employees GROUP BY department) sub
WHERE avg_sal > 80000;

-- Correlated subquery with EXISTS
SELECT c.name FROM customers c
WHERE EXISTS (
    SELECT 1 FROM orders o
    WHERE o.customer_id = c.id
      AND o.total > 1000
);

Key Takeaways: - Non-correlated subqueries run once; correlated subqueries run once per outer row. - EXISTS is often faster than IN for large datasets because it short-circuits. - Derived tables in FROM must have an alias.

Exercises

  1. Above-Average Products

    Find all products priced above the average price of their own category using a correlated subquery.

    Hint

    In the WHERE clause, compare price to a subquery that computes AVG(price) for the same category.

  2. Customers With Recent Orders

    Use EXISTS to find customers who placed an order in the last 30 days.

    Hint

    EXISTS (SELECT 1 FROM orders WHERE customer_id = c.id AND order_date > CURRENT_DATE - 30).

Common Problems & Solutions

1 Subquery returns more than one row when a scalar is expected.
Cause: Using = with a subquery that returns multiple rows instead of a single value.
Solution: Use IN, ANY, or ALL instead of =, or add LIMIT 1 / aggregation to ensure a single value.
2 Correlated subquery is extremely slow.
Cause: The subquery executes once per row in the outer query with no supporting index.
Solution: Add an index on the correlated column, or rewrite as a JOIN or CTE for better performance.
