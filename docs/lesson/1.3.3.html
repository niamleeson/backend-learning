Properties and Encapsulation

Intuition

Properties are like smart attributes: they look like regular attributes from the outside, but behind the scenes they run code for validation, computation, or access control.

Properties let you define managed attributes with getter, setter, and deleter methods while maintaining a clean attribute-access syntax. This is Python's approach to encapsulation.

class Temperature:
    def __init__(self, celsius: float):
        self._celsius = celsius  # convention: _ = internal

    @property
    def celsius(self) -> float:
        return self._celsius

    @celsius.setter
    def celsius(self, value: float):
        if value < -273.15:
            raise ValueError("Below absolute zero!")
        self._celsius = value

    @property
    def fahrenheit(self) -> float:
        return self._celsius * 9/5 + 32

t = Temperature(100)
print(t.fahrenheit)    # 212.0
t.celsius = -300       # raises ValueError
  • Properties replace getters/setters with clean attribute syntax
  • Single underscore _ is a convention meaning "private, don't touch"
  • Double underscore __ triggers name mangling for stronger hiding
  • Start with plain attributes; add properties later without changing the API

Exercises

  1. Validated Email

    Create a User class with a name and an email property that validates the email contains '@' on assignment.

    Hint

    Raise ValueError in the setter if '@' not in value.

  2. Read-Only Property

    Create a Circle class where radius is settable but area is a read-only property computed from the radius.

    Hint

    Define area with only @property (no setter). Attempting to set it will raise AttributeError.

Common Problems & Solutions

1 RecursionError when property accesses itself
Cause: The property getter references `self.celsius` instead of `self._celsius`, creating infinite recursion.
Solution: Use a different backing attribute name (conventionally prefixed with `_`).
2 AttributeError: can't set attribute
Cause: Trying to assign to a property that has no setter defined.
Solution: Add a `@property_name.setter` method, or make it intentionally read-only and document it.
