Microservices with NestJS: Transports and Patterns

Intuition

NestJS microservice transports are like different types of mail service. TCP is like a direct phone call — you wait for an answer. Redis pub/sub is like a bulletin board — you post a notice and whoever is interested reads it. RabbitMQ is like certified mail — guaranteed delivery with a receipt. Kafka is like a newspaper — published once, read by many subscribers at their own pace.

Microservices with NestJS: Transports and Patterns

Introduction

NestJS provides first-class support for building microservices with multiple transport layers (TCP, Redis, RabbitMQ, Kafka, gRPC, NATS). This lesson covers how to build, connect, and communicate between NestJS microservices using different transports and communication patterns.

TCP Transport (Simplest)

Service (Responder)

// user-service/src/main.ts
import { NestFactory } from '@nestjs/core';
import { MicroserviceOptions, Transport } from '@nestjs/microservices';
import { UserModule } from './user.module';

async function bootstrap() {
  const app = await NestFactory.createMicroservice<MicroserviceOptions>(
    UserModule,
    {
      transport: Transport.TCP,
      options: {
        host: '0.0.0.0',
        port: 3001,
      },
    },
  );
  await app.listen();
  console.log('User microservice listening on port 3001');
}
bootstrap();

// user-service/src/user.controller.ts
import { Controller } from '@nestjs/common';
import { MessagePattern, Payload, EventPattern } from '@nestjs/microservices';

@Controller()
export class UserController {
  constructor(private readonly userService: UserService) {}

  // Request-Response pattern
  @MessagePattern('get_user')
  async getUser(@Payload() data: { id: string }) {
    return this.userService.findById(data.id);
  }

  @MessagePattern('create_user')
  async createUser(@Payload() data: CreateUserDto) {
    return this.userService.create(data);
  }

  @MessagePattern('search_users')
  async searchUsers(@Payload() data: { query: string; page: number }) {
    return this.userService.search(data.query, data.page);
  }

  // Event pattern (fire-and-forget)
  @EventPattern('order_created')
  async handleOrderCreated(@Payload() data: { userId: string; orderId: string }) {
    await this.userService.incrementOrderCount(data.userId);
  }
}

Client (Requester)

// api-gateway/src/gateway.module.ts
import { Module } from '@nestjs/common';
import { ClientsModule, Transport } from '@nestjs/microservices';

@Module({
  imports: [
    ClientsModule.register([
      {
        name: 'USER_SERVICE',
        transport: Transport.TCP,
        options: { host: 'user-service', port: 3001 },
      },
      {
        name: 'ORDER_SERVICE',
        transport: Transport.TCP,
        options: { host: 'order-service', port: 3002 },
      },
    ]),
  ],
  controllers: [GatewayController],
})
export class GatewayModule {}

// api-gateway/src/gateway.controller.ts
import { Controller, Get, Post, Body, Param, Inject } from '@nestjs/common';
import { ClientProxy } from '@nestjs/microservices';
import { firstValueFrom } from 'rxjs';

@Controller('api')
export class GatewayController {
  constructor(
    @Inject('USER_SERVICE') private userClient: ClientProxy,
    @Inject('ORDER_SERVICE') private orderClient: ClientProxy,
  ) {}

  @Get('users/:id')
  async getUser(@Param('id') id: string) {
    // send() returns Observable — use firstValueFrom for Promise
    return firstValueFrom(
      this.userClient.send('get_user', { id })
    );
  }

  @Post('orders')
  async createOrder(@Body() data: CreateOrderDto) {
    const order = await firstValueFrom(
      this.orderClient.send('create_order', data)
    );

    // Fire-and-forget event to user service
    this.userClient.emit('order_created', {
      userId: data.userId,
      orderId: order.id,
    });

    return order;
  }
}

RabbitMQ Transport

// Service using RabbitMQ
const app = await NestFactory.createMicroservice(PaymentModule, {
  transport: Transport.RMQ,
  options: {
    urls: ['amqp://rabbitmq:5672'],
    queue: 'payment_queue',
    queueOptions: {
      durable: true,
    },
    prefetchCount: 10,
    noAck: false,
  },
});

// Controller with manual ack
@Controller()
export class PaymentController {
  @MessagePattern('process_payment')
  async processPayment(@Payload() data: any, @Ctx() context: RmqContext) {
    const channel = context.getChannelRef();
    const originalMsg = context.getMessage();

    try {
      const result = await this.paymentService.process(data);
      channel.ack(originalMsg);
      return result;
    } catch (err) {
      // Requeue on failure
      channel.nack(originalMsg, false, true);
      throw err;
    }
  }
}

// Client
ClientsModule.register([{
  name: 'PAYMENT_SERVICE',
  transport: Transport.RMQ,
  options: {
    urls: ['amqp://rabbitmq:5672'],
    queue: 'payment_queue',
    queueOptions: { durable: true },
  },
}]);

// Kafka microservice
const app = await NestFactory.createMicroservice(AnalyticsModule, {
  transport: Transport.KAFKA,
  options: {
    client: {
      clientId: 'analytics-service',
      brokers: ['kafka:9092'],
    },
    consumer: {
      groupId: 'analytics-consumer-group',
    },
  },
});

// Controller
@Controller()
export class AnalyticsController {
  @EventPattern('user.activity')
  async handleUserActivity(@Payload() message: any) {
    await this.analyticsService.track(message.value);
  }

  @MessagePattern('get_analytics')
  async getAnalytics(@Payload() data: { userId: string; period: string }) {
    return this.analyticsService.getReport(data.userId, data.period);
  }
}

Hybrid Application (HTTP + Microservice)

// A service that exposes both HTTP and microservice interfaces
async function bootstrap() {
  const app = await NestFactory.create(OrderModule);

  // Connect microservice transport
  app.connectMicroservice({
    transport: Transport.TCP,
    options: { host: '0.0.0.0', port: 3002 },
  });

  // Connect another transport (events)
  app.connectMicroservice({
    transport: Transport.REDIS,
    options: { host: 'redis', port: 6379 },
  });

  await app.startAllMicroservices();
  await app.listen(3000); // HTTP
}

// Controller handles both HTTP and microservice messages
@Controller()
export class OrderController {
  // HTTP endpoint
  @Get('orders/:id')
  @HttpCode(200)
  async getOrderHttp(@Param('id') id: string) {
    return this.orderService.findById(id);
  }

  // Microservice message pattern
  @MessagePattern('get_order')
  async getOrderMs(@Payload() data: { id: string }) {
    return this.orderService.findById(data.id);
  }

  // Event from other services
  @EventPattern('payment_completed')
  async handlePaymentCompleted(@Payload() data: any) {
    await this.orderService.markAsPaid(data.orderId);
  }
}

Common Pitfalls

  1. Not handling service unavailability: Use circuit breakers and timeouts. NestJS ClientProxy has built-in retry options.
  2. Mixing send and emit: send() waits for a response (request-response). emit() is fire-and-forget (event). Use the right one.
  3. Forgetting to call startAllMicroservices(): In hybrid apps, microservice transports don't start until you call this method.
  4. Blocking event handlers: Event patterns should not return values to the caller. Process quickly or offload to a queue.

Summary

NestJS microservices support multiple transports (TCP, Redis, RabbitMQ, Kafka, gRPC) with a consistent decorator-based API. Use @MessagePattern for request-response and @EventPattern for events. Hybrid applications can serve both HTTP and microservice interfaces. Choose your transport based on requirements: TCP for simplicity, Redis for pub/sub, RabbitMQ for reliable queuing, Kafka for high-throughput event streaming.

Exercises

  1. TCP Microservice Pair

    Build two NestJS microservices communicating over TCP: a User Service and an Order Service. The Order Service calls the User Service to validate users before creating orders.

    Hint

    Use ClientsModule.register with Transport.TCP. Use firstValueFrom to convert Observable to Promise.

  2. Event-Driven with Redis

    Build a system where an Order Service emits events via Redis transport, and both a Notification Service and an Analytics Service consume them independently.

    Hint

    Use @EventPattern in both consumers. Each service registers with a different consumer group.

  3. Hybrid HTTP + Microservice

    Create a NestJS service that exposes both REST endpoints for external clients and microservice message patterns for internal services. Use app.connectMicroservice for the TCP transport.

    Hint

    Use NestFactory.create for HTTP, then connectMicroservice for TCP. Call startAllMicroservices before listen.

Common Problems & Solutions

1 ClientProxy.send() hangs and never resolves
Cause: The target microservice is not running or not listening on the expected host:port.
Solution: Add timeout to requests. Implement health checks. Use `firstValueFrom` with `timeout` operator from rxjs.

import { timeout } from 'rxjs';\nfirstValueFrom(\n this.client.send('get_user', { id }).pipe(timeout(5000))\n);

2 Events being processed multiple times
Cause: Multiple instances of the same service are consuming from the same Redis/RabbitMQ channel without consumer groups.
Solution: Use consumer groups (Kafka) or exclusive queues (RabbitMQ). For Redis, use different channel names or implement idempotent handlers.
