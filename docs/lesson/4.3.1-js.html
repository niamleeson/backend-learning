Guards: Authentication and Authorization

Intuition

Guards are like bouncers at a club. The first bouncer (AuthGuard) checks your ID to verify who you are. The second bouncer (RolesGuard) checks if your name is on the VIP list. Only if you pass both checks do you get inside to the actual party (route handler).

Guards: Authentication and Authorization

Introduction

Guards are NestJS components that determine whether a request should be handled by the route handler. They run after middleware but before interceptors and pipes. Their primary use case is authentication (is the user who they claim to be?) and authorization (does the user have permission to access this resource?).

How Guards Work

A guard is a class that implements the CanActivate interface:

import { Injectable, CanActivate, ExecutionContext } from '@nestjs/common';

@Injectable()
export class AuthGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean | Promise<boolean> {
    const request = context.switchToHttp().getRequest();
    // Return true to allow, false to deny
    return this.validateRequest(request);
  }

  private validateRequest(request: any): boolean {
    const token = request.headers.authorization;
    return !!token; // Simple check: does a token exist?
  }
}

If canActivate() returns false or throws an exception, NestJS returns a 403 Forbidden response automatically.

Guard Execution in the Request Pipeline

Middleware -> GUARDS -> Interceptors -> Pipes -> Handler -> Interceptors -> Filters
                ^
                |
         We are here

Guards have access to the ExecutionContext, which provides information about the current request, the handler that will execute, and the class the handler belongs to.

ExecutionContext

ExecutionContext extends ArgumentsHost and adds methods for inspecting the current handler:

canActivate(context: ExecutionContext): boolean {
  // Get the HTTP request
  const request = context.switchToHttp().getRequest();
  const response = context.switchToHttp().getResponse();

  // Get the handler method and class
  const handler = context.getHandler();   // e.g., findAll method
  const classRef = context.getClass();     // e.g., UsersController class

  // Useful for reading custom metadata (decorators)
  return true;
}

Binding Guards

Method-Level

@Controller('users')
export class UsersController {
  @UseGuards(AuthGuard)
  @Get('profile')
  getProfile() {
    return { message: 'Protected profile' };
  }
}

Controller-Level

@Controller('admin')
@UseGuards(AuthGuard)
export class AdminController {
  @Get('dashboard')
  getDashboard() { /* protected */ }

  @Get('settings')
  getSettings() { /* also protected */ }
}

Global

// main.ts
const app = await NestFactory.create(AppModule);
app.useGlobalGuards(new AuthGuard());

// Or via DI (preferred -- allows injection):
// app.module.ts
@Module({
  providers: [
    {
      provide: APP_GUARD,
      useClass: AuthGuard,
    },
  ],
})
export class AppModule {}

Authentication Guard with JWT

import {
  Injectable, CanActivate, ExecutionContext,
  UnauthorizedException,
} from '@nestjs/common';
import { JwtService } from '@nestjs/jwt';

@Injectable()
export class JwtAuthGuard implements CanActivate {
  constructor(private jwtService: JwtService) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const request = context.switchToHttp().getRequest();
    const token = this.extractToken(request);

    if (!token) {
      throw new UnauthorizedException('No token provided');
    }

    try {
      const payload = await this.jwtService.verifyAsync(token);
      request.user = payload; // Attach user to request
    } catch {
      throw new UnauthorizedException('Invalid token');
    }

    return true;
  }

  private extractToken(request: any): string | null {
    const [type, token] = request.headers.authorization?.split(' ') ?? [];
    return type === 'Bearer' ? token : null;
  }
}

Role-Based Authorization Guard

Combine guards with custom metadata using SetMetadata and the Reflector:

// decorators/roles.decorator.ts
import { SetMetadata } from '@nestjs/common';

export const ROLES_KEY = 'roles';
export const Roles = (...roles: string[]) => SetMetadata(ROLES_KEY, roles);

// guards/roles.guard.ts
import { Injectable, CanActivate, ExecutionContext } from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import { ROLES_KEY } from '../decorators/roles.decorator';

@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    const requiredRoles = this.reflector.getAllAndOverride<string[]>(
      ROLES_KEY,
      [context.getHandler(), context.getClass()],
    );

    // If no roles are required, allow access
    if (!requiredRoles) {
      return true;
    }

    const { user } = context.switchToHttp().getRequest();
    return requiredRoles.some((role) => user.roles?.includes(role));
  }
}

Usage:

@Controller('admin')
@UseGuards(JwtAuthGuard, RolesGuard)
export class AdminController {
  @Get('dashboard')
  @Roles('admin')
  getDashboard() {
    return { message: 'Admin dashboard' };
  }

  @Get('reports')
  @Roles('admin', 'manager')
  getReports() {
    return { message: 'Reports' };
  }
}

Public Routes with Global Guard

When using a global auth guard, you need a way to mark public routes:

// decorators/public.decorator.ts
import { SetMetadata } from '@nestjs/common';

export const IS_PUBLIC_KEY = 'isPublic';
export const Public = () => SetMetadata(IS_PUBLIC_KEY, true);

// guards/jwt-auth.guard.ts
@Injectable()
export class JwtAuthGuard implements CanActivate {
  constructor(
    private jwtService: JwtService,
    private reflector: Reflector,
  ) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    // Check if route is marked as public
    const isPublic = this.reflector.getAllAndOverride<boolean>(
      IS_PUBLIC_KEY,
      [context.getHandler(), context.getClass()],
    );

    if (isPublic) return true;

    // ... normal JWT validation
    const request = context.switchToHttp().getRequest();
    const token = this.extractToken(request);
    if (!token) throw new UnauthorizedException();

    try {
      request.user = await this.jwtService.verifyAsync(token);
    } catch {
      throw new UnauthorizedException();
    }
    return true;
  }

  private extractToken(request: any): string | null {
    const [type, token] = request.headers.authorization?.split(' ') ?? [];
    return type === 'Bearer' ? token : null;
  }
}

@Controller('products')
export class ProductsController {
  @Public()
  @Get()
  findAll() { /* No auth needed */ }

  @Post()
  create() { /* Requires auth (global guard) */ }
}

Permission-Based Guard (ABAC)

// decorators/permissions.decorator.ts
export const PERMISSIONS_KEY = 'permissions';
export const RequirePermissions = (...permissions: string[]) =>
  SetMetadata(PERMISSIONS_KEY, permissions);

// guards/permissions.guard.ts
@Injectable()
export class PermissionsGuard implements CanActivate {
  constructor(
    private reflector: Reflector,
    private permissionsService: PermissionsService,
  ) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const required = this.reflector.getAllAndOverride<string[]>(
      PERMISSIONS_KEY,
      [context.getHandler(), context.getClass()],
    );

    if (!required) return true;

    const { user } = context.switchToHttp().getRequest();
    const userPermissions = await this.permissionsService.getUserPermissions(
      user.id,
    );

    return required.every((perm) => userPermissions.includes(perm));
  }
}

Usage:

@Controller('articles')
export class ArticlesController {
  @Post()
  @RequirePermissions('articles:create')
  create(@Body() dto: CreateArticleDto) {}

  @Delete(':id')
  @RequirePermissions('articles:delete')
  remove(@Param('id') id: string) {}
}

Resource Ownership Guard

@Injectable()
export class OwnerGuard implements CanActivate {
  constructor(private postsService: PostsService) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const request = context.switchToHttp().getRequest();
    const user = request.user;
    const postId = request.params.id;

    const post = await this.postsService.findOne(postId);
    if (!post) return false;

    return post.authorId === user.id;
  }
}

// Usage
@Controller('posts')
@UseGuards(JwtAuthGuard)
export class PostsController {
  @Put(':id')
  @UseGuards(OwnerGuard)
  update(@Param('id') id: string, @Body() dto: UpdatePostDto) {}

  @Delete(':id')
  @UseGuards(OwnerGuard)
  remove(@Param('id') id: string) {}
}

Multiple Guards

Guards execute in order. If any guard returns false, the request is denied:

@UseGuards(JwtAuthGuard, RolesGuard, PermissionsGuard)
@Get('sensitive-data')
getSensitiveData() {}

Execution: JwtAuthGuard -> RolesGuard -> PermissionsGuard. All must pass.

Custom @CurrentUser Decorator

import { createParamDecorator, ExecutionContext } from '@nestjs/common';

export const CurrentUser = createParamDecorator(
  (data: string | undefined, ctx: ExecutionContext) => {
    const request = ctx.switchToHttp().getRequest();
    const user = request.user;
    return data ? user?.[data] : user;
  },
);

// Usage
@Get('profile')
@UseGuards(JwtAuthGuard)
getProfile(@CurrentUser() user: User) {
  return user;
}

@Get('profile/name')
@UseGuards(JwtAuthGuard)
getProfileName(@CurrentUser('name') name: string) {
  return { name };
}

Common Pitfalls

  1. Guard order matters -- Auth guard must run before roles guard, since roles guard needs request.user.
  2. Throwing the wrong exception -- Guards should throw UnauthorizedException (401) for auth failures and ForbiddenException (403) for authorization failures.
  3. Forgetting Reflector -- To read custom metadata (roles, permissions), you need to inject Reflector.
  4. Global guards without @Public() -- Every route becomes protected. Health checks, login, and signup endpoints need the @Public() decorator.

Exercises

  1. Build a JWT Auth Guard

    Create a JwtAuthGuard that extracts a Bearer token from the Authorization header, verifies it using JwtService, and attaches the decoded payload to request.user. Register it as a global guard using APP_GUARD. Add a @Public() decorator for unauthenticated routes.

    Hint

    Use @nestjs/jwt JwtService. Register the guard with { provide: APP_GUARD, useClass: JwtAuthGuard }. Use Reflector to check for the IS_PUBLIC_KEY metadata.

  2. Implement Role-Based Access Control

    Create a @Roles() decorator and RolesGuard. Define three roles: 'user', 'moderator', 'admin'. Protect endpoints so that only admins can delete resources, moderators and admins can edit, and all authenticated users can read.

    Hint

    Use SetMetadata to store roles. In the guard, use reflector.getAllAndOverride() to read roles from both handler and class. Check if the user has at least one required role.

  3. Resource Ownership Guard

    Build an OwnerGuard that checks if the current user owns the resource being accessed. The guard should work for any entity by accepting a service token and ID parameter name via metadata.

    Hint

    Create a custom decorator like @CheckOwnership('postsService', 'id'). In the guard, use ModuleRef to dynamically get the service and call findOne().

Common Problems & Solutions

1 Guard returns 403 Forbidden instead of 401 Unauthorized
Cause: The guard returns `false` instead of throwing `UnauthorizedException`. When a guard returns false, NestJS defaults to 403.
Solution: Throw `new UnauthorizedException()` for authentication failures instead of returning false. Return false only for authorization failures (which correctly results in 403).
2 RolesGuard always denies access even with correct roles
Cause: The RolesGuard runs before the AuthGuard, so request.user is undefined when roles are checked.
Solution: Ensure AuthGuard is listed BEFORE RolesGuard in the @UseGuards() decorator: `@UseGuards(AuthGuard, RolesGuard)`.
3 Global guard can't inject dependencies
Cause: The guard is registered in main.ts with `app.useGlobalGuards(new AuthGuard())`, bypassing the DI container.
Solution: Register the guard via DI in a module: `{ provide: APP_GUARD, useClass: AuthGuard }`. This allows NestJS to inject dependencies like JwtService and Reflector.
