Async Patterns: Semaphores, Queues, and Error Handling

Intuition

Semaphores are like a bouncer limiting how many people enter a club at once. Async queues are conveyor belts connecting producers and consumers. Together they build controlled, resilient pipelines.

Async Patterns: Semaphores, Queues, and Error Handling

Advanced async patterns help manage concurrency limits, work distribution, and robust error handling in production asyncio code.

import asyncio
from contextlib import asynccontextmanager

# Semaphore: limit concurrent operations
async def fetch_with_limit(sem: asyncio.Semaphore, url: str) -> str:
    async with sem:
        print(f"Fetching {url} (concurrency limited)")
        await asyncio.sleep(1)  # simulated request
        return f"data from {url}"

async def limited_fetch(urls: list[str], max_concurrent: int = 5):
    sem = asyncio.Semaphore(max_concurrent)
    tasks = [fetch_with_limit(sem, url) for url in urls]
    return await asyncio.gather(*tasks, return_exceptions=True)

# Async Queue: producer-consumer pattern
async def producer(queue: asyncio.Queue, items: list[str]):
    for item in items:
        await queue.put(item)
    await queue.put(None)  # sentinel

async def consumer(queue: asyncio.Queue):
    while True:
        item = await queue.get()
        if item is None:
            break
        print(f"Processing: {item}")
        queue.task_done()

async def main():
    queue = asyncio.Queue(maxsize=10)
    await asyncio.gather(
        producer(queue, ["a", "b", "c"]),
        consumer(queue),
    )

Key Takeaways: - Semaphore(n) limits concurrency to n simultaneous operations - asyncio.Queue enables async producer-consumer patterns - Always use return_exceptions=True in gather for resilient pipelines - Use asyncio.shield() to protect critical coroutines from cancellation - Implement retry logic with exponential backoff for transient failures

Common Problems & Solutions

1 Semaphore does not release when an exception occurs inside `async with sem`.
Cause: This is actually not the problem; `async with` handles exceptions. The real issue is exceptions going unhandled in tasks.
Solution: Use `return_exceptions=True` in `gather`, or wrap task bodies in try/except. The semaphore context manager correctly releases on exceptions.
2 Async queue consumer hangs forever waiting for items.
Cause: The producer finished but did not send a sentinel value to signal completion.
Solution: Send a sentinel (e.g., `None`) per consumer, or use `queue.join()` with `task_done()` calls. Set a timeout on `queue.get()`.
