Sets and Frozensets

Intuition

Sets are dicts without values — they use the same hash-table machinery to guarantee uniqueness and O(1) membership checks.

Sets are unordered collections of unique, hashable elements backed by hash tables. Frozensets are their immutable counterpart and can be used as dict keys or set members.

a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

print(a & b)   # intersection: {3, 4}
print(a | b)   # union: {1, 2, 3, 4, 5, 6}
print(a - b)   # difference: {1, 2}
print(a ^ b)   # symmetric difference: {1, 2, 5, 6}

# Frozenset as a dict key
cache = {}
key = frozenset([1, 2, 3])
cache[key] = "result"
  • Membership testing (in) is O(1) average — much faster than scanning a list.
  • Set elements must be hashable; you cannot put a list inside a set.
  • Use frozenset when you need an immutable set (e.g., as a dict key or inside another set).
  • Set comprehensions follow the same syntax as dict comprehensions but without colons: {x**2 for x in range(5)}.

Exercises

  1. Find Common Elements

    Given two lists, write a function common(a, b) that returns a sorted list of elements present in both, without duplicates.

    Hint

    Convert both lists to sets, take the intersection, then sort the result.

  2. Remove Duplicates Preserving Order

    Write a function unique_ordered(lst) that returns a new list with duplicates removed while preserving the first occurrence order.

    Hint

    Iterate through the list, maintaining a seen set. Append to result only if the item is not in seen.

Common Problems & Solutions

1 `{}` creates an empty dict, not an empty set.
Cause: Python's syntax overloads `{}` for dicts. An empty set requires `set()`.
Solution: Use `set()` to create an empty set. `{1, 2}` works for non-empty set literals.
2 Adding a list to a set raises `TypeError: unhashable type: 'list'`.
Cause: Set elements must be hashable; mutable types like lists are not.
Solution: Convert the list to a tuple or frozenset before adding.
