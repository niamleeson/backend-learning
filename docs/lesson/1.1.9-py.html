Comprehensions: List, Dict, Set

Intuition

A comprehension is a compact recipe: 'give me [this transformation] for [each item] if [this condition holds].' It reads almost like English once you get the pattern.

Comprehensions provide a concise, readable way to create lists, dicts, and sets from iterables. They replace common patterns involving loops and conditional appends.

# List comprehension
squares = [x**2 for x in range(10)]
evens = [x for x in range(20) if x % 2 == 0]

# Dict comprehension
word_lengths = {w: len(w) for w in ["hello", "world", "python"]}

# Set comprehension
unique_vowels = {c for c in "hello world" if c in "aeiou"}

# Nested comprehension (flatten a matrix)
matrix = [[1, 2], [3, 4], [5, 6]]
flat = [num for row in matrix for num in row]  # [1,2,3,4,5,6]

# With walrus operator (Python 3.8+)
results = [y for x in range(10) if (y := x**2) > 20]
  • Comprehensions are more readable and often faster than equivalent loops
  • Keep comprehensions simple; if it needs more than 2 lines, use a regular loop
  • Use generator expressions (x for x in ...) for large sequences to save memory
  • Nested comprehensions read left to right, same order as nested for loops

Exercises

  1. Pythagorean Triples

    Use a list comprehension to find all Pythagorean triples (a, b, c) where a, b, c are all less than or equal to 20 and a < b < c.

    Hint

    Triple nested: [(a,b,c) for a in range(1,21) for b in range(a+1,21) for c in range(b+1,21) if a**2+b**2==c**2].

  2. Invert a Dictionary

    Given d = {'a': 1, 'b': 2, 'c': 3}, use a dict comprehension to swap keys and values.

    Hint

    Use {v: k for k, v in d.items()}.

Common Problems & Solutions

1 Comprehension is unreadable with too many conditions and loops
Cause: Overusing comprehensions for complex logic reduces readability.
Solution: If a comprehension spans more than ~80 characters or has 3+ clauses, refactor into a regular loop.
2 MemoryError from a huge list comprehension
Cause: A list comprehension materializes all elements in memory at once.
Solution: Use a generator expression `(x for x in ...)` to yield items lazily.
