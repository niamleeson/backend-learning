Design a Food Delivery System (DoorDash)

Intuition

Think of a food delivery platform as a three-way coordination problem, like an air traffic control tower. The customer is the departing passenger, the restaurant is the terminal kitchen, and the driver is the airplane. The dispatch engine is the control tower that must decide which plane to assign to which gate, considering where each plane is right now, how fast it is moving, which direction it is heading, and how long the food will take to prepare. The live tracking map is like the flight tracker -- it shows the customer exactly where their food is. The state machine (PLACED, PREPARING, PICKED_UP, DELIVERED) is like the flight status board (BOARDING, DEPARTED, IN FLIGHT, LANDED). Everything must happen within a tight time window because, unlike a package, food cannot sit on a shelf for days.

Design a Food Delivery System (DoorDash)

Introduction

A food delivery system like DoorDash connects three distinct user types -- customers, restaurants, and delivery drivers -- in a real-time, location-aware marketplace. The technical challenges include real-time order tracking, driver dispatch optimization, dynamic ETA calculation, geospatial queries, and managing the complex state machine of an order as it moves through multiple stages. Unlike pure e-commerce, food delivery has hard time constraints: food quality degrades rapidly, so efficiency in matching, routing, and delivery is paramount.

Step 1: Requirements Gathering

Functional Requirements

  • Customer: Browse nearby restaurants, view menus, place orders, track delivery in real-time, rate experience
  • Restaurant: Manage menu, receive orders, update preparation status, set availability hours
  • Driver: Go online/offline, receive delivery offers, navigate to pickup/dropoff, confirm pickup/delivery
  • Order Management: Create, pay, assign driver, track through lifecycle
  • ETA Estimation: Estimated time for food preparation, pickup, and delivery
  • Search & Discovery: Search restaurants by cuisine, filters, ratings, delivery time
  • Promotions: Coupons, surge pricing, restaurant promotions

Non-Functional Requirements

  • Latency: Nearby restaurants load < 300ms, driver assignment < 30s
  • Availability: 99.99% uptime during meal rush hours
  • Real-time: Driver location updates every 5 seconds, order status push notifications
  • Scale: 10M+ orders/day, 2M active drivers, 500K restaurants

Step 2: Capacity Estimation

Assumptions:
- 50M registered customers, 10M DAU
- 500K restaurants
- 2M registered drivers, 500K active during peak
- 10M orders/day, 70% during lunch (11-2) and dinner (5-9)
- Peak: 7M orders in 7 hours = ~280 orders/sec

Traffic:
- Restaurant search: 10M * 3 searches/day = 30M/day = ~350 QPS
- Peak search: ~1,000 QPS
- Order creation: ~280 TPS peak
- Driver location updates: 500K drivers * 1 update/5s = 100K updates/sec
- Order tracking: 1M active orders * 1 check/10s = 100K QPS

Storage:
- Restaurant data: 500K * 20KB = ~10GB
- Menu items: 500K * 50 items * 2KB = ~50GB
- Orders: 10M/day * 5KB * 365 = ~18TB/year
- Driver locations: 100K updates/sec * 100 bytes = ~10MB/sec (time-series)
- Images: 500K restaurants * 60 images = ~15TB (CDN)

Bandwidth:
- Driver location ingestion: ~10MB/sec
- Location push to customers (tracking): ~5MB/sec

Restaurant Search

import express from 'express';
const router = express.Router();

// GET /api/restaurants?lat=37.78&lng=-122.41&cuisine=italian
//   &maxDeliveryTime=30&minRating=4.0&sort=relevance&page=1
router.get('/restaurants', async (req, res) => {
  const {
    lat, lng, cuisine, query,
    maxDeliveryTime, minRating,
    sort = 'relevance', page = 1, limit = 20,
  } = req.query;

  if (!lat || !lng) {
    return res.status(400).json({ error: 'Location required' });
  }

  const maxDistance = '5km'; // 5km delivery radius

  const searchBody = {
    from: (page - 1) * limit,
    size: limit,
    query: {
      bool: {
        must: [
          { term: { status: 'OPEN' } },
          ...(query ? [{ multi_match: {
            query, fields: ['name^3', 'cuisine', 'menuItems'],
          } }] : []),
        ],
        filter: [
          { geo_distance: {
            distance: maxDistance,
            location: { lat: parseFloat(lat), lon: parseFloat(lng) },
          } },
          ...(cuisine ? [{ term: { cuisine } }] : []),
          ...(minRating ? [{ range: { rating: { gte: parseFloat(minRating) } } }] : []),
        ],
      },
    },
    sort: sort === 'distance'
      ? [{ _geo_distance: { location: { lat: parseFloat(lat), lon: parseFloat(lng) }, order: 'asc' } }]
      : sort === 'rating'
        ? [{ rating: 'desc' }]
        : ['_score'],
  };

  const results = await elasticsearch.search({
    index: 'restaurants',
    body: searchBody,
  });

  // Enrich with real-time data (delivery time estimates, open status)
  const restaurants = await Promise.all(
    results.hits.hits.map(async (hit) => {
      const restaurant = { id: hit._id, ...hit._source };
      const eta = await etaService.getRestaurantETA(
        restaurant.id, parseFloat(lat), parseFloat(lng)
      );
      return {
        ...restaurant,
        estimatedDeliveryTime: eta.totalMinutes,
        deliveryFee: calculateDeliveryFee(eta.distanceKm),
        distance: hit.sort?.[0],
      };
    })
  );

  // Filter by max delivery time if specified
  const filtered = maxDeliveryTime
    ? restaurants.filter(r => r.estimatedDeliveryTime <= parseInt(maxDeliveryTime))
    : restaurants;

  res.json({ data: filtered, meta: { page, limit, total: results.hits.total.value } });
});

Order Placement

// POST /api/orders
router.post('/orders', authenticate, async (req, res) => {
  const {
    restaurantId, items, deliveryAddress,
    paymentMethodId, specialInstructions, tip,
  } = req.body;

  // 1. Validate restaurant is open and items are available
  const restaurant = await restaurantService.getRestaurant(restaurantId);
  if (!restaurant || restaurant.status !== 'OPEN') {
    return res.status(400).json({ error: 'Restaurant is closed' });
  }

  // 2. Calculate pricing
  const menuItems = await prisma.menuItem.findMany({
    where: { id: { in: items.map(i => i.menuItemId) } },
  });

  const subtotal = items.reduce((sum, item) => {
    const menuItem = menuItems.find(m => m.id === item.menuItemId);
    if (!menuItem) throw new Error(`Item ${item.menuItemId} not found`);
    return sum + parseFloat(menuItem.price) * item.quantity;
  }, 0);

  const deliveryFee = calculateDeliveryFee(
    getDistance(restaurant, deliveryAddress)
  );
  const serviceFee = subtotal * 0.15;
  const taxRate = 0.0875; // example tax
  const tax = (subtotal + serviceFee) * taxRate;
  const total = subtotal + deliveryFee + serviceFee + tax + (tip || 0);

  // 3. Create order
  const order = await prisma.order.create({
    data: {
      userId: req.user.id,
      restaurantId,
      status: 'PLACED',
      subtotal,
      deliveryFee,
      serviceFee,
      tax,
      tip: tip || 0,
      total,
      deliveryAddress: JSON.stringify(deliveryAddress),
      specialInstructions,
      items: {
        create: items.map(item => {
          const menuItem = menuItems.find(m => m.id === item.menuItemId);
          return {
            menuItemId: item.menuItemId,
            quantity: item.quantity,
            price: menuItem.price,
            customizations: item.customizations || {},
          };
        }),
      },
    },
    include: { items: true },
  });

  // 4. Process payment
  const payment = await paymentService.charge({
    amount: total,
    currency: 'USD',
    paymentMethodId,
    metadata: { orderId: order.id },
  });

  // 5. Notify restaurant
  await kafka.produce('order.events', {
    type: 'order.placed',
    orderId: order.id,
    restaurantId,
  });

  // 6. Start dispatch process (find a driver)
  await dispatchQueue.add('find-driver', {
    orderId: order.id,
    restaurantLocation: {
      lat: restaurant.latitude,
      lng: restaurant.longitude,
    },
    deliveryLocation: deliveryAddress,
  });

  // 7. Calculate initial ETA
  const eta = await etaService.calculateOrderETA(order.id);

  res.status(201).json({ ...order, estimatedDeliveryTime: eta });
});

// POST /api/drivers/location (called every 5 seconds from driver app)
router.post('/drivers/location', authenticateDriver, async (req, res) => {

  const driverId = req.driver.id;

  await redis.geoadd('drivers:active', longitude, latitude, driverId);

    lng: longitude.toString(),
    heading: heading.toString(),
    speed: speed.toString(),
    updatedAt: Date.now().toString(),
  });

  const activeOrderId = await redis.get(`driver:${driverId}:activeOrder`);
  if (activeOrderId) {
    await redis.publish(`order:${activeOrderId}:tracking`, JSON.stringify({
      driverId,
      latitude,
      longitude,
      heading,
      speed,
      timestamp: Date.now(),
    }));

    // Update ETA
    const eta = await etaService.updateDriverETA(activeOrderId, {
      lat: latitude, lng: longitude, speed,
    });
    await redis.publish(`order:${activeOrderId}:eta`, JSON.stringify({ eta }));
  }

  res.status(204).send();
});

Real-Time Order Tracking (WebSocket)

import { WebSocketServer } from 'ws';
import Redis from 'ioredis';

const wss = new WebSocketServer({ port: 8080 });
const subscriber = new Redis();

wss.on('connection', (ws, req) => {
  const orderId = new URL(req.url, 'http://localhost').searchParams.get('orderId');
  if (!orderId) return ws.close();

  const trackingChannel = `order:${orderId}:tracking`;
  const statusChannel = `order:${orderId}:status`;
  const etaChannel = `order:${orderId}:eta`;

  const handler = (channel, message) => {
    if ([trackingChannel, statusChannel, etaChannel].includes(channel)) {
      ws.send(JSON.stringify({
        type: channel.split(':').pop(), // 'tracking', 'status', or 'eta'
        data: JSON.parse(message),
      }));
    }
  };

  subscriber.subscribe(trackingChannel, statusChannel, etaChannel);
  subscriber.on('message', handler);

  ws.on('close', () => {
    subscriber.unsubscribe(trackingChannel, statusChannel, etaChannel);
    subscriber.removeListener('message', handler);
  });
});

Step 6: Deep Dive - Driver Dispatch

The dispatch engine is the heart of the system. It must efficiently match orders with nearby available drivers.

Dispatch Algorithm:

1. Get order's restaurant location
2. Find available drivers within radius (start 2km, expand to 5km)
3. Score each driver:
   - Distance to restaurant (closer = better)
   - Driver rating
   - Vehicle type match
   - Current heading (already going that direction?)
4. Offer to best-scoring driver
5. Driver has 30 seconds to accept
6. If declined/timeout, offer to next driver
7. After 3 rejections, expand radius and try again

class DispatchEngine {
  async findDriver(orderId, restaurantLocation, deliveryLocation) {
    const maxAttempts = 5;
    let radius = 2; // km

    for (let attempt = 0; attempt < maxAttempts; attempt++) {
      // Find nearby available drivers using Redis geospatial
      const nearbyDrivers = await redis.georadius(
        'drivers:active',
        restaurantLocation.lng,
        restaurantLocation.lat,
        radius,
        'km',
        'WITHDIST',
        'ASC',
        'COUNT', 20
      );

      // Filter to only available drivers
      const availableDrivers = [];
      for (const [driverId, distance] of nearbyDrivers) {
        const driverStatus = await redis.hget(`driver:${driverId}:status`, 'status');
        if (driverStatus === 'AVAILABLE') {
          const driverData = await redis.hgetall(`driver:${driverId}:location`);
          availableDrivers.push({
            id: driverId,
            distance: parseFloat(distance),
            ...driverData,
          });
        }
      }

      if (availableDrivers.length === 0) {
        radius = Math.min(radius * 1.5, 10); // expand radius
        continue;
      }

      // Score and sort drivers
      const scored = availableDrivers.map(driver => ({
        ...driver,
        score: this.scoreDriver(driver, restaurantLocation, deliveryLocation),
      })).sort((a, b) => b.score - a.score);

      // Offer to drivers in order
      for (const driver of scored.slice(0, 3)) {
        const accepted = await this.offerToDriver(driver.id, orderId);
        if (accepted) {
          return driver;
        }
      }

      radius = Math.min(radius * 1.5, 10);
    }

    return null; // No driver found
  }

  scoreDriver(driver, restaurantLoc, deliveryLoc) {
    let score = 100;

    // Distance penalty: -10 per km
    score -= driver.distance * 10;

    // Heading bonus: driver moving toward restaurant gets +15
    if (this.isHeadingToward(driver, restaurantLoc)) {
      score += 15;
    }

    return Math.max(score, 0);
  }

  async offerToDriver(driverId, orderId) {
    // Send push notification to driver
    await redis.publish(`driver:${driverId}:offers`, JSON.stringify({
      orderId,
      type: 'delivery_offer',
      expiresIn: 30,
    }));

    // Wait for response (30 second timeout)
    return new Promise((resolve) => {
      const timeout = setTimeout(() => resolve(false), 30000);
      const channel = `driver:${driverId}:response:${orderId}`;

      const sub = new Redis();
      sub.subscribe(channel);
      sub.on('message', (ch, msg) => {
        clearTimeout(timeout);
        sub.unsubscribe(channel);
        sub.disconnect();
        resolve(JSON.parse(msg).accepted);
      });
    });
  }

  isHeadingToward(driver, target) {
    const bearing = this.calculateBearing(
      parseFloat(driver.lat), parseFloat(driver.lng),
      target.lat, target.lng
    );
    const headingDiff = Math.abs(parseFloat(driver.heading) - bearing);
    return headingDiff < 45 || headingDiff > 315;
  }

  calculateBearing(lat1, lng1, lat2, lng2) {
    const dLng = (lng2 - lng1) * Math.PI / 180;
    const y = Math.sin(dLng) * Math.cos(lat2 * Math.PI / 180);
    const x = Math.cos(lat1 * Math.PI / 180) * Math.sin(lat2 * Math.PI / 180) -
              Math.sin(lat1 * Math.PI / 180) * Math.cos(lat2 * Math.PI / 180) * Math.cos(dLng);
    return (Math.atan2(y, x) * 180 / Math.PI + 360) % 360;
  }
}

// Order state machine
const ORDER_TRANSITIONS = {
  PLACED:              ['ACCEPTED', 'CANCELLED'],
  ACCEPTED:            ['PREPARING', 'CANCELLED'],
  PREPARING:           ['READY_FOR_PICKUP', 'CANCELLED'],
  READY_FOR_PICKUP:    ['DRIVER_EN_ROUTE_PICKUP'],
  DRIVER_EN_ROUTE_PICKUP: ['PICKED_UP'],
  PICKED_UP:           ['EN_ROUTE_DELIVERY'],
  EN_ROUTE_DELIVERY:   ['DELIVERED'],
  DELIVERED:           [],
  CANCELLED:           [],
};

async function updateOrderStatus(orderId, newStatus, metadata = {}) {
  const order = await prisma.order.findUnique({ where: { id: orderId } });
  if (!order) throw new Error('Order not found');

  const allowedTransitions = ORDER_TRANSITIONS[order.status];
  if (!allowedTransitions?.includes(newStatus)) {
    throw new Error(`Cannot transition from ${order.status} to ${newStatus}`);
  }

  const timestampField = {
    ACCEPTED: 'acceptedAt',
    PICKED_UP: 'pickedUpAt',
    DELIVERED: 'deliveredAt',
    CANCELLED: 'cancelledAt',
  }[newStatus];

  const updateData = {
    status: newStatus,
    ...(timestampField && { [timestampField]: new Date() }),
    ...(metadata.driverId && { driverId: metadata.driverId }),
    ...(metadata.cancellationReason && { cancellationReason: metadata.cancellationReason }),
  };

  const updated = await prisma.order.update({
    where: { id: orderId },
    data: updateData,
  });

  // Broadcast status change
  await redis.publish(`order:${orderId}:status`, JSON.stringify({
    status: newStatus,
    timestamp: new Date().toISOString(),
    ...metadata,
  }));

  // Emit event for downstream processing
  await kafka.produce('order.events', {
    type: `order.${newStatus.toLowerCase()}`,
    orderId,
    data: updated,
  });

  return updated;
}

Step 9: ETA Estimation

class ETAService {
  async calculateOrderETA(orderId) {
    const order = await prisma.order.findUnique({
      where: { id: orderId },
      include: { restaurant: true },
    });

    const deliveryAddress = JSON.parse(order.deliveryAddress);

    // 1. Restaurant prep time (from historical data + current queue)
    const prepTime = await this.estimatePrepTime(order.restaurantId);

    // 2. Driver to restaurant time
    const driverToRestaurant = order.driverId
      ? await this.estimateTravelTime(
          await this.getDriverLocation(order.driverId),
          { lat: order.restaurant.latitude, lng: order.restaurant.longitude }
        )
      : 5; // assume 5 min if no driver assigned yet

    // 3. Restaurant to delivery address time
    const restaurantToCustomer = await this.estimateTravelTime(
      { lat: order.restaurant.latitude, lng: order.restaurant.longitude },
      deliveryAddress
    );

    // Pickup happens when both prep and driver arrival are complete
    const pickupReady = Math.max(prepTime, driverToRestaurant);
    const totalETA = pickupReady + restaurantToCustomer + 3; // +3 min buffer

    return {
      prepTime,
      driverToRestaurant,
      restaurantToCustomer,
      totalMinutes: Math.round(totalETA),
      estimatedDeliveryAt: new Date(Date.now() + totalETA * 60000),
    };
  }

  async estimatePrepTime(restaurantId) {
    // Use historical average + current queue depth
    const avgPrepTime = await redis.get(`restaurant:${restaurantId}:avgPrepTime`);
    const activeOrders = await prisma.order.count({
      where: {
        restaurantId,
        status: { in: ['ACCEPTED', 'PREPARING'] },
      },
    });

    const basePrepTime = parseFloat(avgPrepTime) || 20;
    return basePrepTime + (activeOrders * 3); // +3 min per active order
  }

  async estimateTravelTime(from, to) {
    // Use Haversine distance + traffic factor
    const distanceKm = this.haversineDistance(
      from.lat, from.lng, to.lat, to.lng
    );
    const avgSpeedKmPerHour = 25; // urban average
    return (distanceKm / avgSpeedKmPerHour) * 60; // convert to minutes
  }

  haversineDistance(lat1, lng1, lat2, lng2) {
    const R = 6371; // Earth's radius in km
    const dLat = (lat2 - lat1) * Math.PI / 180;
    const dLng = (lng2 - lng1) * Math.PI / 180;
    const a = Math.sin(dLat / 2) ** 2 +
              Math.cos(lat1 * Math.PI / 180) * Math.cos(lat2 * Math.PI / 180) *
              Math.sin(dLng / 2) ** 2;
    return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
  }
}

Summary

A food delivery system is a real-time, three-sided marketplace requiring geospatial services, real-time tracking, intelligent dispatch, and complex order state management. The core components are: Redis geospatial indexes for driver location (handles 100K updates/sec), a dispatch engine that scores drivers based on distance and heading, WebSocket connections for live order tracking, and a state machine ensuring valid order transitions. Regional architecture is essential -- each city operates somewhat independently, reducing cross-region latency. The ETA service combines restaurant prep time, driver travel time, and traffic conditions to provide accurate delivery estimates.

Common Problems & Solutions

1 Drivers receiving delivery offers for restaurants that are far away, leading to high decline rates
Symptoms: Average driver acceptance rate below 50%. Orders waiting more than 10 minutes for driver assignment. Driver complaints about unprofitable long-distance pickups.
Cause: The dispatch engine uses a fixed radius that is too large, or the scoring algorithm does not sufficiently weight distance. High decline rates mean orders wait longer for a driver, degrading customer experience.
Solution: Start with a small radius (2km) and gradually expand. Weight distance heavily in the scoring formula. Consider the driver's current heading -- a driver 3km away but heading toward the restaurant is better than one 1km away heading the opposite direction. Track acceptance rates per radius and tune dynamically.
2 Real-time tracking showing stale or jumping driver positions
Cause: Location updates from the driver app are inconsistent due to GPS drift, cellular connectivity gaps, or the app being backgrounded by the OS. The client interpolates incorrectly between infrequent updates.
Solution: Implement client-side location smoothing (interpolate between known points). Detect anomalous jumps (> 500m in 5 seconds) and discard them. Use Kalman filtering on the server side to smooth GPS data. Show a wider accuracy circle when confidence is low. Fall back to road-snapping using a routing API.
3 ETA estimates are consistently inaccurate during peak hours
Cause: The ETA model uses static average speeds that do not account for traffic, weather, or restaurant being overwhelmed during dinner rush. Restaurant prep times increase as order volume rises but the model does not adapt.
Solution: Use dynamic prep time that increases with the restaurant's active order count. Incorporate real-time traffic data from a mapping API. Track historical ETA accuracy and apply correction factors by time of day and day of week. Add a restaurant-specific queue depth multiplier. Always show a range (e.g., '25-35 min') rather than a precise number.
