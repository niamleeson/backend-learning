Flask-Migrate: Database Migrations

Intuition

Migrations are version control for your database schema. Just as git tracks code changes, Alembic tracks schema changes, letting you move forward to any version or roll back when something goes wrong.

Flask-Migrate and Database Migrations

Flask-Migrate wraps Alembic to provide database migration support for Flask-SQLAlchemy. Migrations track schema changes over time, letting you upgrade and downgrade your database safely. This is essential for evolving your schema in production without data loss.

# app.py
from flask import Flask
from flask_sqlalchemy import SQLAlchemy
from flask_migrate import Migrate

db = SQLAlchemy()
migrate = Migrate()

def create_app():
    app = Flask(__name__)
    app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///app.db'
    db.init_app(app)
    migrate.init_app(app, db)
    return app

# CLI commands after setup:
# flask db init          # Create migrations folder (once)
# flask db migrate -m "Initial migration"  # Auto-generate migration
# flask db upgrade       # Apply migration to database
# flask db downgrade     # Revert last migration

Key Takeaways: - Always review auto-generated migrations before applying; Alembic cannot detect all changes (e.g., renamed columns). - Each migration has upgrade() and downgrade() functions to move forward or rollback. - Run flask db migrate to detect changes and flask db upgrade to apply them.

Exercises

  1. Schema Evolution Workflow

    Start with a User model (id, username, email). Add a 'bio' field, generate a migration, and apply it. Then add an 'avatar_url' field and generate a second migration. Practice upgrading and downgrading between versions.

    Hint

    Use flask db migrate -m 'description' after each model change. Use flask db history to see all versions and flask db downgrade to go back.

  2. Data Migration

    Write a migration that splits an existing 'full_name' column into 'first_name' and 'last_name' columns, populating them from the existing data before dropping the original column.

    Hint

    Use op.execute() for raw SQL to move data between columns within the migration's upgrade() function. Always add new columns before dropping old ones.

Common Problems & Solutions

1 flask db migrate generates an empty migration with no changes detected.
Cause: The model changes are not imported or the model module is not loaded when Alembic runs.
Solution: Ensure all model modules are imported in your app factory or in the migrations/env.py file so Alembic can detect them.
2 Multiple developers create conflicting migrations with the same parent revision.
Cause: Two branches generate migrations from the same head revision, creating a fork in the migration chain.
Solution: Use flask db merge to create a merge migration that combines both branches, then apply the merged migration.
