Document Model and CRUD Operations

Intuition

MongoDB collections are like filing cabinets where each folder can have different contents. You query by describing what the document looks like rather than joining tables.

MongoDB: Document Model and CRUD

MongoDB stores data as flexible JSON-like documents (BSON) in collections. CRUD operations work on individual documents or in bulk.

from pymongo import MongoClient

client = MongoClient('mongodb://localhost:27017')
db = client['myapp']
collection = db['users']

# Create
result = collection.insert_one({
    'name': 'Alice', 'email': 'alice@example.com',
    'age': 30, 'tags': ['admin', 'active']
})
collection.insert_many([{'name': 'Bob'}, {'name': 'Carol'}])

# Read
user = collection.find_one({'email': 'alice@example.com'})
active = collection.find({'tags': 'active'}).sort('name', 1).limit(10)

# Update
collection.update_one(
    {'_id': result.inserted_id},
    {'$set': {'age': 31}, '$push': {'tags': 'verified'}}
)

# Delete
collection.delete_one({'name': 'Bob'})
collection.delete_many({'tags': 'inactive'})

Key Takeaways: - Documents are schema-flexible; fields can vary between documents in the same collection. - Use $set, $push, $inc, $unset operators for partial updates without rewriting the whole document. - find() returns a cursor; chain .sort(), .limit(), .skip() for pagination.

Exercises

  1. Build a REST-like CRUD Layer

    Write Python functions for create, find_by_id, update, and delete operations on a 'products' collection with proper error handling.

    Hint

    Use insert_one, find_one({'_id': ObjectId(id)}), update_one with $set, delete_one. Handle None results.

  2. Query with Operators

    Write queries using $gt, $in, $regex, and $exists to find products matching complex criteria.

    Hint

    {'price': {'$gt': 50}, 'category': {'$in': ['books', 'tech']}, 'description': {'$exists': True}}.

Common Problems & Solutions

1 find() returns a cursor object instead of a list.
Cause: find() lazily returns a cursor for memory efficiency.
Solution: Use list(collection.find(...)) to materialize results, or iterate the cursor directly.
2 Update replaces the entire document.
Cause: Passing a plain dict to update_one without $set replaces the document (except _id).
Solution: Always use update operators like $set, $push, $inc for partial updates.
