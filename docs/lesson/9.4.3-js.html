Design a Web Crawler (Google)

Intuition

A web crawler is like a team of librarians exploring an infinite library. Each librarian (worker) picks a room (URL) from a shared to-do list (frontier), reads the books (downloads content), writes down references to other rooms (extracts links), and adds new rooms to the to-do list. A coordinator ensures no two librarians visit the same room, and everyone waits politely before re-entering any building (domain) they recently visited.

Design a Web Crawler (Google)

Introduction

A web crawler (also called a spider or bot) systematically browses the internet, downloading web pages and extracting links to discover new pages. Google's crawler (Googlebot) discovers and indexes billions of pages. This system design problem tests your ability to handle massive scale, politeness constraints, deduplication, distributed coordination, and storage of crawled content.

Step 1: Requirements Gathering

Functional Requirements

  • Crawl the web starting from a set of seed URLs
  • Extract and follow hyperlinks from crawled pages
  • Store the content of crawled pages for indexing
  • Respect robots.txt rules and crawl delays
  • Support configurable crawl depth and domain filters
  • Detect and handle duplicate pages
  • Re-crawl pages periodically to keep content fresh

Non-Functional Requirements

  • Scale: Crawl 1 billion pages per day
  • Throughput: 10,000+ pages per second
  • Politeness: No more than 1 request per second per domain
  • Fault tolerance: Survive worker node failures without losing progress
  • Extensibility: Easy to add content extractors (images, PDFs, etc.)

Out of Scope

  • Full-text search indexing
  • Page ranking algorithms
  • JavaScript rendering (headless browser)

Step 2: Capacity Estimation

Assumptions:
- Target: 1 billion pages per day
- Average page size: 100 KB (HTML only)
- Average 50 links extracted per page
- Average fetch time: 500ms per page

Throughput:
- 1 billion / 86,400 seconds = ~11,570 pages/second
- Round up to 15,000 pages/second for peak headroom

Bandwidth:
- Download: 15,000 pages/s x 100 KB = 1.5 GB/s inbound
- Daily download: 1B x 100 KB = 100 TB/day

Storage:
- Raw HTML storage: 100 TB/day
- With compression (5x): ~20 TB/day
- 30-day retention: 600 TB compressed
- URL frontier (10 billion URLs): 10B x 100 bytes = 1 TB
- URL dedup set: 10B x 8 bytes (hash) = 80 GB

Compute:
- If each worker crawls 20 pages/s: 15,000 / 20 = 750 worker instances
- DNS resolution: cache needed to avoid DNS bottleneck
- robots.txt cache: ~100 million domains x 1 KB = 100 GB

Step 4: Core Components

URL Frontier

The URL frontier is a priority queue of URLs to be crawled. It must balance two concerns: priority (important pages first) and politeness (don't overwhelm any single domain).

import Redis from 'ioredis';

class URLFrontier {
  constructor(redis) {
    this.redis = redis;
  }

  // Add URL with priority score (lower = higher priority)
  async enqueue(url, priority = 5) {
    const domain = new URL(url).hostname;

    // Per-domain queue for politeness
    await this.redis.zadd(`frontier:domain:${domain}`, priority, url);

    // Global domain queue for round-robin across domains
    await this.redis.sadd('frontier:domains', domain);
  }

  // Get next URL to crawl, respecting politeness
  async dequeue() {
    // Round-robin across domains
    const domain = await this.redis.spop('frontier:domains');
    if (!domain) return null;

    // Check if domain is on cooldown
    const cooldownKey = `frontier:cooldown:${domain}`;
    const onCooldown = await this.redis.exists(cooldownKey);
    if (onCooldown) {
      // Put domain back and try another
      await this.redis.sadd('frontier:domains', domain);
      return null;
    }

    // Pop highest-priority URL from this domain
    const results = await this.redis.zpopmin(`frontier:domain:${domain}`, 1);
    if (results.length === 0) return null;

    const url = results[0];

    // Set cooldown for this domain (1 second politeness delay)
    await this.redis.set(cooldownKey, '1', 'PX', 1000);

    // If domain has more URLs, put it back in the global set
    const remaining = await this.redis.zcard(`frontier:domain:${domain}`);
    if (remaining > 0) {
      await this.redis.sadd('frontier:domains', domain);
    }

    return { url, domain };
  }

  async size() {
    const domains = await this.redis.smembers('frontier:domains');
    let total = 0;
    for (const domain of domains) {
      total += await this.redis.zcard(`frontier:domain:${domain}`);
    }
    return total;
  }
}

URL Deduplication with Bloom Filter

import { BloomFilter } from 'bloom-filters';
import crypto from 'node:crypto';

class URLDeduplicator {
  constructor() {
    // Bloom filter: 10 billion items, 0.01% false positive rate
    // For production, use a distributed Bloom filter (Redis BF.ADD)
    this.bloomFilter = BloomFilter.create(10_000_000_000, 0.0001);
    this.redis = new Redis();
  }

  normalizeURL(url) {
    const parsed = new URL(url);
    // Remove fragments, trailing slashes, normalize
    parsed.hash = '';
    let normalized = parsed.toString();
    if (normalized.endsWith('/')) {
      normalized = normalized.slice(0, -1);
    }
    return normalized.toLowerCase();
  }

  fingerprint(url) {
    return crypto.createHash('sha256').update(url).digest('hex').slice(0, 16);
  }

  async isDuplicate(url) {
    const normalized = this.normalizeURL(url);
    const fp = this.fingerprint(normalized);

    // Fast check with Bloom filter
    if (!this.bloomFilter.has(fp)) {
      return false; // Definitely not seen
    }

    // Bloom filter says maybe seen - verify with Redis
    const exists = await this.redis.sismember('crawled:urls', fp);
    return exists === 1;
  }

  async markCrawled(url) {
    const normalized = this.normalizeURL(url);
    const fp = this.fingerprint(normalized);
    this.bloomFilter.add(fp);
    await this.redis.sadd('crawled:urls', fp);
  }
}

Fetcher Worker

import * as cheerio from 'cheerio';

class FetcherWorker {
  constructor(frontier, dedup, contentStore, robotsCache) {
    this.frontier = frontier;
    this.dedup = dedup;
    this.contentStore = contentStore;
    this.robotsCache = robotsCache;
    this.userAgent = 'MyBot/1.0 (+https://mybot.example.com)';
  }

  async run() {
    while (true) {
      const item = await this.frontier.dequeue();
      if (!item) {
        await new Promise(r => setTimeout(r, 100));
        continue;
      }

      try {
        await this.crawl(item.url, item.domain);
      } catch (err) {
        console.error(`Failed to crawl ${item.url}:`, err.message);
      }
    }
  }

  async crawl(url, domain) {
    // Check robots.txt
    const allowed = await this.robotsCache.isAllowed(url, this.userAgent);
    if (!allowed) return;

    // Fetch the page
    const controller = new AbortController();
    const timeout = setTimeout(() => controller.abort(), 10000);

    const response = await fetch(url, {
      headers: {
        'User-Agent': this.userAgent,
        'Accept': 'text/html',
      },
      signal: controller.signal,
      redirect: 'follow',
    });
    clearTimeout(timeout);

    if (!response.ok) return;

    const contentType = response.headers.get('content-type') || '';
    if (!contentType.includes('text/html')) return;

    const html = await response.text();
    if (html.length > 5_000_000) return; // Skip pages > 5MB

    // Store content
    await this.contentStore.store(url, html, {
      statusCode: response.status,
      contentType,
      crawledAt: new Date().toISOString(),
    });

    // Mark URL as crawled
    await this.dedup.markCrawled(url);

    // Extract and enqueue new links
    const links = this.extractLinks(html, url);
    for (const link of links) {
      const isDup = await this.dedup.isDuplicate(link);
      if (!isDup) {
        const priority = this.computePriority(link);
        await this.frontier.enqueue(link, priority);
      }
    }
  }

  extractLinks(html, baseURL) {
    const $ = cheerio.load(html);
    const links = new Set();

    $('a[href]').each((_, el) => {
      try {
        const href = $(el).attr('href');
        const absolute = new URL(href, baseURL).toString();
        // Only follow HTTP/HTTPS links
        if (absolute.startsWith('http://') || absolute.startsWith('https://')) {
          links.add(absolute);
        }
      } catch {
        // Invalid URL, skip
      }
    });

    return Array.from(links);
  }

  computePriority(url) {
    // Higher priority for well-known domains
    const domain = new URL(url).hostname;
    if (domain.endsWith('.edu') || domain.endsWith('.gov')) return 1;
    if (domain.includes('wikipedia')) return 2;
    // Lower priority for deep paths
    const depth = url.split('/').length;
    return Math.min(depth, 10);
  }
}

Robots.txt Cache

class RobotsTxtCache {
  constructor(redis) {
    this.redis = redis;
    this.TTL = 86400; // Cache for 24 hours
  }

  async isAllowed(url, userAgent) {
    const parsed = new URL(url);
    const robotsUrl = `${parsed.protocol}//${parsed.host}/robots.txt`;
    const cacheKey = `robots:${parsed.host}`;

    let rules = await this.redis.get(cacheKey);
    if (!rules) {
      try {
        const response = await fetch(robotsUrl, { signal: AbortSignal.timeout(5000) });
        rules = response.ok ? await response.text() : '';
      } catch {
        rules = ''; // Allow crawling if robots.txt is inaccessible
      }
      await this.redis.set(cacheKey, rules, 'EX', this.TTL);
    }

    return this.parseAndCheck(rules, url, userAgent);
  }

  parseAndCheck(robotsTxt, url, userAgent) {
    // Simplified robots.txt parsing
    const path = new URL(url).pathname;
    const lines = robotsTxt.split('\n');
    let relevant = false;
    const disallowed = [];

    for (const line of lines) {
      const trimmed = line.trim();
      if (trimmed.toLowerCase().startsWith('user-agent:')) {
        const agent = trimmed.split(':')[1].trim();
        relevant = agent === '*' || userAgent.includes(agent);
      } else if (relevant && trimmed.toLowerCase().startsWith('disallow:')) {
        const pattern = trimmed.split(':')[1].trim();
        if (pattern && path.startsWith(pattern)) return false;
      }
    }
    return true;
  }
}

Content Deduplication

Many pages have identical content (mirrors, syndication). Use content fingerprinting:

import crypto from 'node:crypto';

function simhash(text) {
  // SimHash for near-duplicate detection
  const tokens = text.toLowerCase().split(/\s+/);
  const hashBits = 64;
  const vector = new Array(hashBits).fill(0);

  for (const token of tokens) {
    const hash = crypto.createHash('md5').update(token).digest();
    for (let i = 0; i < hashBits; i++) {
      const byteIndex = Math.floor(i / 8);
      const bitIndex = i % 8;
      const bit = (hash[byteIndex] >> bitIndex) & 1;
      vector[i] += bit ? 1 : -1;
    }
  }

  let fingerprint = BigInt(0);
  for (let i = 0; i < hashBits; i++) {
    if (vector[i] > 0) {
      fingerprint |= BigInt(1) << BigInt(i);
    }
  }
  return fingerprint;
}

// Two pages are near-duplicates if Hamming distance < 3
function hammingDistance(a, b) {
  let xor = a ^ b;
  let count = 0;
  while (xor > 0n) {
    count += Number(xor & 1n);
    xor >>= 1n;
  }
  return count;
}

Re-crawl Scheduling

Not all pages need to be re-crawled at the same frequency:

function computeRecrawlInterval(page) {
  // News sites: re-crawl every hour
  if (page.domain.includes('news') || page.changeFrequency === 'hourly') {
    return 3600 * 1000;
  }
  // Frequently changing pages: daily
  if (page.contentChangedLastCrawl) {
    return 24 * 3600 * 1000;
  }
  // Static pages: weekly
  return 7 * 24 * 3600 * 1000;
}

Trap Detection

Some websites generate infinite URLs (calendar pages, query parameter combinations). Detect and avoid URL traps:

  • Limit crawl depth per domain
  • Detect URL patterns that grow infinitely (e.g., /page/1, /page/2, ...)
  • Limit total pages per domain
  • Detect soft-404 pages (200 status but generic content)

Summary

A web crawler at scale requires a URL frontier with politeness enforcement, distributed fetcher workers, URL deduplication via Bloom filters, robots.txt compliance, and efficient content storage. The key challenges are scale (billions of pages), politeness (don't overwhelm target servers), deduplication (avoid re-crawling identical content), and freshness (re-crawl changing pages more frequently). Geographic distribution of crawler clusters reduces latency and bandwidth costs.

Common Problems & Solutions

1 Crawler gets stuck in URL traps (infinite URLs from the same domain)
Symptoms: The frontier grows unboundedly for certain domains. The crawler spends all its time on a few domains while ignoring others.
Cause: Calendar pages, search result pages, or dynamically generated paths create infinite URL variations (e.g., /calendar?date=2026-01-01, /calendar?date=2026-01-02, ...).
Solution: Implement max URL depth per domain, detect repeating URL patterns with regex, limit pages per domain, and use URL fingerprinting that ignores query parameter values to detect structural duplicates.
2 DNS resolution becomes a bottleneck at high crawl rates
Cause: Each page fetch requires a DNS lookup, and the default system resolver cannot handle 15,000 lookups per second. DNS timeouts cause cascading slowdowns.
Solution: Implement an in-process DNS cache with TTL. Use a dedicated DNS resolver (like dnsmasq) on each crawler node. Pre-resolve DNS for domains in the frontier. Batch DNS lookups.
3 Downloaded content consumes storage faster than expected
Cause: No deduplication of page content across URLs, and no filtering of low-value pages (error pages, login walls, pagination duplicates).
Solution: Use content hashing (SHA-256) to detect exact duplicates and SimHash for near-duplicates. Filter out pages below a minimum content threshold. Compress stored HTML with gzip/brotli.
