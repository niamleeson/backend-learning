Defining Functions and Return Values

Intuition

A function is a named recipe: you give it ingredients (arguments), it follows the steps (body), and hands back a result (return value). If no return, it hands back nothing (None).

Functions are reusable blocks of code defined with def. They take inputs, perform work, and return outputs. Functions are first-class objects in Python.

def calculate_bmi(weight_kg: float, height_m: float) -> float:
    """Calculate Body Mass Index."""
    if height_m <= 0:
        raise ValueError("Height must be positive")
    return weight_kg / (height_m ** 2)

# Functions can return multiple values (tuple)
def min_max(numbers: list) -> tuple:
    return min(numbers), max(numbers)

lo, hi = min_max([3, 1, 4, 1, 5, 9])

# Functions without explicit return give None
def greet(name):
    print(f"Hello, {name}")

result = greet("Alice")  # result is None
  • Every function returns something; without return, it returns None
  • Use type hints for clarity: def func(x: int) -> str:
  • Write docstrings for any non-trivial function
  • Keep functions short and focused on a single responsibility

Exercises

  1. Temperature Converter

    Write two functions: celsius_to_fahrenheit(c) and fahrenheit_to_celsius(f). Include type hints and docstrings.

    Hint

    F = C * 9/5 + 32 and C = (F - 32) * 5/9.

  2. Multiple Return Values

    Write a function stats(numbers) that returns the mean, median, and standard deviation of a list as a named tuple.

    Hint

    Use collections.namedtuple('Stats', ['mean', 'median', 'stdev']).

Common Problems & Solutions

1 Function returns None unexpectedly
Cause: Forgetting the `return` statement or having a conditional path that doesn't return.
Solution: Ensure all code paths return a value. Use a linter to catch missing returns.
2 Mutable default argument shares state between calls
Cause: Default arguments are evaluated once at function definition, not per call.
Solution: Use `None` as default and create the mutable inside: `if items is None: items = []`.
