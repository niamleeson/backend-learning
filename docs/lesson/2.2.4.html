Test Coverage and Testing Best Practices

Intuition

Coverage tells you what code your tests touch, but not whether the tests are any good. High coverage with weak assertions is a false sense of security. Focus on testing behavior and edge cases.

Test Coverage and Testing Best Practices

Code coverage measures how much of your code is executed during tests. Combined with best practices, it ensures both quantity and quality of tests.

# Install: pip install pytest-cov
# Run: pytest --cov=mypackage --cov-report=html tests/

# .coveragerc or pyproject.toml config
# [tool.coverage.run]
# source = ["src"]
# omit = ["*/tests/*", "*/migrations/*"]
#
# [tool.coverage.report]
# fail_under = 80
# show_missing = true

# Example: well-structured test following AAA pattern
def test_discount_calculation():
    # Arrange
    cart = ShoppingCart()
    cart.add_item("widget", price=100, quantity=2)

    # Act
    total = cart.calculate_total(discount=0.1)

    # Assert
    assert total == 180.0  # 200 - 10%

# Test naming convention
def test_calculate_total_with_discount_returns_reduced_price():
    pass  # descriptive name tells what is being tested

def test_calculate_total_with_empty_cart_returns_zero():
    pass  # another descriptive name

Key Takeaways: - Follow the AAA pattern: Arrange, Act, Assert - Aim for 80%+ coverage but focus on critical paths, not 100% - Use --cov-fail-under in CI to enforce minimum coverage - Name tests descriptively: test_<what>_<condition>_<expected> - Test behavior, not implementation details

Exercises

  1. Achieve 90% Coverage

    Write a module with at least 3 functions including error handling. Write tests to reach 90% branch coverage. Run pytest --cov --cov-branch and identify uncovered branches.

    Hint

    Use --cov-report=term-missing to see which lines are not covered. Focus on testing both the happy path and error branches.

  2. Refactor Tests to AAA

    Take a test file with messy tests (mixed setup, assertions, and logic) and refactor each test to follow the Arrange-Act-Assert pattern clearly.

    Hint

    Each test should have one clear section for setup, one function call, and assertions grouped at the end. Extract shared setup into fixtures.

Common Problems & Solutions

1 Coverage is 95% but bugs still slip through to production.
Cause: Tests have high line coverage but weak assertions that do not verify correctness.
Solution: Add mutation testing (e.g., `mutmut`) to verify tests actually catch changes. Ensure every test has meaningful assertions.
2 Coverage report includes third-party code and test files, inflating numbers.
Cause: Coverage is not configured to exclude non-source directories.
Solution: Set `source = ["src"]` and `omit = ["*/tests/*"]` in coverage configuration to measure only your source code.
