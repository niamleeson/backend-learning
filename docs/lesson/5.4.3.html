CORS Configuration and Security Headers

Intuition

CORS is like a guest list at a private event: only approved domains get through the browser's security check. Security headers are instructions to the browser about how to handle your content safely, like telling it to never display your page in a frame.

CORS Configuration and Security Headers

CORS (Cross-Origin Resource Sharing) controls which domains can access your API from browsers. Security headers provide additional protection against common web vulnerabilities.

from fastapi import FastAPI, Request
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

# CORS configuration
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://myapp.com", "https://staging.myapp.com"],
    allow_credentials=True,
    allow_methods=["GET", "POST", "PUT", "DELETE"],
    allow_headers=["Authorization", "Content-Type"],
    max_age=600,  # Cache preflight for 10 minutes
)

# Security headers middleware
@app.middleware("http")
async def add_security_headers(request: Request, call_next):
    response = await call_next(request)
    response.headers["X-Content-Type-Options"] = "nosniff"
    response.headers["X-Frame-Options"] = "DENY"
    response.headers["X-XSS-Protection"] = "1; mode=block"
    response.headers["Strict-Transport-Security"] = (
        "max-age=31536000; includeSubDomains"
    )
    response.headers["Content-Security-Policy"] = "default-src 'self'"
    return response

@app.get("/api/data")
async def get_data():
    return {"message": "Secured response"}

Key Takeaways: - Never use allow_origins=["*"] with allow_credentials=True in production - CORS is enforced by browsers, not by your server; it prevents unauthorized frontend access - Security headers like HSTS, CSP, and X-Frame-Options defend against common attacks - max_age caches preflight responses, reducing OPTIONS request overhead

Exercises

  1. Environment-Based CORS

    Create a CORS configuration that reads allowed origins from an environment variable. In development, allow localhost origins. In production, only allow your specific domain. Use Pydantic Settings to manage the configuration.

    Hint

    Use pydantic_settings.BaseSettings to load CORS_ORIGINS from an env variable as a comma-separated string. Split it into a list and pass it to allow_origins.

  2. Security Audit Endpoint

    Create a /security/headers-check endpoint that returns an analysis of the current security headers configuration, flagging any missing recommended headers as warnings.

    Hint

    Make a request to another endpoint using httpx.AsyncClient(app=app) inside the handler, inspect the response headers, and compare them against a list of recommended security headers.

Common Problems & Solutions

1 CORS preflight fails with 405 Method Not Allowed
Cause: The CORS middleware is not installed, or another middleware is intercepting the OPTIONS request before the CORS middleware can respond.
Solution: Ensure `CORSMiddleware` is added to the app before any other middleware that might intercept requests. Verify the middleware is imported from `fastapi.middleware.cors`.
2 Credentials not sent with cross-origin requests
Cause: The frontend is not setting `credentials: 'include'` in fetch options, or the backend does not have `allow_credentials=True`.
Solution: Set `allow_credentials=True` in CORS middleware and use specific origins (not `*`). On the frontend, add `credentials: 'include'` to fetch or `withCredentials: true` for axios.
