Project Layout and Packaging with pyproject.toml

Intuition

pyproject.toml is Python's answer to package.json — a single declarative file that tells build tools, linters, and installers everything about your project.

Modern Python projects use pyproject.toml as the single configuration file for build systems, metadata, and tool settings, replacing the legacy setup.py and setup.cfg.

# pyproject.toml
# [build-system]
# requires = ["hatchling"]
# build-backend = "hatchling.build"
#
# [project]
# name = "myapp"
# version = "0.1.0"
# description = "My application"
# requires-python = ">=3.10"
# dependencies = ["requests>=2.28", "click>=8.0"]
#
# [project.optional-dependencies]
# dev = ["pytest", "ruff", "mypy"]
#
# [project.scripts]
# myapp = "myapp.cli:main"

# Recommended project layout:
# myapp/
#   pyproject.toml
#   src/
#     myapp/
#       __init__.py
#       cli.py
#   tests/
#     test_cli.py
  • pyproject.toml is the PEP 621 standard for project metadata — prefer it over setup.py.
  • The src/ layout prevents accidental imports of the uninstalled package during testing.
  • Use [project.scripts] to define CLI entry points that get installed as executables.
  • Optional dependency groups (e.g., dev, test) keep production installs lean.

Exercises

  1. Create a pyproject.toml

    Write a pyproject.toml for a project named taskmanager with version 1.0.0, requiring Python >=3.10, depending on click and rich, with a dev dependency group containing pytest.

    Hint

    Follow the PEP 621 format with [build-system], [project], and [project.optional-dependencies] sections.

  2. Build and Install Locally

    Using the src layout, create a minimal package with one module and install it in editable mode with pip install -e ..

    Hint

    Create src/mypackage/__init__.py, add a pyproject.toml, then run pip install -e ..

Common Problems & Solutions

1 `pip install -e .` fails with 'no build backend specified'.
Cause: The `[build-system]` section is missing from `pyproject.toml`.
Solution: Add `[build-system]` with `requires` and `build-backend` (e.g., `hatchling`, `setuptools>=64`).
2 Tests import the installed package instead of the local source code.
Cause: Without the `src/` layout, Python may find the top-level package directory before the installed version.
Solution: Use the `src/` layout so that the package is only importable when properly installed.
