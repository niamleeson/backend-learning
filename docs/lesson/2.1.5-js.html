Window Functions: ROW_NUMBER, RANK, LAG, LEAD

Intuition

Imagine you're in a line of people ordered by height. GROUP BY would merge everyone into groups and only report the group average. A window function lets each person stay in line but peek at their neighbors — seeing who's taller, what their running average is, or what their rank is — without merging anyone together.

Window Functions: ROW_NUMBER, RANK, LAG, LEAD

Introduction

Window functions perform calculations across a set of rows that are related to the current row — without collapsing rows into groups like GROUP BY does. They are one of SQL's most powerful features, enabling ranking, running totals, moving averages, and comparisons with neighboring rows.

Window Functions vs GROUP BY

GROUP BY collapses rows:
| city     | count |          3 input rows -> 1 output row per group
|----------|-------|
| New York | 47    |

Window functions KEEP all rows:
| name  | city     | rank |    3 input rows -> 3 output rows
|-------|----------|------|    (each row gets a computed value)
| Alice | New York | 1    |
| Bob   | New York | 2    |
| Carol | New York | 3    |

Basic Syntax

function_name() OVER (
  PARTITION BY column     -- optional: divide rows into groups
  ORDER BY column         -- optional: define order within partition
  ROWS/RANGE frame        -- optional: define window frame
)

Ranking Functions

ROW_NUMBER()

Assigns a unique sequential number to each row:

SELECT
  name,
  department,
  salary,
  ROW_NUMBER() OVER (ORDER BY salary DESC) AS overall_rank,
  ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS dept_rank
FROM employees;

// Top 3 earners per department
async function getTopEarnersPerDept() {
  const result = await pool.query(`
    SELECT * FROM (
      SELECT
        name,
        department,
        salary,
        ROW_NUMBER() OVER (
          PARTITION BY department
          ORDER BY salary DESC
        ) AS rank
      FROM employees
    ) ranked
    WHERE rank <= 3
    ORDER BY department, rank
  `);
  return result.rows;
}

RANK() and DENSE_RANK()

SELECT
  name,
  score,
  ROW_NUMBER() OVER (ORDER BY score DESC) AS row_num,   -- 1, 2, 3, 4
  RANK()       OVER (ORDER BY score DESC) AS rank,       -- 1, 2, 2, 4 (skips)
  DENSE_RANK() OVER (ORDER BY score DESC) AS dense_rank  -- 1, 2, 2, 3 (no skip)
FROM students;

Ranking comparison (scores: 95, 90, 90, 85):

  name   | score | ROW_NUMBER | RANK | DENSE_RANK
  -------|-------|------------|------|-----------
  Alice  | 95    | 1          | 1    | 1
  Bob    | 90    | 2          | 2    | 2
  Carol  | 90    | 3          | 2    | 2
  Dan    | 85    | 4          | 4    | 3
                                 ^        ^
                              skips 3   no skip

NTILE()

Divides rows into N roughly equal buckets:

-- Divide customers into 4 quartiles by spending
SELECT
  name,
  total_spent,
  NTILE(4) OVER (ORDER BY total_spent DESC) AS quartile
FROM customer_spending;
-- quartile 1 = top 25%, quartile 4 = bottom 25%

Offset Functions: LAG and LEAD

Access values from previous or subsequent rows:

SELECT
  date,
  revenue,
  LAG(revenue, 1) OVER (ORDER BY date) AS prev_day_revenue,
  LEAD(revenue, 1) OVER (ORDER BY date) AS next_day_revenue,
  revenue - LAG(revenue, 1) OVER (ORDER BY date) AS daily_change
FROM daily_revenue;

async function getRevenueWithTrends() {
  const result = await pool.query(`
    SELECT
      TO_CHAR(date, 'YYYY-MM-DD') AS date,
      revenue::numeric(12,2),
      LAG(revenue, 1) OVER (ORDER BY date)::numeric(12,2) AS prev_day,
      CASE
        WHEN LAG(revenue, 1) OVER (ORDER BY date) > 0
        THEN ROUND(
          (revenue - LAG(revenue, 1) OVER (ORDER BY date))
          / LAG(revenue, 1) OVER (ORDER BY date) * 100, 2
        )
        ELSE NULL
      END AS pct_change
    FROM daily_revenue
    ORDER BY date DESC
    LIMIT 30
  `);
  return result.rows;
}

FIRST_VALUE and LAST_VALUE

SELECT
  name,
  department,
  salary,
  FIRST_VALUE(name) OVER (
    PARTITION BY department ORDER BY salary DESC
  ) AS highest_paid_in_dept,
  salary - FIRST_VALUE(salary) OVER (
    PARTITION BY department ORDER BY salary DESC
  ) AS diff_from_top
FROM employees;

Aggregate Window Functions

Use regular aggregates as window functions:

SELECT
  date,
  revenue,
  SUM(revenue) OVER (ORDER BY date) AS running_total,
  AVG(revenue) OVER (
    ORDER BY date
    ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
  ) AS moving_avg_7day,
  SUM(revenue) OVER () AS grand_total,
  ROUND(revenue / SUM(revenue) OVER () * 100, 2) AS pct_of_total
FROM daily_revenue;

Window Frames:

  ROWS BETWEEN 6 PRECEDING AND CURRENT ROW   (7-day moving window)

  Day 1: [*]
  Day 2: [*, *]
  Day 3: [*, *, *]
  ...
  Day 7: [*, *, *, *, *, *, *]     <- full window
  Day 8: [   *, *, *, *, *, *, *]  <- slides forward

Named Windows

Reuse window definitions:

SELECT
  date,
  revenue,
  SUM(revenue) OVER w AS running_total,
  AVG(revenue) OVER w AS running_avg,
  ROW_NUMBER() OVER w AS row_num
FROM daily_revenue
WINDOW w AS (ORDER BY date)
ORDER BY date;

Real-World Example: User Retention Analysis

async function getUserRetentionCohorts() {
  const result = await pool.query(`
    WITH user_activity AS (
      SELECT
        user_id,
        DATE_TRUNC('month', MIN(created_at)) AS cohort_month,
        DATE_TRUNC('month', activity_date) AS activity_month
      FROM user_sessions
      GROUP BY user_id, DATE_TRUNC('month', activity_date)
    )
    SELECT
      TO_CHAR(cohort_month, 'YYYY-MM') AS cohort,
      EXTRACT(MONTH FROM AGE(activity_month, cohort_month))::int AS months_later,
      COUNT(DISTINCT user_id)::int AS active_users,
      FIRST_VALUE(COUNT(DISTINCT user_id)) OVER (
        PARTITION BY cohort_month ORDER BY activity_month
      )::int AS cohort_size,
      ROUND(
        COUNT(DISTINCT user_id)::numeric
        / FIRST_VALUE(COUNT(DISTINCT user_id)) OVER (
            PARTITION BY cohort_month ORDER BY activity_month
          ) * 100, 1
      ) AS retention_pct
    FROM user_activity
    GROUP BY cohort_month, activity_month
    ORDER BY cohort_month, activity_month
  `);
  return result.rows;
}

Exercises

  1. Top-N Per Group Pattern

    Write a query that returns the top 5 best-selling products per category. Use ROW_NUMBER() partitioned by category, ordered by total_sold descending.

    Hint

    Wrap the window function in a subquery and filter WHERE rank <= 5 in the outer query. You can't filter on window functions in the same SELECT.

  2. 7-Day Moving Average

    Calculate a 7-day moving average of daily signups. Fill in days with zero signups using generate_series. Build this as a Node.js function that returns the last 30 days of data.

    Hint

    Use ROWS BETWEEN 6 PRECEDING AND CURRENT ROW in the window frame. Join with generate_series to fill date gaps.

  3. Month-over-Month Growth

    Write a query that shows monthly revenue and the percentage change from the previous month using LAG(). Format as: month, revenue, prev_revenue, growth_pct.

    Hint

    Use LAG(revenue) OVER (ORDER BY month) to get previous month's revenue, then calculate the percentage change.

Common Problems & Solutions

1 Cannot use window function in WHERE clause
Cause: Window functions are evaluated after WHERE, so they can't be used to filter rows directly.
Solution: Wrap the query in a subquery or CTE, then filter in the outer query: `SELECT * FROM (SELECT ..., ROW_NUMBER() OVER (...) AS rn FROM ...) sub WHERE rn <= 5`.
2 LAST_VALUE returns unexpected results
Cause: The default window frame is `RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`, which means LAST_VALUE only sees up to the current row.
Solution: Explicitly set the frame: `LAST_VALUE(col) OVER (ORDER BY col ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING)`.
3 Window function query is very slow on large tables
Cause: Window functions need to sort data. Without proper indexes, sorting millions of rows is expensive, especially with PARTITION BY.
Solution: Create indexes that match your PARTITION BY + ORDER BY columns. For `PARTITION BY dept ORDER BY salary DESC`, create: `CREATE INDEX idx_emp_dept_salary ON employees (department, salary DESC)`.
