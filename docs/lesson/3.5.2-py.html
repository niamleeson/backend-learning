Sessions: add, commit, rollback, flush

Intuition

The Session is like a shopping cart. You add items (add), the store checks availability (flush), and you pay at checkout (commit). If something goes wrong, you empty the cart (rollback).

Sessions: add, commit, rollback, flush

The Session is the ORM's workspace. It tracks changes to objects and synchronizes them with the database through flush and commit.

from sqlalchemy.orm import Session

with Session(engine) as session:
    user = User(name='Alice', email='alice@example.com')
    session.add(user)
    session.flush()           # sends INSERT, but no commit
    print(user.id)            # id is now available
    session.commit()          # persists the transaction

# Rollback on error
with Session(engine) as session:
    try:
        session.add(User(name='Bob', email='bob@example.com'))
        session.add(User(name='Dup', email='bob@example.com'))
        session.commit()
    except Exception:
        session.rollback()
        raise

# Bulk operations
with Session(engine) as session:
    session.add_all([
        User(name='C', email='c@x.com'),
        User(name='D', email='d@x.com')
    ])
    session.commit()

# Delete
with Session(engine) as session:
    user = session.get(User, 1)
    session.delete(user)
    session.commit()

Key Takeaways: - add() stages an object; flush() sends SQL without committing; commit() finalizes the transaction. - After rollback(), the session is usable again but previously pending objects are expunged. - Use session.get(Model, pk) for primary key lookups; it checks the identity map first.

Exercises

  1. CRUD Operations

    Write functions for create, read, update, and delete operations on a User model using Session.

    Hint

    create: add + commit. read: session.get or session.execute(select(User)). update: modify attribute + commit.

  2. Batch Insert with Error Handling

    Insert 100 users in a single session, handling potential unique constraint violations gracefully.

    Hint

    Wrap in try/except, rollback on IntegrityError, or use savepoints with session.begin_nested().

Common Problems & Solutions

1 DetachedInstanceError when accessing attributes outside session.
Cause: The object was expunged or the session was closed, and lazy-loaded attributes cannot be fetched.
Solution: Use eager loading (joinedload/selectinload) before closing session, or use expire_on_commit=False.
2 Session state is corrupted after an exception.
Cause: Continuing to use a session after a failed flush without calling rollback().
Solution: Always call session.rollback() after an exception before reusing the session.
