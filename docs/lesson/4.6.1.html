Flask-SQLAlchemy: Setup and Models

Intuition

Flask-SQLAlchemy models are the Python representation of your database schema. Each class is a table, each attribute is a column, and relationships are the ORM's way of expressing JOINs as simple object attribute access.

Flask-SQLAlchemy Setup and Models

Flask-SQLAlchemy integrates SQLAlchemy ORM with Flask, providing a simplified API for defining models, querying data, and managing database connections. Models are Python classes that map to database tables, and relationships define how tables connect.

from flask import Flask
from flask_sqlalchemy import SQLAlchemy
from datetime import datetime, timezone

db = SQLAlchemy()

def create_app():
    app = Flask(__name__)
    app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///blog.db'
    app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = False
    db.init_app(app)
    return app

class User(db.Model):
    __tablename__ = 'users'
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(80), unique=True, nullable=False)
    email = db.Column(db.String(120), unique=True, nullable=False)
    created_at = db.Column(db.DateTime, default=lambda: datetime.now(timezone.utc))
    posts = db.relationship('Post', backref='author', lazy='select',
                            cascade='all, delete-orphan')

    def __repr__(self):
        return f'<User {self.username}>'

class Post(db.Model):
    __tablename__ = 'posts'
    id = db.Column(db.Integer, primary_key=True)
    title = db.Column(db.String(200), nullable=False)
    body = db.Column(db.Text, nullable=False)
    published = db.Column(db.Boolean, default=False)
    user_id = db.Column(db.Integer, db.ForeignKey('users.id'), nullable=False)
    tags = db.relationship('Tag', secondary='post_tags', backref='posts')

post_tags = db.Table('post_tags',
    db.Column('post_id', db.Integer, db.ForeignKey('post.id')),
    db.Column('tag_id', db.Integer, db.ForeignKey('tag.id')))

class Tag(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(50), unique=True)

Key Takeaways: - Use db.init_app(app) with the factory pattern to avoid circular imports. - Relationships define ORM-level links: db.relationship() on the parent, db.ForeignKey() on the child. - Many-to-many relationships use an association table defined with db.Table().

Exercises

  1. Define a Blog Data Model

    Create User, Post, Comment, and Tag models with proper relationships: User has many Posts, Post has many Comments, and Posts and Tags have a many-to-many relationship. Include created_at timestamps.

    Hint

    Use db.relationship() with backref for one-to-many and a secondary association table for many-to-many. Add db.ForeignKey on the 'many' side.

  2. CRUD Operations

    Write Flask routes that create a user, add posts to that user, query posts with filters (published=True), and delete a user (cascading to their posts).

    Hint

    Use db.session.add(), db.session.commit(), and Post.query.filter_by(published=True).all() for queries. Set cascade='all, delete-orphan' on the relationship.

Common Problems & Solutions

1 OperationalError: no such table when querying a model.
Cause: The database tables have not been created. db.create_all() was never called or was called outside an app context.
Solution: Run db.create_all() inside an app context: with app.app_context(): db.create_all(). For production, use Flask-Migrate instead of create_all().
2 DetachedInstanceError when accessing a relationship attribute outside a request.
Cause: The SQLAlchemy session is closed and the lazy-loaded relationship cannot be fetched.
Solution: Use eager loading with lazy='joined' or lazy='subquery' on the relationship, or access all needed data within the request context before the session closes.
