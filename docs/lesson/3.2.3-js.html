Third-Party Middleware: cors, helmet, morgan, compression

Intuition

These middleware packages are like the security and facilities team for an office building. Helmet is the security guard setting access rules at every door. CORS is the receptionist deciding which visitors from other companies can enter. Morgan is the front desk logging every visitor. Compression is the mail room that shrinks packages before sending them out.

Third-Party Middleware: cors, helmet, morgan, compression

Introduction

While Express provides minimal built-in middleware, the ecosystem has mature, battle-tested packages for common needs. Four of the most essential third-party middleware packages are: cors (Cross-Origin Resource Sharing), helmet (security headers), morgan (HTTP request logging), and compression (response compression). Nearly every production Express API uses some combination of these.

cors — Cross-Origin Resource Sharing

By default, browsers block requests from one origin (domain) to another. CORS middleware adds the necessary headers to allow cross-origin requests.

npm install cors
npm install -D @types/cors

Basic Usage

import cors from 'cors';

// Allow all origins (development only!)
app.use(cors());

// Allow specific origin
app.use(cors({
  origin: 'https://myapp.com'
}));

Advanced Configuration

app.use(cors({
  // Multiple allowed origins
  origin: ['https://myapp.com', 'https://admin.myapp.com'],

  // Dynamic origin check
  // origin: (origin, callback) => {
  //   const allowedOrigins = ['https://myapp.com'];
  //   if (!origin || allowedOrigins.includes(origin)) {
  //     callback(null, true);
  //   } else {
  //     callback(new Error('Not allowed by CORS'));
  //   }
  // },

  // Allowed HTTP methods
  methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE'],

  // Allowed headers
  allowedHeaders: ['Content-Type', 'Authorization'],

  // Expose custom headers to the client
  exposedHeaders: ['X-Total-Count', 'X-Request-Id'],

  // Allow credentials (cookies, authorization headers)
  credentials: true,

  // Cache preflight request results (seconds)
  maxAge: 86400, // 24 hours

  // Pass preflight to the next handler
  preflightContinue: false,

  // Success status for legacy browsers
  optionsSuccessStatus: 204,
}));

Per-Route CORS

// Different CORS for different routes
app.use('/api', cors({ origin: 'https://myapp.com', credentials: true }));
app.use('/public', cors()); // Allow all origins for public endpoints

helmet — Security Headers

Helmet sets various HTTP security headers to protect against common web vulnerabilities:

npm install helmet

import helmet from 'helmet';

// Apply all default security headers
app.use(helmet());

This single line sets approximately 15 security headers. Key ones include:

X-Content-Type-Options: nosniff       - Prevents MIME sniffing
X-Frame-Options: SAMEORIGIN           - Prevents clickjacking
X-XSS-Protection: 0                   - Disables buggy browser XSS filter
Strict-Transport-Security: ...        - Forces HTTPS
Content-Security-Policy: ...          - Controls resource loading
X-DNS-Prefetch-Control: off           - Controls DNS prefetching

Custom Configuration

app.use(helmet({
  // Customize Content Security Policy
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      scriptSrc: ["'self'", "'unsafe-inline'", 'cdn.example.com'],
      styleSrc: ["'self'", "'unsafe-inline'"],
      imgSrc: ["'self'", 'data:', '*.amazonaws.com'],
    },
  },

  // Configure HSTS
  hsts: {
    maxAge: 31536000,  // 1 year
    includeSubDomains: true,
    preload: true,
  },

  // Disable specific middleware
  crossOriginEmbedderPolicy: false,
}));

// For API-only servers, you can simplify CSP
app.use(helmet({
  contentSecurityPolicy: false, // Not needed for pure APIs
}));

morgan — HTTP Request Logging

Morgan logs incoming HTTP requests, invaluable for debugging and monitoring:

npm install morgan
npm install -D @types/morgan

Predefined Formats

import morgan from 'morgan';

// Development: colored, concise output
app.use(morgan('dev'));
// GET /api/users 200 12.345 ms - 256

// Combined: Apache-style logs (good for production)
app.use(morgan('combined'));
// ::1 - - [01/Jan/2025:12:00:00 +0000] "GET /api/users HTTP/1.1" 200 256 "-" "curl/7.88"

// Common: similar to combined without referrer/user-agent
app.use(morgan('common'));

// Short: shorter than default, includes response time
app.use(morgan('short'));

// Tiny: minimal output
app.use(morgan('tiny'));
// GET /api/users 200 256 - 12.345 ms

Custom Format

// Custom token format
app.use(morgan(':method :url :status :res[content-length] - :response-time ms'));

// Custom tokens
morgan.token('request-id', (req: Request) => req.requestId || 'N/A');
morgan.token('user-id', (req: Request) => req.user?.id?.toString() || 'anonymous');

app.use(morgan(':request-id :user-id :method :url :status :response-time ms'));

Conditional Logging

// Skip logging for health checks
app.use(morgan('dev', {
  skip: (req, res) => req.path === '/health',
}));

// Only log errors (status >= 400)
app.use(morgan('combined', {
  skip: (req, res) => res.statusCode < 400,
}));

// Use different formats per environment
if (process.env.NODE_ENV === 'production') {
  app.use(morgan('combined'));
} else {
  app.use(morgan('dev'));
}

Streaming Logs to a File

import fs from 'fs';
import path from 'path';

const accessLogStream = fs.createWriteStream(
  path.join(__dirname, '../logs/access.log'),
  { flags: 'a' } // append mode
);

app.use(morgan('combined', { stream: accessLogStream }));

compression — Response Compression

Compresses response bodies to reduce bandwidth usage:

npm install compression
npm install -D @types/compression

import compression from 'compression';

// Compress all responses
app.use(compression());

Configuration

app.use(compression({
  // Compression level (0-9, -1 for default)
  level: 6,

  // Minimum size to compress (bytes)
  threshold: 1024, // Don't compress responses smaller than 1KB

  // Filter: which responses to compress
  filter: (req, res) => {
    // Don't compress if client requests no compression
    if (req.headers['x-no-compression']) {
      return false;
    }
    // Use default filter (compresses text-based content types)
    return compression.filter(req, res);
  },
}));

When NOT to Compress

// Skip compression for already-compressed formats
app.use(compression({
  filter: (req, res) => {
    const contentType = res.getHeader('Content-Type') as string || '';
    // Skip images, videos, and other binary formats
    if (/image|video|audio/.test(contentType)) {
      return false;
    }
    return compression.filter(req, res);
  }
}));

Putting It All Together

import express from 'express';
import cors from 'cors';
import helmet from 'helmet';
import morgan from 'morgan';
import compression from 'compression';

const app = express();

// Security headers first
app.use(helmet({
  contentSecurityPolicy: false, // API-only server
}));

// CORS
app.use(cors({
  origin: process.env.CORS_ORIGIN || 'http://localhost:5173',
  credentials: true,
}));

// Compression
app.use(compression());

// Logging
app.use(morgan(process.env.NODE_ENV === 'production' ? 'combined' : 'dev'));

// Body parsing
app.use(express.json({ limit: '1mb' }));
app.use(express.urlencoded({ extended: true }));

// Routes
app.use('/api', apiRouter);

export default app;

Common Pitfalls

CORS: Credentials with Wildcard Origin

// WRONG: browsers reject this combination
app.use(cors({ origin: '*', credentials: true }));

// CORRECT: specify exact origin when using credentials
app.use(cors({ origin: 'https://myapp.com', credentials: true }));

Helmet Breaking Frontend Resources

// Helmet's CSP may block inline scripts, external CDNs, etc.
// If your API also serves a frontend, configure CSP carefully
app.use(helmet({
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      scriptSrc: ["'self'", "'unsafe-inline'"], // Allow inline scripts
    }
  }
}));

Summary

These four middleware packages handle cross-cutting concerns every API needs: cors for cross-origin access control, helmet for security headers, morgan for request logging, and compression for reducing response sizes. Install them, configure them appropriately for your environment, and register them early in your middleware stack.

Exercises

  1. Production-Ready Middleware Stack

    Configure a complete middleware stack for a production Express API: helmet with API-appropriate settings, CORS allowing specific origins with credentials, morgan logging to both console and file, and compression with a 1KB threshold. Use environment variables for configuration.

    Hint

    Create a writable stream for morgan file logging. Use different morgan formats for dev vs production. Read CORS_ORIGIN from process.env.

  2. Dynamic CORS Configuration

    Implement a CORS setup that reads allowed origins from a database or config file. The origin callback should check the incoming origin against the allowed list and reject unknown origins with a proper error.

    Hint

    Use the origin option as a function: (origin, callback) => { ... }. The first argument to callback is an error (null for success), the second is a boolean.

  3. Custom Morgan Tokens

    Create custom morgan tokens for: request ID (from X-Request-Id header), response time in seconds (not ms), request body size, and authenticated user email. Create a custom format string using these tokens.

    Hint

    Use morgan.token('name', (req, res) => value) to define custom tokens. Reference them in the format string as :name.

Common Problems & Solutions

1 CORS errors in the browser: 'No Access-Control-Allow-Origin header'
Cause: The cors middleware is not configured, not matching the requesting origin, or the middleware is registered after the route.
Solution: Ensure cors() is registered before routes. Check that the origin option matches the frontend URL exactly (including protocol and port). Enable credentials if using cookies.
2 Helmet's Content-Security-Policy blocks API responses
Cause: Helmet sets a restrictive CSP by default that can interfere with API responses or frontend resources served by the same server.
Solution: For API-only servers, disable CSP: helmet({ contentSecurityPolicy: false }). For servers that also serve frontend, customize the directives to allow your resources.
3 Compression not working — responses are not compressed
Cause: The client does not send Accept-Encoding: gzip/deflate, the response is too small (below threshold), or the content type is not compressible.
Solution: Check that the client sends Accept-Encoding header (browsers do this automatically). Lower the threshold for testing. Verify the content type is text-based (JSON, HTML, etc.).
