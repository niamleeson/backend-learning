Request and Response Objects

Intuition

The request object is a read-only snapshot of everything the client sent, while the response object is your canvas for everything you send back. Together they form the core input/output contract of every Flask endpoint.

Request and Response Objects

Flask provides the request proxy to access incoming data and the Response class (or helpers like jsonify and make_response) to craft outgoing responses. Understanding these objects is essential for handling form data, JSON payloads, headers, and cookies.

from flask import Flask, request, jsonify, make_response

app = Flask(__name__)

@app.route('/echo', methods=['POST'])
def echo():
    # Access different parts of the request
    content_type = request.content_type
    user_agent = request.headers.get('User-Agent')
    query_param = request.args.get('format', 'json')

    if request.is_json:
        data = request.get_json()
    else:
        data = request.form.to_dict()

    # Build a custom response
    response = make_response(jsonify({
        'received': data,
        'content_type': content_type,
        'user_agent': user_agent
    }))
    response.headers['X-Custom-Header'] = 'flask-demo'
    response.set_cookie('visited', 'true', max_age=3600, httponly=True)
    return response, 200

@app.route('/upload', methods=['POST'])
def upload():
    file = request.files.get('document')
    if file and file.filename:
        file.save(f'/tmp/{file.filename}')
        return jsonify({'saved': file.filename}), 201
    return jsonify({'error': 'No file provided'}), 400

Key Takeaways: - request.args holds query parameters, request.form holds form data, and request.get_json() parses JSON bodies. - make_response() gives you a mutable Response object for setting headers and cookies. - Always validate incoming data before processing it.

Exercises

  1. Request Inspector Endpoint

    Build an endpoint that returns a JSON summary of the incoming request: method, all headers, query parameters, and body content (form or JSON).

    Hint

    Use dict(request.headers) to convert headers and request.data.decode() for the raw body.

  2. Cookie-Based Visit Counter

    Create a route that reads a 'visits' cookie, increments it, sets the updated cookie on the response, and returns the count as JSON.

    Hint

    Use request.cookies.get('visits', '0'), convert to int, increment, and set it with response.set_cookie().

Common Problems & Solutions

1 request.get_json() returns None even though the body contains JSON.
Cause: The client is not sending the Content-Type: application/json header, so Flask does not attempt to parse JSON.
Solution: Either set the correct Content-Type header on the client, or use request.get_json(force=True) to parse regardless of content type.
2 UnicodeDecodeError when reading request.data.
Cause: The request body contains binary data (like a file upload) that cannot be decoded as UTF-8.
Solution: Use request.files for file uploads instead of reading request.data directly. For binary data, access request.get_data(as_text=False).
