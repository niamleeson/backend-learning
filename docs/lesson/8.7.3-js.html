EventEmitter Patterns and Domain Events

Intuition

Domain events with EventEmitter are like a company-wide announcement system. When something important happens (new order placed), an announcement goes out over the PA system. Different departments (email, analytics, inventory) each hear the same announcement and take their own appropriate action, without the announcer needing to know who is listening or what they will do.

EventEmitter Patterns and Domain Events

Introduction

Before reaching for Kafka or RabbitMQ, Node.js provides a built-in pub/sub mechanism: the EventEmitter. For in-process event-driven architecture, domain events with EventEmitter provide loose coupling between modules without the infrastructure overhead of an external message broker. This pattern is the foundation of domain-driven design in Node.js.

Node.js EventEmitter Basics

import { EventEmitter } from 'node:events';

const emitter = new EventEmitter();

// Subscribe
emitter.on('user.created', (user) => {
  console.log('Send welcome email to:', user.email);
});

emitter.on('user.created', (user) => {
  console.log('Initialize user preferences for:', user.id);
});

// Publish
emitter.emit('user.created', { id: 1, email: 'alice@example.com' });

Typed Event Bus

// Type-safe event bus with TypeScript
class TypedEventBus {
  constructor() {
    this.emitter = new EventEmitter();
    this.emitter.setMaxListeners(50);
  }

  on(event, handler) {
    this.emitter.on(event, handler);
    return () => this.emitter.off(event, handler); // Return unsubscribe fn
  }

  once(event, handler) {
    this.emitter.once(event, handler);
  }

  emit(event, data) {
    this.emitter.emit(event, data);
  }

  async emitAsync(event, data) {
    const listeners = this.emitter.listeners(event);
    await Promise.all(
      listeners.map(listener => listener(data))
    );
  }

  removeAllListeners(event) {
    this.emitter.removeAllListeners(event);
  }
}

// Singleton event bus
export const eventBus = new TypedEventBus();

Domain Events Pattern

Domain events represent something meaningful that happened in the business domain:

// Define domain events
class DomainEvent {
  constructor(type, data) {
    this.type = type;
    this.data = data;
    this.timestamp = new Date();
    this.id = crypto.randomUUID();
  }
}

class OrderCreatedEvent extends DomainEvent {
  constructor(order) {
    super('order.created', {
      orderId: order.id,
      userId: order.userId,
      items: order.items,
      total: order.total,
    });
  }
}

class OrderPaidEvent extends DomainEvent {
  constructor(orderId, paymentId) {
    super('order.paid', { orderId, paymentId });
  }
}

class UserRegisteredEvent extends DomainEvent {
  constructor(user) {
    super('user.registered', {
      userId: user.id,
      email: user.email,
      name: user.name,
    });
  }
}

Event-Driven Service Layer

// Order service emits events after business operations
class OrderService {
  constructor(orderRepo, eventBus) {
    this.orderRepo = orderRepo;
    this.eventBus = eventBus;
  }

  async createOrder(userId, items) {
    const order = await this.orderRepo.create({ userId, items });

    // Emit domain event
    await this.eventBus.emitAsync('order.created', new OrderCreatedEvent(order));

    return order;
  }

  async payOrder(orderId, paymentDetails) {
    const order = await this.orderRepo.findById(orderId);
    const payment = await this.paymentGateway.charge(paymentDetails);

    await this.orderRepo.update(orderId, { status: 'PAID' });

    await this.eventBus.emitAsync('order.paid', new OrderPaidEvent(orderId, payment.id));

    return order;
  }
}

// Handlers are registered at module initialization
class EmailHandler {
  constructor(eventBus, emailService) {
    // Register event handlers
    eventBus.on('order.created', this.onOrderCreated.bind(this));
    eventBus.on('order.paid', this.onOrderPaid.bind(this));
    eventBus.on('user.registered', this.onUserRegistered.bind(this));
  }

  async onOrderCreated(event) {
    await this.emailService.send({
      to: event.data.userId,
      template: 'order-confirmation',
      data: { orderId: event.data.orderId, total: event.data.total },
    });
  }

  async onOrderPaid(event) {
    await this.emailService.send({
      to: event.data.userId,
      template: 'payment-receipt',
      data: { orderId: event.data.orderId },
    });
  }

  async onUserRegistered(event) {
    await this.emailService.send({
      to: event.data.email,
      template: 'welcome',
      data: { name: event.data.name },
    });
  }
}

class AnalyticsHandler {
  constructor(eventBus, analyticsService) {
    eventBus.on('order.created', this.trackOrderCreated.bind(this));
    eventBus.on('user.registered', this.trackUserRegistered.bind(this));
  }

  async trackOrderCreated(event) {
    await this.analyticsService.track('order_created', {
      orderId: event.data.orderId,
      total: event.data.total,
    });
  }

  async trackUserRegistered(event) {
    await this.analyticsService.track('user_registered', {
      userId: event.data.userId,
    });
  }
}

NestJS Event-Driven with CQRS Module

import { EventBus, EventsHandler, IEventHandler } from '@nestjs/cqrs';

// Event
class OrderCreatedEvent {
  constructor(orderId, userId, total) {
    this.orderId = orderId;
    this.userId = userId;
    this.total = total;
  }
}

// Handler 1: Send email
@EventsHandler(OrderCreatedEvent)
class OrderCreatedEmailHandler {
  constructor(private emailService: EmailService) {}

  async handle(event) {
    await this.emailService.sendOrderConfirmation(event.userId, event.orderId);
  }
}

// Handler 2: Update analytics
@EventsHandler(OrderCreatedEvent)
class OrderCreatedAnalyticsHandler {
  constructor(private analyticsService: AnalyticsService) {}

  async handle(event) {
    await this.analyticsService.track('order_created', {
      orderId: event.orderId,
      total: event.total,
    });
  }
}

// Service publishes event
@Injectable()
class OrderService {
  constructor(private eventBus: EventBus) {}

  async createOrder(data) {
    const order = await this.orderRepo.create(data);
    this.eventBus.publish(new OrderCreatedEvent(order.id, data.userId, order.total));
    return order;
  }
}

Bridging In-Process Events to External Systems

// Bridge: EventEmitter events → Kafka/Redis for cross-service communication
class EventBridge {
  constructor(eventBus, kafkaProducer, redis) {
    this.eventBus = eventBus;
    this.producer = kafkaProducer;
    this.redis = redis;

    // Bridge specific events to Kafka
    this.bridgeToKafka('order.created', 'orders');
    this.bridgeToKafka('order.paid', 'orders');
    this.bridgeToKafka('user.registered', 'user-events');

    // Bridge to Redis pub/sub for real-time notifications
    this.bridgeToRedis('order.status.changed');
  }

  bridgeToKafka(eventType, topic) {
    this.eventBus.on(eventType, async (event) => {
      await this.producer.send({
        topic,
        messages: [{
          key: event.data.orderId || event.data.userId,
          value: JSON.stringify(event),
          headers: { 'event-type': eventType },
        }],
      });
    });
  }

  bridgeToRedis(eventType) {
    this.eventBus.on(eventType, async (event) => {
      await this.redis.publish(eventType, JSON.stringify(event));
    });
  }
}

Error Handling in Event Handlers

class ResilientEventBus extends TypedEventBus {
  async emitAsync(event, data) {
    const listeners = this.emitter.listeners(event);

    const results = await Promise.allSettled(
      listeners.map(listener => listener(data))
    );

    // Log failures but don't fail the main operation
    const failures = results.filter(r => r.status === 'rejected');
    if (failures.length > 0) {
      console.error(`${failures.length} handler(s) failed for event '${event}':`,
        failures.map(f => f.reason.message)
      );
    }

    return {
      total: results.length,
      succeeded: results.filter(r => r.status === 'fulfilled').length,
      failed: failures.length,
    };
  }
}

Common Pitfalls

  1. Event handler errors crashing the emitter: Use Promise.allSettled in async handlers. Unhandled errors in EventEmitter crash the process.
  2. Memory leaks from unremoved listeners: Always remove listeners when modules are destroyed. Set max listeners appropriately.
  3. Circular event chains: Event A triggers handler that emits Event B, which triggers handler that emits Event A. Add cycle detection.
  4. Assuming delivery order: EventEmitter delivers synchronously in registration order, but async handlers may complete in any order.

Summary

EventEmitter provides lightweight in-process pub/sub for domain events within a Node.js application. Domain events decouple business logic from side effects (emails, analytics, notifications). For cross-service communication, bridge in-process events to Kafka or Redis. Use NestJS CQRS module for structured event handling in NestJS applications. Always handle errors in event handlers gracefully to prevent one failed handler from affecting others.

Common Problems & Solutions

1 Unhandled error in event handler crashes the process
Cause: An async event handler throws an error that is not caught. Node.js EventEmitter propagates errors.
Solution: Always wrap event handlers in try/catch. Use emitAsync with Promise.allSettled. Add a global 'error' event handler on the emitter.
2 Memory leak warning: MaxListenersExceededWarning
Cause: Too many listeners registered on a single event, often due to listeners not being removed when modules are destroyed.
Solution: Call emitter.removeListener() or use the unsubscribe function returned by on(). Set emitter.setMaxListeners() to an appropriate value.
