Integration Testing and Test Architecture

Intuition

Unit tests check individual bricks, integration tests check that the bricks hold together as a wall. You need both: unit tests for speed and precision, integration tests for confidence.

Integration Testing and Test Architecture

Integration tests verify that components work together correctly. A solid test architecture balances unit, integration, and end-to-end tests in a testing pyramid.

import pytest
import sqlite3

@pytest.fixture(scope='session')
def db():
    conn = sqlite3.connect(':memory:')
    conn.execute('CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT)')
    yield conn
    conn.close()

@pytest.fixture(autouse=True)
def clean_db(db):
    yield
    db.execute('DELETE FROM users')
    db.commit()

def test_user_creation_and_retrieval(db):
    """Integration test: insert + query across layers."""
    db.execute('INSERT INTO users (name) VALUES (?)', ('Alice',))
    db.commit()
    cursor = db.execute('SELECT name FROM users WHERE name = ?', ('Alice',))
    assert cursor.fetchone()[0] == 'Alice'

def test_user_uniqueness_constraint(db):
    db.execute('INSERT INTO users (id, name) VALUES (1, "Bob")')
    db.commit()
    with pytest.raises(sqlite3.IntegrityError):
        db.execute('INSERT INTO users (id, name) VALUES (1, "Dup")')

Key Takeaways: - Testing pyramid: many unit tests, fewer integration tests, fewest E2E tests - Use in-memory databases or containers for integration tests - autouse=True fixtures handle cleanup automatically - Separate test directories: tests/unit/, tests/integration/ - Use markers to run integration tests separately: pytest -m integration

Exercises

  1. Build a Test Pyramid

    For a simple user registration system (validate, hash password, store in DB), write 3 unit tests (one per function) and 1 integration test that runs the full flow end-to-end.

    Hint

    Unit tests mock dependencies. The integration test uses a real in-memory SQLite database and calls the top-level register_user() function.

  2. Organize Tests by Type

    Create a directory structure with tests/unit/ and tests/integration/. Add conftest.py files and pytest markers so pytest -m unit and pytest -m integration work.

    Hint

    Register markers in pytest.ini or pyproject.toml under [tool.pytest.ini_options]. Use pytestmark = pytest.mark.integration at the module level.

Common Problems & Solutions

1 Integration tests are slow and flaky, causing CI to fail intermittently.
Cause: Tests depend on external services or shared state that varies between runs.
Solution: Use in-memory databases, Docker containers (testcontainers), or mock servers. Ensure each test cleans up after itself.
2 Integration tests pass individually but fail when run together.
Cause: Tests share mutable state (e.g., database rows) that leaks between tests.
Solution: Use per-test cleanup fixtures with `autouse=True` or database transactions that roll back after each test.
