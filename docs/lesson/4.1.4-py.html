Blueprints and Application Structure

Intuition

Blueprints are like plug-in modules for your Flask app -- each one encapsulates a feature area so you can develop, test, and reuse them independently. They enforce separation of concerns at the architectural level.

Blueprints and Application Structure

Blueprints let you organize a Flask application into modular components. Each blueprint can have its own routes, templates, static files, and error handlers. This is essential for large projects where a single file becomes unmanageable.

# project/auth/routes.py
from flask import Blueprint, jsonify, request

auth_bp = Blueprint('auth', __name__, url_prefix='/auth')

@auth_bp.route('/login', methods=['POST'])
def login():
    data = request.get_json()
    return jsonify({'token': 'abc123', 'user': data['username']})

@auth_bp.route('/logout', methods=['POST'])
def logout():
    return jsonify({'message': 'Logged out'}), 200

# project/api/routes.py
from flask import Blueprint, jsonify

api_bp = Blueprint('api', __name__, url_prefix='/api/v1')

@api_bp.route('/items')
def list_items():
    return jsonify({'items': ['widget', 'gadget']})

# project/__init__.py
from flask import Flask

def create_app():
    app = Flask(__name__)
    from .auth.routes import auth_bp
    from .api.routes import api_bp
    app.register_blueprint(auth_bp)
    app.register_blueprint(api_bp)
    return app

Key Takeaways: - Blueprints define a collection of routes and handlers that are registered on the app later. - url_prefix on a blueprint namespaces all its routes under a common path. - A recommended structure groups each blueprint in its own package with routes, models, and templates subdirectories.

Exercises

  1. Multi-Blueprint Application

    Create a Flask app with three blueprints: 'auth' (login/register), 'blog' (list/create posts), and 'admin' (user management). Each should have its own url_prefix and at least two routes.

    Hint

    Create a separate Python file for each blueprint and register them all inside create_app().

  2. Blueprint Error Handlers

    Add blueprint-specific error handlers for 404 and 400 errors on one of your blueprints, returning JSON error messages. Verify they only trigger for that blueprint's routes.

    Hint

    Use @blueprint.app_errorhandler(404) for app-wide or @blueprint.errorhandler(404) for blueprint-scoped handling (note: blueprint-scoped only works for errors raised explicitly).

Common Problems & Solutions

1 Routes from a blueprint are not accessible; 404 on every request.
Cause: The blueprint was defined but never registered with the Flask application via app.register_blueprint().
Solution: Call app.register_blueprint(your_bp) inside the app factory. Check that url_prefix is set correctly and not duplicated.
2 url_for() returns incorrect URLs for blueprint routes.
Cause: The endpoint name must include the blueprint name as a prefix, e.g., 'auth.login' not just 'login'.
Solution: Use url_for('blueprint_name.function_name') when referencing blueprint routes from outside that blueprint.
