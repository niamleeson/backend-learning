tRPC Fundamentals: Routers, Procedures, and Context

Intuition

tRPC is like having a direct phone line between two people who speak the same language, versus REST which is like sending letters that need to be translated at both ends. Since both sides speak TypeScript, there is no need for a translation layer (schema, codegen) -- you just talk directly.

tRPC Fundamentals: Routers, Procedures, and Context

tRPC (TypeScript Remote Procedure Call) provides end-to-end type safety between your backend and frontend without code generation, schemas, or runtime validation overhead. If you control both the client and server in TypeScript, tRPC might be the most productive API layer available.

Why tRPC?

REST / GraphQL workflow:
1. Define API schema/types
2. Implement server handlers
3. Generate client types (or write manually)
4. Keep everything in sync (manual effort)

tRPC workflow:
1. Write server procedures with TypeScript types
2. Import the type on the client
3. Done. Types are always in sync.

+------------------+          Type inference          +------------------+
|   tRPC Server    | -------------------------------->|   tRPC Client    |
|  (defines types) |    No codegen, no runtime cost   | (auto-complete!) |
+------------------+                                  +------------------+

Setting Up tRPC

import { initTRPC, TRPCError } from '@trpc/server';
import { Context } from './context';

const t = initTRPC.context<Context>().create();

export const router = t.router;
export const publicProcedure = t.procedure;
export const middleware = t.middleware;

Context Setup

// server/context.ts
import { CreateExpressContextOptions } from '@trpc/server/adapters/express';
import { PrismaClient } from '@prisma/client';
import jwt from 'jsonwebtoken';

const prisma = new PrismaClient();

export interface Context {
  prisma: PrismaClient;
  userId: string | null;
  userRole: string | null;
}

export function createContext({ req }: CreateExpressContextOptions): Context {
  let userId: string | null = null;
  let userRole: string | null = null;

  const token = req.headers.authorization?.replace('Bearer ', '');
  if (token) {
    try {
      const decoded = jwt.verify(token, process.env.JWT_SECRET!) as any;
      userId = decoded.sub;
      userRole = decoded.role;
    } catch {
      // Invalid token
    }
  }

  return { prisma, userId, userRole };
}

Procedures

tRPC has three types of procedures:

  • query - Read operations (like GET)
  • mutation - Write operations (like POST/PUT/DELETE)
  • subscription - Real-time updates (like WebSocket)
// server/routers/user.ts
import { z } from 'zod';
import { router, publicProcedure } from '../trpc';
import { TRPCError } from '@trpc/server';

export const userRouter = router({
  // Query: fetch a user by ID
  getById: publicProcedure
    .input(z.object({ id: z.string().uuid() }))
    .query(async ({ input, ctx }) => {
      const user = await ctx.prisma.user.findUnique({
        where: { id: input.id },
      });
      if (!user) {
        throw new TRPCError({
          code: 'NOT_FOUND',
          message: `User with id '${input.id}' not found`,
        });
      }
      return user;
    }),

  // Query: list users with pagination
  list: publicProcedure
    .input(
      z.object({
        limit: z.number().min(1).max(100).default(20),
        cursor: z.string().uuid().optional(),
      })
    )
    .query(async ({ input, ctx }) => {
      const users = await ctx.prisma.user.findMany({
        take: input.limit + 1,
        ...(input.cursor && {
          cursor: { id: input.cursor },
          skip: 1,
        }),
        orderBy: { createdAt: 'desc' },
      });

      let nextCursor: string | undefined;
      if (users.length > input.limit) {
        const nextItem = users.pop();
        nextCursor = nextItem!.id;
      }

      return { users, nextCursor };
    }),

  // Mutation: create a user
  create: publicProcedure
    .input(
      z.object({
        email: z.string().email(),
        name: z.string().min(2).max(100),
        bio: z.string().max(500).optional(),
      })
    )
    .mutation(async ({ input, ctx }) => {
      return ctx.prisma.user.create({ data: input });
    }),

  // Mutation: update a user
  update: publicProcedure
    .input(
      z.object({
        id: z.string().uuid(),
        name: z.string().min(2).max(100).optional(),
        bio: z.string().max(500).optional(),
      })
    )
    .mutation(async ({ input, ctx }) => {
      const { id, ...data } = input;
      return ctx.prisma.user.update({ where: { id }, data });
    }),
});

Routers and Merging

// server/routers/post.ts
export const postRouter = router({
  getById: publicProcedure
    .input(z.object({ id: z.string().uuid() }))
    .query(async ({ input, ctx }) => {
      return ctx.prisma.post.findUnique({
        where: { id: input.id },
        include: { author: true, comments: true },
      });
    }),

  create: publicProcedure
    .input(
      z.object({
        title: z.string().min(5).max(200),
        body: z.string().min(50),
        tags: z.array(z.string()).max(10).optional(),
      })
    )
    .mutation(async ({ input, ctx }) => {
      if (!ctx.userId) {
        throw new TRPCError({ code: 'UNAUTHORIZED' });
      }
      return ctx.prisma.post.create({
        data: { ...input, authorId: ctx.userId },
      });
    }),

  delete: publicProcedure
    .input(z.object({ id: z.string().uuid() }))
    .mutation(async ({ input, ctx }) => {
      if (!ctx.userId) {
        throw new TRPCError({ code: 'UNAUTHORIZED' });
      }
      const post = await ctx.prisma.post.findUnique({ where: { id: input.id } });
      if (!post || post.authorId !== ctx.userId) {
        throw new TRPCError({ code: 'FORBIDDEN' });
      }
      await ctx.prisma.post.delete({ where: { id: input.id } });
      return { success: true };
    }),
});

import { userRouter } from './user';
import { postRouter } from './post';

export const appRouter = router({
  user: userRouter,
  post: postRouter,
});

// Export type for client usage
export type AppRouter = typeof appRouter;

Middleware

tRPC middleware runs before procedure execution, similar to Express middleware:

import { middleware, publicProcedure } from '../trpc';

// Auth middleware
const isAuthenticated = middleware(async ({ ctx, next }) => {
  if (!ctx.userId) {
    throw new TRPCError({ code: 'UNAUTHORIZED' });
  }
  return next({
    ctx: {
      ...ctx,
      userId: ctx.userId, // now guaranteed non-null
    },
  });
});

// Protected procedure (requires authentication)
export const protectedProcedure = publicProcedure.use(isAuthenticated);

// Admin middleware
const isAdmin = middleware(async ({ ctx, next }) => {
  if (ctx.userRole !== 'ADMIN') {
    throw new TRPCError({ code: 'FORBIDDEN' });
  }
  return next({ ctx });
});

export const adminProcedure = protectedProcedure.use(isAdmin);

// Logging middleware
const logMiddleware = middleware(async ({ path, type, next }) => {
  const start = Date.now();
  const result = await next();
  const duration = Date.now() - start;
  console.log(`${type} ${path} - ${duration}ms`);
  return result;
});

export const loggedProcedure = publicProcedure.use(logMiddleware);

Using Protected Procedures

export const postRouter = router({
  // Anyone can read
  getById: publicProcedure
    .input(z.object({ id: z.string() }))
    .query(/* ... */),

  // Must be logged in to create
  create: protectedProcedure
    .input(z.object({ title: z.string(), body: z.string() }))
    .mutation(async ({ input, ctx }) => {
      // ctx.userId is guaranteed non-null here!
      return ctx.prisma.post.create({
        data: { ...input, authorId: ctx.userId },
      });
    }),

  // Must be admin to delete any post
  adminDelete: adminProcedure
    .input(z.object({ id: z.string() }))
    .mutation(async ({ input, ctx }) => {
      await ctx.prisma.post.delete({ where: { id: input.id } });
      return { success: true };
    }),
});

Connecting to Express

// server/index.ts
import express from 'express';
import * as trpcExpress from '@trpc/server/adapters/express';
import { appRouter } from './routers';
import { createContext } from './context';

const app = express();

app.use(
  '/trpc',
  trpcExpress.createExpressMiddleware({
    router: appRouter,
    createContext,
    onError: ({ error, path }) => {
      console.error(`tRPC error on ${path}:`, error.message);
    },
  })
);

app.listen(4000, () => console.log('tRPC server on port 4000'));

Exercises

  1. Build a tRPC CRUD API

    Create a tRPC router with full CRUD operations for a 'todo' resource: list (with cursor pagination), getById, create, update, toggleComplete, and delete. Use Zod for input validation and Prisma for data access.

    Hint

    Define input schemas with Zod for each procedure. Use z.object() for structured inputs.

  2. Implement Auth Middleware Chain

    Create three tRPC middleware: logging (logs duration), authentication (checks JWT), and rate limiting (max 100 requests per minute per user). Chain them together to create a rateLimitedProtectedProcedure.

    Hint

    Middleware chains with publicProcedure.use(log).use(auth).use(rateLimit). Each returns next() with potentially modified context.

  3. Nested Routers with Shared Context

    Create nested routers: appRouter -> admin (adminProcedure base), appRouter -> public (publicProcedure base). Admin routes should include user management and analytics. Test that auth is enforced on admin routes.

    Hint

    Use adminProcedure as the base for all procedures in the admin router.

Common Problems & Solutions

1 TypeScript errors when importing AppRouter type on the client
Cause: The client cannot import server code directly; it needs only the type, not the runtime code.
Solution: Export only the type: 'export type AppRouter = typeof appRouter'. Use 'import type { AppRouter }' on the client to ensure no runtime code is imported.
2 Context types are 'any' in procedure handlers
Cause: The tRPC instance was not initialized with the context type.
Solution: Pass the context type when initializing: initTRPC.context().create(). This propagates types to all procedures.
3 Zod validation errors are not descriptive on the client
Cause: By default, tRPC wraps Zod errors in a BAD_REQUEST TRPCError without the field-level details.
Solution: The Zod errors are included in the error's 'cause' field. On the client, access error.data?.zodError for structured validation errors.
