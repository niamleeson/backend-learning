AWS Fundamentals: EC2, RDS, S3

Intuition

Think of AWS like renting office space. EC2 is renting a desk with a computer (you manage the software). RDS is hiring an IT team to manage your database server (they handle backups and updates). S3 is an infinitely expandable filing cabinet where you pay only for the drawers you use.

AWS Fundamentals: EC2, RDS, S3

AWS is the most widely used cloud platform. Three core services handle compute (EC2), databases (RDS), and storage (S3) -- the building blocks of most backend deployments.

EC2 - Virtual Servers

# Launch and connect to an EC2 instance
aws ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --instance-type t3.micro \
  --key-name my-key \
  --security-groups my-sg

ssh -i my-key.pem ubuntu@<public-ip>

# Using boto3 to manage EC2
import boto3

ec2 = boto3.client('ec2', region_name='us-east-1')

# List running instances
response = ec2.describe_instances(
    Filters=[{'Name': 'instance-state-name', 'Values': ['running']}]
)
for reservation in response['Reservations']:
    for instance in reservation['Instances']:
        print(f"{instance['InstanceId']}: {instance['PublicIpAddress']}")

RDS - Managed Databases

# Create a PostgreSQL RDS instance
aws rds create-db-instance \
  --db-instance-identifier myapp-db \
  --db-instance-class db.t3.micro \
  --engine postgres \
  --master-username admin \
  --master-user-password secretpass \
  --allocated-storage 20

S3 - Object Storage

import boto3

s3 = boto3.client('s3')

# Upload a file
s3.upload_file('report.pdf', 'my-bucket', 'reports/2024/report.pdf')

# Generate a pre-signed URL (temporary access)
url = s3.generate_presigned_url(
    'get_object',
    Params={'Bucket': 'my-bucket', 'Key': 'reports/2024/report.pdf'},
    ExpiresIn=3600  # 1 hour
)

Common Mistakes

  • Leaving EC2 security groups wide open (0.0.0.0/0 on all ports)
  • Making RDS publicly accessible -- always keep it in a private subnet
  • Storing AWS credentials in code instead of using IAM roles or environment variables

Key Takeaways

  • Use IAM roles for EC2 instances instead of hardcoded credentials
  • RDS handles backups, patching, and failover -- don't run databases on raw EC2
  • Use S3 pre-signed URLs for temporary, secure file access

Exercises

  1. Deploy a Python App to EC2

    Launch a t3.micro EC2 instance, SSH in, install Python and your application, set up a systemd service, and configure a security group to allow HTTP traffic on port 80.

    Hint

    Use user data scripts for automated setup. Open port 80 in the security group and set up nginx to proxy to your app.

  2. S3 File Upload API

    Build a FastAPI endpoint that accepts file uploads, stores them in S3 with a unique key, and returns a pre-signed download URL valid for 1 hour.

    Hint

    Use boto3's upload_fileobj for streaming uploads and generate_presigned_url for download links.

  3. RDS Connection from Python

    Create an RDS PostgreSQL instance, configure security groups to allow access from your EC2 instance, and write a Python script using SQLAlchemy to connect and run queries.

    Hint

    Put EC2 and RDS in the same VPC. Use the RDS endpoint as the database host in your connection string.

Common Problems & Solutions

1 Cannot connect to RDS from EC2
Symptoms: ['Connection timed out to RDS endpoint', 'psycopg2.OperationalError: connection timed out']
Cause: Security group on RDS doesn't allow inbound connections from the EC2 instance's security group.
Solution: Edit the RDS security group to allow inbound PostgreSQL (port 5432) from the EC2 instance's security group ID.
2 S3 access denied despite correct credentials
Symptoms: ['botocore.exceptions.ClientError: An error occurred (AccessDenied)']
Cause: The IAM policy doesn't grant the required S3 actions, or the bucket policy denies access.
Solution: Check the IAM policy attached to the user/role. Ensure it includes `s3:PutObject`, `s3:GetObject`, etc. for the specific bucket ARN.
