Path Parameters, Query Parameters, and Request Body

Intuition

Path parameters identify specific resources, query parameters filter or modify responses, and request bodies carry complex data for creation or updates. Think of them as the address, the options, and the package contents of an HTTP request.

Path Parameters, Query Parameters, and Request Body in FastAPI

FastAPI provides an elegant way to handle different types of HTTP input: path parameters, query parameters, and request bodies. Each serves a distinct purpose in API design.

from fastapi import FastAPI, Query, Path
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    price: float
    description: str | None = None

@app.get("/items/{item_id}")
async def read_item(
    item_id: int = Path(..., gt=0, description="The ID of the item"),
    q: str | None = Query(None, max_length=50),
    skip: int = Query(0, ge=0)
):
    return {"item_id": item_id, "q": q, "skip": skip}

@app.post("/items/")
async def create_item(item: Item):
    return {"item": item, "price_with_tax": item.price * 1.1}

Key Takeaways: - Path parameters are part of the URL path and are always required - Query parameters are appended after ? and can have defaults - Request bodies are declared using Pydantic models and sent as JSON - Path() and Query() provide validation and metadata for documentation

Exercises

  1. Build a Search Endpoint

    Create a GET endpoint /search/{category} that accepts a path parameter category, query parameters q (required string), page (default 1), and limit (default 10, max 100). Return all parameters in the response.

    Hint

    Use Query(...) with the ellipsis to make q required, and Query(10, le=100) to set a max value for limit.

  2. Multi-Body Endpoint

    Create a POST endpoint that accepts two separate Pydantic models in the request body: User (name, email) and Address (street, city, zip_code). Return the combined data.

    Hint

    When you declare multiple Pydantic model parameters, FastAPI expects a JSON body with keys matching the parameter names.

Common Problems & Solutions

1 422 Validation Error when sending a POST request with JSON body
Cause: The request body does not match the Pydantic model schema, or the Content-Type header is not set to application/json.
Solution: Ensure the JSON payload matches the model fields exactly. Check that your HTTP client sets `Content-Type: application/json` and verify field types match the model definition.
2 Path parameter not being captured correctly, returning 404
Cause: Route ordering matters in FastAPI. A fixed path like `/items/latest` might be shadowed by `/items/{item_id}` if declared after it.
Solution: Declare fixed path routes before parameterized routes. Place `/items/latest` above `/items/{item_id}` in your code.
