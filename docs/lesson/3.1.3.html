Aggregations: GROUP BY, HAVING, COUNT, SUM, AVG

Intuition

GROUP BY is like sorting items into buckets, then measuring each bucket. HAVING is picking which buckets to keep based on those measurements.

Aggregations: GROUP BY and HAVING

Aggregation functions collapse multiple rows into summary values. GROUP BY defines the grouping and HAVING filters the aggregated results.

-- Count orders per customer
SELECT customer_id, COUNT(*) AS order_count,
       SUM(total) AS total_spent,
       AVG(total) AS avg_order
FROM orders
GROUP BY customer_id
HAVING COUNT(*) > 5
ORDER BY total_spent DESC;

-- Multiple grouping columns
SELECT EXTRACT(YEAR FROM created_at) AS yr,
       EXTRACT(MONTH FROM created_at) AS mo,
       category, COUNT(*) AS cnt
FROM products
GROUP BY yr, mo, category;

Key Takeaways: - Common aggregates: COUNT, SUM, AVG, MIN, MAX. COUNT(*) counts all rows; COUNT(col) skips NULLs. - Every non-aggregated column in SELECT must appear in GROUP BY. - WHERE filters rows before aggregation; HAVING filters groups after aggregation.

Exercises

  1. Top Spending Categories

    Write a query that shows each product category with its total revenue, but only categories with revenue over $10,000, sorted descending.

    Hint

    GROUP BY category, SUM(price * quantity) AS revenue, HAVING revenue > 10000.

  2. Monthly Active Users

    Count distinct users who logged in each month in 2024. Return month and user_count.

    Hint

    Use COUNT(DISTINCT user_id) and GROUP BY EXTRACT(MONTH FROM login_date) with a WHERE on the year.

Common Problems & Solutions

1 Error: column must appear in the GROUP BY clause or be used in an aggregate function.
Cause: A column in SELECT is neither grouped nor aggregated.
Solution: Add the column to GROUP BY or wrap it in an aggregate function like MAX() or ANY_VALUE().
2 HAVING clause seems to have no effect.
Cause: Confusing HAVING with WHERE; using HAVING on non-aggregated columns.
Solution: Use WHERE for row-level filtering before grouping, HAVING only for conditions on aggregate results.
