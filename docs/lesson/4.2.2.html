WTForms: Form Handling and Validation

Intuition

WTForms turns form handling from scattered request parsing into a declarative, self-validating data structure. Each form field knows what data it expects and how to reject bad input, centralizing validation logic in one place.

WTForms Form Handling and Validation

Flask-WTF integrates WTForms with Flask, providing form classes with declarative field definitions and built-in validators. Forms handle rendering, data binding, and validation in a structured way that reduces boilerplate and prevents common mistakes.

from flask import Flask, render_template, redirect, url_for, flash
from flask_wtf import FlaskForm
from wtforms import StringField, PasswordField, EmailField, SubmitField
from wtforms.validators import DataRequired, Email, Length, EqualTo

app = Flask(__name__)
app.config['SECRET_KEY'] = 'dev-secret-key'

class RegistrationForm(FlaskForm):
    username = StringField('Username', validators=[
        DataRequired(), Length(min=3, max=25)
    ])
    email = EmailField('Email', validators=[DataRequired(), Email()])
    password = PasswordField('Password', validators=[
        DataRequired(), Length(min=8)
    ])
    confirm = PasswordField('Confirm Password', validators=[
        DataRequired(), EqualTo('password', message='Passwords must match')
    ])
    submit = SubmitField('Register')

@app.route('/register', methods=['GET', 'POST'])
def register():
    form = RegistrationForm()
    if form.validate_on_submit():
        # form.username.data, form.email.data, etc.
        flash('Registration successful!', 'success')
        return redirect(url_for('register'))
    return render_template('register.html', form=form)

Key Takeaways: - FlaskForm automatically handles CSRF protection when SECRET_KEY is set. - validate_on_submit() checks both that the request is POST and that all validators pass. - Validators are composable: stack DataRequired, Length, Email, and custom validators on any field.

Exercises

  1. Login Form with Validation

    Create a LoginForm with email and password fields, both required. Build a route that renders the form on GET and validates on POST, flashing appropriate success or error messages.

    Hint

    Use flash() to show messages and iterate over form.field.errors in the template to display validation errors.

  2. Custom Validator

    Write a custom WTForms validator that checks if a username contains only alphanumeric characters and underscores. Apply it to a registration form.

    Hint

    Define a function that takes form and field as arguments and raises ValidationError if the check fails, or use a class-based validator with init and call.

Common Problems & Solutions

1 validate_on_submit() always returns False even with valid data.
Cause: The CSRF token is missing from the HTML form, or SECRET_KEY is not configured.
Solution: Include {{ form.hidden_tag() }} inside your
element and ensure app.config['SECRET_KEY'] is set.
2 Form fields show no error messages after failed validation.
Cause: The template does not iterate over field.errors to display them.
Solution: Add {% for error in form.field_name.errors %}{{ error }}{% endfor %} below each field in your template.
