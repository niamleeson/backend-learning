Winston: Structured Logging, Transports, and Log Levels

Intuition

Winston is like a sophisticated mail sorting facility. Every log message is a letter that arrives at the facility with a priority stamp (log level). The facility has multiple outbound trucks (transports) -- one for the console display, one for daily file archives, one for the error department. Each truck only accepts letters above a certain priority. The formatting machine stamps every letter with a timestamp and return address (metadata) before it goes out, ensuring every destination gets consistently formatted mail.

Winston: Structured Logging, Transports, and Log Levels

Why Structured Logging Matters

When your Node.js application is running in production, console.log is not enough. You need logs that are searchable, filterable, and parseable by log aggregation tools like the ELK stack, Datadog, or Splunk. Structured logging means outputting log entries as JSON objects with consistent fields -- timestamp, level, message, and contextual metadata -- instead of arbitrary strings.

# Unstructured (useless at scale)
User logged in
Order placed for $49.99
Database connection failed

# Structured (searchable, filterable, alertable)
{"timestamp":"2025-03-15T10:30:00Z","level":"info","message":"User logged in","userId":"u_abc123","ip":"192.168.1.50","service":"auth"}
{"timestamp":"2025-03-15T10:30:01Z","level":"info","message":"Order placed","orderId":"ord_xyz","amount":49.99,"userId":"u_abc123","service":"orders"}
{"timestamp":"2025-03-15T10:30:02Z","level":"error","message":"Database connection failed","host":"db-primary","port":5432,"error":"ECONNREFUSED","service":"orders"}

Basic Setup

import winston from 'winston';

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
  defaultMeta: { service: 'my-api' },
  transports: [
    new winston.transports.Console(),
    new winston.transports.File({ filename: 'logs/error.log', level: 'error' }),
    new winston.transports.File({ filename: 'logs/combined.log' }),
  ],
});

// Usage
logger.info('Server started', { port: 3000 });
logger.error('Database connection failed', { host: 'localhost', error: 'ECONNREFUSED' });
logger.warn('Rate limit approaching', { userId: 'u_123', requests: 95, limit: 100 });
logger.debug('Query executed', { sql: 'SELECT * FROM users', duration: 42 });

Log Levels

Winston uses the npm logging levels by default (RFC 5424 syslog levels are also available):

┌──────────┬────────┬──────────────────────────────────────────┐
│  Level   │ Number │ When to Use                              │
├──────────┼────────┼──────────────────────────────────────────┤
│ error    │   0    │ Application errors requiring attention   │
│ warn     │   1    │ Potential issues, degraded behavior      │
│ info     │   2    │ Normal operations, audit events          │
│ http     │   3    │ HTTP request/response logging            │
│ verbose  │   4    │ More detail than info, less than debug   │
│ debug    │   5    │ Development debugging, detailed traces   │
│ silly    │   6    │ Extremely verbose, rarely used           │
└──────────┴────────┴──────────────────────────────────────────┘

Setting level: 'info' means you see error, warn, and info.
Messages at debug and below are silently dropped.

Formats

Winston formats are composable transforms that process log entries:

import winston from 'winston';
const { combine, timestamp, json, colorize, printf, errors, metadata } = winston.format;

// JSON format for production (machine-readable)
const productionFormat = combine(
  timestamp({ format: 'YYYY-MM-DDTHH:mm:ss.SSSZ' }),
  errors({ stack: true }),  // capture stack traces
  json()
);

// Pretty format for development (human-readable)
const developmentFormat = combine(
  colorize({ all: true }),
  timestamp({ format: 'HH:mm:ss' }),
  printf(({ timestamp, level, message, service, ...meta }) => {
    const metaStr = Object.keys(meta).length ? JSON.stringify(meta, null, 2) : '';
    return `${timestamp} [${level}] ${message} ${metaStr}`;
  })
);

// Choose format based on environment
const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: process.env.NODE_ENV === 'production'
    ? productionFormat
    : developmentFormat,
  defaultMeta: { service: 'my-api' },
  transports: [new winston.transports.Console()],
});

Transports

Transports determine where log entries are sent:

import winston from 'winston';
import 'winston-daily-rotate-file';  // npm install winston-daily-rotate-file

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
  transports: [
    // Console: always
    new winston.transports.Console({
      level: process.env.NODE_ENV === 'production' ? 'info' : 'debug',
    }),

    // Error file: only errors
    new winston.transports.File({
      filename: 'logs/error.log',
      level: 'error',
      maxsize: 10 * 1024 * 1024,  // 10MB
      maxFiles: 5,
    }),

    // Daily rotation: all logs
    new winston.transports.DailyRotateFile({
      filename: 'logs/app-%DATE%.log',
      datePattern: 'YYYY-MM-DD',
      maxSize: '20m',
      maxFiles: '14d',  // keep 14 days
      zippedArchive: true,
    }),
  ],
});

Child Loggers and Contextual Logging

Child loggers inherit the parent's configuration but add context:

// Create the root logger
const logger = winston.createLogger({
  level: 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
  defaultMeta: { service: 'my-api' },
  transports: [new winston.transports.Console()],
});

// Create a child logger for a specific module
const authLogger = logger.child({ module: 'auth' });
const orderLogger = logger.child({ module: 'orders' });

// Each log includes the module field automatically
authLogger.info('User logged in', { userId: 'u_123' });
// {"timestamp":"...","level":"info","message":"User logged in","service":"my-api","module":"auth","userId":"u_123"}

orderLogger.error('Payment failed', { orderId: 'o_456', reason: 'card_declined' });
// {"timestamp":"...","level":"error","message":"Payment failed","service":"my-api","module":"orders","orderId":"o_456","reason":"card_declined"}

Express Integration with Request Context

import express from 'express';
import crypto from 'node:crypto';
import winston from 'winston';

const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
  defaultMeta: { service: 'my-api' },
  transports: [new winston.transports.Console()],
});

const app = express();

// Request logging middleware
app.use((req, res, next) => {
  // Generate or extract request ID
  const requestId = req.headers['x-request-id'] || crypto.randomUUID();
  req.requestId = requestId;
  res.setHeader('X-Request-ID', requestId);

  // Create a request-scoped child logger
  req.log = logger.child({
    requestId,
    method: req.method,
    url: req.originalUrl,
    ip: req.ip,
  });

  // Log request start
  req.log.info('Request received');

  // Log response when finished
  const startTime = Date.now();
  res.on('finish', () => {
    const duration = Date.now() - startTime;
    const logLevel = res.statusCode >= 500 ? 'error'
      : res.statusCode >= 400 ? 'warn'
      : 'info';

    req.log[logLevel]('Request completed', {
      statusCode: res.statusCode,
      duration,
      contentLength: res.get('Content-Length'),
    });
  });

  next();
});

// Use req.log in route handlers
app.get('/api/users/:id', async (req, res) => {
  req.log.info('Fetching user', { targetUserId: req.params.id });

  try {
    const user = await findUser(req.params.id);
    req.log.info('User found', { targetUserId: user.id });
    res.json(user);
  } catch (err) {
    req.log.error('Failed to fetch user', {
      targetUserId: req.params.id,
      error: err.message,
      stack: err.stack,
    });
    res.status(500).json({ error: 'Internal server error' });
  }
});

Sensitive Data Redaction

Never log passwords, tokens, credit cards, or PII:

const sensitiveKeys = ['password', 'token', 'authorization', 'cookie', 'secret', 'ssn', 'creditCard'];

const redactFormat = winston.format((info) => {
  // Deep clone to avoid mutating the original
  const redacted = JSON.parse(JSON.stringify(info));

  function redactObject(obj) {
    for (const key of Object.keys(obj)) {
      if (sensitiveKeys.some(sk => key.toLowerCase().includes(sk))) {
        obj[key] = '[REDACTED]';
      } else if (typeof obj[key] === 'object' && obj[key] !== null) {
        redactObject(obj[key]);
      }
    }
  }

  redactObject(redacted);
  return redacted;
});

const logger = winston.createLogger({
  format: winston.format.combine(
    redactFormat(),
    winston.format.timestamp(),
    winston.format.json()
  ),
  transports: [new winston.transports.Console()],
});

logger.info('Login attempt', {
  email: 'user@example.com',
  password: 'secret123',
  token: 'eyJhbG...',
});
// Output: {"email":"user@example.com","password":"[REDACTED]","token":"[REDACTED]"}

Error Logging Best Practices

// DO: Log errors with full context
try {
  await processOrder(orderId);
} catch (err) {
  logger.error('Order processing failed', {
    orderId,
    userId: req.user.id,
    error: err.message,
    stack: err.stack,
    code: err.code,
  });
}

// DON'T: Log just the error object
// logger.error(err);                    // Missing context
// logger.error('Error: ' + err);        // Loses stack trace
// logger.error(err.toString());          // Loses structured data

// DO: Use the errors format for automatic stack capture
const logger = winston.createLogger({
  format: winston.format.combine(
    winston.format.errors({ stack: true }),
    winston.format.timestamp(),
    winston.format.json()
  ),
  transports: [new winston.transports.Console()],
});

// Now you can pass Error objects directly
logger.error('Database query failed', { error: new Error('Connection timeout') });

Testing with Winston

import winston from 'winston';

// Create a silent logger for tests
const testLogger = winston.createLogger({
  silent: true,  // suppresses all output
});

// Or capture logs for assertions
class TestTransport extends winston.Transport {
  constructor(opts) {
    super(opts);
    this.logs = [];
  }

  log(info, callback) {
    this.logs.push(info);
    callback();
  }

  clear() {
    this.logs = [];
  }
}

// In your test
const testTransport = new TestTransport();
const logger = winston.createLogger({
  transports: [testTransport],
});

logger.info('test message', { key: 'value' });
assert(testTransport.logs.length === 1);
assert(testTransport.logs[0].message === 'test message');

Production Configuration

A complete production-ready logger setup:

import winston from 'winston';
import 'winston-daily-rotate-file';

const isProduction = process.env.NODE_ENV === 'production';

const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || (isProduction ? 'info' : 'debug'),
  format: winston.format.combine(
    winston.format.timestamp({ format: 'YYYY-MM-DDTHH:mm:ss.SSSZ' }),
    winston.format.errors({ stack: true }),
    isProduction
      ? winston.format.json()
      : winston.format.combine(
          winston.format.colorize(),
          winston.format.printf(({ timestamp, level, message, ...meta }) => {
            const metaStr = Object.keys(meta).length
              ? '\n' + JSON.stringify(meta, null, 2)
              : '';
            return `${timestamp} ${level}: ${message}${metaStr}`;
          })
        )
  ),
  defaultMeta: {
    service: process.env.SERVICE_NAME || 'my-api',
    version: process.env.APP_VERSION || '1.0.0',
    environment: process.env.NODE_ENV || 'development',
  },
  transports: [
    new winston.transports.Console(),
    ...(isProduction ? [
      new winston.transports.DailyRotateFile({
        filename: 'logs/app-%DATE%.log',
        datePattern: 'YYYY-MM-DD',
        maxSize: '50m',
        maxFiles: '30d',
        zippedArchive: true,
      }),
      new winston.transports.DailyRotateFile({
        filename: 'logs/error-%DATE%.log',
        datePattern: 'YYYY-MM-DD',
        level: 'error',
        maxSize: '20m',
        maxFiles: '90d',
        zippedArchive: true,
      }),
    ] : []),
  ],
  // Handle uncaught exceptions and rejections
  exceptionHandlers: [
    new winston.transports.File({ filename: 'logs/exceptions.log' }),
  ],
  rejectionHandlers: [
    new winston.transports.File({ filename: 'logs/rejections.log' }),
  ],
});

export default logger;

Summary

Winston is the most flexible logging library in the Node.js ecosystem. Its transport system lets you route logs to console, files, rotation, and external services simultaneously. Use JSON format in production, pretty-print in development, child loggers for contextual data, and always redact sensitive information. Combined with request ID tracking, Winston gives you the observability you need to debug production issues efficiently.

Common Problems & Solutions

1 Winston logs are not appearing in the console
Cause: The log level is set higher than the level being logged. For example, level is 'error' but you are calling logger.info().
Solution: Check the logger's level setting. Winston only outputs messages at or above the configured level. Set level to 'debug' during development: `winston.createLogger({ level: 'debug' })`. You can also set level per-transport.
2 Stack traces are missing from error logs
Cause: Winston does not automatically serialize Error objects' stack traces. Passing an Error as metadata without the errors format loses the stack.
Solution: Add `winston.format.errors({ stack: true })` to your format chain, and make sure it comes BEFORE the json() or printf() format. Then pass errors as: `logger.error('Failed', { error: err })` or `logger.error(err)`.
3 Log files grow indefinitely and fill up disk space
Cause: Using the basic File transport without rotation or size limits.
Solution: Install and use winston-daily-rotate-file: `new DailyRotateFile({ maxSize: '20m', maxFiles: '14d', zippedArchive: true })`. This rotates logs daily, compresses old files, and automatically deletes files older than the retention period.
