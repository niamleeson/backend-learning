HTTP Methods, Status Codes, and Content Negotiation

Intuition

HTTP methods and status codes form a shared vocabulary between client and server. When used correctly, any developer can predict your API's behavior without reading documentation, because the conventions are universally understood.

HTTP Methods, Status Codes, and Content Negotiation

Using the correct HTTP methods and status codes makes your API predictable and self-documenting. Content negotiation lets clients request different representations (JSON, XML) of the same resource. Mastering these conventions is fundamental to building professional APIs.

from flask import Flask, jsonify, request, abort

app = Flask(__name__)
items = {1: {'id': 1, 'name': 'Widget', 'price': 9.99}}

@app.route('/api/items', methods=['POST'])
def create_item():
    if not request.is_json:
        return jsonify({'error': 'Content-Type must be application/json'}), 415
    data = request.get_json()
    new_id = max(items.keys()) + 1 if items else 1
    items[new_id] = {'id': new_id, **data}
    return jsonify(items[new_id]), 201

@app.route('/api/items/<int:item_id>', methods=['PUT'])
def replace_item(item_id):
    if item_id not in items:
        abort(404)
    items[item_id] = {'id': item_id, **request.get_json()}
    return jsonify(items[item_id]), 200

@app.route('/api/items/<int:item_id>', methods=['PATCH'])
def update_item(item_id):
    if item_id not in items:
        abort(404)
    items[item_id].update(request.get_json())
    return jsonify(items[item_id]), 200

@app.route('/api/items/<int:item_id>', methods=['DELETE'])
def delete_item(item_id):
    if item_id not in items:
        abort(404)
    del items[item_id]
    return '', 204

@app.route('/api/items/<int:item_id>')
def get_item(item_id):
    if item_id not in items:
        abort(404)
    accept = request.accept_mimetypes
    if accept.best == 'text/html':
        return f"<h1>{items[item_id]['name']}</h1>", 200
    return jsonify(items[item_id])

Key Takeaways: - PUT replaces an entire resource; PATCH updates specific fields. Use the right one. - Status codes: 200 OK, 201 Created, 204 No Content, 400 Bad Request, 404 Not Found, 415 Unsupported Media Type. - Check request.accept_mimetypes for content negotiation.

Exercises

  1. Full CRUD with Proper Status Codes

    Implement a complete CRUD API for a 'products' resource. Use correct status codes: 201 for creation, 204 for deletion, 404 for missing resources, and 409 for duplicate creation.

    Hint

    Check if a product with the same name already exists before creating, and return 409 Conflict if it does.

  2. Content Negotiation

    Build an endpoint that returns JSON by default but returns CSV when the client sends Accept: text/csv. Test with curl using the -H flag.

    Hint

    Use request.accept_mimetypes.best_match(['application/json', 'text/csv']) to determine the preferred format.

Common Problems & Solutions

1 PUT request clears fields that were not included in the request body.
Cause: PUT is meant to replace the entire resource. If the client omits fields, they are lost.
Solution: Use PATCH for partial updates and PUT only for full replacements. Document this distinction clearly in your API.
2 DELETE returns a response body but the client expected 204 No Content.
Cause: Returning jsonify() with a 204 status code still sends a body, which some clients may not handle.
Solution: Return an empty string with 204: return '', 204. Do not include a response body with 204 status.
