Background Tasks and Streaming Responses

Intuition

Background tasks let you say 'I will handle this later' so the user gets an instant response. Streaming responses are like reading a book aloud page by page instead of waiting to photocopy the entire book before handing it over.

Background Tasks and Streaming Responses

FastAPI supports background tasks that run after a response is sent, and streaming responses that send data incrementally. Both are essential for responsive, high-performance APIs.

from fastapi import FastAPI, BackgroundTasks
from fastapi.responses import StreamingResponse
import asyncio

app = FastAPI()

def send_email_notification(email: str, message: str):
    # Simulate slow email sending
    import time
    time.sleep(3)
    print(f"Email sent to {email}: {message}")

@app.post("/register")
async def register(
    email: str, background_tasks: BackgroundTasks
):
    background_tasks.add_task(send_email_notification, email, "Welcome!")
    return {"message": "Registration successful"}  # Returns immediately

async def event_generator():
    for i in range(10):
        yield f"data: Event {i}\n\n"
        await asyncio.sleep(1)

@app.get("/stream")
async def stream_events():
    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream"
    )

Key Takeaways: - BackgroundTasks runs functions after the response is returned to the client - Background tasks are ideal for email, logging, and non-critical side effects - StreamingResponse sends data chunk by chunk using an async generator - Server-Sent Events (SSE) use text/event-stream media type for real-time updates

Exercises

  1. File Processing Pipeline

    Create an upload endpoint that accepts a file, immediately returns a task ID, and processes the file (word count, line count) in a background task. Create a second endpoint to check the processing status and results by task ID.

    Hint

    Use a shared dictionary to store task statuses. The background task updates the status from 'processing' to 'complete' with results. The status endpoint reads from the same dictionary.

  2. Live Log Streaming

    Create an SSE endpoint that streams the contents of a log file in real-time. Use an async generator that watches for new lines appended to the file and yields them as SSE events.

    Hint

    Use aiofiles to read the file and track the file position. In a loop, seek to the last position, read new content, yield it if non-empty, then await asyncio.sleep(0.5) before checking again.

Common Problems & Solutions

1 Background task exceptions are silently swallowed
Cause: FastAPI does not propagate exceptions from background tasks to the client since the response has already been sent.
Solution: Add try/except blocks inside your background task functions and log errors explicitly. Consider using a task queue like Celery for critical tasks that need retry logic.
2 StreamingResponse sends all data at once instead of streaming
Cause: The generator yields all data immediately without any delays or I/O waits, and middleware or a reverse proxy may buffer the response.
Solution: Add appropriate `await asyncio.sleep()` in the generator for time-based streaming. Disable response buffering in your reverse proxy with headers like `X-Accel-Buffering: no` for nginx.
