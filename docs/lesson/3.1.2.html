JOINs: INNER, LEFT, RIGHT, FULL, CROSS

Intuition

Think of JOINs as overlapping Venn diagrams: INNER is the overlap, LEFT keeps everything on the left plus the overlap, and FULL keeps both circles entirely.

JOINs: INNER, LEFT, RIGHT, FULL, CROSS

JOINs combine rows from two or more tables based on a related column. Choosing the right join type determines which rows appear in the result.

-- INNER JOIN: only matching rows
SELECT o.id, c.name, o.total
FROM orders o
INNER JOIN customers c ON o.customer_id = c.id;

-- LEFT JOIN: all left rows, NULLs for no match
SELECT c.name, o.id AS order_id
FROM customers c
LEFT JOIN orders o ON c.id = o.customer_id;

-- FULL OUTER JOIN: all rows from both tables
SELECT a.name, b.name
FROM table_a a
FULL OUTER JOIN table_b b ON a.key = b.key;

-- CROSS JOIN: cartesian product
SELECT s.size, c.color
FROM sizes s CROSS JOIN colors c;

Key Takeaways: - INNER JOIN returns only matching rows. LEFT JOIN keeps all rows from the left table. - FULL OUTER JOIN keeps unmatched rows from both sides. CROSS JOIN produces every combination. - Always specify the join condition with ON to avoid accidental cartesian products.

Exercises

  1. Find Customers Without Orders

    Write a query using LEFT JOIN to find all customers who have never placed an order.

    Hint

    LEFT JOIN orders on customer_id, then filter WHERE orders.id IS NULL.

  2. Product-Category Matrix

    Use a CROSS JOIN to generate all possible combinations of product sizes and colors, then filter to only combinations that exist in inventory.

    Hint

    Start with CROSS JOIN then add a WHERE EXISTS subquery checking inventory.

Common Problems & Solutions

1 JOIN returns far more rows than expected.
Cause: The join key is not unique on one side, creating a many-to-many multiplication.
Solution: Verify cardinality of join keys. Use DISTINCT or aggregate if duplicates are expected, or add extra join conditions.
2 LEFT JOIN behaves like INNER JOIN.
Cause: A WHERE clause on the right table filters out NULLs, effectively converting it to an INNER JOIN.
Solution: Move the filter condition from WHERE into the ON clause of the LEFT JOIN.
