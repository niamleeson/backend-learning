Python Installation & Environment Setup

Intuition

Think of virtual environments as separate toolboxes for each project. Each toolbox has exactly the tools (packages) that project needs, so they never interfere with each other.

Setting up a proper Python environment is the foundation of productive development. A clean installation and well-configured environment prevent countless headaches down the road.

# Check Python version
import sys
print(sys.version)

# Create a virtual environment
# $ python3 -m venv myproject
# $ source myproject/bin/activate  # macOS/Linux
# $ myproject\Scripts\activate     # Windows

# Verify pip is available
import pip
print(pip.__version__)
  • Always use virtual environments to isolate project dependencies
  • Use python3 explicitly to avoid conflicts with Python 2
  • pip freeze > requirements.txt captures your dependency snapshot
  • Prefer pyenv for managing multiple Python versions on one machine

Exercises

  1. Create and Activate a Virtual Environment

    Create a new virtual environment called 'practice_env', activate it, install the 'requests' library, and verify it's installed with pip list.

    Hint

    Use python3 -m venv practice_env then source practice_env/bin/activate on macOS/Linux.

  2. Check Your Python Setup

    Write a script that prints your Python version, pip version, and the path to the Python executable being used.

    Hint

    Use sys.version, sys.executable, and pip.__version__.

Common Problems & Solutions

1 Running `python` opens Python 2 instead of Python 3
Cause: The system default `python` command is aliased to Python 2 on older systems.
Solution: Use `python3` explicitly, or install pyenv and set the global version to 3.x.
2 ModuleNotFoundError after installing a package
Cause: The package was installed in a different Python environment than the one running your script.
Solution: Activate the correct virtual environment before running your script, or verify with `which python`.
