AWS Lambda and Serverless Python

Intuition

Lambda is like a vending machine versus a restaurant. A restaurant (EC2) is always open, paying rent and staff even when empty. A vending machine (Lambda) sits idle at zero cost and only activates when someone inserts a coin. Great for sporadic use, but for a packed dinner rush, the restaurant is more efficient.

AWS Lambda and Serverless Python

AWS Lambda runs your code without managing servers. You pay only for compute time consumed -- no traffic, no cost.

Lambda Function Basics

import json

def lambda_handler(event, context):
    """Handler receives event data and runtime context."""
    name = event.get('queryStringParameters', {}).get('name', 'World')
    return {
        'statusCode': 200,
        'headers': {'Content-Type': 'application/json'},
        'body': json.dumps({'message': f'Hello, {name}!'})
    }

Deploying with AWS CLI

zip function.zip lambda_function.py

aws lambda create-function \
  --function-name hello-api \
  --runtime python3.12 \
  --handler lambda_function.lambda_handler \
  --zip-file fileb://function.zip \
  --role arn:aws:iam::123456789:role/lambda-role

Common Mistakes

  • Putting heavy initialization inside the handler instead of outside (cold start optimization)
  • Not setting memory/timeout limits -- default 3s timeout may be too short
  • Including unnecessary files, hitting the 50MB package limit

Key Takeaways

  • Initialize clients outside the handler to reuse across invocations
  • Use Lambda Layers for shared dependencies
  • Lambda is best for event-driven tasks; for always-on APIs, consider ECS

Common Problems & Solutions

1 Lambda function times out
Symptoms: ['Task timed out after 3.00 seconds']
Cause: Default timeout is 3 seconds. Operations like database queries or API calls may take longer.
Solution: Increase the timeout (max 15 minutes): `aws lambda update-function-configuration --timeout 30`. For long tasks, consider Step Functions or SQS.
2 Import error for third-party packages
Symptoms: ["Unable to import module 'lambda_function': No module named 'requests'"]
Cause: Dependencies are not included in the deployment package.
Solution: Install packages into the deployment zip: `pip install -r requirements.txt -t package/`. Or use Lambda Layers for shared dependencies.
3 Cold start latency is too high
Symptoms: ['First request takes 5-10 seconds', 'Subsequent requests are fast']
Cause: Lambda loads your code and initializes the runtime on the first invocation after idle time.
Solution: Use Provisioned Concurrency to keep instances warm. Minimize package size. Move initialization (imports, DB connections) outside the handler function.
