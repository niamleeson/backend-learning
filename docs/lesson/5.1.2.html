Pydantic Models and Data Validation

Intuition

Pydantic models act as strict contracts for your data, catching invalid input before it ever reaches your business logic. They turn runtime type errors into clear, structured validation messages automatically.

Pydantic Models and Data Validation

Pydantic is the backbone of FastAPI's data validation. It uses Python type hints to validate incoming data, serialize outgoing data, and generate JSON Schema for documentation.

from pydantic import BaseModel, Field, field_validator, EmailStr
from datetime import datetime

class UserCreate(BaseModel):
    username: str = Field(..., min_length=3, max_length=50)
    email: EmailStr
    age: int = Field(..., ge=13, le=120)
    tags: list[str] = Field(default_factory=list)

    @field_validator("username")
    @classmethod
    def username_alphanumeric(cls, v: str) -> str:
        if not v.isalnum():
            raise ValueError("Username must be alphanumeric")
        return v.lower()

class UserResponse(BaseModel):
    id: int
    username: str
    email: EmailStr
    created_at: datetime

    model_config = {"from_attributes": True}

Key Takeaways: - Field() adds validation constraints like min/max length and numeric bounds - Custom validators with @field_validator allow complex validation logic - model_config = {"from_attributes": True} enables ORM model compatibility - Pydantic v2 uses model_validate() instead of the older from_orm()

Exercises

  1. Nested Model Validation

    Create a Pydantic model Order containing a customer field (a Customer model with name and email) and an items field (a list of OrderItem models with product_name, quantity >= 1, and unit_price > 0). Add a computed property for total price.

    Hint

    Use @computed_field or a @property for the total. Nest models by using one model class as a type annotation in another.

  2. Custom Serialization

    Build a Product model that stores price as an integer (cents) internally but serializes to a float (dollars) in JSON output. Use a field serializer to customize the output.

    Hint

    Use @field_serializer('price') to divide by 100 during serialization, and a validator to multiply by 100 when receiving float input.

Common Problems & Solutions

1 ValidationError not showing which field failed in a nested model
Cause: Pydantic v2 error messages include location paths, but deeply nested models can produce long error chains that are hard to parse.
Solution: Use a custom exception handler in FastAPI to format Pydantic validation errors into user-friendly responses. Access `exc.errors()` to get structured error details with field paths.
2 ORM objects not converting to Pydantic models
Cause: The Pydantic model is missing `from_attributes = True` in the model config, so it cannot read attributes from ORM objects.
Solution: Add `model_config = {"from_attributes": True}` to your response model, and use `UserResponse.model_validate(orm_obj)` for explicit conversion.
