Inheritance and Method Resolution Order

Intuition

Inheritance is like a family tree of behavior. When you call a method, Python walks up the MRO chain until it finds a class that implements it, like asking your parent if you don't know the answer.

Inheritance lets a class derive behavior from a parent class. Python supports multiple inheritance and uses the Method Resolution Order (MRO) to determine which method to call.

class Animal:
    def __init__(self, name: str):
        self.name = name
    def speak(self) -> str:
        raise NotImplementedError

class Dog(Animal):
    def speak(self) -> str:
        return f"{self.name} says Woof!"

class Cat(Animal):
    def speak(self) -> str:
        return f"{self.name} says Meow!"

# Multiple inheritance and MRO
class Pet:
    def info(self):
        return "I'm a pet"

class PetDog(Dog, Pet):
    pass

print(PetDog.__mro__)
# (PetDog, Dog, Animal, Pet, object)

dog = PetDog("Rex")
print(dog.speak())  # from Dog
print(dog.info())   # from Pet
  • Use super() to call parent methods without hardcoding the parent class name
  • MRO follows C3 linearization: children before parents, left before right
  • Check MRO with ClassName.__mro__ or ClassName.mro()
  • Avoid deep inheritance hierarchies; prefer composition for complex relationships

Exercises

  1. Shape Hierarchy

    Create a Shape base class with an area() method. Derive Rectangle and Circle. Then create Square inheriting from Rectangle.

    Hint

    Square's __init__ can call super().__init__(side, side).

  2. Diamond Problem

    Create a diamond inheritance: A -> B, A -> C, B + C -> D. Add __init__ with super() in each. Print the MRO and verify initialization order.

    Hint

    Use super().__init__() in all classes and add print statements to track the call order.

Common Problems & Solutions

1 TypeError: Cannot create a consistent method resolution order
Cause: The inheritance hierarchy has conflicting orderings that C3 linearization cannot resolve.
Solution: Restructure the hierarchy so the base class order is consistent across all subclasses.
2 Parent __init__ not called in child
Cause: Forgetting to call `super().__init__()` in the child's `__init__`.
Solution: Always call `super().__init__(...)` in the child to properly initialize parent attributes.
