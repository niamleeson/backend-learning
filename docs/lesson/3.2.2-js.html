Built-in Middleware: express.json, express.static

Intuition

Built-in middleware is like the standard equipment in a car. express.json() is the fuel injector that converts raw fuel (request bytes) into usable energy (parsed objects). express.static() is the trunk — a dedicated space for carrying cargo (files) that passengers (API routes) do not need to handle.

Built-in Middleware: express.json, express.static

Introduction

Express ships with a small set of built-in middleware functions. While Express is famously "unopinionated," these built-in utilities cover the most universal needs: parsing request bodies and serving static files. Understanding what they do, how to configure them, and their security implications is important for every Express developer.

express.json()

Parses incoming requests with JSON payloads. It reads the request body stream, parses it as JSON, and populates req.body:

import express from 'express';

const app = express();

// Parse JSON bodies
app.use(express.json());

app.post('/api/users', (req, res) => {
  console.log(req.body); // { name: "Alice", email: "alice@example.com" }
  res.status(201).json({ user: req.body });
});

Configuration Options

app.use(express.json({
  // Maximum request body size (default: "100kb")
  limit: '10mb',

  // Only parse when Content-Type matches (default: "application/json")
  type: 'application/json',

  // Enable strict mode - only accepts arrays and objects (default: true)
  strict: true,

  // Custom reviver function for JSON.parse
  reviver: (key, value) => {
    // Convert date strings to Date objects
    if (typeof value === 'string' && /^\d{4}-\d{2}-\d{2}T/.test(value)) {
      return new Date(value);
    }
    return value;
  },
}));

Body Size Limits

Setting appropriate body size limits is a security measure against denial-of-service attacks:

// Different limits for different routes
app.use('/api/uploads', express.json({ limit: '50mb' }));
app.use('/api', express.json({ limit: '1mb' }));

// The first matching middleware applies

What Happens Without express.json()

// Without body parser
app.post('/api/data', (req, res) => {
  console.log(req.body); // undefined
  res.json({ received: req.body }); // { received: undefined }
});

// With body parser
app.use(express.json());
app.post('/api/data', (req, res) => {
  console.log(req.body); // { key: "value" }
  res.json({ received: req.body }); // { received: { key: "value" } }
});

express.urlencoded()

Parses URL-encoded bodies (HTML form submissions):

app.use(express.urlencoded({
  extended: true,  // Use qs library (supports nested objects)
  limit: '1mb',
}));

// Form submission: name=Alice&email=alice%40example.com
app.post('/submit', (req, res) => {
  console.log(req.body); // { name: "Alice", email: "alice@example.com" }
});

extended: true vs false

// extended: false - uses querystring library (flat key-value pairs)
// Form: colors[0]=red&colors[1]=blue
// Result: { 'colors[0]': 'red', 'colors[1]': 'blue' }

// extended: true - uses qs library (nested objects and arrays)
// Form: colors[0]=red&colors[1]=blue
// Result: { colors: ['red', 'blue'] }

// Form: user[name]=Alice&user[address][city]=NYC
// extended true: { user: { name: 'Alice', address: { city: 'NYC' } } }
// extended false: { 'user[name]': 'Alice', 'user[address][city]': 'NYC' }

express.raw() and express.text()

// Parse body as a Buffer
app.use('/webhooks', express.raw({ type: 'application/octet-stream' }));

app.post('/webhooks/stripe', (req, res) => {
  const rawBody = req.body; // Buffer
  // Useful for signature verification (Stripe, GitHub webhooks)
});

// Parse body as plain text
app.use('/text', express.text({ type: 'text/plain' }));

app.post('/text/process', (req, res) => {
  const text = req.body; // string
  res.json({ length: text.length });
});

express.static()

Serves static files (HTML, CSS, JS, images) from a directory:

import path from 'path';

// Serve files from the 'public' directory
app.use(express.static(path.join(__dirname, 'public')));

// Files in public/ are now accessible:
// public/index.html   -> GET /index.html (or GET /)
// public/css/style.css -> GET /css/style.css
// public/img/logo.png -> GET /img/logo.png

Configuration Options

app.use(express.static(path.join(__dirname, 'public'), {
  // Set max-age cache header (in milliseconds)
  maxAge: '1d',  // or 86400000

  // Send index.html for directory requests (default: 'index.html')
  index: 'index.html',

  // Enable or disable directory dot files
  dotfiles: 'ignore', // 'allow', 'deny', or 'ignore'

  // Custom headers
  setHeaders: (res, filePath) => {
    if (filePath.endsWith('.html')) {
      res.set('Cache-Control', 'no-cache');
    }
    if (filePath.endsWith('.js') || filePath.endsWith('.css')) {
      res.set('Cache-Control', 'public, max-age=31536000'); // 1 year
    }
  },

  // Enable ETag generation (default: true)
  etag: true,

  // Fall through to next middleware if file not found (default: true)
  fallthrough: true,

  // Redirect to trailing "/" when path is a directory (default: true)
  redirect: true,
}));

Serving with a Path Prefix

// Files served under /static prefix
app.use('/static', express.static(path.join(__dirname, 'public')));

// public/css/style.css -> GET /static/css/style.css

Multiple Static Directories

// Express checks directories in order
app.use(express.static(path.join(__dirname, 'public')));
app.use(express.static(path.join(__dirname, 'uploads')));
app.use(express.static(path.join(__dirname, 'assets')));

// First directory with matching file wins

SPA (Single Page Application) Fallback

// Serve static files first
app.use(express.static(path.join(__dirname, 'client/build')));

// API routes
app.use('/api', apiRouter);

// SPA fallback: serve index.html for all non-API, non-file routes
app.get('*', (req, res) => {
  res.sendFile(path.join(__dirname, 'client/build', 'index.html'));
});

Typical Middleware Registration Order

const app = express();

// 1. Security headers
app.use(helmet());

// 2. CORS
app.use(cors());

// 3. Request logging
app.use(morgan('dev'));

// 4. Body parsing
app.use(express.json({ limit: '10mb' }));
app.use(express.urlencoded({ extended: true }));

// 5. Static files
app.use(express.static(path.join(__dirname, 'public')));

// 6. API routes
app.use('/api', apiRouter);

// 7. SPA fallback (if applicable)

// 8. 404 handler
app.use((req, res) => {
  res.status(404).json({ error: 'Not found' });
});

// 9. Error handler (MUST be last, MUST have 4 params)
app.use(errorHandler);

Security Considerations

Body Size Limits

// Prevent large payload attacks
app.use(express.json({ limit: '1mb' })); // Reasonable for most APIs
// Default is 100kb which is fine for typical JSON APIs

Static File Security

// Never serve your project root!
// WRONG:
app.use(express.static(__dirname));  // Exposes source code, .env, etc!

// CORRECT: serve a dedicated public directory
app.use(express.static(path.join(__dirname, 'public')));

// Prevent dotfile access
app.use(express.static('public', { dotfiles: 'deny' }));

Common Pitfalls

1. Body Parser Not Matching Content-Type

// Client sends Content-Type: text/plain with JSON body
// express.json() ignores it — req.body stays undefined

// Solution: configure the type option
app.use(express.json({ type: ['application/json', 'text/plain'] }));

2. Static Files Taking Precedence Over Routes

// If public/api/users.json exists:
app.use(express.static('public'));
app.get('/api/users', handler); // Never reached if file exists!

// Solution: mount static files at a non-conflicting path
app.use('/assets', express.static('public'));
app.get('/api/users', handler); // Works fine

Summary

Express built-in middleware covers essential needs: express.json() and express.urlencoded() for body parsing, express.static() for file serving, plus express.raw() and express.text() for special cases. Configure body size limits for security, use dedicated public directories for static files, and register middleware in the correct order.

Exercises

  1. Configure Body Parsing for Multiple Content Types

    Set up an Express app that handles JSON bodies (limit 1mb), URL-encoded form data (extended: true), and raw binary data for a /webhooks endpoint. Create test routes that echo back the parsed body for each type.

    Hint

    Use express.json(), express.urlencoded(), and express.raw() with different type options. You can apply express.raw() only to the /webhooks path.

  2. Static File Server with Caching

    Configure express.static to serve a public directory with proper caching: HTML files should have no-cache, CSS/JS should have max-age of 1 year, and images should have max-age of 1 week. Use the setHeaders option.

    Hint

    Check the file extension in the setHeaders callback using path.extname(filePath) or string methods. Set different Cache-Control headers based on the extension.

  3. SPA with API Routes

    Build an Express app that serves a React/Vue SPA from a build directory, has API routes under /api, and falls back to index.html for all non-API routes (for client-side routing). Ensure API 404s return JSON, not HTML.

    Hint

    Order matters: static files first, then API routes, then the SPA fallback catch-all. The API 404 handler should be mounted on the /api router, not globally.

Common Problems & Solutions

1 req.body is an empty object {} instead of the expected data
Cause: The Content-Type header does not match what express.json() expects, or the body is malformed JSON.
Solution: Verify the client sends Content-Type: application/json. Test with curl -H 'Content-Type: application/json' -d '{"key":"value"}'. Check for syntax errors in the JSON body.
2 413 Payload Too Large error
Cause: The request body exceeds the limit set in express.json() or express.urlencoded(). The default limit is 100kb.
Solution: Increase the limit: express.json({ limit: '10mb' }). Set different limits for different routes if needed. For file uploads, use multer instead of body parsers.
