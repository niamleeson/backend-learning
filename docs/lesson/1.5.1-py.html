Exceptions: try, except, finally, else

Intuition

The try/except/else/finally pattern separates the happy path (else) from error recovery (except) and guaranteed cleanup (finally), making intent explicit.

Python's try/except/else/finally blocks give you fine-grained control over error handling. Understanding the execution order is essential for writing robust code.

def safe_divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("Cannot divide by zero")
        return None
    except TypeError as e:
        print(f"Type error: {e}")
        return None
    else:
        print("Division succeeded")  # runs only if no exception
        return result
    finally:
        print("Cleanup here")       # ALWAYS runs

print(safe_divide(10, 3))   # succeeds
print(safe_divide(10, 0))   # ZeroDivisionError caught
  • else runs only when no exception was raised — keep non-error logic here to avoid catching unrelated exceptions.
  • finally always executes, even after return, break, or continue.
  • Catch specific exceptions, not bare except: — that catches SystemExit and KeyboardInterrupt too.
  • You can chain multiple except clauses or catch a tuple: except (ValueError, KeyError).

Exercises

  1. Safe Integer Parser

    Write a function parse_int(s) that converts a string to an integer, returning a default value of 0 if the string is not a valid integer.

    Hint

    Wrap int(s) in a try/except catching ValueError.

  2. File Reader with Cleanup

    Write a function that reads a file and returns its contents, prints 'not found' if the file doesn't exist, and always prints 'done' at the end using finally.

    Hint

    Catch FileNotFoundError in the except block and put the print in finally.

Common Problems & Solutions

1 Using bare `except:` catches KeyboardInterrupt, making the program impossible to Ctrl+C.
Cause: Bare except catches all exceptions, including `BaseException` subclasses.
Solution: Always catch specific exceptions, or at minimum use `except Exception:`.
2 `finally` block overrides the return value from `try` or `except`.
Cause: If `finally` contains a `return`, it replaces any previous return value.
Solution: Avoid `return` statements inside `finally` blocks.
