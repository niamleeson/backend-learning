Immutability Patterns and Pure Functions

Intuition

Pure functions are like math equations: 2+3 always equals 5. Immutable data is like a printed document. You cannot change it, but you can make a new copy with edits. This predictability eliminates entire classes of bugs.

Immutability Patterns and Pure Functions

Pure functions always return the same output for the same input and have no side effects. Immutability patterns reduce bugs by preventing accidental state changes.

from dataclasses import dataclass
from typing import NamedTuple

# Immutable data with NamedTuple
class Point(NamedTuple):
    x: float
    y: float

# Immutable dataclass
@dataclass(frozen=True)
class User:
    name: str
    email: str
    age: int

# Pure function: no side effects, same input -> same output
def add_tax(price: float, rate: float = 0.1) -> float:
    return round(price * (1 + rate), 2)

# Returning new objects instead of mutating
def update_email(user: User, new_email: str) -> User:
    return User(name=user.name, email=new_email, age=user.age)

# Immutable collections
from types import MappingProxyType

config = MappingProxyType({"host": "localhost", "port": 8080})
# config["host"] = "x"  # Raises TypeError

# Avoiding mutation pitfalls
def append_safe(items: list[int], value: int) -> list[int]:
    return [*items, value]  # New list, original unchanged

Key Takeaways: - Pure functions are easier to test, debug, and reason about - Use frozen=True dataclasses or NamedTuple for immutable data - Return new objects instead of mutating arguments - MappingProxyType creates read-only dict views - Avoid mutable default arguments (def f(items=[])); use None instead

Common Problems & Solutions

1 Mutable default argument is shared across all calls: `def f(items=[])` accumulates data.
Cause: Default mutable arguments are created once at function definition time, not per call.
Solution: Use `None` as default and create a new object inside: `def f(items=None): items = items or []`.
2 Frozen dataclass cannot be updated; `FrozenInstanceError` on attribute assignment.
Cause: Frozen dataclasses prevent all attribute modification by design.
Solution: Use `dataclasses.replace(obj, field=new_value)` to create a new instance with the modified field.
