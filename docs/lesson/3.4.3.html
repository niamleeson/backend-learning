SQL Expression Language: select, insert, update, delete

Intuition

The expression language is like writing SQL with Python building blocks. Each method call adds a clause, and the final object compiles to proper SQL for your database.

SQLAlchemy SQL Expression Language

The expression language builds SQL statements in Python using Table and Column objects, giving you composable, database-agnostic queries.

from sqlalchemy import select, insert, update, delete, func, and_, or_

stmt = (
    select(users.c.name, func.count(posts.c.id).label('post_count'))
    .join(posts, users.c.id == posts.c.user_id)
    .where(users.c.created_at >= '2024-01-01')
    .group_by(users.c.name)
    .having(func.count(posts.c.id) > 5)
    .order_by(func.count(posts.c.id).desc())
)

with engine.connect() as conn:
    rows = conn.execute(stmt).fetchall()

# INSERT, UPDATE, DELETE
stmt = insert(users).values(name='Alice', email='alice@example.com')
stmt = update(users).where(users.c.id == 1).values(name='Bob')
stmt = delete(users).where(users.c.active == False)

# Composable: build queries dynamically
stmt = select(users)
if filter_active:
    stmt = stmt.where(users.c.active == True)
if search_name:
    stmt = stmt.where(users.c.name.ilike(f'%{search_name}%'))

Key Takeaways: - select(), insert(), update(), delete() create statement objects that compile to SQL. - Use .c to access columns: table.c.column_name. Chain .where(), .join(), .order_by(). - Statements are composable: add filters conditionally by reassigning the statement variable.

Exercises

  2. Bulk Insert with Returning

    Insert multiple rows using insert().values([...]) and retrieve the generated IDs using .returning(table.c.id).

    Hint

    stmt = insert(users).values([{...}, {...}]).returning(users.c.id); result = conn.execute(stmt).

Common Problems & Solutions

1 TypeError when comparing Column to Python None.
Cause: Using == None instead of .is_(None) for NULL checks.
Solution: Use column.is_(None) and column.is_not(None) for NULL comparisons.
2 AND/OR logic produces wrong SQL.
Cause: Python's and/or keywords do not work with SQLAlchemy expressions.
Solution: Use and_(), or_() functions or the & and | operators with parentheses.
