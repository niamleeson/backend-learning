Decorators: Building and Stacking

Intuition

A decorator is a gift-wrapping machine: the original function is the gift, and the decorator adds wrapping (extra behavior) around it without changing what's inside.

Decorators are functions that wrap other functions to extend their behavior without modifying their source code. They use the @ syntax and are essential for logging, auth, caching, and more.

import functools
import time

def timer(func):
    """Measure execution time of a function."""
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"{func.__name__} took {elapsed:.4f}s")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(1)
    return "done"

# Stacking decorators (applied bottom-up)
@timer
@functools.lru_cache(maxsize=128)
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)
  • Always use @functools.wraps(func) to preserve the original function's metadata
  • @decorator is syntactic sugar for func = decorator(func)
  • Stacked decorators apply bottom-up: the lowest decorator wraps first
  • Use functools.lru_cache for memoization out of the box

Exercises

  1. Retry Decorator

    Write a @retry(max_attempts=3) decorator that catches exceptions and retries the function up to max_attempts times before re-raising.

    Hint

    You need a decorator factory: a function that returns a decorator. Use a for loop with try/except inside the wrapper.

  2. Debug Logger

    Write a @debug decorator that prints the function name, arguments, and return value each time it's called.

    Hint

    In the wrapper, print func.__name__, args, kwargs, call the function, print the result, and return it.

Common Problems & Solutions

1 Decorated function loses its __name__ and __doc__
Cause: Without `@functools.wraps`, the wrapper replaces the original function's metadata.
Solution: Add `@functools.wraps(func)` to the wrapper function inside your decorator.
2 Decorator with arguments requires three levels of nesting
Cause: A parameterized decorator needs a factory function that returns the actual decorator.
Solution: Structure as: `def decorator(args) -> def actual_decorator(func) -> def wrapper(*a, **kw)`.
