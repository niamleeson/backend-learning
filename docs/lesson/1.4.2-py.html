Dictionaries: Internals and Patterns

Intuition

A dict is a hash table where keys are hashed to bucket indices. This gives O(1) average lookups but requires keys to be immutable and hashable.

Dictionaries are hash-map-based collections that map keys to values in average O(1) time. Since Python 3.7, insertion order is guaranteed.

user = {"name": "Alice", "age": 30, "role": "engineer"}

# Safe access patterns
age = user.get("age", 0)           # returns 0 if missing
user.setdefault("dept", "eng")     # insert only if absent

# Merge (Python 3.9+)
defaults = {"role": "viewer", "theme": "dark"}
merged = defaults | user            # user values win

# Dictionary comprehension
squares = {n: n**2 for n in range(6)}
# {0: 0, 1: 1, 2: 4, 3: 9, 4: 16, 5: 25}
  • Keys must be hashable (immutable types like str, int, tuple of hashables).
  • Use .get() or in checks to avoid KeyError in production code.
  • The merge operator | (Python 3.9+) creates a new dict; |= updates in place.
  • Iterating yields keys; use .items() for key-value pairs.

Exercises

  1. Word Frequency Counter

    Write a function word_freq(text) that returns a dictionary mapping each word (lowercased) to its count. Split on whitespace.

    Hint

    Use dict.get(word, 0) + 1 or setdefault to build counts incrementally.

  2. Invert a Dictionary

    Write a function invert(d) that swaps keys and values. If multiple keys share a value, collect them in a list. E.g., invert({'a':1,'b':1,'c':2}) returns {1:['a','b'], 2:['c']}.

    Hint

    Use setdefault on the result dict to initialize lists for each value.

Common Problems & Solutions

1 RuntimeError: dictionary changed size during iteration.
Cause: Adding or deleting keys while iterating mutates the hash table mid-traversal.
Solution: Iterate over `list(d.keys())` or build a new dict instead of modifying in place.
2 Using a mutable type (e.g., list) as a dictionary key raises `TypeError: unhashable type`.
Cause: Lists are mutable and therefore not hashable; dict keys must be hashable.
Solution: Convert the list to a tuple before using it as a key.
