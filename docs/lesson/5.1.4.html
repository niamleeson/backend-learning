Auto-Generated Documentation (Swagger/ReDoc)

Intuition

FastAPI turns your type hints and decorators into a living, interactive contract that frontend developers and API consumers can explore without reading your source code. The documentation is always in sync because it is generated from the code itself.

Auto-Generated Documentation: Swagger and ReDoc

FastAPI automatically generates interactive API documentation from your code. Swagger UI provides a try-it-out interface, while ReDoc offers a clean reference layout. Both are powered by OpenAPI.

from fastapi import FastAPI, Query
from pydantic import BaseModel, Field

app = FastAPI(
    title="My API",
    description="A production-ready API with auto-generated docs",
    version="1.0.0",
    docs_url="/docs",
    redoc_url="/redoc",
    openapi_tags=[
        {"name": "users", "description": "User management operations"},
        {"name": "items", "description": "Item CRUD operations"},
    ]
)

class ItemCreate(BaseModel):
    """Schema for creating a new item."""
    name: str = Field(..., example="Widget", description="Item name")
    price: float = Field(..., gt=0, example=9.99)

@app.post("/items/", tags=["items"], summary="Create an item",
          description="Create a new item with name and price.")
async def create_item(item: ItemCreate):
    return {"item_id": 1, **item.model_dump()}

Key Takeaways: - Swagger UI is available at /docs and ReDoc at /redoc by default - Docstrings, summary, description, and tags enrich documentation - Pydantic Field(example=...) populates example values in the docs - OpenAPI JSON schema is available at /openapi.json for client generation

Exercises

  1. Customize OpenAPI Schema

    Create a FastAPI app with at least 3 endpoints organized into tagged groups. Add detailed descriptions, examples, and custom response descriptions using the responses parameter in the route decorator.

    Hint

    Use the responses dict in the decorator: responses={404: {"description": "Not found", "content": {"application/json": {"example": {"detail": "Item not found"}}}}} to document error responses.

  2. Generate a TypeScript Client

    Export the OpenAPI JSON from a FastAPI app and use openapi-typescript-codegen to generate a TypeScript HTTP client. Verify the generated types match your Pydantic models.

    Hint

    Fetch /openapi.json from your running app, save it to a file, then run npx openapi-typescript-codegen --input openapi.json --output ./client.

Common Problems & Solutions

1 Swagger UI not loading, returning 404 at /docs
Cause: The `docs_url` was set to `None` or a custom path, or a root path prefix is misconfigured behind a reverse proxy.
Solution: Check that `docs_url` is not set to `None` in the FastAPI constructor. If behind a proxy, set `root_path` in FastAPI or via the `--root-path` flag in uvicorn.
2 Example values not appearing in Swagger UI
Cause: Using `schema_extra` from Pydantic v1 syntax instead of v2's `json_schema_extra` or `Field(example=...)`.
Solution: In Pydantic v2, use `Field(..., examples=["value"])` or set `model_config = {"json_schema_extra": {"examples": [...]}}` on the model class.
