Request and Response Objects

Intuition

Think of req and res as two halves of a conversation. req is the incoming letter — it carries the sender's address (URL), the message (body), and any special instructions (headers). res is your reply — you choose the tone (status code), attach notes (headers), and write your answer (body).

Request and Response Objects

Introduction

Every Express route handler receives two powerful objects: req (the Request) and res (the Response). These objects wrap Node.js's native http.IncomingMessage and http.ServerResponse with a rich API for reading incoming data and crafting outgoing responses. Mastering these objects is essential — they are the interface between your application logic and the HTTP protocol.

The Request Object (req)

The request object represents the HTTP request from the client. Express enhances Node.js's native request with many convenience properties and methods.

Core Request Properties

import { Request, Response } from 'express';

app.get('/api/users/:id', (req: Request, res: Response) => {
  // URL and path information
  req.path;          // "/api/users/42"
  req.baseUrl;       // "" (or prefix if using Router)
  req.originalUrl;   // "/api/users/42?fields=name,email"
  req.url;           // "/api/users/42?fields=name,email"
  req.method;        // "GET"
  req.protocol;      // "http" or "https"
  req.hostname;      // "localhost"
  req.ip;            // "127.0.0.1"

  // Route parameters
  req.params;        // { id: "42" }
  req.params.id;     // "42"

  // Query string
  req.query;         // { fields: "name,email" }

  // Request body (requires body-parsing middleware)
  req.body;          // { name: "Alice" } (for POST/PUT/PATCH)

  // Headers
  req.headers;       // { "content-type": "application/json", ... }
  req.get('Content-Type');  // "application/json"
  req.get('Authorization'); // "Bearer eyJhbG..."
});

Reading Headers

app.get('/api/data', (req: Request, res: Response) => {
  // req.get() / req.header() - case-insensitive header access
  const contentType = req.get('content-type');
  const authHeader = req.get('authorization');
  const userAgent = req.get('user-agent');

  // Check accepted content types
  req.accepts('json');          // "json" or false
  req.accepts('html');          // "html" or false
  req.accepts(['json', 'xml']); // best match or false

  // Check content type of the request body
  req.is('json');               // "json" or false
  req.is('application/json');   // "application/json" or false

  res.json({ contentType, authHeader });
});

Request Body

The request body is only available if body-parsing middleware is registered:

// Must register these BEFORE route handlers
app.use(express.json());                         // Parse JSON bodies
app.use(express.urlencoded({ extended: true }));  // Parse form data
app.use(express.raw({ type: 'application/octet-stream' })); // Raw binary
app.use(express.text({ type: 'text/plain' }));   // Plain text

app.post('/api/users', (req: Request, res: Response) => {
  // For JSON request: Content-Type: application/json
  // Body: {"name": "Alice", "email": "alice@example.com"}
  const { name, email } = req.body;

  res.status(201).json({ user: { name, email } });
});

Cookies

import cookieParser from 'cookie-parser';

app.use(cookieParser());

app.get('/api/profile', (req: Request, res: Response) => {
  const sessionId = req.cookies.sessionId;
  const preferences = req.signedCookies.preferences;
  res.json({ sessionId });
});

Custom Properties on req

Middleware often attaches custom data to the request object:

// Extend the Express Request type
declare global {
  namespace Express {
    interface Request {
      user?: { id: number; email: string; role: string };
      requestId?: string;
    }
  }
}

// Auth middleware attaches user
const authMiddleware = (req: Request, res: Response, next: Function) => {
  const token = req.get('authorization')?.replace('Bearer ', '');
  if (token) {
    req.user = { id: 1, email: 'alice@example.com', role: 'admin' };
  }
  next();
};

app.get('/api/profile', authMiddleware, (req: Request, res: Response) => {
  res.json({ user: req.user });
});

The Response Object (res)

The response object is used to send data back to the client.

Sending Responses

app.get('/examples', (req: Request, res: Response) => {

  // JSON response (most common for APIs)
  res.json({ message: 'Hello' });
  // Sets Content-Type: application/json automatically

  // Plain text
  res.send('Hello World');

  // HTML
  res.send('<h1>Hello World</h1>');

  // Send with specific status
  res.status(201).json({ created: true });

  // No body (for DELETE responses)
  res.status(204).send();
  // or
  res.sendStatus(204); // Sets status AND sends status text as body
});

Status Codes

// res.status() sets the HTTP status code and is chainable
res.status(200).json({ data: [] });     // OK
res.status(201).json({ id: 1 });        // Created
res.status(204).send();                  // No Content
res.status(400).json({ error: '...' }); // Bad Request
res.status(401).json({ error: '...' }); // Unauthorized
res.status(403).json({ error: '...' }); // Forbidden
res.status(404).json({ error: '...' }); // Not Found
res.status(409).json({ error: '...' }); // Conflict
res.status(422).json({ error: '...' }); // Unprocessable Entity
res.status(500).json({ error: '...' }); // Internal Server Error

// res.sendStatus() sends status + default status message as body
res.sendStatus(200); // Sends "OK"
res.sendStatus(404); // Sends "Not Found"

Setting Headers

app.get('/api/data', (req: Request, res: Response) => {
  // Set a single header
  res.set('X-Request-Id', '12345');
  res.set('Cache-Control', 'no-cache');

  // Set multiple headers
  res.set({
    'X-Powered-By': 'My API',
    'X-Response-Time': '42ms'
  });

  // Shorthand (alias for res.set)
  res.header('X-Custom', 'value');

  // Append to existing header
  res.append('Set-Cookie', 'foo=bar');
  res.append('Set-Cookie', 'baz=qux');

  res.json({ data: 'value' });
});

Cookies

app.post('/login', (req: Request, res: Response) => {
  // Set a cookie
  res.cookie('sessionId', 'abc123', {
    httpOnly: true,   // Not accessible via JavaScript
    secure: true,     // Only sent over HTTPS
    sameSite: 'strict',
    maxAge: 24 * 60 * 60 * 1000, // 1 day in milliseconds
    path: '/'
  });

  // Clear a cookie
  res.clearCookie('sessionId');

  res.json({ message: 'Logged in' });
});

Redirects

// Temporary redirect (302)
res.redirect('/new-location');

// Permanent redirect (301)
res.redirect(301, '/new-permanent-location');

// Redirect back to referrer
res.redirect('back');

File Downloads and Streaming

// Send a file
app.get('/download', (req, res) => {
  res.download('/path/to/file.pdf', 'report.pdf');
});

// Send a file inline (display in browser)
app.get('/file', (req, res) => {
  res.sendFile('/absolute/path/to/file.pdf');
});

// Stream a response
app.get('/stream', (req, res) => {
  res.set('Content-Type', 'text/plain');
  res.write('Chunk 1\n');
  res.write('Chunk 2\n');
  res.write('Chunk 3\n');
  res.end();
});

TypeScript: Typing Request and Response

Express generics let you fully type your handlers:

// Request<Params, ResBody, ReqBody, Query>
interface CreateUserBody {
  name: string;
  email: string;
}

interface UserResponse {
  id: number;
  name: string;
  email: string;
}

app.post(
  '/api/users',
  (req: Request<{}, UserResponse, CreateUserBody>, res: Response<UserResponse>) => {
    const { name, email } = req.body; // Typed as CreateUserBody
    const user: UserResponse = { id: 1, name, email };
    res.json(user); // Must match UserResponse
  }
);

Common Patterns

Consistent API Response Format

// Helper function for consistent responses
function sendSuccess<T>(res: Response, data: T, statusCode = 200) {
  res.status(statusCode).json({
    success: true,
    data
  });
}

function sendError(res: Response, message: string, statusCode = 400) {
  res.status(statusCode).json({
    success: false,
    error: message
  });
}

app.get('/api/users/:id', (req, res) => {
  const user = { id: 1, name: 'Alice' };
  if (!user) return sendError(res, 'User not found', 404);
  sendSuccess(res, user);
});

Content Negotiation

app.get('/api/data', (req: Request, res: Response) => {
  const data = { name: 'Express', version: '4.x' };

  res.format({
    'application/json': () => {
      res.json(data);
    },
    'text/html': () => {
      res.send('<h1>' + data.name + '</h1>');
    },
    'text/plain': () => {
      res.send(data.name + ' ' + data.version);
    },
    default: () => {
      res.status(406).send('Not Acceptable');
    }
  });
});

Common Pitfalls

1. Sending Multiple Responses

// WRONG
app.get('/test', (req, res) => {
  res.json({ a: 1 });
  res.json({ b: 2 }); // Error: Cannot set headers after they are sent
});

// CORRECT: only send one response
app.get('/test', (req, res) => {
  if (someCondition) {
    return res.json({ a: 1 });
  }
  res.json({ b: 2 });
});

2. Forgetting Async Error Handling

// WRONG: unhandled promise rejection
app.get('/data', async (req, res) => {
  const data = await fetchData(); // If this throws, Express won't catch it
  res.json(data);
});

// CORRECT: wrap in try-catch
app.get('/data', async (req, res, next) => {
  try {
    const data = await fetchData();
    res.json(data);
  } catch (error) {
    next(error); // Forward to error handler
  }
});

Summary

The request and response objects are your primary tools for handling HTTP communication. The request provides access to parameters, query strings, headers, body, and cookies. The response lets you set status codes, headers, cookies, and send data in various formats. Type them properly with TypeScript and use consistent patterns for clean, maintainable API code.

Exercises

  1. Request Inspector Endpoint

    Create a GET /inspect endpoint that returns a JSON object containing: method, path, full URL, all query parameters, all headers, IP address, and the user-agent string. This is useful for debugging.

    Hint

    Use req.method, req.path, req.originalUrl, req.query, req.headers, req.ip, and req.get('user-agent').

  2. Content Negotiation Handler

    Build a GET /api/info endpoint that returns data in different formats based on the Accept header. Return JSON for application/json, plain text for text/plain, and XML-like string for application/xml. Use res.format() to handle negotiation.

    Hint

    res.format() accepts an object where keys are MIME types and values are handler functions. Include a 'default' key for unsupported types.

  3. Typed Request Handlers

    Create a POST /api/products endpoint with full TypeScript typing. Define interfaces for the request body (name, price, category), response body (product with id), and query parameters. Use Express generic types: Request.

    Hint

    The Request generic order is: Request. Use {} for unused type parameters.

Common Problems & Solutions

1 req.body is undefined even though express.json() is registered
Cause: The client is not sending the Content-Type: application/json header, so Express does not parse the body as JSON.
Solution: Ensure the client sends the Content-Type header. In fetch/axios, this is usually automatic for JSON. For curl, add -H 'Content-Type: application/json'. You can also check with req.is('json').
2 res.json() throws 'Cannot set headers after they are sent to the client'
Cause: A response has already been sent earlier in the handler (or in middleware), and the code tries to send another response.
Solution: Use 'return' statements after sending responses in conditional branches. Check that middleware calling next() does not also send a response.
