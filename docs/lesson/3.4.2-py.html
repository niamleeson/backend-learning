MetaData, Table, and Column Definitions

Intuition

MetaData is your schema catalog in Python. It knows every table and column, and can either define new ones or learn about existing ones by peeking at the database.

SQLAlchemy Core: MetaData, Table, Column

MetaData is a collection of Table objects that describe your database schema in Python. Tables and columns are defined programmatically or reflected from the database.

from sqlalchemy import MetaData, Table, Column, Integer, String, ForeignKey, DateTime, func

metadata = MetaData()

users = Table('users', metadata,
    Column('id', Integer, primary_key=True),
    Column('name', String(100), nullable=False),
    Column('email', String(255), unique=True, nullable=False),
    Column('created_at', DateTime, server_default=func.now())
)

posts = Table('posts', metadata,
    Column('id', Integer, primary_key=True),
    Column('user_id', Integer, ForeignKey('users.id'), nullable=False),
    Column('title', String(200)),
    Column('body', String)
)

metadata.create_all(engine)

# Reflect existing tables from database
metadata_reflected = MetaData()
metadata_reflected.reflect(bind=engine)
existing_table = metadata_reflected.tables['users']

Key Takeaways: - MetaData holds all Table definitions; create_all() generates DDL only for tables that do not yet exist. - Column types map to SQL types: Integer, String, Text, DateTime, Boolean, Numeric, etc. - reflect() introspects an existing database and builds Table objects automatically.

Common Problems & Solutions

1 create_all() does not update existing tables with new columns.
Cause: create_all() only creates tables that do not exist; it does not alter existing ones.
Solution: Use Alembic migrations for schema changes, or manually run ALTER TABLE statements.
2 reflect() fails or returns empty tables.
Cause: Wrong schema name, or the database user lacks SELECT privileges on information_schema.
Solution: Specify schema='public' in reflect(), and ensure the DB user has adequate permissions.
