WebSocket Endpoints and Connection Management

Intuition

WebSockets are like phone calls compared to HTTP's postal mail: once the line is open, both sides can talk freely without re-dialing. The connection manager is your switchboard operator, knowing who is on the line and routing messages.

WebSocket Endpoints and Connection Management

FastAPI supports WebSocket connections for real-time bidirectional communication. Unlike HTTP, WebSockets maintain a persistent connection, requiring explicit connection lifecycle management.

from fastapi import FastAPI, WebSocket, WebSocketDisconnect

app = FastAPI()

class ConnectionManager:
    def __init__(self):
        self.active_connections: list[WebSocket] = []

    async def connect(self, websocket: WebSocket):
        await websocket.accept()
        self.active_connections.append(websocket)

    def disconnect(self, websocket: WebSocket):
        self.active_connections.remove(websocket)

    async def send_personal(self, message: str, websocket: WebSocket):
        await websocket.send_text(message)

    async def broadcast(self, message: str):
        for conn in self.active_connections:
            await conn.send_text(message)

manager = ConnectionManager()

@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    await manager.connect(websocket)
    try:
        while True:
            data = await websocket.receive_text()
            await manager.broadcast(f"Message: {data}")
    except WebSocketDisconnect:
        manager.disconnect(websocket)
        await manager.broadcast("A client disconnected")

Key Takeaways: - WebSocket endpoints use @app.websocket() instead of HTTP method decorators - Always call await websocket.accept() before sending or receiving data - Handle WebSocketDisconnect to clean up resources when clients disconnect - A connection manager pattern keeps track of all active connections

Exercises

  1. Authenticated WebSocket

    Create a WebSocket endpoint that requires a JWT token passed as a query parameter during the handshake. Validate the token before accepting the connection, and close with code 4001 if invalid.

    Hint

    Access the token via websocket.query_params.get('token'). Validate it before calling accept(). If invalid, use await websocket.close(code=4001, reason='Unauthorized').

  2. Heartbeat Mechanism

    Implement a heartbeat system where the server sends a 'ping' every 30 seconds and expects a 'pong' response within 10 seconds. Disconnect clients that fail to respond.

    Hint

    Use asyncio.create_task to run a background coroutine per connection that sends pings on a timer. Use asyncio.wait_for(websocket.receive_text(), timeout=10) to detect missing pongs.

Common Problems & Solutions

1 WebSocket connection drops silently without triggering disconnect handler
Cause: Network interruptions or client crashes can cause the TCP connection to hang without sending a close frame.
Solution: Implement a heartbeat/ping mechanism. Use `websocket.send_text('ping')` periodically and close the connection if no response arrives within a timeout. This detects dead connections proactively.
2 ConnectionManager not thread-safe with multiple workers
Cause: Each Uvicorn worker process has its own ConnectionManager instance, so connections on one worker cannot broadcast to connections on another.
Solution: Use a shared pub/sub backend like Redis. Each worker subscribes to a Redis channel. On broadcast, publish to Redis, and each worker forwards the message to its local connections.
