Pub/Sub and Lua Scripting

Intuition

Pub/Sub is like a radio broadcast: anyone tuned in hears the message. Lua scripting is like handing the server a recipe card to execute all steps without interruption.

Pub/Sub and Lua Scripting in Redis

Pub/Sub enables real-time messaging between services. Lua scripting executes atomic multi-step operations server-side.

import redis, threading

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# Publisher
r.publish('notifications', 'New order received')

# Subscriber (runs in its own thread)
def subscribe_loop():
    pubsub = r.pubsub()
    pubsub.subscribe('notifications', 'alerts')
    for message in pubsub.listen():
        if message['type'] == 'message':
            print(f"[{message['channel']}] {message['data']}")

thread = threading.Thread(target=subscribe_loop, daemon=True)
thread.start()

# Lua scripting: atomic rate limiter
rate_limit_script = """
local key = KEYS[1]
local limit = tonumber(ARGV[1])
local window = tonumber(ARGV[2])
local current = tonumber(redis.call('GET', key) or '0')
if current >= limit then
    return 0
end
redis.call('INCR', key)
if current == 0 then
    redis.call('EXPIRE', key, window)
end
return 1
"""
rate_limit = r.register_script(rate_limit_script)
allowed = rate_limit(keys=['rate:user:42'], args=[10, 60])

Key Takeaways: - Pub/Sub is fire-and-forget; messages are lost if no subscriber is listening. Use Streams for persistence. - Lua scripts execute atomically on the server, avoiding race conditions in multi-step operations. - register_script() caches the script SHA for efficient repeated execution.

Exercises

  1. Chat Room with Pub/Sub

    Build a simple chat system where users subscribe to a room channel and publish messages to it.

    Hint

    Each user runs pubsub.subscribe('room:general'). Publishing is r.publish('room:general', message).

  2. Atomic Token Bucket

    Implement a token bucket rate limiter using a Lua script that atomically checks and decrements available tokens.

    Hint

    Lua script: GET tokens, if > 0 DECR and return 1, else return 0. Use EXPIRE for refill window.

Common Problems & Solutions

1 Subscriber misses messages published before it connected.
Cause: Pub/Sub does not buffer messages; only active subscribers receive them.
Solution: Use Redis Streams (XADD/XREAD) for durable message queues with consumer groups.
2 Lua script blocks all other Redis commands while running.
Cause: Redis is single-threaded; Lua scripts hold the event loop.
Solution: Keep Lua scripts short and fast. Avoid unbounded loops or large data operations in scripts.
