Working with JSON

Intuition

JSON is a text format with a direct mapping to Python dicts and lists. The json module is just a translator between Python objects and that text format.

JSON is the lingua franca of web APIs and configuration files. Python's json module handles serialization and deserialization between Python objects and JSON strings.

import json

data = {
    "name": "Alice",
    "scores": [95, 87, 92],
    "active": True,
    "address": None
}

# Serialize to string
json_str = json.dumps(data, indent=2)

# Write to file
with open("data.json", "w", encoding="utf-8") as f:
    json.dump(data, f, indent=2)

# Read from file
with open("data.json", encoding="utf-8") as f:
    loaded = json.load(f)

print(loaded["name"])  # Alice
print(type(loaded["address"]))  # <class 'NoneType'>
  • json.dumps() returns a string; json.dump() writes directly to a file.
  • json.loads() parses a string; json.load() reads from a file.
  • JSON maps to Python: object -> dict, array -> list, null -> None, true/false -> True/False.
  • Custom objects need a serializer: pass default=str or define a custom encoder for types like datetime.

Common Problems & Solutions

1 `TypeError: Object of type datetime is not JSON serializable`.
Cause: The default encoder only handles basic types (dict, list, str, int, float, bool, None).
Solution: Pass `default=str` for quick fixes, or write a custom encoder for production code.
2 JSON keys are always strings; integer keys become strings after round-tripping.
Cause: JSON spec only allows string keys, so `{1: 'a'}` becomes `{'1': 'a'}`.
Solution: Convert keys back to int after loading, or use string keys from the start.
