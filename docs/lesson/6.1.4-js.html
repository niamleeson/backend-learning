Shell Scripting Basics for Backend Devs

Intuition

A shell script is like a recipe card for your server. Instead of manually typing commands every time you deploy, back up, or check health, you write them down once and let the script repeat the exact same steps perfectly every time. The set -euo pipefail at the top is like telling the chef: 'If any step fails, stop immediately and tell me -- don't try to serve a half-cooked dish.'

Shell Scripting Basics for Backend Devs

Why Shell Scripting?

As a backend developer, you will write shell scripts for deployment automation, server setup, backup procedures, health checks, and CI/CD pipelines. While Node.js scripts can handle many tasks, shell scripts are the lingua franca of server automation -- they work everywhere, require no runtime installation, and integrate naturally with system tools.

Shell Script Fundamentals

Your First Script

#!/bin/bash
# deploy.sh - Deploy application to production
# Usage: ./deploy.sh [environment]

set -euo pipefail    # The most important line in any bash script

echo "Starting deployment..."

The set -euo pipefail flags: - -e: Exit immediately on any error - -u: Treat unset variables as errors - -o pipefail: Pipe fails if any command in the pipeline fails

Variables

#!/bin/bash
set -euo pipefail

# Variable assignment (no spaces around =)
APP_NAME="myapp"
PORT=3000
DEPLOY_DIR="/var/www/${APP_NAME}"

# Command substitution
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
GIT_HASH=$(git rev-parse --short HEAD)
NODE_VERSION=$(node --version)

# Default values
ENVIRONMENT=${1:-production}          # First arg, default "production"
LOG_LEVEL=${LOG_LEVEL:-info}          # Env var with default

# String operations
FILENAME="server.js"
echo "${FILENAME%.js}"                # Remove suffix: "server"
echo "${FILENAME##*.}"                # Get extension: "js"
echo "${APP_NAME^^}"                  # Uppercase: "MYAPP"
echo "${APP_NAME,,}"                  # Lowercase: "myapp"

# Arrays
SERVERS=("web1" "web2" "web3")
echo "${SERVERS[0]}"                  # First element
echo "${SERVERS[@]}"                  # All elements
echo "${#SERVERS[@]}"                 # Length: 3

Conditionals

#!/bin/bash
set -euo pipefail

# if/elif/else
if [[ "$NODE_ENV" == "production" ]]; then
    echo "Running in production"
elif [[ "$NODE_ENV" == "staging" ]]; then
    echo "Running in staging"
else
    echo "Running in development"
fi

# File tests
if [[ -f ".env" ]]; then
    echo ".env file exists"
fi

if [[ -d "node_modules" ]]; then
    echo "Dependencies installed"
fi

if [[ ! -f "dist/server.js" ]]; then
    echo "Build not found, building..."
    npm run build
fi

# Numeric comparisons
if [[ $PORT -lt 1024 ]]; then
    echo "Port requires root privileges"
fi

# String checks
if [[ -z "$DATABASE_URL" ]]; then
    echo "ERROR: DATABASE_URL is not set"
    exit 1
fi

if [[ -n "$REDIS_URL" ]]; then
    echo "Redis URL configured"
fi

# Command success/failure
if command -v docker &> /dev/null; then
    echo "Docker is installed"
else
    echo "Docker is NOT installed"
    exit 1
fi

# Combining conditions
if [[ -f ".env" ]] && [[ "$NODE_ENV" == "production" ]]; then
    source .env
fi

Loops

#!/bin/bash
set -euo pipefail

# For loop
SERVERS=("web1.example.com" "web2.example.com" "web3.example.com")
for server in "${SERVERS[@]}"; do
    echo "Deploying to $server..."
    ssh "deploy@${server}" "cd /var/www/app && git pull && npm ci && pm2 reload all"
done

# C-style for loop
for ((i=1; i<=5; i++)); do
    echo "Attempt $i of 5..."
    if curl -sf http://localhost:3000/health > /dev/null; then
        echo "Health check passed!"
        break
    fi
    sleep 2
done

# While loop
RETRIES=0
MAX_RETRIES=10
while [[ $RETRIES -lt $MAX_RETRIES ]]; do
    if pg_isready -h localhost -p 5432 > /dev/null 2>&1; then
        echo "PostgreSQL is ready!"
        break
    fi
    echo "Waiting for PostgreSQL... (attempt $((RETRIES+1))/$MAX_RETRIES)"
    RETRIES=$((RETRIES + 1))
    sleep 2
done

if [[ $RETRIES -eq $MAX_RETRIES ]]; then
    echo "PostgreSQL did not become ready in time"
    exit 1
fi

# Read file line by line
while IFS= read -r line; do
    echo "Processing: $line"
done < servers.txt

# Iterate over files
for file in src/**/*.ts; do
    echo "TypeScript file: $file"
done

Functions

#!/bin/bash
set -euo pipefail

# Function definition
log() {
    local level=$1
    shift
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [$level] $*"
}

log "INFO" "Starting deployment"
log "ERROR" "Something went wrong"

# Function with return value
check_dependency() {
    local cmd=$1
    if command -v "$cmd" &> /dev/null; then
        return 0
    else
        return 1
    fi
}

if check_dependency "docker"; then
    log "INFO" "Docker found"
fi

# Function with output capture
get_free_port() {
    local port
    port=$(python3 -c 'import socket; s=socket.socket(); s.bind(("",0)); print(s.getsockname()[1]); s.close()')
    echo "$port"
}

FREE_PORT=$(get_free_port)
echo "Using port: $FREE_PORT"

# Cleanup function with trap
cleanup() {
    log "INFO" "Cleaning up temporary files..."
    rm -rf "$TMP_DIR"
}

TMP_DIR=$(mktemp -d)
trap cleanup EXIT    # Runs cleanup on script exit (even on error)

Real-World Scripts

Deployment Script

#!/bin/bash
set -euo pipefail

# Configuration
APP_DIR="/var/www/myapp"
RELEASES_DIR="$APP_DIR/releases"
CURRENT_LINK="$APP_DIR/current"
SHARED_DIR="$APP_DIR/shared"
KEEP_RELEASES=5

TIMESTAMP=$(date +%Y%m%d%H%M%S)
RELEASE_DIR="$RELEASES_DIR/$TIMESTAMP"

log() { echo "[$(date '+%H:%M:%S')] $*"; }

# Create release directory
log "Creating release $TIMESTAMP"
mkdir -p "$RELEASE_DIR"

# Clone/copy application
log "Copying application files..."
git archive HEAD | tar -x -C "$RELEASE_DIR"

# Install dependencies
log "Installing dependencies..."
cd "$RELEASE_DIR"
npm ci --production

# Build
log "Building..."
npm run build

# Link shared files
log "Linking shared resources..."
ln -sf "$SHARED_DIR/.env" "$RELEASE_DIR/.env"
ln -sf "$SHARED_DIR/uploads" "$RELEASE_DIR/uploads"

# Run migrations
log "Running database migrations..."
npx prisma migrate deploy

# Switch symlink (atomic deployment)
log "Switching to new release..."
ln -sfn "$RELEASE_DIR" "$CURRENT_LINK"

# Restart application
log "Restarting application..."
sudo systemctl restart myapp

# Health check
log "Running health check..."
sleep 3
if curl -sf http://localhost:3000/health > /dev/null; then
    log "Deployment successful!"
else
    log "Health check failed! Rolling back..."
    PREVIOUS=$(ls -1t "$RELEASES_DIR" | sed -n '2p')
    if [[ -n "$PREVIOUS" ]]; then
        ln -sfn "$RELEASES_DIR/$PREVIOUS" "$CURRENT_LINK"
        sudo systemctl restart myapp
        log "Rolled back to $PREVIOUS"
    fi
    exit 1
fi

# Clean up old releases
log "Cleaning old releases..."
cd "$RELEASES_DIR"
ls -1t | tail -n +$((KEEP_RELEASES + 1)) | xargs -r rm -rf

log "Done!"

Database Backup Script

#!/bin/bash
set -euo pipefail

# Configuration
DB_NAME="myapp_production"
DB_USER="myapp"
BACKUP_DIR="/var/backups/postgresql"
S3_BUCKET="s3://myapp-backups/database"
RETENTION_DAYS=30

TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/${DB_NAME}_${TIMESTAMP}.sql.gz"

log() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*"; }

# Create backup directory
mkdir -p "$BACKUP_DIR"

# Create backup
log "Starting backup of $DB_NAME..."
pg_dump -U "$DB_USER" -h localhost "$DB_NAME" | gzip > "$BACKUP_FILE"

# Verify backup
BACKUP_SIZE=$(stat -f%z "$BACKUP_FILE" 2>/dev/null || stat -c%s "$BACKUP_FILE")
if [[ $BACKUP_SIZE -lt 1000 ]]; then
    log "ERROR: Backup file is suspiciously small ($BACKUP_SIZE bytes)"
    exit 1
fi

log "Backup created: $BACKUP_FILE ($BACKUP_SIZE bytes)"

# Upload to S3 (if aws cli is available)
if command -v aws &> /dev/null; then
    log "Uploading to S3..."
    aws s3 cp "$BACKUP_FILE" "$S3_BUCKET/"
    log "Upload complete"
fi

# Clean up old local backups
log "Removing backups older than $RETENTION_DAYS days..."
find "$BACKUP_DIR" -name "*.sql.gz" -mtime +$RETENTION_DAYS -delete

log "Backup completed successfully"

Health Check Script

#!/bin/bash
set -euo pipefail

SERVICES=(
    "http://localhost:3000/health|Node.js API"
    "http://localhost:5432|PostgreSQL"
    "http://localhost:6379|Redis"
)

STATUS=0

check_http() {
    local url=$1
    local name=$2
    local response
    local http_code

    http_code=$(curl -sf -o /dev/null -w "%{http_code}" --max-time 5 "$url" 2>/dev/null) || true

    if [[ "$http_code" == "200" ]]; then
        echo "  [OK]   $name ($url)"
    else
        echo "  [FAIL] $name ($url) - HTTP $http_code"
        STATUS=1
    fi
}

check_port() {
    local host=$1
    local port=$2
    local name=$3

    if nc -z -w5 "$host" "$port" 2>/dev/null; then
        echo "  [OK]   $name ($host:$port)"
    else
        echo "  [FAIL] $name ($host:$port)"
        STATUS=1
    fi
}

echo "=== Health Check Report ==="
echo "Time: $(date)"
echo ""

# Check HTTP endpoints
check_http "http://localhost:3000/health" "Node.js API"

# Check ports
check_port "localhost" 5432 "PostgreSQL"
check_port "localhost" 6379 "Redis"

# Check disk space
DISK_USAGE=$(df / | tail -1 | awk '{print $5}' | tr -d '%')
if [[ $DISK_USAGE -gt 90 ]]; then
    echo "  [WARN] Disk usage is at ${DISK_USAGE}%"
    STATUS=1
else
    echo "  [OK]   Disk usage: ${DISK_USAGE}%"
fi

# Check memory
MEM_AVAILABLE=$(free -m | awk '/Mem:/ {print $7}')
if [[ $MEM_AVAILABLE -lt 256 ]]; then
    echo "  [WARN] Low memory: ${MEM_AVAILABLE}MB available"
    STATUS=1
else
    echo "  [OK]   Memory available: ${MEM_AVAILABLE}MB"
fi

echo ""
if [[ $STATUS -eq 0 ]]; then
    echo "All checks passed!"
else
    echo "Some checks FAILED!"
fi

exit $STATUS

Script Safety Patterns

Input Validation

#!/bin/bash
set -euo pipefail

# Require arguments
if [[ $# -lt 1 ]]; then
    echo "Usage: $0 <environment>"
    echo "Environments: staging, production"
    exit 1
fi

ENVIRONMENT=$1

# Validate input
case "$ENVIRONMENT" in
    staging|production)
        echo "Deploying to $ENVIRONMENT"
        ;;
    *)
        echo "ERROR: Invalid environment '$ENVIRONMENT'"
        echo "Must be 'staging' or 'production'"
        exit 1
        ;;
esac

Confirmation Prompts

if [[ "$ENVIRONMENT" == "production" ]]; then
    read -r -p "Deploy to PRODUCTION? Type 'yes' to confirm: " confirm
    if [[ "$confirm" != "yes" ]]; then
        echo "Deployment cancelled."
        exit 0
    fi
fi

Lock Files

LOCKFILE="/tmp/deploy.lock"

if [[ -f "$LOCKFILE" ]]; then
    PID=$(cat "$LOCKFILE")
    if kill -0 "$PID" 2>/dev/null; then
        echo "Deployment already in progress (PID $PID)"
        exit 1
    else
        echo "Stale lock file found, removing"
        rm "$LOCKFILE"
    fi
fi

echo $$ > "$LOCKFILE"
trap 'rm -f "$LOCKFILE"' EXIT

Debugging Shell Scripts

# Run with debug output
bash -x deploy.sh

# Enable debug in script
set -x          # Turn on debug
# ... commands ...
set +x          # Turn off debug

# Print commands as they execute (at top of script)
#!/bin/bash -x

Common Pitfalls

  1. Forgetting set -euo pipefail - Without this, scripts continue after errors, potentially doing damage.
  2. Not quoting variables - $VAR vs "$VAR". Unquoted variables break on spaces.
  3. Using == in [ ] vs [[ ]] - Always use [[ ]] in bash; [ ] is POSIX but less powerful.
  4. Not using local in functions - Variables leak to global scope without local.
  5. Assuming GNU tools on macOS - macOS uses BSD versions of sed, find, etc. They differ from Linux. Use Homebrew's coreutils or write portable commands.

Exercises

  1. Build a Deployment Script

    Write a bash deployment script that: (1) accepts an environment argument (staging/production), (2) validates the argument, (3) asks for confirmation if production, (4) creates a timestamped release directory, (5) runs npm ci and npm run build, (6) runs a health check, and (7) rolls back on failure.

    Hint

    Use set -euo pipefail, trap for cleanup, functions for each step, and curl for health checks.

  2. Server Setup Script

    Write a script that sets up a fresh Ubuntu server for Node.js: install Node.js (via nvm or NodeSource), install PM2 globally, create a deploy user, set up the application directory structure, and configure firewall rules (ufw).

    Hint

    Use apt-get for package installation. Check if each tool is already installed before installing. Use adduser for user creation.

  3. Log Rotation Script

    Write a script that rotates application logs: compress logs older than 1 day, delete compressed logs older than 30 days, and report the total space recovered. The script should use flock to prevent concurrent execution.

    Hint

    Use find with -mtime, gzip for compression, and stat to calculate file sizes before/after.

Common Problems & Solutions

1 Script works when run manually but fails in cron or CI
Cause: The script depends on environment variables, PATH, or working directory that are set in your interactive shell but not available in cron/CI environments.
Solution: Always use full paths to executables. Set PATH explicitly at the top of the script. Source environment files explicitly. Use `cd` to set the working directory. Never rely on ~ (use $HOME instead).

!/bin/bash\nset -euo pipefail\nexport PATH=/usr/local/bin:/usr/bin:/bin\ncd /var/www/myapp

2 Script fails with 'unexpected end of file'
Cause: Unclosed quotes, missing 'fi', 'done', or 'esac', or the file has Windows line endings (\r\n).
Solution: Check for matching if/fi, while/done, case/esac pairs. Fix line endings with `sed -i 's/\r$//' script.sh` or `dos2unix script.sh`. Use shellcheck for static analysis: `shellcheck script.sh`.
3 Variable expansion not working as expected
Cause: Using single quotes instead of double quotes. Single quotes prevent all variable expansion.
Solution: Use double quotes for strings with variables: `echo "Hello $NAME"`. Use single quotes only for literal strings. For complex cases, use ${variable} syntax for clarity.

Wrong:\necho 'Hello $NAME' # Outputs: Hello $NAME\n# Right:\necho "Hello $NAME" # Outputs: Hello World
