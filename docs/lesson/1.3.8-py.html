Dataclasses and Named Tuples

Intuition

Dataclasses are like filling out a form template: you define the fields, and Python auto-generates all the boring code (init, repr, equality). Named tuples are the same idea but immutable and tuple-like.

Dataclasses and named tuples reduce boilerplate for classes that primarily hold data. They auto-generate __init__, __repr__, __eq__, and more.

from dataclasses import dataclass, field
from typing import NamedTuple

@dataclass
class User:
    name: str
    email: str
    age: int = 0
    tags: list = field(default_factory=list)

    @property
    def is_adult(self) -> bool:
        return self.age >= 18

# Frozen (immutable) dataclass
@dataclass(frozen=True)
class Point:
    x: float
    y: float

# Named tuple (immutable, lighter weight)
class Color(NamedTuple):
    red: int
    green: int
    blue: int

u = User("Alice", "alice@test.com", 30)
p = Point(1.0, 2.0)
c = Color(255, 128, 0)
print(u)  # User(name='Alice', email='alice@test.com', age=30, tags=[])
print(c.red, c[0])  # 255 255 (both attribute and index access)
  • Use @dataclass for mutable data containers with auto-generated methods
  • Use field(default_factory=list) for mutable defaults (never tags: list = [])
  • frozen=True makes instances immutable and hashable
  • NamedTuples are immutable, support indexing, and use less memory than dataclasses

Exercises

  1. Config Dataclass

    Create a @dataclass for app configuration with host, port (default 8080), debug (default False), and allowed_origins (default empty list). Add a url property.

    Hint

    Use field(default_factory=list) for allowed_origins. Property returns f'http://{self.host}:{self.port}'.

  2. Named Tuple vs Dataclass

    Implement the same Coordinate(lat, lon) as both a NamedTuple and a frozen dataclass. Compare their behavior with indexing, equality, and as dict keys.

    Hint

    NamedTuple supports c[0]; frozen dataclass doesn't. Both are hashable.

Common Problems & Solutions

1 ValueError: mutable default for field tags is not allowed
Cause: Using `tags: list = []` in a dataclass would share the same list across instances.
Solution: Use `tags: list = field(default_factory=list)`.
2 FrozenInstanceError: cannot assign to field
Cause: Trying to modify an attribute of a `frozen=True` dataclass.
Solution: Use `dataclasses.replace(obj, field=new_value)` to create a modified copy.
