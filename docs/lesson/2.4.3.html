Async I/O: aiohttp, aiofiles

Intuition

aiohttp and aiofiles are async wrappers that make network and file operations play nicely with the event loop. Without them, synchronous I/O would block all other coroutines.

Async I/O: aiohttp and aiofiles

aiohttp and aiofiles provide non-blocking HTTP requests and file operations for use with asyncio, enabling truly concurrent I/O.

import asyncio
import aiohttp
import aiofiles

async def fetch_url(session: aiohttp.ClientSession, url: str) -> str:
    async with session.get(url) as response:
        return await response.text()

async def fetch_many(urls: list[str]) -> list[str]:
    async with aiohttp.ClientSession() as session:
        tasks = [fetch_url(session, url) for url in urls]
        return await asyncio.gather(*tasks)

async def write_results(filename: str, data: str) -> None:
    async with aiofiles.open(filename, 'w') as f:
        await f.write(data)

async def read_file(filename: str) -> str:
    async with aiofiles.open(filename, 'r') as f:
        return await f.read()

async def main():
    urls = [
        "https://httpbin.org/get",
        "https://httpbin.org/ip",
    ]
    results = await fetch_many(urls)
    await write_results("output.txt", "\n".join(results))
    content = await read_file("output.txt")
    print(f"Read {len(content)} characters")

asyncio.run(main())

Key Takeaways: - Reuse a single aiohttp.ClientSession for connection pooling - Always use async with for sessions and responses (ensures cleanup) - aiofiles wraps file I/O to prevent blocking the event loop - Combine aiohttp + aiofiles for pipelines that fetch and store data - Set timeouts: aiohttp.ClientTimeout(total=30) to prevent hanging

Common Problems & Solutions

1 `aiohttp.ClientConnectorError: Cannot connect to host` when making many concurrent requests.
Cause: Too many simultaneous connections overwhelm the server or exhaust local sockets.
Solution: Use `aiohttp.TCPConnector(limit=50)` to cap concurrent connections, and add `asyncio.Semaphore` for rate limiting.
2 `RuntimeError: Session is closed` when reusing an aiohttp session.
Cause: The session was used outside its `async with` block, or the block exited before tasks completed.
Solution: Ensure all requests complete before the `async with` block exits. Use `gather()` inside the context manager.
