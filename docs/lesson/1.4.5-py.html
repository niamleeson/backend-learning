collections Module: Counter, defaultdict, OrderedDict, deque

Intuition

The collections module gives you battle-tested data structures that eliminate the 'check-then-initialize' pattern and provide O(1) operations where lists would be O(n).

The collections module provides specialized container types that solve common patterns more efficiently than plain dicts and lists.

from collections import Counter, defaultdict, OrderedDict, deque

# Counter: count occurrences
words = ["apple", "banana", "apple", "cherry", "banana", "apple"]
counts = Counter(words)
print(counts.most_common(2))  # [('apple', 3), ('banana', 2)]

# defaultdict: auto-initialize missing keys
graph = defaultdict(list)
graph["a"].append("b")  # no KeyError

# deque: O(1) append/pop from both ends
dq = deque([1, 2, 3], maxlen=5)
dq.appendleft(0)       # deque([0, 1, 2, 3])
dq.rotate(1)            # deque([3, 0, 1, 2])
  • Counter supports arithmetic: c1 + c2 merges counts, c1 - c2 subtracts them.
  • defaultdict(factory) calls factory() for missing keys — eliminates boilerplate setdefault calls.
  • deque is ideal for queues and sliding windows; list insert(0, x) is O(n) but deque.appendleft is O(1).
  • OrderedDict retains insertion order (redundant since 3.7) but still useful for move_to_end() and equality semantics.

Exercises

    Write a function top_k(text, k) that returns the k most frequent words in a string using Counter.

    Hint

    Hint

Common Problems & Solutions

1 `defaultdict` silently creates entries for any key access, inflating the dict.
Cause: Even a read like `d[key]` triggers the default factory if the key is missing.
Solution: Use `d.get(key)` or `key in d` for lookups that should not create entries.
2 `deque` does not support efficient random access by index.
Cause: Deque is implemented as a doubly-linked list of blocks; indexed access is O(n).
Solution: Use a list if you need frequent indexed access; use deque only for queue/stack patterns.
