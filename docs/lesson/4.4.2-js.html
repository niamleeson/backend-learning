JWT Strategy and Auth Module

Intuition

JWT auth is like a theme park wristband. At the entrance (login), they check your ticket (credentials) and give you a wristband (JWT). For every ride (API endpoint), you just show your wristband -- no need to re-check the ticket. But wristbands expire at closing time (token expiry), and if yours is fake (invalid signature), security (JwtStrategy) catches it immediately.

JWT Strategy and Auth Module

Introduction

JSON Web Tokens (JWT) are the most common authentication mechanism for stateless APIs. Unlike session-based auth, JWTs are self-contained -- they carry all the information needed to authenticate a user without querying a database on every request. This lesson covers building a complete JWT authentication module in NestJS.

JWT Fundamentals

A JWT consists of three parts separated by dots:

JWT Strategy

// auth/strategies/jwt.strategy.ts
import { Injectable, UnauthorizedException } from '@nestjs/common';
import { PassportStrategy } from '@nestjs/passport';
import { ExtractJwt, Strategy } from 'passport-jwt';
import { ConfigService } from '@nestjs/config';
import { UsersService } from '../../users/users.service';

@Injectable()
export class JwtStrategy extends PassportStrategy(Strategy) {
  constructor(
    private config: ConfigService,
    private usersService: UsersService,
  ) {
    super({
      jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(),
      ignoreExpiration: false,
      secretOrKey: config.get<string>('JWT_SECRET'),
    });
  }

  async validate(payload: JwtPayload): Promise<any> {
    // payload is the decoded JWT (after verification)
    // This method is called ONLY if the token is valid

    // Optional: Check if user still exists
    const user = await this.usersService.findById(payload.sub);
    if (!user) {
      throw new UnauthorizedException('User no longer exists');
    }

    // Optional: Check if user is active
    if (!user.isActive) {
      throw new UnauthorizedException('Account is deactivated');
    }

    // What you return here becomes request.user
    return {
      id: payload.sub,
      email: payload.email,
      roles: payload.roles,
    };
  }
}

// Types
export interface JwtPayload {
  sub: string;       // User ID
  email: string;
  roles: string[];
  iat: number;       // Issued at
  exp: number;       // Expires at
}

Complete Auth Module

// auth/auth.module.ts
import { Module } from '@nestjs/common';
import { JwtModule } from '@nestjs/jwt';
import { PassportModule } from '@nestjs/passport';
import { ConfigModule, ConfigService } from '@nestjs/config';
import { AuthService } from './auth.service';
import { AuthController } from './auth.controller';
import { JwtStrategy } from './strategies/jwt.strategy';
import { LocalStrategy } from './strategies/local.strategy';
import { UsersModule } from '../users/users.module';

@Module({
  imports: [
    UsersModule,
    PassportModule.register({ defaultStrategy: 'jwt' }),
    JwtModule.registerAsync({
      imports: [ConfigModule],
      useFactory: async (config: ConfigService) => ({
        secret: config.get<string>('JWT_SECRET'),
        signOptions: {
          expiresIn: config.get<string>('JWT_EXPIRES_IN', '15m'),
        },
      }),
      inject: [ConfigService],
    }),
  ],
  controllers: [AuthController],
  providers: [AuthService, JwtStrategy, LocalStrategy],
  exports: [AuthService, JwtModule],
})
export class AuthModule {}

Auth Service with Token Generation

// auth/auth.service.ts
import { Injectable, UnauthorizedException, ConflictException } from '@nestjs/common';
import { JwtService } from '@nestjs/jwt';
import { ConfigService } from '@nestjs/config';
import { UsersService } from '../users/users.service';
import * as bcrypt from 'bcrypt';

@Injectable()
export class AuthService {
  constructor(
    private usersService: UsersService,
    private jwtService: JwtService,
    private config: ConfigService,
  ) {}

  async validateUser(email: string, password: string) {
    const user = await this.usersService.findByEmail(email);
    if (!user) return null;

    const isValid = await bcrypt.compare(password, user.password);
    if (!isValid) return null;

    const { password: _, ...result } = user;
    return result;
  }

  async login(user: any) {
    const tokens = await this.generateTokens(user);

    return {
      ...tokens,
      user: {
        id: user.id,
        email: user.email,
        name: user.name,
        roles: user.roles,
      },
    };
  }

  async register(dto: RegisterDto) {
    const existing = await this.usersService.findByEmail(dto.email);
    if (existing) {
      throw new ConflictException('Email already registered');
    }

    const hashedPassword = await bcrypt.hash(dto.password, 12);
    const user = await this.usersService.create({
      ...dto,
      password: hashedPassword,
    });

    return this.login(user);
  }

  private async generateTokens(user: any) {
    const payload: JwtPayload = {
      sub: user.id,
      email: user.email,
      roles: user.roles || ['user'],
    };

    const [accessToken, refreshToken] = await Promise.all([
      this.jwtService.signAsync(payload),
      this.jwtService.signAsync(payload, {
        secret: this.config.get('JWT_REFRESH_SECRET'),
        expiresIn: '7d',
      }),
    ]);

    return { accessToken, refreshToken };
  }
}

Auth Controller

// auth/auth.controller.ts
import {
  Controller, Post, Body, UseGuards, Get, Request,
  HttpCode, HttpStatus,
} from '@nestjs/common';
import { AuthService } from './auth.service';
import { LocalAuthGuard } from './guards/local-auth.guard';
import { JwtAuthGuard } from './guards/jwt-auth.guard';
import { Public } from './decorators/public.decorator';
import { CurrentUser } from './decorators/current-user.decorator';
import { RegisterDto } from './dto/register.dto';
import { LoginDto } from './dto/login.dto';

@Controller('auth')
export class AuthController {
  constructor(private authService: AuthService) {}

  @Public()
  @Post('register')
  async register(@Body() dto: RegisterDto) {
    return this.authService.register(dto);
  }

  @Public()
  @UseGuards(LocalAuthGuard)
  @Post('login')
  @HttpCode(HttpStatus.OK)
  async login(@Request() req: any) {
    return this.authService.login(req.user);
  }

  @UseGuards(JwtAuthGuard)
  @Get('profile')
  getProfile(@CurrentUser() user: any) {
    return user;
  }

  @UseGuards(JwtAuthGuard)
  @Post('change-password')
  async changePassword(
    @CurrentUser('id') userId: string,
    @Body() dto: ChangePasswordDto,
  ) {
    return this.authService.changePassword(userId, dto);
  }
}

DTOs

// auth/dto/register.dto.ts
import { IsEmail, IsString, MinLength, MaxLength } from 'class-validator';

export class RegisterDto {
  @IsString()
  @MinLength(2)
  @MaxLength(50)
  name: string;

  @IsEmail()
  email: string;

  @IsString()
  @MinLength(8)
  password: string;
}

// auth/dto/login.dto.ts
export class LoginDto {
  @IsEmail()
  email: string;

  @IsString()
  password: string;
}

Global JWT Auth Guard

Protect all routes by default:

// auth/guards/jwt-auth.guard.ts
import { ExecutionContext, Injectable, UnauthorizedException } from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import { AuthGuard } from '@nestjs/passport';
import { IS_PUBLIC_KEY } from '../decorators/public.decorator';

@Injectable()
export class JwtAuthGuard extends AuthGuard('jwt') {
  constructor(private reflector: Reflector) {
    super();
  }

  canActivate(context: ExecutionContext) {
    const isPublic = this.reflector.getAllAndOverride<boolean>(IS_PUBLIC_KEY, [
      context.getHandler(),
      context.getClass(),
    ]);

    if (isPublic) return true;

    return super.canActivate(context);
  }

  handleRequest(err: any, user: any, info: any) {
    if (err || !user) {
      throw err || new UnauthorizedException(
        info?.name === 'TokenExpiredError'
          ? 'Token has expired'
          : 'Invalid authentication token',
      );
    }
    return user;
  }
}

Register globally:

// app.module.ts
@Module({
  imports: [AuthModule, UsersModule],
  providers: [
    {
      provide: APP_GUARD,
      useClass: JwtAuthGuard,
    },
  ],
})
export class AppModule {}

Token Extraction Options

// From Authorization header (most common)
ExtractJwt.fromAuthHeaderAsBearerToken()

// From query parameter
ExtractJwt.fromUrlQueryParameter('token')

// From cookie
jwtFromRequest: (req) => req.cookies?.access_token

// From custom header
ExtractJwt.fromHeader('x-auth-token')

// Try multiple sources
jwtFromRequest: ExtractJwt.fromExtractors([
  ExtractJwt.fromAuthHeaderAsBearerToken(),
  (req) => req.cookies?.access_token,
])

Asymmetric Keys (RS256)

For production, consider using RSA key pairs:

import { readFileSync } from 'fs';

@Module({
  imports: [
    JwtModule.registerAsync({
      useFactory: (config: ConfigService) => ({
        privateKey: readFileSync(config.get('JWT_PRIVATE_KEY_PATH')),
        publicKey: readFileSync(config.get('JWT_PUBLIC_KEY_PATH')),
        signOptions: {
          algorithm: 'RS256',
          expiresIn: '15m',
        },
      }),
      inject: [ConfigService],
    }),
  ],
})
export class AuthModule {}

// In JwtStrategy:
super({
  jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(),
  secretOrKey: readFileSync(config.get('JWT_PUBLIC_KEY_PATH')),
  algorithms: ['RS256'],
});

RS256 allows services to verify tokens without knowing the private signing key.

Common Pitfalls

  1. Short-lived tokens without refresh -- Access tokens should expire quickly (15 min). Without refresh tokens, users must re-login constantly.
  2. Storing tokens in localStorage -- Vulnerable to XSS attacks. Consider httpOnly cookies for web apps.
  3. Including sensitive data in JWT payload -- JWTs are base64-encoded, not encrypted. Anyone can read the payload. Never include passwords or sensitive PII.
  4. Not checking user existence in validate() -- A valid token for a deleted user should be rejected.

Exercises

  1. Build a Complete JWT Auth Module

    Create a full authentication module with: register (with email uniqueness check), login (with JWT generation), profile endpoint (protected), and change-password endpoint. Include proper DTOs with validation, bcrypt password hashing, and error handling.

    Hint

    Follow the module structure in this lesson. Use bcrypt with a salt round of 12. Generate both access and refresh tokens. Return user info (without password) on login.

  2. Implement Global Auth with Public Routes

    Register JwtAuthGuard globally using APP_GUARD. Create a @Public() decorator to mark open routes. Verify that: /auth/login and /auth/register are public, /users endpoints require auth, and /health is public.

    Hint

    Use SetMetadata for the @Public() decorator. In the guard's canActivate(), use Reflector to check for the public metadata before calling super.canActivate().

  3. Add Token Extraction from Cookies

    Modify the JWT strategy to extract tokens from httpOnly cookies as a fallback when no Bearer token is present. Set the cookie on login. Include CSRF protection for cookie-based auth.

    Hint

    Use a custom extractor function that checks cookies: (req) => req.cookies?.access_token. Set the cookie with res.cookie('access_token', token, { httpOnly: true, secure: true }). Use the csurf package for CSRF protection.

Common Problems & Solutions

1 Token expired error even with a fresh token
Cause: Server time is out of sync, or the JWT_EXPIRES_IN value is too short (e.g., '1s' instead of '1h').
Solution: Check the expiresIn value in JwtModule configuration. Ensure server clock is synchronized (use NTP). Decode the token at jwt.io to inspect the exp claim.
2 JwtStrategy validate() is never called
Cause: The token is invalid or expired, so Passport rejects it before calling validate(). Or the JwtStrategy is not registered as a provider.
Solution: 1) Add JwtStrategy to the module's providers. 2) Verify the JWT_SECRET matches between token signing and strategy. 3) Check that the token format is 'Bearer ' in the Authorization header.
3 Different JWT secret between auth service and strategy
Cause: JwtModule.register() and JwtStrategy use different values for the secret, causing token verification to fail.
Solution: Use ConfigService in both places to ensure the same secret. Better yet, use JwtModule.registerAsync() and inject ConfigService in the strategy.
