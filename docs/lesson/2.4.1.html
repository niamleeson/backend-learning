The Event Loop and Coroutines

Intuition

The event loop is like a restaurant host managing one waiter. The waiter (event loop) takes orders from multiple tables (coroutines) and serves whoever is ready next, never standing idle.

Event Loop and Coroutines

Asyncio uses an event loop to run coroutines cooperatively. Coroutines are defined with async def and yield control with await, enabling concurrent I/O without threads.

import asyncio

async def fetch_data(name: str, delay: float) -> str:
    print(f"Start fetching {name}")
    await asyncio.sleep(delay)  # Non-blocking wait
    print(f"Done fetching {name}")
    return f"{name}_data"

async def main():
    # Coroutines run concurrently when awaited together
    results = await asyncio.gather(
        fetch_data("users", 2),
        fetch_data("posts", 1),
        fetch_data("comments", 1.5),
    )
    print(results)  # ['users_data', 'posts_data', 'comments_data']

asyncio.run(main())  # Entry point: starts the event loop

# Understanding the event loop
async def show_loop():
    loop = asyncio.get_running_loop()
    print(f"Loop running: {loop.is_running()}")
    # Schedule a callback
    loop.call_later(1.0, lambda: print("Callback fired"))
    await asyncio.sleep(2)

Key Takeaways: - async def creates a coroutine function; calling it returns a coroutine object - await suspends the coroutine and lets the event loop run other tasks - asyncio.run() creates an event loop, runs the coroutine, and cleans up - The event loop is single-threaded: only one coroutine runs at a time - Coroutines are concurrent (interleaved) but not parallel (simultaneous)

Exercises

  1. Sequential vs Concurrent

    Write two versions of a function that makes 5 async calls: one that awaits them sequentially (one by one) and one that uses asyncio.gather. Time both and compare.

    Hint

    Sequential: for url in urls: await fetch(url). Concurrent: await asyncio.gather(*[fetch(url) for url in urls]). The concurrent version should be ~5x faster.

  2. Custom Event Loop Callbacks

    Use loop.call_later and loop.call_soon to schedule 3 callbacks with different delays. Observe the execution order and explain why it differs from submission order.

    Hint

    Get the loop with asyncio.get_running_loop(). call_soon runs on the next iteration, call_later(delay, fn) runs after delay seconds.

Common Problems & Solutions

1 `RuntimeError: This event loop is already running` when calling `asyncio.run()` inside async code.
Cause: You cannot start a new event loop when one is already running (e.g., in Jupyter notebooks or nested async).
Solution: Use `await` directly in existing async code. In Jupyter, use `await main()` directly. For legacy code, use `nest_asyncio` package.
2 Coroutine was never awaited: `RuntimeWarning: coroutine 'func' was never awaited`.
Cause: Calling `async_func()` without `await` creates a coroutine object but never executes it.
Solution: Always `await` coroutine calls: `result = await async_func()`. Or wrap in a task: `asyncio.create_task(async_func())`.
