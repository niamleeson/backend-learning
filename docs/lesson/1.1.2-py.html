Variables, Constants, and Naming Conventions

Intuition

Variables are like labeled sticky notes on objects. Reassigning a variable just moves the sticky note to a different object; it doesn't change the original object.

Variables in Python are names that reference objects in memory. Following consistent naming conventions makes your code readable and professional.

# Variables are dynamically typed
user_name = "Alice"        # snake_case for variables
MAX_RETRIES = 5            # UPPER_SNAKE for constants
_internal_cache = {}       # leading underscore = private by convention

# Multiple assignment
x, y, z = 1, 2, 3
a = b = c = 0

# Type hints (optional but recommended)
count: int = 10
name: str = "Bob"
is_active: bool = True
  • Python has no true constants; UPPER_CASE is a convention signaling "do not modify"
  • Use descriptive names: user_count over uc, is_valid over flag
  • Avoid shadowing built-ins like list, str, id, type
  • Leading double underscore triggers name mangling in classes

Exercises

  1. Naming Convention Practice

    Refactor these variable names to follow PEP 8: MyVariable, SOME_value, class, listOfItems. Assign each a sensible value.

    Hint

    Use snake_case for regular variables, UPPER_SNAKE for constants, and avoid reserved keywords.

  2. Swap Without a Temp Variable

    Given a = 10 and b = 20, swap their values using Python's tuple unpacking. Print both before and after.

    Hint

    Python allows a, b = b, a in a single statement.

Common Problems & Solutions

1 SyntaxError: invalid syntax when naming a variable `class`
Cause: `class` is a reserved keyword in Python and cannot be used as a variable name.
Solution: Use a different name like `klass`, `class_name`, or `cls`.
2 Accidentally overwriting a built-in function like `list`
Cause: Assigning `list = [1, 2, 3]` shadows the built-in `list()` constructor.
Solution: Use descriptive names like `my_list` or `items` instead. Use `del list` to restore the built-in.
