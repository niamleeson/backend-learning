Indexes, Schema Design, and pymongo

Intuition

MongoDB schema design is about co-locating data that is read together. Embed for speed, reference for flexibility. Indexes make or break query performance just like in SQL.

MongoDB: Indexes, Schema Design, and pymongo

Proper indexing and schema design are critical for MongoDB performance. pymongo provides the full driver API for Python applications.

from pymongo import MongoClient, ASCENDING, DESCENDING

db = MongoClient()['myapp']

# Create indexes
db.users.create_index('email', unique=True)
db.users.create_index([('status', ASCENDING), ('created_at', DESCENDING)])

# Text index for search
db.articles.create_index([('title', 'text'), ('body', 'text')])
results = db.articles.find({'$text': {'$search': 'python mongodb'}})

# TTL index for auto-expiry
db.sessions.create_index('expires_at', expireAfterSeconds=0)

# Schema design: embed vs reference
# Embed: data accessed together, 1:few
{'name': 'Alice', 'addresses': [
    {'street': '123 Main', 'city': 'Seattle'}
]}
# Reference: many:many, large sub-documents
{'title': 'My Post', 'author_id': ObjectId('...')}

# Bulk operations
from pymongo import InsertOne, UpdateOne
ops = [InsertOne({'x': 1}), UpdateOne({'x': 1}, {'$inc': {'x': 1}})]
db.col.bulk_write(ops, ordered=False)

Key Takeaways: - Create indexes on fields used in queries, sorts, and unique constraints. - Embed related data when it is always accessed together. Reference when data is shared or large. - TTL indexes auto-delete expired documents, perfect for sessions and temporary data.

Exercises

  1. Design an E-Commerce Schema

    Design MongoDB collections for products, orders, and users. Decide which data to embed vs reference.

    Hint

    Embed order_items in orders (always read together). Reference user_id in orders (user data changes independently).

  2. Index Optimization

    Given slow queries on a collection, use explain() to identify missing indexes and create appropriate compound indexes.

    Hint

    collection.find({...}).explain() shows queryPlanner. Look for COLLSCAN (no index) vs IXSCAN (index used).

Common Problems & Solutions

1 Document exceeds 16MB BSON size limit.
Cause: Embedding too many items in an array within a single document.
Solution: Use the bucket pattern or reference pattern. Store large arrays in a separate collection.
2 Query does not use the expected compound index.
Cause: Query fields do not match the index prefix. A compound index (a, b, c) only helps queries starting with a.
Solution: Order compound index fields by equality filters first, then sort, then range filters (ESR rule).
