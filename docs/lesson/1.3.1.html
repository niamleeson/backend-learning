Classes and Objects

Intuition

A class is like a cookie cutter and objects are the cookies. The cutter defines the shape (attributes and methods), but each cookie can have different decorations (attribute values).

Classes are blueprints for creating objects that bundle data (attributes) and behavior (methods). Python's class system is flexible yet straightforward.

class Dog:
    species = "Canis familiaris"  # class attribute

    def __init__(self, name: str, age: int):
        self.name = name          # instance attribute
        self.age = age

    def speak(self) -> str:
        return f"{self.name} says Woof!"

    def __repr__(self) -> str:
        return f"Dog(name={self.name!r}, age={self.age})"

# Creating instances
rex = Dog("Rex", 5)
buddy = Dog("Buddy", 3)

print(rex.speak())          # "Rex says Woof!"
print(rex.species)          # "Canis familiaris" (shared)
print(Dog.species)          # same, accessed via class
  • __init__ initializes instance attributes; it's not a constructor (__new__ is)
  • self refers to the current instance; it's passed automatically
  • Class attributes are shared across all instances; instance attributes are per-object
  • Everything in Python is an object, including classes themselves

Common Problems & Solutions

1 TypeError: __init__() takes 1 positional argument but 2 were given
Cause: Forgot to include `self` as the first parameter in `__init__`.
Solution: Always define methods with `self` as the first parameter: `def __init__(self, name):`.
2 Mutable class attribute shared unexpectedly across instances
Cause: A list or dict as a class attribute is shared by all instances; mutating it affects all.
Solution: Initialize mutable attributes in `__init__` as instance attributes: `self.items = []`.
