Prisma with NestJS: PrismaService and Module

Intuition

If TypeORM is like hand-writing SQL on typed paper, Prisma is like using a smart form that auto-completes and checks your entries. The schema.prisma file is the master template, and prisma generate creates a perfectly typed assistant (PrismaClient) that knows every field, relation, and query possible in your database.

Prisma with NestJS: PrismaService and Module

Introduction

Prisma is a modern, type-safe ORM with an excellent developer experience. Unlike TypeORM which uses decorators on classes, Prisma uses a declarative schema file to generate a fully typed client. The generated PrismaClient provides autocompletion, type checking, and query building that catches errors at compile time rather than runtime.

Installation

npm i prisma @prisma/client
npx prisma init

This creates prisma/schema.prisma and a .env file.

Prisma Schema

// prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        String   @id @default(uuid())
  email     String   @unique
  name      String
  password  String
  role      Role     @default(USER)
  isActive  Boolean  @default(true)
  posts     Post[]
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@index([email])
}

model Post {
  id        String   @id @default(uuid())
  title     String
  content   String
  published Boolean  @default(false)
  author    User     @relation(fields: [authorId], references: [id], onDelete: Cascade)
  authorId  String
  tags      Tag[]
  createdAt DateTime @default(now())

  @@index([authorId])
}

model Tag {
  id    String @id @default(uuid())
  name  String @unique
  posts Post[]
}

enum Role {
  USER
  ADMIN
  MODERATOR
}

Create PrismaService

// prisma/prisma.service.ts
import { Injectable, OnModuleInit, OnModuleDestroy } from '@nestjs/common';
import { PrismaClient } from '@prisma/client';

@Injectable()
export class PrismaService
  extends PrismaClient
  implements OnModuleInit, OnModuleDestroy
{
  constructor() {
    super({
      log: process.env.NODE_ENV === 'development'
        ? ['query', 'info', 'warn', 'error']
        : ['error'],
    });
  }

  async onModuleInit() {
    await this.$connect();
  }

  async onModuleDestroy() {
    await this.$disconnect();
  }
}

Using PrismaService in a Service

// users/users.service.ts
import { Injectable, NotFoundException, ConflictException } from '@nestjs/common';
import { PrismaService } from '../prisma/prisma.service';
import { Prisma } from '@prisma/client';

@Injectable()
export class UsersService {
  constructor(private prisma: PrismaService) {}

  async findAll(params: {
    page?: number;
    limit?: number;
    search?: string;
    role?: string;
  }) {
    const { page = 1, limit = 20, search, role } = params;

    const where: Prisma.UserWhereInput = {
      ...(search && {
        OR: [
          { name: { contains: search, mode: 'insensitive' } },
          { email: { contains: search, mode: 'insensitive' } },
        ],
      }),
      ...(role && { role: role as any }),
    };

    const [data, total] = await Promise.all([
      this.prisma.user.findMany({
        where,
        skip: (page - 1) * limit,
        take: limit,
        orderBy: { createdAt: 'desc' },
        select: {
          id: true,
          name: true,
          email: true,
          role: true,
          isActive: true,
          createdAt: true,
          _count: { select: { posts: true } },
        },
      }),
      this.prisma.user.count({ where }),
    ]);

    return {
      data,
      meta: { total, page, limit, totalPages: Math.ceil(total / limit) },
    };
  }

  async findOne(id: string) {
    const user = await this.prisma.user.findUnique({
      where: { id },
      include: {
        posts: {
          where: { published: true },
          orderBy: { createdAt: 'desc' },
          take: 10,
        },
      },
    });

    if (!user) throw new NotFoundException(`User #${id} not found`);
    return user;
  }

  async create(data: Prisma.UserCreateInput) {
    try {
      return await this.prisma.user.create({
        data,
        select: { id: true, name: true, email: true, role: true },
      });
    } catch (error) {
      if (error instanceof Prisma.PrismaClientKnownRequestError) {
        if (error.code === 'P2002') {
          throw new ConflictException('Email already exists');
        }
      }
      throw error;
    }
  }

  async update(id: string, data: Prisma.UserUpdateInput) {
    try {
      return await this.prisma.user.update({
        where: { id },
        data,
        select: { id: true, name: true, email: true, role: true },
      });
    } catch (error) {
      if (error instanceof Prisma.PrismaClientKnownRequestError) {
        if (error.code === 'P2025') {
          throw new NotFoundException(`User #${id} not found`);
        }
      }
      throw error;
    }
  }

  async remove(id: string) {
    try {
      await this.prisma.user.delete({ where: { id } });
    } catch (error) {
      if (error instanceof Prisma.PrismaClientKnownRequestError) {
        if (error.code === 'P2025') {
          throw new NotFoundException(`User #${id} not found`);
        }
      }
      throw error;
    }
  }
}

Prisma Error Handling

Create a reusable error handler:

// prisma/prisma-error.handler.ts
import {
  ConflictException, NotFoundException, BadRequestException,
} from '@nestjs/common';
import { Prisma } from '@prisma/client';

export function handlePrismaError(error: unknown): never {
  if (error instanceof Prisma.PrismaClientKnownRequestError) {
    switch (error.code) {
      case 'P2002':
        const field = (error.meta?.target as string[])?.join(', ');
        throw new ConflictException(`Unique constraint failed on: ${field}`);
      case 'P2003':
        throw new BadRequestException('Foreign key constraint failed');
      case 'P2025':
        throw new NotFoundException('Record not found');
      default:
        throw error;
    }
  }
  throw error;
}

Middleware (Soft Delete)

// prisma/prisma.service.ts
async onModuleInit() {
  await this.$connect();

  // Soft delete middleware
  this.$use(async (params, next) => {
    if (params.model === 'User') {
      if (params.action === 'delete') {
        params.action = 'update';
        params.args.data = { deletedAt: new Date() };
      }
      if (params.action === 'deleteMany') {
        params.action = 'updateMany';
        if (params.args.data) {
          params.args.data.deletedAt = new Date();
        } else {
          params.args.data = { deletedAt: new Date() };
        }
      }
      // Filter out soft-deleted records
      if (params.action === 'findMany' || params.action === 'findFirst') {
        if (!params.args) params.args = {};
        if (!params.args.where) params.args.where = {};
        params.args.where.deletedAt = null;
      }
    }
    return next(params);
  });
}

Common Pitfalls

  1. Forgetting npx prisma generate -- After changing schema.prisma, you must regenerate the client. Otherwise types are stale.
  2. Not handling Prisma errors -- P2002 (unique), P2025 (not found), P2003 (foreign key) should be caught and converted to HTTP exceptions.
  3. Selecting too much data -- Use select or include to limit returned fields. Don't return passwords or internal fields.
  4. N+1 queries -- Use include to eager-load relations in a single query instead of querying inside a loop.

Exercises

  1. Set Up Prisma in NestJS

    Create a PrismaModule with PrismaService (extending PrismaClient). Mark it as @Global(). Set up a schema with User, Post, and Comment models. Generate the client and implement a full CRUD service for Posts.

    Hint

    Use onModuleInit for $connect and onModuleDestroy for $disconnect. Use select/include to control response shape. Handle P2002 and P2025 errors.

  2. Build Complex Queries

    Implement a search endpoint for posts that supports: text search across title and content, filter by author, filter by published status, filter by date range, sort by multiple fields, and cursor-based pagination.

    Hint

    Use Prisma.PostWhereInput for dynamic filtering. Use cursor and skip: 1 for cursor pagination. Chain conditions with AND/OR.

  3. Prisma Error Handler Middleware

    Create a NestJS interceptor that catches all Prisma errors and converts them to appropriate HTTP exceptions. Map P2002 to 409, P2003 to 400, P2025 to 404. Include the affected field in the error message.

    Hint

    Use catchError in an RxJS interceptor. Check for PrismaClientKnownRequestError. Access error.meta for field names.

Common Problems & Solutions

1 PrismaClient types are outdated or missing
Cause: The Prisma client was not regenerated after schema changes.
Solution: Run `npx prisma generate` after every schema change. Add it as a postinstall script: `"postinstall": "prisma generate"`.
2 Connection pool exhausted under load
Cause: Too many concurrent connections or connection pool too small for the workload.
Solution: Configure the connection pool in DATABASE_URL: `?connection_limit=20`. Consider using PgBouncer in production. Ensure $disconnect is called on shutdown.
3 Prisma throws P2025 on update instead of a custom 404
Cause: Prisma's update() throws when the record doesn't exist, but the raw error isn't user-friendly.
Solution: Catch PrismaClientKnownRequestError with code P2025 and throw NotFoundException. Or use findFirst + update pattern to check existence first.
