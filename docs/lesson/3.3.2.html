Entity-Relationship Diagrams and Schema Design

Intuition

An ERD is a blueprint for your database, just as an architect draws floor plans before building. It captures what exists and how things connect.

ERD and Schema Design

Entity-Relationship Diagrams visually represent database tables, their columns, and relationships. Good schema design starts with understanding the domain.

CREATE TABLE customers (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    name VARCHAR(100) NOT NULL
);

CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    customer_id INTEGER NOT NULL REFERENCES customers(id),
    status VARCHAR(20) DEFAULT 'pending',
    created_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    price NUMERIC(10,2) NOT NULL CHECK (price > 0)
);

CREATE TABLE order_items (
    id SERIAL PRIMARY KEY,
    order_id INTEGER REFERENCES orders(id) ON DELETE CASCADE,
    product_id INTEGER REFERENCES products(id),
    quantity INTEGER CHECK (quantity > 0)
);

Key Takeaways: - Identify entities (nouns), attributes (properties), and relationships (verbs) from domain requirements. - Use junction tables for many-to-many relationships. - ON DELETE CASCADE, SET NULL, or RESTRICT controls referential integrity behavior.

Exercises

  1. Design a Blog Schema

    Draw and implement an ERD for a blog: users, posts, comments, and tags (many-to-many with posts).

    Hint

    Create users, posts (FK to users), comments (FK to posts and users), tags, and post_tags junction table.

  2. Add Soft Delete

    Modify a schema to support soft deletes instead of hard deletes, with appropriate indexes.

    Hint

    Add a deleted_at TIMESTAMPTZ column, partial index WHERE deleted_at IS NULL, and update queries to filter.

Common Problems & Solutions

1 Cannot delete a parent row due to foreign key constraint.
Cause: Child rows still reference the parent, and the default is RESTRICT.
Solution: Use ON DELETE CASCADE to auto-delete children, ON DELETE SET NULL, or delete children first.
2 Schema cannot represent a many-to-many relationship directly.
Cause: SQL tables only support one-to-many via foreign keys.
Solution: Create a junction/bridge table with foreign keys to both tables and a composite primary key.
