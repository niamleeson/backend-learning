Descriptors: __get__, __set__, __delete__

Intuition

Descriptors are like smart gatekeepers for attributes. Instead of storing a value directly, they intercept every get, set, and delete to add validation, transformation, or lazy computation.

Descriptors: get, set, delete

Descriptors are objects that customize attribute access on classes. They power property, classmethod, staticmethod, and ORMs. Any object with __get__, __set__, or __delete__ is a descriptor.

class Validated:
    """A descriptor that validates attribute values."""
    def __init__(self, min_val: float, max_val: float):
        self.min_val = min_val
        self.max_val = max_val

    def __set_name__(self, owner, name):
        self.name = name  # Automatically set by Python

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return obj.__dict__.get(self.name, 0)

    def __set__(self, obj, value):
        if not self.min_val <= value <= self.max_val:
            raise ValueError(
                f"{self.name} must be between {self.min_val} and {self.max_val}"
            )
        obj.__dict__[self.name] = value

    def __delete__(self, obj):
        del obj.__dict__[self.name]

class Product:
    price = Validated(0, 10000)
    weight = Validated(0, 500)

    def __init__(self, price: float, weight: float):
        self.price = price    # Triggers __set__
        self.weight = weight

p = Product(29.99, 1.5)  # OK
# Product(-5, 1.0)  # Raises ValueError

Key Takeaways: - Data descriptors (__get__ + __set__) take priority over instance __dict__ - Non-data descriptors (only __get__) are overridden by instance attributes - __set_name__ is called automatically when the descriptor is assigned to a class - Descriptors must be class-level attributes, not instance attributes - property is implemented as a descriptor under the hood

Exercises

  1. Type-Checked Descriptor

    Create a TypeChecked descriptor that enforces a specific type on assignment. Use it in a class: name = TypeChecked(str), age = TypeChecked(int).

    Hint

    Store the expected type in __init__. In __set__, use isinstance(value, self.expected_type) and raise TypeError on mismatch.

  2. Lazy Property Descriptor

    Write a LazyProperty descriptor that computes a value on first access, caches it in the instance dict, and returns the cached value on subsequent access.

    Hint

    In __get__, check if the value exists in obj.__dict__. If not, compute it and store it. The cached value then shadows the descriptor.

Common Problems & Solutions

1 Descriptor's `__set__` is never called; values go directly to instance `__dict__`.
Cause: The descriptor is stored as an instance attribute instead of a class attribute.
Solution: Define descriptors at the class level: `class Foo: attr = MyDescriptor()`. Not in `__init__`.
2 All instances share the same descriptor value unexpectedly.
Cause: The descriptor stores the value on itself instead of on the instance.
Solution: Store values in `obj.__dict__[self.name]`, not in `self.value`. Use `__set_name__` to get the attribute name.
