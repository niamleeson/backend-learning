Profiling: cProfile, line_profiler, py-spy

Intuition

Profiling is like a time audit for your code. cProfile gives you the company-wide view of where time goes, line_profiler zooms into a single department, and py-spy lets you audit without interrupting work.

Profiling: cProfile, line_profiler, and py-spy

Profiling identifies performance bottlenecks by measuring where your program spends time. Start with cProfile for overview, then drill down with line_profiler or py-spy.

import cProfile
import pstats

def slow_function():
    total = 0
    for i in range(1_000_000):
        total += i ** 0.5
    return total

def main():
    result = slow_function()
    data = [i for i in range(500_000)]
    sorted_data = sorted(data, reverse=True)

# cProfile from code
profiler = cProfile.Profile()
profiler.enable()
main()
profiler.disable()

stats = pstats.Stats(profiler)
stats.sort_stats('cumulative')
stats.print_stats(10)  # Top 10 functions

# Command line: python -m cProfile -s cumulative script.py

# line_profiler (pip install line_profiler)
# @profile  # decorator for line_profiler
# def slow_function():
#     ...
# Run: kernprof -l -v script.py

# py-spy: attach to running process
# py-spy top --pid 12345
# py-spy record -o profile.svg --pid 12345

Key Takeaways: - cProfile is built-in and low-overhead; use for initial profiling - Sort by cumulative time to find the biggest bottlenecks - line_profiler shows time per line for targeted function analysis - py-spy attaches to running processes with zero code changes - Always profile before optimizing; do not guess at bottlenecks

Exercises

  1. Profile and Optimize

    Write a deliberately slow function (nested loops, repeated string concatenation). Profile it with cProfile, identify the bottleneck, and optimize it. Compare before/after profiles.

    Hint

    String concatenation in a loop is O(n^2). Replace with ''.join(parts). Use stats.sort_stats('tottime') to find the hotspot.

  2. Flame Graph with py-spy

    Run a Python script and generate a flame graph SVG using py-spy. Identify the tallest stack (deepest call chain) and the widest bar (most time spent).

    Hint

    Install py-spy with pip install py-spy. Run py-spy record -o flame.svg -- python your_script.py. Open the SVG in a browser.

Common Problems & Solutions

1 cProfile shows `` for most time-consuming entries, making it hard to identify your code.
Cause: Your code calls built-in C functions heavily, and cProfile attributes time to them.
Solution: Use `stats.print_callers()` to see which of your functions call the slow built-ins. Or use line_profiler on suspect functions.
2 Profiling overhead changes the performance characteristics of the code.
Cause: cProfile adds overhead per function call, which can distort results for code with many small function calls.
Solution: Use sampling profilers like `py-spy` which have minimal overhead and do not distort timing. Reserve cProfile for initial analysis.
