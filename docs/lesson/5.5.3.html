Database Testing with FastAPI

Intuition

Database testing with dependency overrides is like a fire drill: you swap real alarms with test ones so you can verify the evacuation procedure works without calling the actual fire department. Every test gets a fresh building.

Database Testing with FastAPI

Testing FastAPI applications with databases requires isolated test databases, proper fixture management, and dependency overrides to swap production dependencies with test ones.

import pytest
import pytest_asyncio
from httpx import AsyncClient, ASGITransport
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker

TEST_DB_URL = "sqlite+aiosqlite:///./test.db"
test_engine = create_async_engine(TEST_DB_URL)
TestSession = async_sessionmaker(test_engine, expire_on_commit=False)

@pytest_asyncio.fixture
async def db_session():
    async with test_engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    async with TestSession() as session:
        yield session
    async with test_engine.begin() as conn:
        await conn.run_sync(Base.metadata.drop_all)

@pytest_asyncio.fixture
async def client(db_session):
    def override_get_db():
        return db_session
    app.dependency_overrides[get_db] = override_get_db
    transport = ASGITransport(app=app)
    async with AsyncClient(transport=transport, base_url="http://test") as c:
        yield c
    app.dependency_overrides.clear()

@pytest.mark.asyncio
async def test_create_user(client):
    response = await client.post("/users/", json={"name": "Alice", "email": "a@b.com"})
    assert response.status_code == 201

Key Takeaways: - Use dependency_overrides to inject test database sessions - Create and drop tables per test for full isolation - httpx.AsyncClient with ASGITransport tests the full ASGI stack without a server - Use pytest-asyncio for async test support

Exercises

  1. Transactional Test Fixtures

    Create a test fixture that wraps each test in a transaction that is rolled back after the test completes, instead of creating and dropping tables. This is faster for large test suites.

    Hint

    Use await connection.begin() to start a transaction, bind the session to that connection, yield the session, then await transaction.rollback(). Override the get_db dependency with this session.

  2. Factory Boy Integration

    Set up Factory Boy (or a similar library) with async SQLAlchemy to create test data factories for your models. Write tests that use factories to generate realistic test data for complex scenarios.

    Hint

    Create a UserFactory using factory.alchemy.SQLAlchemyModelFactory with class Meta: model = User; sqlalchemy_session_persistence = 'commit'. Pass the test session to the factory.

Common Problems & Solutions

1 Test database state leaking between tests
Cause: Tests are sharing the same database session or tables are not being reset between tests.
Solution: Use the create/drop pattern in fixtures with function scope, or use transaction rollback for each test. Ensure fixtures have `scope='function'` (the default) for per-test isolation.
2 async test hanging or timing out
Cause: The event loop is not properly configured for pytest, or there is a deadlock between async fixtures and the test runner.
Solution: Add `asyncio_mode = 'auto'` in `pyproject.toml` under `[tool.pytest.ini_options]`. Ensure all async fixtures use `@pytest_asyncio.fixture` and tests use `@pytest.mark.asyncio`.
