__init_subclass__ and Dynamic Class Creation

Intuition

init_subclass is like a registration desk that every subclass must check in at. It gives the parent class a chance to inspect, modify, or record each new subclass as it is created.

init_subclass and Dynamic Class Creation

__init_subclass__ is a simpler alternative to metaclasses for customizing subclass creation. Combined with type(), you can create classes dynamically at runtime.

class Plugin:
    """Base class that auto-registers subclasses."""
    _registry: dict[str, type] = {}

    def __init_subclass__(cls, *, plugin_name: str = "", **kwargs):
        super().__init_subclass__(**kwargs)
        name = plugin_name or cls.__name__.lower()
        Plugin._registry[name] = cls
        print(f"Registered plugin: {name}")

    @classmethod
    def get_plugin(cls, name: str) -> type:
        return cls._registry[name]

class JSONPlugin(Plugin, plugin_name="json"):
    def parse(self, data: str) -> dict:
        import json
        return json.loads(data)

class XMLPlugin(Plugin, plugin_name="xml"):
    def parse(self, data: str) -> dict:
        return {"xml": data}  # simplified

# Dynamic class creation with type()
def make_model(name: str, fields: dict[str, type]) -> type:
    def init(self, **kwargs):
        for key, val in kwargs.items():
            setattr(self, key, val)
    def repr(self):
        attrs = ', '.join(f'{k}={getattr(self, k)!r}' for k in fields)
        return f"{name}({attrs})"

    namespace = {'__init__': init, '__repr__': repr}
    return type(name, (), namespace)

User = make_model('User', {'name': str, 'age': int})
u = User(name='Alice', age=30)

Key Takeaways: - __init_subclass__ is called whenever a class is subclassed - It can accept keyword arguments from the class definition line - Prefer __init_subclass__ over metaclasses for simple hooks - type(name, bases, dict) creates classes dynamically at runtime - Dynamic classes are useful for ORMs, plugin systems, and configuration-driven code

Exercises

  1. Auto-Registering Commands

    Create a Command base class using __init_subclass__ that registers each subclass by a command_name keyword. Build a dispatcher that looks up and runs commands by name.

    Hint

    Accept command_name as a keyword arg in __init_subclass__. Store in a class-level dict. The dispatcher calls Command._registry[name]().execute().

  2. Dynamic Model Factory

    Write a make_dataclass(name, fields) function that uses type() to create a class with __init__, __repr__, and __eq__ based on a dict of field names and types.

    Hint

    Generate __init__ that sets each field. __eq__ compares all field values. Or use dataclasses.make_dataclass() for the standard library approach.

Common Problems & Solutions

1 `__init_subclass__` keyword arguments cause `TypeError: __init_subclass__() got an unexpected keyword argument`.
Cause: The base class's `__init_subclass__` does not accept the keyword argument or does not pass `**kwargs` to `super()`.
Solution: Always include `**kwargs` in the signature and call `super().__init_subclass__(**kwargs)` to support multiple inheritance.
2 Dynamically created class with `type()` does not work with `isinstance` checks as expected.
Cause: The dynamic class is a new type object each time `type()` is called, even with the same name.
Solution: Cache created classes by name or parameters. Ensure the same `type()` call is not repeated for the same class definition.
