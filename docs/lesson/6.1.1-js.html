Essential Linux Commands for Developers

Intuition

Linux commands are like a Swiss Army knife for servers. Each tool does one thing well, and by combining them with pipes (like snapping tools together), you can accomplish complex tasks that would take hundreds of lines of code in a programming language.

Essential Linux Commands for Developers

Why Linux Matters for Backend Developers

As a backend developer, you will spend a significant portion of your career working with Linux servers. Whether you are deploying applications, debugging production issues, or managing infrastructure, Linux command-line proficiency is non-negotiable. Most cloud servers, Docker containers, and CI/CD environments run Linux.

The Linux File System Hierarchy

/
├── bin/        # Essential user binaries (ls, cp, mv)
├── etc/        # Configuration files
├── home/       # User home directories
├── var/        # Variable data (logs, databases)
│   ├── log/    # System and application logs
│   └── www/    # Web server files
├── tmp/        # Temporary files
├── usr/        # User programs and data
│   ├── bin/    # Most user commands
│   ├── lib/    # Libraries
│   └── local/  # Locally installed software
├── opt/        # Optional/third-party software
└── proc/       # Virtual filesystem for process info

Understanding this hierarchy helps you know where to find configuration files (/etc), logs (/var/log), and installed software (/usr/local).

Navigation and File Operations

Moving Around

# Print current directory
pwd

# Change directory
cd /var/log
cd ~           # Go to home directory
cd -           # Go to previous directory
cd ..          # Go up one level

# List files
ls             # Basic listing
ls -la         # Long format, show hidden files
ls -lah        # Human-readable file sizes
ls -lt         # Sort by modification time

File and Directory Operations

# Create directories
mkdir myproject
mkdir -p src/controllers/api    # Create nested directories

# Create files
touch server.js
echo "Hello" > greeting.txt     # Create with content

# Copy files and directories
cp source.txt dest.txt
cp -r src/ backup/              # Copy directory recursively

# Move and rename
mv old-name.js new-name.js
mv file.txt /var/data/

# Remove files and directories
rm file.txt
rm -r directory/                # Remove directory recursively
rm -rf node_modules/            # Force remove (use with caution!)

Viewing and Searching File Content

Reading Files

# Display entire file
cat server.js

# Display with line numbers
cat -n server.js

# View large files page by page
less /var/log/syslog
# In less: / to search, n for next match, q to quit

# View first/last lines
head -n 20 server.js            # First 20 lines
tail -n 50 /var/log/app.log     # Last 50 lines
tail -f /var/log/app.log        # Follow log in real-time (crucial for debugging)

Searching with grep

# Search for pattern in file
grep "error" /var/log/app.log

# Case-insensitive search
grep -i "error" /var/log/app.log

# Recursive search in directory
grep -r "TODO" src/

# Show line numbers
grep -n "function" server.js

# Show context (lines before/after match)
grep -B 3 -A 3 "Error" app.log

# Invert match (show lines NOT matching)
grep -v "DEBUG" app.log

# Count matches
grep -c "error" app.log

# Search with regex
grep -E "error|warning|critical" app.log

Finding Files with find

# Find by name
find /var/log -name "*.log"

# Find by type (f=file, d=directory)
find . -type f -name "*.js"

# Find recently modified files (last 24 hours)
find . -type f -mtime -1

# Find large files (over 100MB)
find / -type f -size +100M

# Find and execute command
find . -name "*.tmp" -exec rm {} \;

# Find and delete (simpler syntax)
find . -name "node_modules" -type d -prune -exec rm -rf {} +

Text Processing

sed - Stream Editor

# Replace first occurrence on each line
sed 's/old/new/' file.txt

# Replace all occurrences
sed 's/old/new/g' file.txt

# Edit file in place
sed -i 's/localhost/0.0.0.0/g' config.js

# Delete lines matching pattern
sed '/^#/d' config.txt          # Remove comment lines

# Replace in specific line range
sed '10,20s/foo/bar/g' file.txt

awk - Pattern Processing

# Print specific columns
awk '{print $1, $3}' access.log

# Filter by condition
awk '$9 == 500' access.log      # Lines where 9th field is 500

# Sum a column
awk '{sum += $5} END {print sum}' data.txt

# Custom field separator
awk -F',' '{print $2}' data.csv

Pipes and Redirection

Pipes are the backbone of Linux command composition.

Command A ──|pipe|──> Command B ──|pipe|──> Command C
  stdout          stdin/stdout          stdin

# Pipe output of one command to another
cat access.log | grep "POST" | wc -l

# Count unique IP addresses
cat access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -10

# Redirection
echo "log entry" > file.txt     # Overwrite
echo "another entry" >> file.txt # Append
command 2> errors.txt            # Redirect stderr
command > output.txt 2>&1        # Redirect both stdout and stderr
command &> all.txt               # Same as above (bash shortcut)

Process Management

# List running processes
ps aux                          # All processes
ps aux | grep node              # Filter for Node.js

# Interactive process viewer
top                             # Basic
htop                            # Better (if installed)

# Find process by port
lsof -i :3000                   # What is using port 3000?
netstat -tlnp                   # All listening ports
ss -tlnp                        # Modern alternative to netstat

# Kill processes
kill PID                        # Graceful (SIGTERM)
kill -9 PID                     # Force kill (SIGKILL)
killall node                    # Kill all node processes

# Run in background
node server.js &                # Run in background
nohup node server.js &          # Survives terminal close

Disk and Memory

# Disk usage
df -h                           # Filesystem usage (human-readable)
du -sh /var/log                 # Size of directory
du -sh * | sort -h              # Size of each item, sorted

# Memory usage
free -h                         # RAM usage

Networking Commands

# Test connectivity
ping google.com
curl -I https://api.example.com  # HTTP headers only
curl -X POST -H "Content-Type: application/json" \
  -d '{"key":"value"}' http://localhost:3000/api

# DNS lookup
nslookup example.com
dig example.com

# Download files
wget https://example.com/file.tar.gz
curl -O https://example.com/file.tar.gz

# Check open ports
nc -zv localhost 5432            # Check if PostgreSQL port is open

Environment Variables

# View all environment variables
env
printenv

# Set variable (current session)
export NODE_ENV=production
export PORT=3000

# Use in commands
echo $NODE_ENV

# Set for a single command
NODE_ENV=test npm test

# Persist in shell config
echo 'export NODE_ENV=production' >> ~/.bashrc
source ~/.bashrc                 # Reload config

Archiving and Compression

# Create tar archive
tar -czf backup.tar.gz ./myproject/

# Extract tar archive
tar -xzf backup.tar.gz

# List contents without extracting
tar -tzf backup.tar.gz

# Flags: c=create, x=extract, z=gzip, f=file, v=verbose

Practical Scenarios

Analyzing Node.js Application Logs

# Find all errors in the last hour
find /var/log/myapp -name "*.log" -mmin -60 -exec grep -l "ERROR" {} \;

# Count errors by type
grep "ERROR" app.log | awk -F'ERROR: ' '{print $2}' | sort | uniq -c | sort -rn

# Watch logs in real-time, filtering for errors
tail -f /var/log/myapp/app.log | grep --line-buffered "ERROR\|FATAL"

# Find which endpoint is getting the most 500 errors
grep "500" access.log | awk '{print $7}' | sort | uniq -c | sort -rn | head

Server Health Check Script

#!/bin/bash
echo "=== Server Health Check ==="
echo "Uptime: $(uptime)"
echo "Disk Usage:"
df -h / | tail -1
echo "Memory:"
free -h | grep Mem
echo "Node.js Processes:"
ps aux | grep node | grep -v grep | wc -l
echo "Port 3000:"
ss -tlnp | grep 3000 || echo "Not listening"

Common Pitfalls

  1. Running rm -rf / or rm -rf * in wrong directory - Always double-check pwd before destructive commands
  2. Forgetting quotes around variables - rm $FILE is dangerous if FILE contains spaces; use rm "$FILE"
  3. Using kill -9 as first resort - Always try kill (SIGTERM) first to allow graceful shutdown
  4. Not using tail -f for debugging - Many developers cat entire log files instead of following them
  5. Ignoring exit codes - Use $? to check if the last command succeeded (0 = success)

Exercises

  1. Log Analysis Challenge

    Create a sample access log file with 100+ lines containing various HTTP status codes (200, 301, 404, 500). Write a one-liner pipeline that outputs the top 5 most frequent status codes with their counts, sorted in descending order.

    Hint

    Use awk to extract the status code column, pipe to sort, then uniq -c, then sort -rn, then head -5.

  2. Find and Clean Build Artifacts

    Write a command that finds all node_modules directories and .next directories under your home folder, prints their sizes, and calculates the total space used. Do NOT delete them yet - just report.

    Hint

    Use find with -type d -name 'node_modules', pipe to xargs du -sh, and use awk to sum sizes.

  3. Process Detective

    Write a shell script that takes a port number as an argument and reports: (1) whether anything is listening on that port, (2) the PID and process name, (3) how much memory that process is using.

    Hint

    Use lsof -i :$PORT or ss -tlnp to find the process, then ps aux with the PID to get memory info.

  4. Environment Setup Script

    Write a bash script that checks if Node.js, npm, Docker, and Git are installed, prints their versions, and warns if any are missing. The script should exit with code 1 if any required tool is missing.

    Hint

    Use 'command -v' or 'which' to check if a command exists. Use $? to check exit codes.

Common Problems & Solutions

1 Permission denied when running a script
Cause: The script file does not have execute permissions.
Solution: Add execute permission with `chmod +x script.sh`. For directories, ensure you have execute permission to enter them (`chmod +x directory/`). Check file ownership with `ls -la`.

chmod +x deploy.sh && ./deploy.sh

2 Command not found even though the program is installed
Symptoms: bash: node: command not found
Cause: The program's binary is not in your PATH environment variable.
Solution: Find the binary location with `which program` or `find / -name program 2>/dev/null`. Add its directory to PATH: `export PATH=$PATH:/path/to/bin`. Make it permanent by adding to ~/.bashrc or ~/.zshrc.
3 Disk space full on production server
Cause: Log files, Docker images, or temporary files consuming all available space.
Solution: Identify large files/directories with `du -sh /* | sort -h`. Common culprits: `/var/log` (rotate logs), Docker (`docker system prune`), `/tmp` (clean old files). Use `find /var/log -name '*.log' -mtime +30 -delete` to remove old logs.
