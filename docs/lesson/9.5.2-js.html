Design a Ride-Sharing Service (Uber)

Intuition

Think of a ride-sharing system like a pizza delivery dispatch center. Customers call in orders (ride requests), the dispatcher (matching service) looks at a map of available drivers (delivery drivers) and picks the closest one. The map is constantly updating as drivers move around. During the Super Bowl (surge), there are way more orders than drivers, so prices go up to attract more drivers to work. The whole system is a two-sided marketplace where you must balance supply and demand in real-time across geography.

Design a Ride-Sharing Service (Uber)

Introduction

A ride-sharing service connects riders who need transportation with nearby drivers in real-time. This is one of the most challenging system design problems because it combines real-time location tracking, geospatial matching, dynamic pricing, payments, and ETA computation — all with strict latency requirements. Uber processes over 20 million trips per day across 70+ countries.

This problem tests your ability to handle geospatial data at scale, design real-time matching algorithms, manage state machines for trip lifecycle, and handle the unique challenges of a two-sided marketplace.

Requirements

Functional Requirements

  1. Rider requests a ride — Specify pickup and dropoff locations, see estimated fare and ETA
  2. Match rider with nearby driver — Find the optimal available driver based on proximity, rating, and vehicle type
  3. Real-time tracking — Both rider and driver see each other's location on a map during the trip
  4. Trip lifecycle — Request, match, driver en route, pickup, in progress, dropoff, payment, rating
  5. Dynamic pricing (surge) — Adjust prices based on supply/demand in a geographic area
  6. Payment processing — Charge rider, pay driver, handle tips
  7. Driver availability — Drivers toggle online/offline, system tracks their current status
  8. ETA calculation — Estimated time of arrival for driver to pickup and ride to destination

Non-Functional Requirements

  1. Low latency matching — Match rider to driver in <5 seconds
  2. Real-time location updates — Location updates every 3-5 seconds
  3. High availability — 99.99% uptime
  4. Consistency — A driver must never be matched to two riders simultaneously (strong consistency for matching)
  5. Scalability — Support 1M concurrent active drivers and 5M concurrent riders

Capacity Estimation

  - Drivers: 1M * 1 update/4s = 250,000 updates/sec
  - Active rides: 500K * 1 update/4s = 125,000 updates/sec
  - Total location writes: ~375,000/sec

Storage per location update: lat(8B) + lng(8B) + timestamp(8B) + driver_id(8B) = 32B
Location data per day:       375K * 32B * 86400 = ~1 TB/day (if stored; typically sampled)

Trip record:                 ~2KB per trip
Trip storage per day:        20M * 2KB = 40 GB/day

Matching queries:            ~1,000/sec (one per new ride request at peak)
ETA queries:                 ~5,000/sec (riders checking before requesting)

API Design

import express from 'express';
const router = express.Router();

// Request a ride
router.post('/rides', async (req, res) => {
  // POST /api/v1/rides
  // Body: {
  //   pickup: { lat, lng, address },
  //   dropoff: { lat, lng, address },
  //   vehicle_type: 'standard' | 'xl' | 'premium'
  // }
  // Response: { ride_id, estimated_fare, estimated_eta, surge_multiplier, status }
  const { pickup, dropoff, vehicle_type } = req.body;

  const surge = await pricingService.getSurgeMultiplier(pickup);
  const fare = await pricingService.estimateFare(pickup, dropoff, vehicle_type, surge);
  const eta = await etaService.estimatePickupETA(pickup);

  const ride = await rideService.createRide({
    rider_id: req.user.id,
    pickup,
    dropoff,
    vehicle_type,
    estimated_fare: fare,
    surge_multiplier: surge,
  });

  // Trigger async matching
  await matchingService.findDriver(ride);

  res.status(201).json({
    ride_id: ride.id,
    estimated_fare: fare,
    estimated_eta: eta,
    surge_multiplier: surge,
    status: 'matching',
  });
});

// Get ride status
router.get('/rides/:rideId', async (req, res) => {
  // Response: { ride_id, status, driver, pickup, dropoff, current_location, eta }
  const ride = await rideService.getRideWithDetails(req.params.rideId);
  res.json(ride);
});

// Driver accepts a ride
router.post('/rides/:rideId/accept', async (req, res) => {
  // Called by driver
  const result = await matchingService.acceptRide(
    req.params.rideId,
    req.user.id // driver_id
  );
  res.json(result);
});

// Driver updates location
router.post('/drivers/location', async (req, res) => {
  // POST /api/v1/drivers/location
  // Body: { lat, lng, heading, speed }
  await locationService.updateDriverLocation(
    req.user.id,
    req.body
  );
  res.status(204).send();
});

// Driver goes online/offline
router.put('/drivers/status', async (req, res) => {
  // Body: { status: 'online' | 'offline' }
  await driverService.updateStatus(req.user.id, req.body.status);
  res.json({ status: req.body.status });
});

// Complete ride and process payment
router.post('/rides/:rideId/complete', async (req, res) => {
  const result = await rideService.completeRide(req.params.rideId);
  res.json(result);
});

// Rate a ride
router.post('/rides/:rideId/rating', async (req, res) => {
  // Body: { rating: 1-5, comment?: string }
  await rideService.addRating(req.params.rideId, req.user.id, req.body);
  res.status(204).send();
});

// Fare estimate (before requesting)
router.post('/rides/estimate', async (req, res) => {
  // Body: { pickup: { lat, lng }, dropoff: { lat, lng }, vehicle_type }
  const { pickup, dropoff, vehicle_type } = req.body;
  const surge = await pricingService.getSurgeMultiplier(pickup);
  const fare = await pricingService.estimateFare(pickup, dropoff, vehicle_type, surge);
  const eta = await etaService.estimatePickupETA(pickup);
  res.json({ estimated_fare: fare, surge_multiplier: surge, estimated_eta: eta });
});

export default router;

Core Implementation: Geospatial Matching

import Redis from 'ioredis';

const redis = new Redis({ host: 'redis-cluster', port: 6379 });

class LocationService {
  // Update driver location using Redis GEO
  async updateDriverLocation(driverId, { lat, lng, heading, speed }) {
    // GEOADD stores location in a sorted set with geohash scoring
    await redis.geoadd('drivers:available', lng, lat, driverId.toString());

    // Store metadata separately
    await redis.hset(`driver:${driverId}:meta`, {
      lat: lat.toString(),
      lng: lng.toString(),
      heading: heading.toString(),
      speed: speed.toString(),
      updated_at: Date.now().toString(),
    });
  }

  // Find nearby available drivers
  async findNearbyDrivers(lat, lng, radiusKm = 5, limit = 10) {
    // GEOSEARCH returns drivers within radius, sorted by distance
    const results = await redis.geosearch(
      'drivers:available',
      'FROMLONLAT', lng, lat,
      'BYRADIUS', radiusKm, 'km',
      'ASC',
      'COUNT', limit,
      'WITHDIST',
      'WITHCOORD'
    );

    return results.map(([driverId, distance, [driverLng, driverLat]]) => ({
      driver_id: driverId,
      distance_km: parseFloat(distance),
      lat: parseFloat(driverLat),
      lng: parseFloat(driverLng),
    }));
  }

  // Remove driver from available pool
  async removeFromAvailable(driverId) {
    await redis.zrem('drivers:available', driverId.toString());
  }
}

class MatchingService {
  constructor(locationService, rideService) {
    this.locationService = locationService;
    this.rideService = rideService;
  }

  async findDriver(ride) {
    const candidates = await this.locationService.findNearbyDrivers(
      ride.pickup_lat,
      ride.pickup_lng,
      5, // 5km radius
      20  // top 20 candidates
    );

    if (candidates.length === 0) {
      // Expand search radius or notify rider
      await this.rideService.updateStatus(ride.id, 'no_drivers');
      return null;
    }

    // Score candidates: lower distance + higher rating = better match
    const scored = await Promise.all(
      candidates.map(async (c) => {
        const driver = await this.getDriverProfile(c.driver_id);
        return {
          ...c,
          score: c.distance_km * 0.7 - (driver.rating_avg * 0.3),
          vehicle_type: driver.vehicle_type,
        };
      })
    );

    // Filter by vehicle type and sort by score
    const matched = scored
      .filter(s => s.vehicle_type === ride.vehicle_type)
      .sort((a, b) => a.score - b.score);

    // Try to lock the best driver (prevent double-matching)
    for (const candidate of matched) {
      const locked = await this.lockDriver(candidate.driver_id, ride.id);
      if (locked) {
        await this.rideService.assignDriver(ride.id, candidate.driver_id);
        await this.locationService.removeFromAvailable(candidate.driver_id);
        // Notify driver via WebSocket
        await this.notifyDriver(candidate.driver_id, ride);
        return candidate;
      }
    }

    return null; // No driver available after locking attempts
  }

  // Distributed lock to prevent double-matching
  async lockDriver(driverId, rideId) {
    // SET with NX (only if not exists) and EX (expire after 30s)
    const result = await redis.set(
      `driver_lock:${driverId}`,
      rideId.toString(),
      'EX', 30,
      'NX'
    );
    return result === 'OK';
  }
}

class PricingService {
  // Compute surge multiplier based on supply/demand in area
  async getSurgeMultiplier(pickup) {
    const geohash = this.computeGeohash(pickup.lat, pickup.lng, 6);

    // Count riders requesting in this zone (last 5 min)
    const demand = await redis.get(`demand:${geohash}`) || 0;
    // Count available drivers in this zone
    const supply = await this.locationService.findNearbyDrivers(
      pickup.lat, pickup.lng, 2
    );

    const ratio = parseInt(demand) / Math.max(supply.length, 1);

    if (ratio > 3) return 2.5;
    if (ratio > 2) return 2.0;
    if (ratio > 1.5) return 1.5;
    if (ratio > 1) return 1.25;
    return 1.0;
  }

  async estimateFare(pickup, dropoff, vehicleType, surge) {
    const distance = this.haversineDistance(pickup, dropoff);
    const baseFare = { standard: 2.50, xl: 4.00, premium: 6.00 };
    const perKm = { standard: 1.20, xl: 1.80, premium: 2.50 };
    const perMin = { standard: 0.30, xl: 0.40, premium: 0.60 };
    const estimatedMinutes = (distance / 30) * 60; // Assume 30 km/h avg

    const fare =
      baseFare[vehicleType] +
      distance * perKm[vehicleType] +
      estimatedMinutes * perMin[vehicleType];

    return Math.round(fare * surge * 100) / 100;
  }

  haversineDistance(point1, point2) {
    const R = 6371; // Earth radius in km
    const dLat = ((point2.lat - point1.lat) * Math.PI) / 180;
    const dLng = ((point2.lng - point1.lng) * Math.PI) / 180;
    const a =
      Math.sin(dLat / 2) ** 2 +
      Math.cos((point1.lat * Math.PI) / 180) *
      Math.cos((point2.lat * Math.PI) / 180) *
      Math.sin(dLng / 2) ** 2;
    return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
  }

  computeGeohash(lat, lng, precision) {
    // Simplified: in production use ngeohash library
    return `${Math.floor(lat * 10)}:${Math.floor(lng * 10)}`;
  }
}

Real-Time Communication with WebSockets

import { WebSocketServer } from 'ws';
import Redis from 'ioredis';

const sub = new Redis();
const pub = new Redis();
const connections = new Map();

const wss = new WebSocketServer({ port: 8080 });

wss.on('connection', (ws, req) => {
  const userId = authenticateWS(req);
  connections.set(userId, ws);

  ws.on('close', () => connections.delete(userId));
});

// Subscribe to ride updates via Redis Pub/Sub
sub.psubscribe('ride:*');
sub.on('pmessage', (pattern, channel, message) => {
  const rideId = channel.split(':')[1];
  const data = JSON.parse(message);

  // Forward to connected rider/driver
  const riderWs = connections.get(data.rider_id);
  const driverWs = connections.get(data.driver_id);

  if (riderWs) riderWs.send(JSON.stringify(data));
  if (driverWs) driverWs.send(JSON.stringify(data));
});

// Publish location updates from driver service
async function broadcastDriverLocation(rideId, driverId, location) {
  await pub.publish(`ride:${rideId}`, JSON.stringify({
    type: 'driver_location',
    driver_id: driverId,
    ...location,
    timestamp: Date.now(),
  }));
}

Summary

A ride-sharing system requires real-time geospatial indexing (Redis GEO), distributed locking for driver matching, WebSocket connections for live tracking, and a state machine for trip lifecycle management. The system partitions naturally by city/region. Surge pricing uses supply/demand ratios computed from geohash-bucketed zones. The critical path — matching a rider with a driver — must use Redis-based locking to prevent double-assignment. Location data flows through Redis Pub/Sub to deliver real-time updates to both parties.

Common Problems & Solutions

1 Two riders get matched with the same driver simultaneously
Cause: Race condition in the matching service — two concurrent ride requests both see the same driver as available and attempt to assign them.
Solution: Use Redis SET with NX (set-if-not-exists) as a distributed lock on the driver ID. Only the first request that acquires the lock gets the driver. The second request must fall back to the next candidate. This provides atomic check-and-set semantics without database-level locking.
2 Driver location updates are lagging behind by 10-15 seconds
Cause: WebSocket connection drops and reconnects, or the location service is overwhelmed by write volume at peak hours.
Solution: Use a dedicated Redis cluster for location writes, separate from the matching Redis. Buffer location updates client-side and send in batches (every 3-4 seconds). Implement WebSocket heartbeat with automatic reconnection. Accept slightly stale locations for matching (5s staleness is acceptable).
3 Surge pricing oscillates rapidly, confusing riders
Cause: Supply/demand ratio is recomputed too frequently and small fluctuations cause large multiplier swings.
Solution: Smooth the surge multiplier using a moving average over the last 5 minutes instead of instantaneous values. Apply a dampening factor so the multiplier can only change by +/- 0.25 per computation cycle. Use hysteresis: surge only increases when demand/supply exceeds the threshold for 2 consecutive cycles.
