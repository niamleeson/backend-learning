Boolean Logic and Truthiness

Intuition

Think of truthiness as a simple question: is this thing 'something' or 'nothing'? Empty and zero values are 'nothing' (falsy); everything else is 'something' (truthy).

Python's boolean type has exactly two values: True and False. But every object in Python has a truthiness value, which determines how it behaves in boolean contexts like if statements.

# Falsy values in Python
falsy_values = [False, None, 0, 0.0, "", [], {}, set(), tuple()]
for val in falsy_values:
    assert not val  # all are falsy

# Truthy: everything else
assert bool("hello")  # True
assert bool([1, 2])   # True
assert bool(-1)       # True (any non-zero number)

# Short-circuit evaluation
name = "" or "Anonymous"       # "Anonymous" (first truthy)
result = None and expensive()  # None (skips expensive())

# Custom truthiness
class Bag:
    def __init__(self, items):
        self.items = items
    def __bool__(self):
        return len(self.items) > 0
  • Falsy: False, None, 0, 0.0, "", empty collections
  • Use is to compare with None, not ==
  • and returns the first falsy value or the last value
  • or returns the first truthy value or the last value

Exercises

  1. Truthy/Falsy Quiz

    Without running the code, predict the boolean value of: bool([0]), bool(''), bool(' '), bool({}), bool(None). Then verify.

    Hint

    [0] is a non-empty list (truthy), ' ' is a non-empty string (truthy).

  2. Short-Circuit Defaults

    Write a function greet(name=None) that uses or to default to 'Stranger' when name is falsy, and returns the greeting string.

    Hint

    Use display_name = name or 'Stranger'.

Common Problems & Solutions

1 Using `==` to compare with None gives unexpected results
Cause: Objects can override `__eq__` to return True for None comparisons.
Solution: Always use `is None` or `is not None` for None checks.
2 `if x:` fails to distinguish between 0 and None
Cause: Both `0` and `None` are falsy, so a bare truthiness check treats them the same.
Solution: Be explicit: `if x is not None:` when 0 is a valid value.
