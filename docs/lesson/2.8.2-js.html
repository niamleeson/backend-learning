Aggregation Pipeline

Intuition

The aggregation pipeline is like an assembly line in a factory. Raw materials (documents) enter one end and pass through workstations (stages) that each perform one transformation — sorting, filtering, grouping, reshaping. At the end, you get the finished product (your report or analysis). Each station only cares about what it receives and what it passes on.

Aggregation Pipeline

Introduction

MongoDB's Aggregation Pipeline is a powerful framework for data processing and analysis. Documents pass through a series of stages — each stage transforms the data in some way: filtering, grouping, sorting, reshaping, or computing new fields. It's MongoDB's equivalent of SQL's GROUP BY, HAVING, subqueries, and window functions combined.

Pipeline Concept

Document Flow:

  Collection
    |
  $match (filter)        -- WHERE
    |
  $group (aggregate)     -- GROUP BY
    |
  $sort                  -- ORDER BY
    |
  $limit                 -- LIMIT
    |
  Result

Each stage receives documents from the previous stage
and passes transformed documents to the next stage.

Core Stages

$match (Filter)

const recentOrders = await orders.aggregate([
  {
    $match: {
      status: 'completed',
      createdAt: { $gte: new Date('2024-01-01') },
    },
  },
]).toArray();

$group (Aggregate)

// Revenue by category
const byCategory = await orders.aggregate([
  { $match: { status: 'completed' } },
  { $unwind: '$items' },  // flatten items array
  {
    $group: {
      _id: '$items.category',
      totalRevenue: { $sum: '$items.price' },
      orderCount: { $sum: 1 },
      avgPrice: { $avg: '$items.price' },
      maxPrice: { $max: '$items.price' },
      minPrice: { $min: '$items.price' },
      products: { $addToSet: '$items.productId' },
    },
  },
  { $sort: { totalRevenue: -1 } },
]).toArray();

$project (Reshape)

const userSummary = await users.aggregate([
  {
    $project: {
      name: 1,
      email: 1,
      fullAddress: {
        $concat: ['$address.street', ', ', '$address.city', ', ', '$address.state'],
      },
      tagCount: { $size: { $ifNull: ['$tags', []] } },
      isAdmin: { $eq: ['$role', 'admin'] },
      yearJoined: { $year: '$createdAt' },
    },
  },
]).toArray();

$lookup (JOIN)

// Equivalent of SQL LEFT JOIN
const ordersWithUsers = await orders.aggregate([
  {
    $lookup: {
      from: 'users',            // collection to join
      localField: 'userId',     // field in orders
      foreignField: '_id',      // field in users
      as: 'user',               // output array field
    },
  },
  { $unwind: '$user' },         // convert array to object
  {
    $project: {
      orderId: '$_id',
      total: 1,
      customerName: '$user.name',
      customerEmail: '$user.email',
    },
  },
]).toArray();

// Advanced $lookup with pipeline (subquery)
const usersWithRecentOrders = await users.aggregate([
  {
    $lookup: {
      from: 'orders',
      let: { userId: '$_id' },
      pipeline: [
        { $match: { $expr: { $eq: ['$userId', '$$userId'] } } },
        { $sort: { createdAt: -1 } },
        { $limit: 5 },
        { $project: { total: 1, status: 1, createdAt: 1 } },
      ],
      as: 'recentOrders',
    },
  },
]).toArray();

$unwind (Flatten Arrays)

// Document: { tags: ['a', 'b', 'c'] }
// After $unwind: 3 documents, each with one tag

const tagCounts = await posts.aggregate([
  { $unwind: '$tags' },
  { $group: { _id: '$tags', count: { $sum: 1 } } },
  { $sort: { count: -1 } },
  { $limit: 20 },
]).toArray();

Advanced Stages

$addFields

const enriched = await orders.aggregate([
  {
    $addFields: {
      itemCount: { $size: '$items' },
      isLargeOrder: { $gte: ['$total', 100] },
      dayOfWeek: { $dayOfWeek: '$createdAt' },
    },
  },
]).toArray();

$bucket (Histogram)

const priceDistribution = await products.aggregate([
  {
    $bucket: {
      groupBy: '$price',
      boundaries: [0, 10, 25, 50, 100, 500, Infinity],
      default: 'Other',
      output: {
        count: { $sum: 1 },
        products: { $push: '$name' },
      },
    },
  },
]).toArray();

$facet (Multiple Pipelines)

// Run multiple aggregations in parallel
const dashboard = await orders.aggregate([
  { $match: { createdAt: { $gte: thirtyDaysAgo } } },
  {
    $facet: {
      summary: [
        {
          $group: {
            _id: null,
            totalOrders: { $sum: 1 },
            totalRevenue: { $sum: '$total' },
            avgOrder: { $avg: '$total' },
          },
        },
      ],
      byStatus: [
        { $group: { _id: '$status', count: { $sum: 1 } } },
        { $sort: { count: -1 } },
      ],
      dailyRevenue: [
        {
          $group: {
            _id: { $dateToString: { format: '%Y-%m-%d', date: '$createdAt' } },
            revenue: { $sum: '$total' },
            orders: { $sum: 1 },
          },
        },
        { $sort: { '_id': 1 } },
      ],
      topCustomers: [
        { $group: { _id: '$userId', spent: { $sum: '$total' }, orders: { $sum: 1 } } },
        { $sort: { spent: -1 } },
        { $limit: 10 },
      ],
    },
  },
]).toArray();

const result = dashboard[0];
// { summary: [...], byStatus: [...], dailyRevenue: [...], topCustomers: [...] }

Real-World: Analytics Pipeline

async function getMonthlyAnalytics(year: number) {
  return orders.aggregate([
    {
      $match: {
        createdAt: {
          $gte: new Date(`${year}-01-01`),
          $lt: new Date(`${year + 1}-01-01`),
        },
        status: { $ne: 'cancelled' },
      },
    },
    {
      $group: {
        _id: { $month: '$createdAt' },
        revenue: { $sum: '$total' },
        orders: { $sum: 1 },
        uniqueCustomers: { $addToSet: '$userId' },
        avgOrderValue: { $avg: '$total' },
      },
    },
    {
      $addFields: {
        month: '$_id',
        customerCount: { $size: '$uniqueCustomers' },
      },
    },
    {
      $project: { uniqueCustomers: 0, _id: 0 },
    },
    { $sort: { month: 1 } },
  ]).toArray();
}

Pipeline Performance Tips

1. $match early - filter as much data as possible first
2. $match uses indexes (but only as the first stage)
3. $project early to reduce document size
4. Avoid $unwind on large arrays when possible
5. Use $facet for multiple aggregations in one query
6. Use allowDiskUse: true for large datasets
7. Check with .explain('executionStats')

Exercises

  1. E-Commerce Dashboard

    Build an aggregation pipeline that produces a complete dashboard: total revenue, order count, average order value, revenue by category, daily revenue for the last 30 days, and top 10 customers by spending. Use $facet for a single query.

    Hint

    Use $facet with named pipelines. Each pipeline starts with $group or appropriate stage. Use $dateToString for daily grouping.

  2. User Activity Report

    Given a 'sessions' collection with userId, startTime, endTime, and pages array, build a pipeline that calculates: average session duration, most visited pages, daily active users, and user engagement tiers (based on session count).

    Hint

    Use $subtract for duration, $unwind for pages, $group for DAU. Use $bucket for engagement tiers.

  3. Pipeline vs SQL

    Take 5 complex SQL queries (with JOINs, GROUP BY, HAVING, subqueries) and rewrite them as MongoDB aggregation pipelines. Compare readability and performance.

    Hint

    SQL JOIN = $lookup. SQL GROUP BY = $group. SQL HAVING = $match after $group. SQL subquery = $lookup with pipeline.

Common Problems & Solutions

1 Aggregation returns empty results
Cause: $match stage filters out all documents due to type mismatch (string vs ObjectId), date format issues, or field name typos.
Solution: Debug stage by stage: run with only the first stage, check results, add the next stage, repeat. Use `$match: {}` temporarily to bypass filtering.
2 $lookup returns empty arrays
Cause: The localField and foreignField types don't match (e.g., string vs ObjectId), or the field names are wrong.
Solution: Verify field types match between collections. If one stores ObjectId and the other stores string, convert with $toObjectId or $toString in the pipeline.
3 Aggregation runs out of memory on large datasets
Cause: The pipeline exceeds the 100MB memory limit per stage (e.g., large $group or $sort).
Solution: Pass `{ allowDiskUse: true }` as an option. Also, $match early to reduce data volume, and use $project to reduce document size before heavy stages.
