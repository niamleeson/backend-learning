Jinja2: Templates, Inheritance, and Macros

Intuition

Template inheritance is like class inheritance for HTML -- you define a base structure once and let pages override specific sections. Macros are the template equivalent of utility functions, eliminating copy-paste across pages.

Jinja2 Templates, Inheritance, and Macros

Jinja2 is Flask's default template engine. Template inheritance lets you define a base layout and extend it in child templates, keeping HTML DRY. Macros are reusable template functions for repeated UI components like form fields or cards.

# app.py
from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def index():
    posts = [
        {'title': 'First Post', 'author': 'Alice', 'body': 'Hello world'},
        {'title': 'Second Post', 'author': 'Bob', 'body': 'Flask is great'}
    ]
    return render_template('index.html', posts=posts)

# templates/base.html
# <!DOCTYPE html>
# <html>
# <head><title>{% block title %}My App{% endblock %}</title></head>
# <body>
#   <nav>{% block nav %}{% endblock %}</nav>
#   <main>{% block content %}{% endblock %}</main>
# </body></html>

# templates/macros.html
# {% macro post_card(post) %}
# <article>
#   <h2>{{ post.title }}</h2>
#   <p>By {{ post.author }}</p>
#   <p>{{ post.body }}</p>
# </article>
# {% endmacro %}

# templates/index.html
# {% extends 'base.html' %}
# {% from 'macros.html' import post_card %}
# {% block content %}
#   {% for post in posts %}{{ post_card(post) }}{% endfor %}
# {% endblock %}

Key Takeaways: - {% extends %} creates a parent-child template relationship; child templates fill in {% block %} sections. - Macros act like functions inside templates -- define once, import and call anywhere. - Jinja2 auto-escapes HTML by default, preventing XSS attacks.

Exercises

  1. Build a Template Hierarchy

    Create a base.html with header, sidebar, and content blocks. Then create three child templates (home.html, about.html, contact.html) that each extend base.html and override only the content block.

    Hint

    Use {% extends 'base.html' %} at the top of each child template and {% block content %}...{% endblock %} to inject page-specific content.

  2. Reusable Form Macro

    Write a Jinja2 macro that renders a form field with label, input, and error message. Use it to build a registration form with name, email, and password fields.

    Hint

    Define {% macro form_field(name, label, type='text', error=None) %} and call it with {% from 'macros.html' import form_field %}.

Common Problems & Solutions

1 TemplateNotFound error even though the template file exists.
Cause: The template is not in the 'templates' folder relative to the Flask app or blueprint, or the folder name is misspelled.
Solution: Ensure your templates are in a folder named exactly 'templates' at the same level as your Flask app module. For blueprints, set template_folder in the Blueprint constructor.
2 HTML tags appear as raw text instead of rendered HTML.
Cause: Jinja2's auto-escaping converts HTML characters to safe entities by default.
Solution: Use the |safe filter ({{ content|safe }}) only on trusted content, or wrap with Markup() in Python. Never mark user input as safe.
