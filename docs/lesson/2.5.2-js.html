Prisma Client: CRUD, Filtering, and Sorting

Intuition

Prisma Client is like a personal assistant who knows every table and column in your database by heart. When you say 'find users where orders are more than 100,' the assistant not only writes the perfect query but also checks your request for mistakes at compile time — catching typos in field names, wrong filter types, and missing required fields before you even run the code.

Prisma Client: CRUD, Filtering, and Sorting

Introduction

Prisma Client is the auto-generated, type-safe database client that you use to interact with your database. Every query is fully typed — your IDE gives you autocomplete for model fields, filter options, and relation includes. This eliminates entire classes of bugs that traditional ORMs can't catch.

Setting Up Prisma Client

import { PrismaClient } from '@prisma/client';

// Create a singleton instance
const prisma = new PrismaClient({
  log: ['query', 'warn', 'error'], // log SQL queries
});

export default prisma;

CREATE Operations

// Create a single record
const user = await prisma.user.create({
  data: {
    email: 'alice@example.com',
    name: 'Alice',
    passwordHash: await hash('password123'),
  },
});
// Returns the full User object with all fields

// Create with relation (nested create)
const userWithProfile = await prisma.user.create({
  data: {
    email: 'bob@example.com',
    name: 'Bob',
    passwordHash: await hash('password123'),
    addresses: {
      create: {
        street: '123 Main St',
        city: 'New York',
        postalCode: '10001',
        country: 'US',
      },
    },
  },
  include: {
    addresses: true, // include addresses in response
  },
});

// Create many records
const result = await prisma.user.createMany({
  data: [
    { email: 'user1@test.com', name: 'User 1', passwordHash: 'hash1' },
    { email: 'user2@test.com', name: 'User 2', passwordHash: 'hash2' },
    { email: 'user3@test.com', name: 'User 3', passwordHash: 'hash3' },
  ],
  skipDuplicates: true, // ignore conflicts on unique fields
});
console.log(result.count); // 3

READ Operations

// Find by unique field
const user = await prisma.user.findUnique({
  where: { email: 'alice@example.com' },
});
// Returns User | null

// Find by unique field or throw
const user = await prisma.user.findUniqueOrThrow({
  where: { id: userId },
});
// Throws PrismaClientKnownRequestError if not found

// Find first matching
const admin = await prisma.user.findFirst({
  where: { role: 'ADMIN' },
  orderBy: { createdAt: 'desc' },
});

// Find many with filters
const users = await prisma.user.findMany({
  where: {
    role: 'CUSTOMER',
    createdAt: { gte: new Date('2024-01-01') },
  },
  orderBy: { name: 'asc' },
  take: 20,  // LIMIT
  skip: 40,  // OFFSET
});

// Select specific fields
const emails = await prisma.user.findMany({
  select: {
    id: true,
    email: true,
    name: true,
    // passwordHash is NOT included
  },
});

Filtering

// Comparison operators
const products = await prisma.product.findMany({
  where: {
    price: { gt: 10 },        // greater than
    stock: { gte: 1 },        // greater than or equal
    name: { not: 'Deleted' }, // not equal
  },
});

// String filters
const users = await prisma.user.findMany({
  where: {
    email: { contains: 'gmail', mode: 'insensitive' }, // ILIKE '%gmail%'
    name: { startsWith: 'A' },                          // LIKE 'A%'
  },
});

// List filters
const selected = await prisma.user.findMany({
  where: {
    id: { in: [id1, id2, id3] },    // IN
    role: { notIn: ['ADMIN'] },      // NOT IN
  },
});

// NULL checks
const verified = await prisma.user.findMany({
  where: {
    emailVerifiedAt: { not: null },  // IS NOT NULL
    deletedAt: null,                  // IS NULL (shorthand)
  },
});

// Logical operators: AND, OR, NOT
const results = await prisma.user.findMany({
  where: {
    OR: [
      { email: { contains: 'admin' } },
      { role: 'ADMIN' },
    ],
    AND: [
      { createdAt: { gte: new Date('2024-01-01') } },
      { deletedAt: null },
    ],
    NOT: {
      status: 'BANNED',
    },
  },
});

// Filter on relations
const usersWithOrders = await prisma.user.findMany({
  where: {
    orders: {
      some: { total: { gt: 100 } },  // has at least one order > $100
    },
  },
});

const usersWithoutOrders = await prisma.user.findMany({
  where: {
    orders: { none: {} },  // has zero orders
  },
});

UPDATE Operations

// Update one
const updated = await prisma.user.update({
  where: { id: userId },
  data: { name: 'New Name' },
});

// Update many
const result = await prisma.user.updateMany({
  where: { role: 'CUSTOMER', lastLoginAt: { lt: oneYearAgo } },
  data: { status: 'INACTIVE' },
});
console.log(result.count); // number of updated rows

// Upsert (create or update)
const user = await prisma.user.upsert({
  where: { email: 'alice@example.com' },
  update: { name: 'Alice Updated', lastLoginAt: new Date() },
  create: { email: 'alice@example.com', name: 'Alice', passwordHash: 'hash' },
});

// Atomic number operations
const product = await prisma.product.update({
  where: { id: productId },
  data: {
    stock: { decrement: 1 },    // stock = stock - 1
    viewCount: { increment: 1 }, // view_count = view_count + 1
  },
});

DELETE Operations

// Delete one
const deleted = await prisma.user.delete({
  where: { id: userId },
});

// Delete many
const result = await prisma.user.deleteMany({
  where: { deletedAt: { not: null }, deletedAt: { lt: thirtyDaysAgo } },
});

// Soft delete pattern
const softDeleted = await prisma.user.update({
  where: { id: userId },
  data: { deletedAt: new Date() },
});

Sorting and Pagination

// Multi-field sorting
const products = await prisma.product.findMany({
  orderBy: [
    { isActive: 'desc' },  // active products first
    { price: 'asc' },      // then by price
  ],
  take: 20,
  skip: 0,
});

// Cursor-based pagination (more efficient than offset)
const page = await prisma.post.findMany({
  take: 20,
  cursor: { id: lastSeenId },
  skip: 1, // skip the cursor item itself
  orderBy: { createdAt: 'desc' },
});

Aggregations

const stats = await prisma.order.aggregate({
  _count: true,
  _sum: { total: true },
  _avg: { total: true },
  _min: { total: true },
  _max: { total: true },
  where: { status: 'DELIVERED' },
});
// { _count: 150, _sum: { total: 8500.00 }, _avg: { total: 56.67 }, ... }

// Group by
const byStatus = await prisma.order.groupBy({
  by: ['status'],
  _count: true,
  _sum: { total: true },
  orderBy: { _count: { status: 'desc' } },
});

Include vs Select

// include: adds relation data to all scalar fields
const user = await prisma.user.findUnique({
  where: { id: userId },
  include: {
    orders: { take: 5, orderBy: { createdAt: 'desc' } },
    addresses: true,
  },
});
// Returns all user fields + orders + addresses

// select: only includes specified fields (more efficient)
const user = await prisma.user.findUnique({
  where: { id: userId },
  select: {
    id: true,
    name: true,
    email: true,
    orders: {
      select: { id: true, total: true, status: true },
      take: 5,
    },
  },
});
// Returns ONLY id, name, email, and limited order data

Exercises

  1. Dynamic Filter Builder

    Build a function searchProducts(filters) that accepts optional: search (text match on name/description), minPrice, maxPrice, categoryId, inStock (boolean), sortBy, sortDir, page, pageSize. Construct the Prisma query dynamically.

    Hint

    Build the where object conditionally: const where: Prisma.ProductWhereInput = {}; if (filters.search) where.OR = [{name: {contains: ...}}, {description: {contains: ...}}];

  2. Cursor-Based Pagination API

    Implement a paginated API that returns items with a nextCursor field. The client passes the cursor to get the next page. Handle the first page (no cursor) and last page (no more items) edge cases.

    Hint

    Use take: limit + 1 to check if there's a next page. If you get limit+1 results, there's more data. Return the last item's id as nextCursor.

  3. Aggregation Dashboard

    Build a dashboard endpoint that returns: total users, new users this month, total revenue, revenue by status, top 5 products by order count, average order value by month for the last 12 months.

    Hint

    Use prisma.order.aggregate for totals, prisma.order.groupBy for breakdowns. For complex queries, use prisma.$queryRaw.

Common Problems & Solutions

1 N+1 query problem with nested includes
Cause: Prisma uses separate queries for each relation include (not JOINs). Including deeply nested relations generates many queries.
Solution: Use `select` to limit fetched data. For complex queries with multiple nested relations, consider using `prisma.$queryRaw` with manual JOINs.
2 Type error: Argument type is not assignable
Cause: Passing the wrong type to a Prisma query. Common with Date objects, Decimal values, or enum strings.
Solution: Check Prisma's generated types. Use `new Date()` for DateTime fields, `new Prisma.Decimal('10.50')` for Decimal fields, and enum values from the generated enum.
3 findUnique returns null instead of throwing
Cause: findUnique returns null when no record matches, which can lead to null reference errors downstream.
Solution: Use `findUniqueOrThrow` when you expect a record to exist. Or check for null and throw a custom 404 error.
