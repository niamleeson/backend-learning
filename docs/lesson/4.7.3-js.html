Testing, Documentation, and Production Config

Intuition

Testing, documentation, and production config are like the quality assurance, user manual, and safety features of a car. Tests ensure the engine runs correctly. Swagger docs are the owner's manual for API consumers. Production config (helmet, compression, health checks) are the airbags and seatbelts that protect your app in the real world.

Testing, Documentation, and Production Config

Introduction

The final step in building a production-ready NestJS API is ensuring it's well-tested, properly documented, and configured for deployment. This lesson covers unit testing with Jest, e2e testing with supertest, API documentation with Swagger, health checks, graceful shutdown, and production configuration.

Unit Testing NestJS Services

// posts/posts.service.spec.ts
import { Test, TestingModule } from '@nestjs/testing';
import { PostsService } from './posts.service';
import { PrismaService } from '../prisma/prisma.service';
import { NotFoundException, ForbiddenException } from '@nestjs/common';

describe('PostsService', () => {
  let service: PostsService;
  let prisma: any;

  const mockPrisma = {
    post: {
      findMany: jest.fn(),
      findUnique: jest.fn(),
      create: jest.fn(),
      update: jest.fn(),
      delete: jest.fn(),
      count: jest.fn(),
    },
  };

  beforeEach(async () => {
    const module: TestingModule = await Test.createTestingModule({
      providers: [
        PostsService,
        { provide: PrismaService, useValue: mockPrisma },
      ],
    }).compile();

    service = module.get<PostsService>(PostsService);
    prisma = module.get<PrismaService>(PrismaService);
    jest.clearAllMocks();
  });

  describe('findBySlug', () => {
    it('should return a post when found', async () => {
      const mockPost = {
        id: '1',
        title: 'Test Post',
        slug: 'test-post',
        content: 'Content',
      };
      mockPrisma.post.findUnique.mockResolvedValue(mockPost);

      const result = await service.findBySlug('test-post');
      expect(result).toEqual(mockPost);
      expect(mockPrisma.post.findUnique).toHaveBeenCalledWith(
        expect.objectContaining({ where: { slug: 'test-post' } }),
      );
    });

    it('should throw NotFoundException when not found', async () => {
      mockPrisma.post.findUnique.mockResolvedValue(null);
      await expect(service.findBySlug('nonexistent')).rejects.toThrow(
        NotFoundException,
      );
    });
  });

  describe('update', () => {
    it('should throw ForbiddenException if user is not author', async () => {
      mockPrisma.post.findUnique.mockResolvedValue({
        id: '1',
        authorId: 'author-1',
      });

      await expect(
        service.update('1', 'other-user', { title: 'New Title' }),
      ).rejects.toThrow(ForbiddenException);
    });
  });
});

End-to-End Testing

// test/posts.e2e-spec.ts
import { Test } from '@nestjs/testing';
import { INestApplication, ValidationPipe } from '@nestjs/common';
import * as request from 'supertest';
import { AppModule } from '../src/app.module';
import { PrismaService } from '../src/prisma/prisma.service';

describe('Posts (e2e)', () => {
  let app: INestApplication;
  let prisma: PrismaService;
  let authToken: string;

  beforeAll(async () => {
    const module = await Test.createTestingModule({
      imports: [AppModule],
    }).compile();

    app = module.createNestApplication();
    app.useGlobalPipes(new ValidationPipe({
      whitelist: true,
      transform: true,
    }));
    await app.init();

    prisma = app.get(PrismaService);

    // Clean database
    await prisma.comment.deleteMany();
    await prisma.post.deleteMany();
    await prisma.user.deleteMany();

    // Create test user and get token
    const registerResponse = await request(app.getHttpServer())
      .post('/api/auth/register')
      .send({ name: 'Test User', email: 'test@example.com', password: 'password123' });

    authToken = registerResponse.body.accessToken;
  });

  afterAll(async () => {
    await prisma.comment.deleteMany();
    await prisma.post.deleteMany();
    await prisma.user.deleteMany();
    await app.close();
  });

  describe('POST /api/posts', () => {
    it('should create a post', () => {
      return request(app.getHttpServer())
        .post('/api/posts')
        .set('Authorization', `Bearer ${authToken}`)
        .send({
          title: 'My First Post',
          content: 'Hello World',
          published: true,
        })
        .expect(201)
        .expect((res) => {
          expect(res.body.data.title).toBe('My First Post');
          expect(res.body.data.slug).toBeDefined();
        });
    });

    it('should reject unauthenticated requests', () => {
      return request(app.getHttpServer())
        .post('/api/posts')
        .send({ title: 'Test', content: 'Test' })
        .expect(401);
    });

    it('should validate the request body', () => {
      return request(app.getHttpServer())
        .post('/api/posts')
        .set('Authorization', `Bearer ${authToken}`)
        .send({ title: '' }) // Missing content, empty title
        .expect(400);
    });
  });

  describe('GET /api/posts', () => {
    it('should return paginated posts', () => {
      return request(app.getHttpServer())
        .get('/api/posts?page=1&limit=10')
        .expect(200)
        .expect((res) => {
          expect(res.body.data).toBeInstanceOf(Array);
          expect(res.body.meta).toBeDefined();
          expect(res.body.meta.total).toBeGreaterThanOrEqual(0);
        });
    });
  });
});

// main.ts
import { SwaggerModule, DocumentBuilder } from '@nestjs/swagger';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  const config = new DocumentBuilder()
    .setTitle('Blog API')
    .setDescription('A production-ready blog API built with NestJS')
    .setVersion('1.0')
    .addBearerAuth()
    .addTag('auth', 'Authentication endpoints')
    .addTag('posts', 'Blog post operations')
    .addTag('comments', 'Comment operations')
    .addTag('users', 'User management')
    .build();

  const document = SwaggerModule.createDocument(app, config);
  SwaggerModule.setup('docs', app, document);

  await app.listen(3000);
}

Decorating DTOs and Controllers

import { ApiProperty, ApiPropertyOptional } from '@nestjs/swagger';

export class CreatePostDto {
  @ApiProperty({ example: 'My First Post', minLength: 3, maxLength: 200 })
  @IsString()
  @MinLength(3)
  title: string;

  @ApiProperty({ example: 'The full content of the blog post...' })
  @IsString()
  content: string;

  @ApiPropertyOptional({ example: true, default: false })
  @IsOptional()
  @IsBoolean()
  published?: boolean;

  @ApiPropertyOptional({ example: ['nestjs', 'typescript'], type: [String] })
  @IsOptional()
  @IsArray()
  @IsString({ each: true })
  tagNames?: string[];
}

import { ApiTags, ApiBearerAuth, ApiOperation, ApiResponse, ApiQuery } from '@nestjs/swagger';

@ApiTags('posts')
@Controller('posts')
export class PostsController {
  @ApiOperation({ summary: 'List all published posts' })
  @ApiQuery({ name: 'page', required: false, type: Number })
  @ApiQuery({ name: 'limit', required: false, type: Number })
  @ApiQuery({ name: 'search', required: false, type: String })
  @ApiResponse({ status: 200, description: 'Paginated list of posts' })
  @Public()
  @Get()
  findAll(@Query() query: FilterPostsDto) {
    return this.postsService.findAll(query);
  }

  @ApiOperation({ summary: 'Create a new post' })
  @ApiBearerAuth()
  @ApiResponse({ status: 201, description: 'Post created successfully' })
  @ApiResponse({ status: 401, description: 'Unauthorized' })
  @Post()
  create(@CurrentUser('id') userId: string, @Body() dto: CreatePostDto) {
    return this.postsService.create(userId, dto);
  }
}

Health Checks

npm i @nestjs/terminus

// health/health.controller.ts
import { Controller, Get } from '@nestjs/common';
import {
  HealthCheckService, HealthCheck,
  PrismaHealthIndicator, DiskHealthIndicator,
  MemoryHealthIndicator,
} from '@nestjs/terminus';

@Controller('health')
export class HealthController {
  constructor(
    private health: HealthCheckService,
    private db: PrismaHealthIndicator,
    private disk: DiskHealthIndicator,
    private memory: MemoryHealthIndicator,
  ) {}

  @Get()
  @HealthCheck()
  @Public()
  check() {
    return this.health.check([
      () => this.db.pingCheck('database'),
      () => this.disk.checkStorage('storage', {
        path: '/',
        thresholdPercent: 0.9,
      }),
      () => this.memory.checkHeap('memory_heap', 200 * 1024 * 1024),
    ]);
  }
}

Production Configuration

// main.ts (production-ready)
async function bootstrap() {
  const app = await NestFactory.create(AppModule, {
    logger: process.env.NODE_ENV === 'production'
      ? ['error', 'warn']
      : ['log', 'debug', 'error', 'warn', 'verbose'],
  });

  app.setGlobalPrefix('api');
  app.enableCors({
    origin: process.env.CORS_ORIGIN?.split(',') || [],
    credentials: true,
  });

  app.useGlobalPipes(new ValidationPipe({
    whitelist: true,
    forbidNonWhitelisted: true,
    transform: true,
  }));

  app.enableShutdownHooks();

  // Compression
  app.use(compression());

  // Helmet for security headers
  app.use(helmet());

  const port = process.env.PORT || 3000;
  await app.listen(port);
}

Common Pitfalls

  1. Not testing error paths -- Test that services throw correct exceptions for invalid inputs, missing records, and unauthorized access.
  2. Swagger docs drift from implementation -- Keep API decorators in sync with actual behavior. Use the CLI plugin for automatic DTO detection.
  3. Missing health checks -- Without health checks, orchestrators (Kubernetes, ECS) can't tell if your service is healthy.
  4. Forgetting to clean up in e2e tests -- Always clean the database in beforeAll/afterAll to prevent test pollution.

Exercises

  1. Write Comprehensive Tests

    Write unit tests for PostsService (all CRUD methods, including error cases) and e2e tests for the full posts API (create, read, update, delete, pagination, search). Aim for 80%+ coverage on the service layer.

    Hint

    Mock PrismaService for unit tests. Use the real database for e2e tests with cleanup in beforeAll/afterAll. Test auth scenarios: unauthenticated, wrong user, admin access.

  2. Add Swagger Documentation

    Document the entire API with Swagger. Add descriptions to all endpoints, DTOs, and response types. Include example values. Test the Swagger UI at /docs to ensure all endpoints are properly documented and testable.

    Hint

    Use @ApiTags for grouping, @ApiOperation for descriptions, @ApiProperty for DTO fields, @ApiBearerAuth for protected endpoints. Enable the Swagger CLI plugin in nest-cli.json for automatic DTO detection.

  3. Production-Ready Deployment

    Create a Docker setup with multi-stage builds. Add health check endpoints. Configure compression and helmet. Set up environment-specific logging. Create a docker-compose.yml with the API and PostgreSQL. Verify the app starts and passes health checks in Docker.

    Hint

    Use the Dockerfile template from this lesson. Add @nestjs/terminus for health checks. Test with docker-compose up --build and hit the health endpoint.

Common Problems & Solutions

1 E2E tests fail with database connection errors
Cause: The test environment doesn't have a running database, or the DATABASE_URL in .env.test points to the wrong database.
Solution: Create a .env.test file with a test-specific database URL. Use docker-compose to spin up a test database. Or use an in-memory database for faster tests.
2 Swagger doesn't show DTO properties
Cause: The Swagger CLI plugin isn't enabled, and DTOs don't have @ApiProperty decorators.
Solution: Either add @ApiProperty to every DTO field, or enable the CLI plugin in nest-cli.json: `"plugins": ["@nestjs/swagger"]`. The plugin automatically reads class-validator decorators.
3 Application crashes on shutdown with pending connections
Cause: Shutdown hooks aren't enabled, or services don't properly clean up (close database connections, stop background jobs).
Solution: Call `app.enableShutdownHooks()` in bootstrap. Implement OnModuleDestroy in services that hold connections. For Prisma, call $disconnect in onModuleDestroy.
