asyncio: Tasks, gather, and wait

Intuition

Tasks are like placing multiple orders at a restaurant. gather waits until all dishes arrive, while wait(FIRST_COMPLETED) lets you eat each dish as soon as it is ready.

asyncio Tasks, gather, and wait

Tasks wrap coroutines for concurrent execution and provide control over scheduling, cancellation, and result collection.

import asyncio

async def process(name: str, delay: float) -> str:
    await asyncio.sleep(delay)
    return f"{name} completed"

async def main():
    # Create tasks for concurrent execution
    task1 = asyncio.create_task(process("A", 2))
    task2 = asyncio.create_task(process("B", 1))
    task3 = asyncio.create_task(process("C", 3))

    # gather: wait for all, returns results in order
    results = await asyncio.gather(task1, task2, task3)
    print(results)  # ['A completed', 'B completed', 'C completed']

async def main_with_wait():
    tasks = [
        asyncio.create_task(process("A", 2)),
        asyncio.create_task(process("B", 1)),
        asyncio.create_task(process("C", 3)),
    ]
    # wait: returns done and pending sets
    done, pending = await asyncio.wait(
        tasks, return_when=asyncio.FIRST_COMPLETED
    )
    for task in done:
        print(task.result())  # 'B completed' (finished first)
    for task in pending:
        task.cancel()  # Cancel remaining tasks

asyncio.run(main())

Key Takeaways: - create_task() schedules a coroutine to run concurrently on the event loop - gather() runs tasks and returns results in submission order - wait() returns sets of completed and pending tasks with flexible policies - Use return_when=FIRST_COMPLETED or FIRST_EXCEPTION for early exit - Tasks can be cancelled with task.cancel() which raises CancelledError

Exercises

  1. Task Timeout Manager

    Write a function that runs 5 async tasks with asyncio.wait and a 2-second timeout. Process completed tasks and cancel any that are still pending after the timeout.

    Hint

    Use await asyncio.wait(tasks, timeout=2.0). Iterate over pending and call task.cancel(). Wrap cancellation in try/except CancelledError.

  2. Gather with Error Handling

    Use asyncio.gather(return_exceptions=True) with tasks where some raise exceptions. Filter results into successes and errors.

    Hint

    With return_exceptions=True, exceptions appear as values in the results list. Check isinstance(r, Exception) to separate them.

Common Problems & Solutions

1 `asyncio.gather` raises the first exception and cancels remaining tasks.
Cause: By default, `gather` propagates the first exception and cancels other tasks.
Solution: Use `asyncio.gather(*tasks, return_exceptions=True)` to collect all results including exceptions without cancellation.
2 Tasks silently disappear: results are lost and exceptions go unnoticed.
Cause: Creating a task without storing a reference allows garbage collection to destroy it.
Solution: Always keep a reference to tasks: `tasks.add(asyncio.create_task(coro()))`. Use `gather` or `wait` to collect results.
