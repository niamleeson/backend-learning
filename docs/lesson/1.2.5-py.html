Closures and Nested Functions

Intuition

A closure is like a function carrying a backpack of variables from where it was born. Wherever it goes, it still has access to that backpack.

A closure is a nested function that remembers variables from its enclosing scope even after the outer function has returned. Closures are the foundation of decorators and callback patterns.

def make_multiplier(factor):
    """Returns a function that multiplies by factor."""
    def multiplier(x):
        return x * factor  # 'factor' is captured from enclosing scope
    return multiplier

double = make_multiplier(2)
triple = make_multiplier(3)
print(double(5))   # 10
print(triple(5))   # 15

# Closure with mutable state
def make_accumulator(initial=0):
    total = [initial]  # list to allow mutation without nonlocal
    def add(value):
        total[0] += value
        return total[0]
    return add

acc = make_accumulator()
print(acc(10))  # 10
print(acc(20))  # 30
  • A closure "closes over" variables from the enclosing scope
  • The closed-over variables persist as long as the closure exists
  • Use func.__closure__ to inspect captured variables
  • Closures are a lightweight alternative to classes with a single method

Common Problems & Solutions

1 Loop variable captured by reference, not value
Cause: All closures in a loop share the same loop variable, which ends at the final value.
Solution: Use a default argument to capture the current value: `lambda x, i=i: x + i`.
2 Cannot modify enclosed variable without `nonlocal`
Cause: Assigning to a variable in a closure creates a new local variable instead of modifying the enclosed one.
Solution: Use `nonlocal var_name` or wrap the value in a mutable container like a list.
