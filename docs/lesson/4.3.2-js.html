Pipes: Validation and Transformation with class-validator

Intuition

Pipes are like airport security checkpoints. The metal detector (ParseIntPipe) transforms your bag contents into scannable data. The X-ray machine (ValidationPipe) checks that nothing dangerous is inside. If anything fails the check, you're stopped before reaching the gate (handler).

Pipes: Validation and Transformation with class-validator

Introduction

Pipes are NestJS components that transform or validate data before it reaches the route handler. They operate on the arguments of a controller method -- body, params, query -- and can either return transformed data or throw an exception if validation fails.

Pipes in the Request Pipeline

Middleware -> Guards -> Interceptors -> PIPES -> Handler
                                         ^
                                         |
                                    We are here

Pipes have two primary use cases: 1. Transformation -- Convert input data (e.g., string to number, plain object to class instance) 2. Validation -- Verify input data matches expected shapes and rules

Built-in Pipes

NestJS provides several built-in pipes:

Pipe Purpose
ValidationPipe Validates DTOs using class-validator
ParseIntPipe Converts string to integer
ParseFloatPipe Converts string to float
ParseBoolPipe Converts string to boolean
ParseUUIDPipe Validates UUID format
ParseArrayPipe Parses and validates arrays
ParseEnumPipe Validates enum values
DefaultValuePipe Provides default values for undefined params

Using Built-in Pipes

import {
  Controller, Get, Param, Query,
  ParseIntPipe, ParseUUIDPipe, ParseBoolPipe, DefaultValuePipe,
} from '@nestjs/common';

@Controller('users')
export class UsersController {
  // Parse string param to integer
  @Get(':id')
  findOne(@Param('id', ParseIntPipe) id: number) {
    // id is guaranteed to be a number
    return this.usersService.findOne(id);
  }

  // Validate UUID format
  @Get(':uuid')
  findByUuid(@Param('uuid', ParseUUIDPipe) uuid: string) {
    return this.usersService.findByUuid(uuid);
  }

  // Parse query params with defaults
  @Get()
  findAll(
    @Query('page', new DefaultValuePipe(1), ParseIntPipe) page: number,
    @Query('limit', new DefaultValuePipe(20), ParseIntPipe) limit: number,
    @Query('active', new DefaultValuePipe(true), ParseBoolPipe) active: boolean,
  ) {
    return this.usersService.findAll({ page, limit, active });
  }
}

ValidationPipe with class-validator

The ValidationPipe is the most important pipe. It validates incoming data against DTO classes decorated with class-validator decorators.

Setup

npm i class-validator class-transformer

Enable globally in main.ts:

import { ValidationPipe } from '@nestjs/common';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.useGlobalPipes(
    new ValidationPipe({
      whitelist: true,            // Strip unknown properties
      forbidNonWhitelisted: true, // Error on unknown properties
      transform: true,            // Auto-transform to DTO instances
      transformOptions: {
        enableImplicitConversion: true, // Convert types automatically
      },
    }),
  );
  await app.listen(3000);
}

DTO with Validation Decorators

import {
  IsString, IsEmail, IsOptional, IsInt, IsEnum,
  Min, Max, MinLength, MaxLength, IsArray,
  ValidateNested, IsNotEmpty, Matches, IsUrl,
} from 'class-validator';
import { Type } from 'class-transformer';

export enum UserRole {
  USER = 'user',
  ADMIN = 'admin',
  MODERATOR = 'moderator',
}

export class AddressDto {
  @IsString()
  @IsNotEmpty()
  street: string;

  @IsString()
  @IsNotEmpty()
  city: string;

  @IsString()
  @Matches(/^\d{5}(-\d{4})?$/, { message: 'Invalid ZIP code format' })
  zipCode: string;
}

export class CreateUserDto {
  @IsString()
  @MinLength(2)
  @MaxLength(50)
  name: string;

  @IsEmail({}, { message: 'Please provide a valid email address' })
  email: string;

  @IsString()
  @MinLength(8)
  @Matches(/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)/, {
    message: 'Password must contain at least one uppercase letter, one lowercase letter, and one number',
  })
  password: string;

  @IsOptional()
  @IsEnum(UserRole)
  role?: UserRole;

  @IsOptional()
  @IsInt()
  @Min(0)
  @Max(150)
  age?: number;

  @IsOptional()
  @IsUrl()
  website?: string;

  @IsOptional()
  @ValidateNested()
  @Type(() => AddressDto)
  address?: AddressDto;

  @IsOptional()
  @IsArray()
  @IsString({ each: true })
  tags?: string[];
}

Custom Validation Decorators

import {
  registerDecorator, ValidationOptions, ValidationArguments,
} from 'class-validator';

// Custom decorator: @IsAfterDate('startDate')
export function IsAfterDate(
  property: string,
  validationOptions?: ValidationOptions,
) {
  return function (object: Object, propertyName: string) {
    registerDecorator({
      name: 'isAfterDate',
      target: object.constructor,
      propertyName: propertyName,
      constraints: [property],
      options: validationOptions,
      validator: {
        validate(value: any, args: ValidationArguments) {
          const [relatedPropertyName] = args.constraints;
          const relatedValue = (args.object as any)[relatedPropertyName];
          return value > relatedValue;
        },
        defaultMessage(args: ValidationArguments) {
          const [relatedPropertyName] = args.constraints;
          return `${args.property} must be after ${relatedPropertyName}`;
        },
      },
    });
  };
}

// Usage
export class CreateEventDto {
  @IsDateString()
  startDate: string;

  @IsDateString()
  @IsAfterDate('startDate', { message: 'End date must be after start date' })
  endDate: string;
}

Custom Pipes

Transformation Pipe

import { PipeTransform, Injectable, ArgumentMetadata } from '@nestjs/common';

@Injectable()
export class TrimPipe implements PipeTransform {
  transform(value: any, metadata: ArgumentMetadata) {
    if (typeof value === 'string') {
      return value.trim();
    }
    if (typeof value === 'object' && value !== null) {
      return this.trimObject(value);
    }
    return value;
  }

  private trimObject(obj: Record<string, any>): Record<string, any> {
    const trimmed: Record<string, any> = {};
    for (const [key, val] of Object.entries(obj)) {
      trimmed[key] = typeof val === 'string' ? val.trim() : val;
    }
    return trimmed;
  }
}

Validation Pipe with Business Logic

@Injectable()
export class ParseDateRangePipe implements PipeTransform {
  transform(value: any, metadata: ArgumentMetadata) {
    const { startDate, endDate } = value;

    const start = new Date(startDate);
    const end = new Date(endDate);

    if (isNaN(start.getTime())) {
      throw new BadRequestException('Invalid start date');
    }
    if (isNaN(end.getTime())) {
      throw new BadRequestException('Invalid end date');
    }
    if (end <= start) {
      throw new BadRequestException('End date must be after start date');
    }

    // 90-day maximum range
    const diffDays = (end.getTime() - start.getTime()) / (1000 * 60 * 60 * 24);
    if (diffDays > 90) {
      throw new BadRequestException('Date range cannot exceed 90 days');
    }

    return { startDate: start, endDate: end };
  }
}

// Usage
@Get('reports')
getReport(@Query(ParseDateRangePipe) range: { startDate: Date; endDate: Date }) {
  return this.reportsService.generate(range.startDate, range.endDate);
}

Pipe Binding Levels

// Parameter level
@Get(':id')
findOne(@Param('id', ParseIntPipe) id: number) {}

// Method level
@Post()
@UsePipes(new ValidationPipe({ groups: ['create'] }))
create(@Body() dto: CreateUserDto) {}

// Controller level
@Controller('users')
@UsePipes(TrimPipe)
export class UsersController {}

// Global level (in main.ts)
app.useGlobalPipes(new ValidationPipe());

// Global level with DI
@Module({
  providers: [
    { provide: APP_PIPE, useClass: ValidationPipe },
  ],
})
export class AppModule {}

Validation Groups

Different validation rules for create vs. update:

export class UserDto {
  @IsString()
  @IsNotEmpty({ groups: ['create'] })
  name: string;

  @IsEmail()
  @IsNotEmpty({ groups: ['create'] })
  email: string;

  @IsString()
  @MinLength(8, { groups: ['create'] })
  @IsOptional({ groups: ['update'] })
  password: string;
}

@Controller('users')
export class UsersController {
  @Post()
  @UsePipes(new ValidationPipe({ groups: ['create'] }))
  create(@Body() dto: UserDto) {}

  @Patch(':id')
  @UsePipes(new ValidationPipe({ groups: ['update'] }))
  update(@Param('id') id: string, @Body() dto: UserDto) {}
}

Common Pitfalls

  1. Using interfaces instead of classes for DTOs -- class-validator decorators are runtime artifacts. Interfaces are erased at compile time and can't be validated.
  2. Missing transform: true -- Without this, @Body() returns a plain object, not a class instance. Nested @ValidateNested() won't work.
  3. Missing @Type() for nested objects -- class-transformer needs @Type(() => NestedDto) to know how to transform nested objects.
  4. Pipe order -- Multiple pipes on a parameter run left to right: @Param('id', PipeA, PipeB). PipeA's output becomes PipeB's input.

Exercises

  1. Create Comprehensive DTOs

    Build CreateProductDto with validation for: name (2-100 chars), price (positive number with max 2 decimal places), category (enum), tags (array of strings, max 10), dimensions (nested object with width/height/depth as positive numbers). Add a custom validator for SKU format (e.g., 'PRD-XXXX-XXXX').

    Hint

    Use @Matches() for SKU format. Use @ValidateNested() with @Type(() => DimensionsDto) for nested objects. Use @ArrayMaxSize(10) for tags.

  2. Build a Custom Transformation Pipe

    Create a SanitizePipe that: trims all strings, converts emails to lowercase, removes HTML tags from text fields, and normalizes phone numbers. Apply it globally.

    Hint

    Recursively walk the object in the transform() method. Use regex to strip HTML tags. Check property types before transforming.

  3. Query Parameter Validation

    Create a comprehensive query DTO for a product search endpoint: page, limit, sortBy (enum of allowed fields), sortOrder (asc/desc), minPrice, maxPrice, categories (comma-separated string to array), search (min 3 chars). Write a custom pipe to parse the categories string.

    Hint

    Use @Transform() from class-transformer to split comma-separated values. Use DefaultValuePipe for sensible defaults.

  4. Conditional Validation

    Build a CreatePaymentDto where: if paymentMethod is 'credit_card', then cardNumber, expiry, and cvv are required; if paymentMethod is 'bank_transfer', then accountNumber and routingNumber are required. Use class-validator's conditional validation.

    Hint

    Use @ValidateIf() decorator: @ValidateIf(o => o.paymentMethod === 'credit_card') before the card-specific field validators.

Common Problems & Solutions

1 Validation decorators are silently ignored
Symptoms: Invalid data passes through to the handler without errors.
Cause: The DTO is an interface (erased at runtime), or ValidationPipe is not enabled, or class-validator is not installed.
Solution: 1) Use a class, not an interface. 2) Enable ValidationPipe globally in main.ts. 3) Run `npm i class-validator class-transformer`.
2 Nested object validation doesn't work
Cause: Missing @ValidateNested() or @Type() decorator on the nested property, or `transform: true` is not set on ValidationPipe.
Solution: Add both decorators: `@ValidateNested()` and `@Type(() => NestedDto)`. Ensure ValidationPipe has `transform: true`.
3 Query params are strings instead of numbers
Cause: HTTP query parameters are inherently strings. Without transformation, they stay as strings even with TypeScript type annotations.
Solution: Enable `transform: true` and `transformOptions: { enableImplicitConversion: true }` in ValidationPipe. Or use `@Type(() => Number)` in the DTO.
