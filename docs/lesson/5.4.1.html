OAuth2 with JWT in FastAPI

Intuition

OAuth2 with JWT is like a wristband at a festival: you prove your identity once at the gate, get a stamped wristband, and show it to access different areas without re-identifying yourself. The stamp (signature) prevents forgery.

OAuth2 and JWT Authentication in FastAPI

FastAPI has built-in support for OAuth2 with password flow and JWT tokens. This provides a standards-compliant authentication system that integrates with Swagger UI's authorize button.

from datetime import datetime, timedelta
from fastapi import FastAPI, Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
import jwt

SECRET_KEY = "your-secret-key"
ALGORITHM = "HS256"
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

def create_access_token(data: dict, expires_delta: timedelta):
    to_encode = data.copy()
    to_encode["exp"] = datetime.utcnow() + expires_delta
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)

async def get_current_user(token: str = Depends(oauth2_scheme)):
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        username = payload.get("sub")
        if username is None:
            raise HTTPException(status_code=401)
    except jwt.InvalidTokenError:
        raise HTTPException(status_code=401)
    return {"username": username}

app = FastAPI()

@app.post("/token")
async def login(form_data: OAuth2PasswordRequestForm = Depends()):
    token = create_access_token({"sub": form_data.username}, timedelta(hours=1))
    return {"access_token": token, "token_type": "bearer"}

Key Takeaways: - OAuth2PasswordBearer extracts the token from the Authorization header - JWTs are stateless tokens containing claims, signed with a secret key - Always set expiration (exp) on tokens to limit their validity - Use python-jose or PyJWT library for encoding and decoding JWTs

Exercises

  1. Refresh Token Flow

    Implement a refresh token system alongside access tokens. Issue a short-lived access token (15 min) and a long-lived refresh token (7 days). Create a /refresh endpoint that validates the refresh token and issues a new access token.

    Hint

    Store refresh tokens in a set or database. The refresh endpoint verifies the refresh token, checks it hasn't been revoked, and returns a new access token. Include a type claim in the JWT to distinguish access from refresh tokens.

  2. Scope-Based Authorization

    Extend the OAuth2 flow to include scopes. Add scopes to the JWT payload and create a dependency require_scope(scope: str) that checks if the current user's token includes the required scope.

    Hint

    Use OAuth2PasswordBearer(tokenUrl='token', scopes={'read': 'Read access', 'write': 'Write access'}) and check SecurityScopes in your dependency.

Common Problems & Solutions

1 JWT token always returns 401 Unauthorized
Cause: The token is expired, the secret key used for decoding differs from encoding, or the algorithm parameter doesn't match.
Solution: Verify the secret key and algorithm are identical for encoding and decoding. Check the `exp` claim. Use `jwt.decode` with `options={"verify_exp": False}` temporarily to debug payload issues.
2 Swagger UI authorize button not sending the token
Cause: The `tokenUrl` in `OAuth2PasswordBearer` doesn't match the actual login endpoint path, or the response format is wrong.
Solution: Ensure `tokenUrl` matches your login route exactly. The login endpoint must return `{"access_token": "...", "token_type": "bearer"}` as a JSON response.
