Filtering, Sorting, and Sparse Fieldsets

Intuition

Think of filtering, sorting, and field selection like ordering at a restaurant. Filtering is choosing a section of the menu ('only vegetarian dishes'). Sorting is asking the waiter to list them by price. Sparse fieldsets is saying 'just tell me the name and price, I do not need the full description.'

Filtering, Sorting, and Sparse Fieldsets

Giving API consumers control over which data they receive and how it is ordered is essential for building flexible, performant APIs. This lesson covers the three main data-shaping mechanisms: filtering, sorting, and field selection.

Filtering

Filtering lets clients request only the records they need.

Simple Equality Filters

GET /api/posts?status=published&authorId=42

app.get('/api/posts', async (req, res) => {
  const allowedFilters = ['status', 'authorId', 'category'];
  const where: Record<string, any> = {};

  for (const key of allowedFilters) {
    if (req.query[key] !== undefined) {
      where[key] = req.query[key];
    }
  }

  const posts = await prisma.post.findMany({ where });
  res.json({ data: posts });
});

Operator-Based Filters

For more complex queries, use operators in query parameters:

GET /api/products?price[gte]=10&price[lte]=100&rating[gt]=4
GET /api/posts?createdAt[after]=2024-01-01&title[contains]=javascript

interface FilterOperators {
  eq?: any;
  neq?: any;
  gt?: number;
  gte?: number;
  lt?: number;
  lte?: number;
  contains?: string;
  startsWith?: string;
  in?: any[];
}

function parseFilters(query: Record<string, any>): Record<string, any> {
  const prismaWhere: Record<string, any> = {};

  for (const [key, value] of Object.entries(query)) {
    // Handle operator syntax: field[operator]=value
    const match = key.match(/^(\w+)\[(\w+)\]$/);
    if (match) {
      const [, field, operator] = match;
      if (!prismaWhere[field]) prismaWhere[field] = {};

      switch (operator) {
        case 'eq': prismaWhere[field].equals = value; break;
        case 'neq': prismaWhere[field].not = value; break;
        case 'gt': prismaWhere[field].gt = Number(value); break;
        case 'gte': prismaWhere[field].gte = Number(value); break;
        case 'lt': prismaWhere[field].lt = Number(value); break;
        case 'lte': prismaWhere[field].lte = Number(value); break;
        case 'contains': prismaWhere[field].contains = value; break;
        case 'startsWith': prismaWhere[field].startsWith = value; break;
        case 'in': prismaWhere[field].in = value.split(','); break;
      }
    } else if (typeof value === 'string' && !key.startsWith('sort') && key !== 'fields' && key !== 'page' && key !== 'limit') {
      // Simple equality
      prismaWhere[key] = value;
    }
  }

  return prismaWhere;
}

app.get('/api/products', async (req, res) => {
  const where = parseFilters(req.query as Record<string, any>);
  const products = await prisma.product.findMany({ where });
  res.json({ data: products });
});

Full-Text Search Filter

GET /api/posts?q=javascript+async

app.get('/api/posts', async (req, res) => {
  const searchQuery = req.query.q as string;

  if (searchQuery) {
    // PostgreSQL full-text search via Prisma raw query
    const posts = await prisma.$queryRaw`
      SELECT * FROM posts
      WHERE to_tsvector('english', title || ' ' || body)
        @@ plainto_tsquery('english', ${searchQuery})
      ORDER BY ts_rank(
        to_tsvector('english', title || ' ' || body),
        plainto_tsquery('english', ${searchQuery})
      ) DESC
      LIMIT 20
    `;
    return res.json({ data: posts });
  }

  const posts = await prisma.post.findMany({ take: 20 });
  res.json({ data: posts });
});

Sorting

Allow clients to control the order of results.

GET /api/posts?sort=createdAt       (ascending)
GET /api/posts?sort=-createdAt      (descending, prefix with -)
GET /api/posts?sort=-createdAt,title (multi-field sort)

function parseSortParam(sort: string | undefined, allowedFields: string[]): Record<string, 'asc' | 'desc'>[] {
  if (!sort) return [{ createdAt: 'desc' }]; // default sort

  return sort.split(',').map(field => {
    const direction = field.startsWith('-') ? 'desc' : 'asc';
    const fieldName = field.replace(/^-/, '');

    // Security: only allow sorting on approved fields
    if (!allowedFields.includes(fieldName)) {
      throw new Error(`Cannot sort by '${fieldName}'. Allowed: ${allowedFields.join(', ')}`);
    }

    return { [fieldName]: direction };
  });
}

app.get('/api/posts', async (req, res) => {
  const allowedSortFields = ['createdAt', 'title', 'viewCount', 'likeCount'];

  try {
    const orderBy = parseSortParam(req.query.sort as string, allowedSortFields);
    const posts = await prisma.post.findMany({ orderBy, take: 20 });
    res.json({ data: posts });
  } catch (err) {
    res.status(400).json({ error: err.message });
  }
});

JSON:API Style Sorting

The JSON:API spec uses exactly this convention with comma-separated fields and - prefix for descending. It is a well-understood standard.

Sparse Fieldsets (Field Selection)

Let clients request only the fields they need, reducing bandwidth and improving performance.

GET /api/posts?fields=id,title,createdAt
GET /api/users?fields=id,name,avatar

function parseFields(fields: string | undefined, allowedFields: string[]): Record<string, boolean> | undefined {
  if (!fields) return undefined; // return all fields

  const requested = fields.split(',').map(f => f.trim());
  const select: Record<string, boolean> = {};

  for (const field of requested) {
    if (allowedFields.includes(field)) {
      select[field] = true;
    }
  }

  // Always include id
  select.id = true;

  return select;
}

app.get('/api/posts', async (req, res) => {
  const allowedFields = ['id', 'title', 'body', 'status', 'createdAt', 'updatedAt', 'authorId'];
  const select = parseFields(req.query.fields as string, allowedFields);

  const posts = await prisma.post.findMany({
    ...(select && { select }),
    take: 20,
  });

  res.json({ data: posts });
});

Nested Field Selection

For APIs with relationships:

GET /api/posts?fields=id,title&fields[author]=id,name

function parseNestedFields(query: Record<string, any>): any {
  const select: Record<string, any> = {};

  // Top-level fields
  if (query.fields && typeof query.fields === 'string') {
    for (const f of query.fields.split(',')) {
      select[f.trim()] = true;
    }
  }

  // Nested fields: fields[relation]=field1,field2
  for (const [key, value] of Object.entries(query)) {
    const match = key.match(/^fields\[(\w+)\]$/);
    if (match && typeof value === 'string') {
      const relation = match[1];
      const relationFields: Record<string, boolean> = {};
      for (const f of value.split(',')) {
        relationFields[f.trim()] = true;
      }
      select[relation] = { select: relationFields };
    }
  }

  return Object.keys(select).length > 0 ? select : undefined;
}

Combining Everything: A Reusable Query Builder

import { Request } from 'express';

interface QueryOptions {
  allowedFilters: string[];
  allowedSorts: string[];
  allowedFields: string[];
  defaultSort: string;
  maxLimit: number;
}

function buildQuery(req: Request, options: QueryOptions) {
  const page = Math.max(1, parseInt(req.query.page as string) || 1);
  const limit = Math.min(
    options.maxLimit,
    Math.max(1, parseInt(req.query.limit as string) || 20)
  );

  return {
    where: parseFilters(req.query as Record<string, any>),
    orderBy: parseSortParam(req.query.sort as string || options.defaultSort, options.allowedSorts),
    select: parseFields(req.query.fields as string, options.allowedFields),
    skip: (page - 1) * limit,
    take: limit,
  };
}

// Usage
app.get('/api/products', async (req, res) => {
  const query = buildQuery(req, {
    allowedFilters: ['category', 'status', 'price', 'brand'],
    allowedSorts: ['price', 'createdAt', 'name', 'rating'],
    allowedFields: ['id', 'name', 'price', 'category', 'description', 'rating', 'imageUrl'],
    defaultSort: '-createdAt',
    maxLimit: 100,
  });

  const [products, total] = await Promise.all([
    prisma.product.findMany(query),
    prisma.product.count({ where: query.where }),
  ]);

  res.json({ data: products, pagination: { page, limit, total } });
});

Security Considerations

  1. Always whitelist filterable/sortable fields. Never pass user input directly to the query.
  2. Validate filter values. A price filter should be numeric, a status filter should be an enum.
  3. Limit complexity. Cap the number of simultaneous filters, sort fields, and selected fields.
  4. Beware of SQL injection. When building raw queries, always use parameterized queries.
  5. Rate limit expensive queries. Sorting by non-indexed fields or using contains on large text columns is expensive.

Common Problems & Solutions

1 SQL injection through filter parameters
Cause: User-supplied filter field names or values are concatenated directly into SQL strings.
Solution: Always whitelist allowed filter fields. Use parameterized queries or an ORM. Never interpolate user input into SQL strings.
2 Sorting by non-indexed columns causes slow queries
Cause: The database performs a full table scan to sort results.
Solution: Only allow sorting on indexed columns. Create composite indexes for common sort combinations. Monitor slow query logs.
3 Overly complex filter combinations cause timeouts
Cause: Clients send many simultaneous filters that result in complex, unoptimized query plans.
Solution: Limit the number of simultaneous filters (e.g., max 5). Use EXPLAIN ANALYZE to identify problematic combinations. Add appropriate composite indexes.
