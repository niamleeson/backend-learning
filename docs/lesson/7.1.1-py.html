Essential Linux Commands for Developers

Intuition

Think of Linux commands like kitchen utensils. Each tool does one thing well (a knife cuts, a peeler peels), but combining them in sequence lets you prepare any dish. Pipes are like an assembly line connecting each station.

Essential Linux Commands for Developers

As a backend developer, Linux is your production environment. Most servers run Linux, so fluency with the command line is non-negotiable.

Navigation & File Operations

# Navigate and explore
pwd                    # Print working directory
ls -lah                # List files with details, hidden files, human-readable sizes
cd /var/log            # Change directory
tree -L 2 src/         # Show directory structure, 2 levels deep

# File operations
cp -r src/ backup/     # Copy recursively
mv old.txt new.txt     # Move/rename
rm -rf build/          # Remove recursively (CAREFUL!)
mkdir -p a/b/c         # Create nested directories

Viewing & Searching Files

# View files
cat config.yaml        # Print entire file
less /var/log/syslog   # Paginated viewing
head -n 20 app.log     # First 20 lines
tail -f app.log        # Follow log output in real-time

# Search
grep -rn "ERROR" /var/log/   # Recursive search with line numbers
find / -name "*.py" -mtime -7  # Python files modified in last 7 days
which python3          # Find executable location

Pipes & Redirection

# Chain commands
cat access.log | grep "500" | wc -l        # Count 500 errors
ps aux | grep python | awk '{print $2}'    # Get PIDs of Python processes
echo "backup" >> notes.txt                  # Append to file
curl -s api.example.com 2>/dev/null        # Suppress stderr

Common Mistakes

  • Running rm -rf / or rm -rf * without double-checking the current directory
  • Forgetting -r flag when copying directories
  • Using cat for large files instead of less or tail

Key Takeaways

  • Master grep, find, awk, and sed -- they solve 90% of text-processing tasks
  • Always use tail -f for live log monitoring
  • Pipe (|) is your best friend for composing powerful one-liners

Exercises

  1. Log Analysis One-Liner

    Given a web server access log file, write a single command pipeline that extracts all unique IP addresses that received a 500 status code, sorts them by frequency, and shows the top 10.

    Hint

    Combine grep, awk, sort, uniq -c, sort -rn, and head in a pipeline.

  2. File Organization Script

    Use find and xargs to locate all .log files older than 30 days in /var/log and compress them with gzip.

    Hint

    Use find with -mtime +30 -name '*.log' and pipe to xargs gzip.

  3. System Reconnaissance

    Write a sequence of commands to determine: disk usage of the current directory, available memory, number of CPU cores, and the current Linux distribution.

    Hint

    Use du -sh, free -h, nproc (or cat /proc/cpuinfo), and cat /etc/os-release.

Common Problems & Solutions

1 Permission denied when running a script
Symptoms: ['bash: ./script.sh: Permission denied']
Cause: The script file does not have execute permission.
Solution: Run `chmod +x script.sh` to add execute permission, then run with `./script.sh`.
2 Command not found after installing a tool
Symptoms: ['bash: mytool: command not found']
Cause: The binary is not in your PATH environment variable.
Solution: Find the binary with `which` or `find`, then add its directory to PATH in your ~/.bashrc or ~/.zshrc: `export PATH=$PATH:/new/path`.
3 Accidentally deleted important files with rm
Cause: rm permanently removes files -- there is no trash/recycle bin by default.
Solution: Use `rm -i` for interactive confirmation. Consider aliasing rm to `rm -i` in your shell config. For critical systems, use a trash utility like `trash-cli`.
