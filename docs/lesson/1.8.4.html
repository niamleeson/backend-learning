itertools: Infinite Iterators, Combinatorics, and Recipes

Intuition

itertools gives you Lego bricks for iteration: infinite sequences, combinatorics, grouping, and chaining — all lazy and composable, so you build pipelines without materializing intermediate results.

The itertools module provides a toolkit of fast, memory-efficient iterator building blocks for creating complex iteration patterns.

import itertools

# Infinite iterators
counter = itertools.count(10, 2)     # 10, 12, 14, ...
cycler = itertools.cycle("AB")       # A, B, A, B, ...
repeater = itertools.repeat(42, 3)   # 42, 42, 42

# Combinatorics
print(list(itertools.combinations([1,2,3], 2)))  # [(1,2),(1,3),(2,3)]
print(list(itertools.permutations("AB")))         # [('A','B'),('B','A')]
print(list(itertools.product("01", repeat=3)))    # 8 binary strings

# Practical recipes
data = [1, 1, 2, 3, 3, 3, 4]
for key, group in itertools.groupby(data):
    print(key, list(group))  # 1 [1,1], 2 [2], 3 [3,3,3], 4 [4]

# Chain multiple iterables
combined = itertools.chain([1,2], [3,4], [5])
print(list(combined))  # [1, 2, 3, 4, 5]
  • chain flattens multiple iterables into one; chain.from_iterable works with a single iterable of iterables.
  • groupby groups consecutive identical elements — sort first if you want global grouping.
  • islice(iterable, stop) takes a slice from any iterator without converting to a list.
  • product, combinations, and permutations handle combinatorial generation efficiently.

Exercises

  1. Sliding Window

    Write a function sliding_window(iterable, n) that yields overlapping tuples of size n. E.g., list(sliding_window([1,2,3,4,5], 3)) returns [(1,2,3),(2,3,4),(3,4,5)].

    Hint

    Use itertools.islice with a deque, or use the itertools pairwise recipe extended to size n.

  2. Flatten Arbitrarily Nested Iterables

    Write a generator flatten(iterable) that recursively flattens nested lists/tuples but leaves strings intact.

    Hint

    Check isinstance(item, Iterable) and not isinstance(item, (str, bytes)) before recursing.

Common Problems & Solutions

1 `groupby` groups non-consecutive duplicates separately.
Cause: `groupby` only groups consecutive equal elements, like Unix `uniq`.
Solution: Sort the data first, or use `collections.defaultdict(list)` for global grouping.
2 Consuming an infinite iterator (e.g., `list(itertools.count())`) hangs or crashes.
Cause: Infinite iterators never raise `StopIteration`; converting to list tries to exhaust them.
Solution: Always limit infinite iterators with `islice`, `takewhile`, or manual stopping logic.
